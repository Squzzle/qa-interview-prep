# Как с помощью RestAssured извлечь значение из JSON-ответа?

**Коротко:** Значение извлекается через встроенный JsonPath — методы `.extract().path("...")`, `.jsonPath().get("...")` или прямую десериализацию в объект через `.as(Class)`.

## Развёрнутый ответ

RestAssured предоставляет несколько способов извлечь данные из JSON-ответа:

1. **Через `extract().path("field")` или `extract().jsonPath()`** — используется нотация JsonPath (аналогична XPath, но для JSON), позволяющая обращаться к вложенным полям, массивам, применять фильтрацию.
2. **Через объект `Response` и метод `.jsonPath()`** — если ответ уже получен как `Response`, можно вызвать `response.jsonPath().getString("field")`, `.getInt("field")`, `.getList("items")` и т.д.
3. **Десериализация всего ответа в POJO** — метод `.as(User.class)` автоматически преобразует JSON в Java-объект с помощью Jackson/Gson (в зависимости от того, что подключено в classpath), после чего с полями можно работать через геттеры.
4. **Извлечение из вложенных структур и массивов** — JsonPath поддерживает нотацию с точками и индексами: `"data.items[0].id"`, а также фильтры вроде `"items.find { it.status == 'ACTIVE' }.id"`.

Типовая нотация JsonPath в RestAssured:

- `"field"` — простое поле верхнего уровня;
- `"parent.child"` — вложенное поле;
- `"items[0].name"` — элемент массива по индексу;
- `"items.size()"` — размер массива;
- `"items.findAll { it.price > 100 }"` — фильтрация по условию (используется синтаксис Groovy, так как JsonPath в RestAssured построен на GPath).

Важно: `path()` внутри `then()` возвращает значение и одновременно выполняет неявную проверку (если указать matcher), а `extract().path()` просто возвращает значение без ассерта — это используется, когда значение нужно сохранить в переменную для последующих запросов (например, id созданного ресурса для последующего GET).

## Пример / когда применяется

Извлечение простого поля:

```java
String name = given()
.when()
    .get("/api/users/123")
.then()
    .extract()
    .path("name");
```

Извлечение из вложенного JSON и массива:

```json
{
  "data": {
    "items": [
      { "id": 1, "status": "ACTIVE" },
      { "id": 2, "status": "INACTIVE" }
    ]
  }
}
```

```java
int firstId = given()
.when()
    .get("/api/items")
.then()
    .extract()
    .path("data.items[0].id");
```

Десериализация в объект:

```java
User user = given()
.when()
    .get("/api/users/123")
.then()
    .extract()
    .as(User.class);
```

Практическое применение: часто id, полученный из ответа `POST /orders`, извлекается и используется в следующем запросе `GET /orders/{id}` для проверки, что заказ действительно создан и сохранился.

## На что смотрит интервьюер

- Знание синтаксиса JsonPath (в том числе работу с массивами и фильтрами).
- Понимание разницы между `path()` внутри цепочки проверок и `extract().path()` для сохранения значения в переменную.
- Умение показать пример цепочки запросов, где значение из одного ответа используется в другом запросе.
- Красный флаг: если кандидат предлагает вручную парсить JSON как строку через `substring`/regex вместо использования JsonPath.
