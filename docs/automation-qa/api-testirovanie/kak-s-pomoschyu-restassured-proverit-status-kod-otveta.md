# Как с помощью RestAssured проверить статус-код ответа?

**Коротко:** Статус-код проверяется методом `.statusCode()` в блоке `then()`, куда можно передать точное число или Hamcrest matcher для более гибких условий.

## Развёрнутый ответ

После выполнения запроса в блоке `when()` ответ передаётся в блок `then()`, где вызывается метод `statusCode(int)` для проверки конкретного HTTP-кода ответа (`200`, `201`, `404`, `500` и т.д.). Если фактический код не совпадёт с ожидаемым, тест упадёт с понятной ошибкой, показывающей ожидаемое и фактическое значение.

Существует несколько вариантов проверки:

- **Точное совпадение:** `statusCode(200)` — самый распространённый вариант, когда известен единственный ожидаемый код.
- **С использованием Hamcrest matcher'ов:** `statusCode(equalTo(200))`, `statusCode(anyOf(is(200), is(201)))`, `statusCode(greaterThanOrEqualTo(200))` — полезно, когда нужно проверить диапазон или несколько допустимых значений.
- **Через `Response` объект напрямую**, если ответ был предварительно извлечён (`extract().response()`), можно вызвать `response.getStatusCode()` и сравнить его через сторонний assert (JUnit/AssertJ), что удобно при сложной логике проверки в несколько шагов.
- **Групповая проверка нескольких условий** через `.then().statusCode(200).body(...)` — можно совмещать проверку статус-кода с проверками тела ответа и заголовков в одной цепочке.

Дополнительно важно понимать семантику самих кодов при написании тестов: `2xx` — успех, `4xx` — ошибка клиента (неверные данные, нет прав, ресурс не найден), `5xx` — ошибка сервера. Хороший API-тест должен проверять не только "счастливый путь" (200/201), но и негативные сценарии (400, 401, 403, 404).

## Пример / когда применяется

```java
given()
    .contentType(ContentType.JSON)
.when()
    .get("/api/users/123")
.then()
    .statusCode(200);
```

Проверка с matcher'ом на несколько допустимых кодов:

```java
given()
.when()
    .delete("/api/users/123")
.then()
    .statusCode(anyOf(is(200), is(204)));
```

Проверка через объект Response:

```java
Response response = given()
.when()
    .post("/api/users");

int statusCode = response.getStatusCode();
assertThat(statusCode).isEqualTo(201);
```

## На что смотрит интервьюер

- Знание базового синтаксиса `.then().statusCode(...)`.
- Умение использовать Hamcrest matcher'ы для более гибких проверок, а не только точное сравнение.
- Понимание, что негативные сценарии (4xx/5xx) так же важны, как и успешные (2xx).
- Красный флаг: если кандидат проверяет статус-код через ручной парсинг строки ответа вместо встроенного метода, или не знает, где именно вызывается `statusCode()` (в `then()`, а не в `when()`).
