# Сравните Jackson и Gson. Какие аннотации из Jackson (`@JsonProperty`, `@JsonIgnore`) вы знаете?

**Коротко:** Jackson мощнее, гибче и лучше интегрирован в Java-экосистему (Spring, RestAssured), поддерживает множество аннотаций для тонкой настройки; Gson проще в использовании "из коробки", но менее функционален.

## Развёрнутый ответ

**Jackson:**

- Стандарт де-факто в Spring и большинстве Java REST-фреймворков.
- Поддерживает потоковую обработку (streaming API), что важно при работе с большими JSON-документами.
- Богатый набор аннотаций для управления (де)сериализацией.
- Поддерживает модули для работы с датами/временем (`jackson-datatype-jsr310`), Kotlin, Guava и др.
- Более сложен в настройке для нестандартных случаев, но и более гибок.

**Gson:**

- Разработан Google, минималистичен, не требует аннотаций для базовой работы.
- Проще в освоении, меньше "магии".
- Хуже работает с сложными generic-типами без дополнительного кода (`TypeToken`).
- Меньше готовых интеграций и модулей по сравнению с Jackson.
- Нет встроенной потоковой обработки уровня Jackson (хотя `JsonReader`/`JsonWriter` присутствуют).

**Ключевые аннотации Jackson:**

- `@JsonProperty("name")` — задаёт, под каким именем поле сериализуется/десериализуется в JSON, если имя поля в Java отличается от имени в JSON.
- `@JsonIgnore` — исключает поле из сериализации и десериализации полностью.
- `@JsonIgnoreProperties({"field1", "field2"})` — на уровне класса игнорирует указанные поля или неизвестные свойства (`ignoreUnknown = true`).
- `@JsonCreator` — указывает конструктор или фабричный метод, используемый при десериализации.
- `@JsonInclude(JsonInclude.Include.NON_NULL)` — исключает null-поля из сериализованного JSON.
- `@JsonFormat(pattern = "yyyy-MM-dd")` — задаёт формат сериализации дат.
- `@JsonAlias({"user_name", "userName"})` — позволяет десериализовать из нескольких возможных имён полей.
- `@JsonRootName` — задаёт корневой элемент при обёртывании объекта.

## Пример / когда применяется

```java
public class User {

    @JsonProperty("user_id")
    private int id;

    private String name;

    @JsonIgnore
    private String password;

    @JsonFormat(pattern = "yyyy-MM-dd")
    private LocalDate birthDate;
}
```

При сериализации такого объекта поле `password` не попадёт в JSON, `id` будет сериализовано как `user_id`, а `birthDate` — в формате `2024-05-01`.

Пример результата:

```json
{
  "user_id": 1,
  "name": "Иван",
  "birthDate": "2024-05-01"
}
```

Выбор библиотеки на практике: если проект уже на Spring Boot — почти всегда используется Jackson (входит по умолчанию); Gson выбирают в простых Android-проектах или там, где не нужна гибкая настройка.

## На что смотрит интервьюер

- Знание конкретных аннотаций и их назначения, а не только общих слов "Jackson лучше".
- Понимание практических различий: производительность, гибкость, экосистема.
- Умение привести пример, когда `@JsonIgnore` необходим (например, скрытие пароля в ответах API).
- Красный флаг: если кандидат не может назвать ни одной аннотации Jackson или путает `@JsonIgnore` с `@JsonIgnoreProperties`.
- Follow-up: "Как десериализовать JSON с полем, отсутствующим в классе?", "Как настроить сериализацию enum?".
