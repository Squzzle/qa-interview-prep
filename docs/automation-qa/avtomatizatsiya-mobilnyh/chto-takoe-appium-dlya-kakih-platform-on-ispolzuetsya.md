# Что такое Appium? Для каких платформ он используется?

**Коротко:** Appium — это кроссплатформенный open-source фреймворк для автоматизации UI мобильных, десктопных и гибридных приложений, работающий по протоколу WebDriver и не требующий модификации тестируемого приложения.

## Развёрнутый ответ

Appium — это сервер автоматизации, реализующий спецификацию W3C WebDriver (и её расширения). Он выступает промежуточным звеном между тестовым скриптом и приложением: клиент отправляет HTTP-команды (найти элемент, кликнуть, ввести текст и т.д.) на Appium Server, который транслирует их в нативные команды платформы через соответствующие драйверы.

Appium используется для автоматизации:

- **Android** — нативные, гибридные и веб-приложения (через драйвер UiAutomator2, ранее — Selendroid);
- **iOS** — нативные, гибридные и веб-приложения (через драйвер XCUITest);
- **Windows-приложений** (через WinAppDriver);
- **мобильных веб-приложений** в браузерах Chrome (Android) и Safari (iOS);
- гибридных приложений с WebView-компонентами, где можно переключаться между нативным и веб-контекстом.

Ключевая особенность Appium — независимость от языка программирования: клиентские библиотеки существуют для Java, Python, JavaScript, C#, Ruby и других языков, поскольку взаимодействие идёт через стандартизированный протокол, а не через встроенный SDK.

## Пример / когда применяется

Типичный сценарий: команда QA пишет тесты на Java с использованием Appium Java-client и TestNG, запускает их на Android-эмуляторе и на реальном iOS-устройстве без изменения кода приложений — используется один и тот же подход к написанию тестов, но с разными desired capabilities (`platformName`, `automationName`, `app` и т.д.):

```text
platformName: Android
automationName: UiAutomator2
deviceName: Pixel_6_API_33
app: /path/to/app.apk
```

## На что смотрит интервьюер

Частая ошибка — путать Appium с Espresso/XCUITest, не понимая, что Appium использует их под капотом как драйверы, а не заменяет их. Интервьюер может спросить про архитектуру клиент-сервер, про поддержку кроссплатформенности "один тест для Android и iOS" (это миф — тесты обычно платформо-специфичны из-за разных локаторов), а также про отличие от инструментов уровня фреймворка (Espresso) с точки зрения скорости и стабильности.
