# Что такое Appium Inspector и для чего он используется?

**Коротко:** Appium Inspector — это графический инструмент (десктопное приложение или веб-версия) для визуального осмотра иерархии элементов экрана приложения, подключенный к запущенной Appium-сессии, который помогает находить локаторы и проверять взаимодействия ещё до написания кода тестов.

## Развёрнутый ответ

Appium Inspector подключается к Appium Server и открывает сессию на реальном устройстве или эмуляторе/симуляторе, отображая:

- **скриншот текущего экрана** приложения в реальном времени;
- **дерево элементов (page source / accessibility tree)** — иерархию всех видимых UI-компонентов с их атрибутами: `resource-id`, `class`, `text`, `content-desc`, `bounds`, `xpath` и т.д.;
- **панель атрибутов выбранного элемента**, из которой можно скопировать готовый локатор (Accessibility ID, XPath, UIAutomator selector, iOS Predicate/Class Chain);
- **возможность выполнять действия** прямо из интерфейса — тап, свайп, ввод текста, что позволяет проверить локатор и взаимодействие без написания тестового кода.

Инструмент незаменим на этапе разработки автотестов: вместо того чтобы гадать, какой атрибут использовать для поиска элемента, инженер открывает Inspector, кликает на нужный элемент на скриншоте и получает точный локатор с несколькими вариантами стратегии поиска.

Ранее существовал как отдельный legacy-инструмент внутри Appium Desktop, сейчас распространяется как отдельное приложение Appium Inspector (Electron-based), совместимое с Appium 2.x.

## Пример / когда применяется

Типичный workflow: QA-инженер запускает Appium Server, указывает desired capabilities (`platformName`, `app`, `deviceName`) в Appium Inspector, нажимает "Start Session", видит текущий экран приложения, кликает на кнопку логина и видит в дереве:

```text
resource-id: com.example.app:id/btn_login
class: android.widget.Button
text: "Войти"
xpath: //android.widget.Button[@resource-id="com.example.app:id/btn_login"]
```

Этот `resource-id` затем используется в коде теста как надёжный локатор.

## На что смотрит интервьюер

Важно показать понимание, что Inspector — это инструмент разработки/дебага, а не часть CI-пайплайна выполнения тестов. Частый вопрос — какие стратегии локаторов предпочтительнее (Accessibility ID/resource-id вместо хрупкого XPath по всей иерархии) и как Inspector помогает выявлять проблемы доступности (accessibility) приложения, что важно и для реальных пользователей с ассистивными технологиями.
