# Что такое Kaspresso? Какие преимущества он дает при написании тестов для Android?

**Коротко:** Kaspresso — это open-source обёртка от Kaspersky над Espresso и UI Automator на Kotlin, которая делает Android UI-тесты более стабильными, читаемыми и удобными в поддержке за счёт встроенных retry-механизмов, DSL, скриншотов и логирования ошибок.

## Развёрнутый ответ

Kaspresso решает типичные проблемы "чистого" Espresso:

- **Flaky-тесты.** Espresso по умолчанию падает сразу при `NoMatchingViewException` или похожих ошибках, если элемент ещё не появился или анимация не завершилась. Kaspresso добавляет автоматические retry/interceptors для операций поиска и взаимодействия с элементами, что существенно снижает нестабильность (flakiness) тестов без ручных `Thread.sleep`.

- **Читаемый DSL (Kotlin-style).** Тесты пишутся в декларативном стиле с использованием `Screen`-объектов (по паттерну Page Object / Screen Object), что делает код более структурированным и понятным для новых участников команды.

- **Автоматический логгинг и скриншоты при падении.** Kaspresso сам собирает подробные логи шагов теста и делает скриншот в момент ошибки, что упрощает диагностику причин падения без дополнительной настройки.

- **Интеграция с UI Automator "из коробки".** Позволяет комбинировать возможности Espresso (внутриприложенческие взаимодействия) и UI Automator (системные диалоги, разрешения, другие приложения) в одном тесте без ручной настройки моста между ними.

- **Поддержка Kotlin DSL и Compose.** Хорошо интегрируется с современным стеком Android-разработки, включая Jetpack Compose.

- **Steps/Scenario API** — возможность декомпозировать сложные сценарии на именованные шаги (`step("Открыть экран логина") { ... }`), что улучшает отчётность и позволяет видеть в логах, на каком именно шаге произошло падение.

## Пример / когда применяется

```kotlin
class LoginTest : TestCase() {
    @Test
    fun successfulLoginTest() = run {
        step("Open login screen") {
            LoginScreen {
                usernameField.typeText("user")
                passwordField.typeText("pass")
                loginButton.click()
            }
        }
        step("Verify welcome screen is shown") {
            WelcomeScreen {
                welcomeText.assertIsDisplayed()
            }
        }
    }
}
```

Kaspresso применяется, когда команда уже использует Espresso, но столкнулась с нестабильностью тестов и хочет получить более структурированный, отказоустойчивый фреймворк без перехода на сторонний инструмент вроде Appium.

## На что смотрит интервьюер

Важно показать понимание, что Kaspresso — это не замена Espresso, а надстройка над ним (и UI Automator), решающая проблему flaky-тестов через встроенные interceptors и retry-логику. Может последовать вопрос о том, чем Kaspresso отличается от чистого Espresso с точки зрения архитектуры (Screen Object паттерн, Steps API) и в каких случаях вообще не стоит использовать Kaspresso (например, если нужна кроссплатформенность с iOS — тогда логичнее Appium).
