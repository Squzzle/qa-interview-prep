# Какие программы и инструменты входят в "скоуп" (набор) для запуска мобильных тестов (Appium Server, эмуляторы/симуляторы, реальные устройства)?

**Коротко:** Для запуска мобильных автотестов обычно нужен набор из Appium Server и драйверов, платформенных SDK (Android SDK, Xcode), эмуляторов/симуляторов или реальных устройств, а также инструментов сборки приложения, отладки (ADB, Xcode Instruments) и, опционально, облачных ферм устройств.

## Развёрнутый ответ

Полный стек для запуска мобильных тестов обычно включает:

**1. Appium Server и драйверы**
Сам сервер Appium (устанавливается через npm, `npm i -g appium`) плюс необходимые драйверы: `uiautomator2` для Android, `xcuitest` для iOS. Устанавливаются командой `appium driver install`.

**2. Платформенные SDK**
- **Android SDK** — включает `platform-tools` (adb, fastboot), `build-tools`, эмуляторные образы (system images), а также переменные окружения `ANDROID_HOME`/`ANDROID_SDK_ROOT`.
- **Xcode + Command Line Tools** (только на macOS) — для сборки и запуска iOS-приложений, включает `xcrun simctl` для управления симуляторами и `WebDriverAgent`, который Appium использует под капотом для взаимодействия через XCUITest.

**3. Эмуляторы / симуляторы**
- **Android Emulator (AVD)** — виртуальные устройства, настраиваемые через Android Studio AVD Manager, с выбором версии API и образа (с Google Play или без, x86/arm).
- **iOS Simulator** — входит в состав Xcode, позволяет запускать разные модели iPhone/iPad с разными версиями iOS без физического устройства.

**4. Реальные устройства**
Физические Android/iOS устройства, подключенные по USB или Wi-Fi (для Android — с включённой отладкой по USB и разрешённым режимом разработчика; для iOS — с провижининг-профилем разработчика и доверенным сертификатом подписи).

**5. Инструменты сборки и установки приложения**
`adb install` для APK, `ios-deploy`/`xcrun simctl install` или Xcode для IPA; для CI часто используется Fastlane для сборки и подписи.

**6. Инструменты диагностики**
- **ADB (Android Debug Bridge)** — логи (`adb logcat`), управление устройством, снятие скриншотов.
- **Xcode Instruments / Console** — логи и профилирование iOS-приложений.

**7. Облачные фермы устройств (опционально)**
BrowserStack App Automate, Sauce Labs, AWS Device Farm, Firebase Test Lab — используются, когда нужно масштабировать тесты на множестве реальных устройств без содержания собственной лаборатории.

**8. Тестовый фреймворк и раннер**
TestNG/JUnit (Java), pytest (Python), Mocha/Jasmine (JS через WebdriverIO) — управляет запуском, параллелизацией и отчётностью тестов.

## Пример / когда применяется

Типичный локальный запуск Android-теста:

```bash
appium --address 127.0.0.1 --port 4723 &
emulator -avd Pixel_6_API_33 &
adb wait-for-device
adb install app-debug.apk
mvn test -Dtest=LoginTest
```

Для CI-конвейера часто заменяют локальный эмулятор/устройство на облачную ферму, указывая в capabilities учётные данные BrowserStack/Sauce Labs вместо локального `deviceName`.

## На что смотрит интервьюер

Интервьюер проверяет практическое понимание всей цепочки запуска, а не только теорию про Appium API: важно упомянуть ADB и его роль в диагностике, разницу между эмулятором/симулятором и реальным устройством (производительность, доступ к сенсорам, GPS, камере, специфичные баги, которые проявляются только на реальных устройствах), а также назвать хотя бы один облачный сервис устройств. Красный флаг — если кандидат не может объяснить, зачем нужен WebDriverAgent для iOS или что делает `adb logcat`.
