# Какими командами можно настроить имя пользователя и email в Git?

**Коротко:** Имя и email автора коммитов настраиваются командами `git config user.name` и `git config user.email`, с флагом `--global` для настройки на всю систему или без него — для конкретного репозитория.

## Развёрнутый ответ

Каждый коммит в Git хранит метаданные автора: имя и email. Эта информация не запрашивается автоматически — её нужно один раз настроить через `git config`, иначе Git выдаст ошибку при попытке коммита ("Please tell me who you are").

Git поддерживает три уровня конфигурации, от более общего к более специфичному (более специфичный уровень имеет приоритет):

- **`--system`** — настройки для всех пользователей на данной машине (файл в системной директории Git); используется редко, требует прав администратора.
- **`--global`** — настройки для конкретного пользователя ОС, применяются ко всем репозиториям на этой машине (хранится в `~/.gitconfig`).
- **Без флага (локальный, per-repo)** — настройки только для текущего репозитория (хранится в `.git/config`), переопределяют глобальные.

Основные команды:

```bash
# Глобальная настройка (для всех репозиториев пользователя)
git config --global user.name "Ivan Petrov"
git config --global user.email "ivan.petrov@company.com"

# Локальная настройка только для текущего репозитория
# (полезно, если для рабочих и личных проектов нужен разный email)
git config user.name "Ivan Petrov (work)"
git config user.email "ivan.petrov@work-company.com"

# Просмотр текущих настроек
git config user.name
git config user.email
git config --list                 # все настройки со всех уровней
git config --list --show-origin   # с указанием, из какого файла взята каждая настройка
```

Локальная (per-repo) настройка полезна, когда на одной машине ведётся работа с несколькими организациями/аккаунтами (например, рабочий GitHub и личный GitHub), и нужно, чтобы коммиты в каждом репозитории подписывались правильным email — это особенно важно, если email используется для сопоставления коммитов с аккаунтом на GitHub/GitLab и для CI/CD-нотификаций.

Если имя/email не заданы правильно до коммитов, их можно исправить постфактум через `git commit --amend --author="Name <email>"` для последнего коммита, либо через `git filter-branch`/`git filter-repo`/интерактивный rebase с изменением автора для более глубоких коммитов (что переписывает историю и требует осторожности для уже опубликованных коммитов).

## Пример / когда применяется

Типичная первичная настройка после установки Git на новой машине:

```bash
git config --global user.name "Anna Ivanova"
git config --global user.email "anna.ivanova@company.com"
git config --global init.defaultBranch main
```

Сценарий: QA-инженер работает и над корпоративным репозиторием (рабочий email), и над личным pet-проектом с автотестами (личный email). В корпоративном репозитории он выполняет `git config user.email "personal@corp.com"` без `--global`, чтобы переопределить глобальный email только для этого репозитория.

## На что смотрит интервьюер

- Знание базового синтаксиса `git config user.name`/`user.email` и флага `--global`.
- Понимание иерархии конфигурации (system → global → local) и того, что более специфичный уровень имеет приоритет.
- Знание, зачем нужна локальная (per-repo) настройка при работе с разными аккаунтами/организациями.
- Follow-up: как исправить автора уже сделанного коммита (`commit --amend --author`); где физически хранятся эти настройки (`~/.gitconfig`, `.git/config`).
- Red flag: незнание разницы между `--global` и локальной настройкой, либо путаница с настройкой SSH-ключей/учётных данных вместо имени и email.
