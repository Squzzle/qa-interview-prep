# Что такое CSS-селектор? Приведите примеры.

**Коротко:** CSS-селектор — это шаблон, изначально предназначенный для применения стилей к элементам HTML, но также широко используемый в автоматизации тестирования для поиска элементов по тегу, классу, id, атрибутам и их комбинациям.

## Развёрнутый ответ

CSS-селектор — это выражение, описывающее набор элементов на странице на основе их тега, атрибутов, положения в дереве DOM или состояния. В контексте UI-автоматизации CSS-селекторы используются как альтернатива XPath: они, как правило, компактнее и быстрее обрабатываются браузером, потому что браузерный движок изначально оптимизирован под парсинг CSS (для применения стилей).

Основные виды CSS-селекторов:

- **По тегу**: `input`, `button` — выбирает все элементы данного тега.
- **По id**: `#login` — выбирает элемент с `id="login"`.
- **По классу**: `.btn-primary` — выбирает элементы с классом `btn-primary`.
- **По атрибуту**: `[name='email']`, `[data-testid='submit']` — точное совпадение значения атрибута.
- **Частичное совпадение атрибута**: `[class*='btn']` (содержит подстроку), `[id^='user-']` (начинается с), `[href$='.pdf']` (заканчивается на).
- **Комбинаторы иерархии**: `div > button` (прямой потомок), `form input` (любой потомок), `label + input` (соседний элемент сразу после), `h2 ~ p` (любой последующий "родственный" элемент).
- **Псевдоклассы**: `li:nth-child(2)`, `input:not([disabled])`, `tr:last-child`.
- **Комбинация классов и атрибутов**: `button.btn.btn-primary[disabled]`.

Важное ограничение CSS-селекторов по сравнению с XPath: в стандартном CSS3 нет прямой возможности искать элемент по его тексту (не по атрибуту, а именно по видимому содержимому) и нет возможности перейти от дочернего элемента к родительскому (в отличие от оси `parent::` в XPath). Некоторые движки (например, Playwright) расширяют CSS собственными псевдоселекторами `:has-text()`, `:visible` и т.д., но это уже не стандартный CSS, а фреймворк-специфичное расширение.

## Пример / когда применяется

```css
/* По id */
#login-button

/* По классу */
.form-control.is-invalid

/* По атрибуту точное значение */
input[type='password']

/* По атрибуту, частичное совпадение (содержит подстроку) */
a[href*='logout']

/* Прямой потомок */
div.card > button

/* Комбинация: input с типом email внутри формы с классом signup */
form.signup input[type='email']

/* Playwright-расширение: поиск по тексту */
button:has-text('Отправить')
```

Применяется в тех случаях, когда важна скорость поиска и когда локатор можно построить по стабильным атрибутам (классам, data-атрибутам), без необходимости искать по тексту или переходить вверх по дереву.

## На что смотрит интервьюер

- Знание синтаксиса комбинаторов (`>`, ` `, `+`, `~`) и разницы между ними.
- Понимание ограничений CSS: отсутствие встроенного поиска по тексту и невозможность перехода к родителю в чистом CSS3.
- Частая ошибка — путать `.class` (класс) и `#id`, либо путать комбинатор потомка (`div input`, любой уровень вложенности) с прямым потомком (`div > input`).
- Follow-up: «Как найти элемент по частичному совпадению атрибута?», «Можно ли через CSS найти родителя элемента?» (ответ: нет, в этом ограничение CSS перед XPath).
