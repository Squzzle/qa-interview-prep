# Что такое локаторы и зачем они нужны в UI-автоматизации?

**Коротко:** Локатор — это выражение (id, CSS-селектор, XPath и т.д.), с помощью которого автотест находит конкретный элемент на странице в DOM, чтобы взаимодействовать с ним (кликнуть, ввести текст, прочитать значение).

## Развёрнутый ответ

Локатор — это способ адресации элемента веб-страницы, который используется драйвером (например, Selenium WebDriver или Playwright) для поиска нужного узла в DOM-дереве. Без локатора автотест не сможет понять, к какому элементу применить действие: нажать кнопку, заполнить поле формы, проверить текст сообщения об ошибке и так далее.

Локаторы нужны, потому что браузер и тестовый фреймворк не «видят» страницу так, как человек — глазами. Вместо этого они работают с деревом DOM и должны получить чёткий, программно интерпретируемый адрес элемента. Хороший локатор должен обладать несколькими свойствами:

- **Уникальность** — однозначно указывать на один элемент (или на предсказуемый набор элементов, если это намеренно).
- **Стабильность** — не ломаться при незначительных изменениях вёрстки (например, при смене порядка классов CSS или добавлении обёрточного div).
- **Читаемость** — быть понятным другим разработчикам и легко поддерживаемым.
- **Производительность** — искать элемент максимально быстро (это особенно важно при большом количестве проверок в тесте).

Локаторы являются фундаментом Page Object паттерна: в объектах страниц (Page Object) хранятся именно локаторы, а бизнес-логика теста оперирует уже методами, которые эти локаторы используют внутри.

## Пример / когда применяется

Пример объявления локатора в Selenium (Java):

```java
By loginButton = By.id("login-submit");
WebElement button = driver.findElement(loginButton);
button.click();
```

Пример в Playwright (TypeScript) через CSS-селектор:

```javascript
await page.click('button[data-testid="login-submit"]');
```

Локаторы применяются в любом сценарии UI-автоматизации: заполнение форм, проверка текста на странице, клики по кнопкам, работа с выпадающими списками, drag-and-drop и т.д. — везде, где тесту нужно взаимодействовать с конкретным элементом интерфейса.

## На что смотрит интервьюер

- Понимает ли кандидат разницу между локатором и селектором (локатор — более широкое понятие, включающее стратегию поиска: id, name, className, tagName, linkText, CSS, XPath).
- Знает ли кандидат критерии «хорошего» локатора (устойчивость к изменениям вёрстки, уникальность, читаемость).
- Частая ошибка — путать локатор с самим `WebElement` или объектом, полученным после поиска.
- Follow-up вопросы: «Какие типы локаторов ты используешь чаще всего и почему?», «Как ты обеспечиваешь стабильность локаторов в проекте (data-testid, атрибуты для тестов)?».
