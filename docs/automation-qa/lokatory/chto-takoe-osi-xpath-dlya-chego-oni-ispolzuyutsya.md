# Что такое оси XPath? Для чего они используются? Приведите пример.

**Коротко:** Оси XPath (axes) — это специальные ключевые слова, задающие направление поиска относительно текущего (контекстного) узла: вверх к родителям, вниз к потомкам, в сторону к соседним элементам. Они позволяют находить элементы, которые сложно или невозможно адресовать через CSS-селекторы.

## Развёрнутый ответ

Оси XPath решают задачу навигации по DOM-дереву в любом направлении относительно выбранного (контекстного) узла — в отличие от CSS-селекторов, которые умеют перемещаться только «вниз» по дереву (к потомкам) и «вбок» (к следующим соседям через `+`/`~`), но не «вверх» к родителям.

Основные оси XPath:

- **`parent::`** — прямой родитель контекстного узла.
- **`ancestor::`** — все предки узла (родитель, дед и т.д.) вплоть до корня.
- **`ancestor-or-self::`** — предки плюс сам узел.
- **`child::`** — прямые дочерние узлы (используется по умолчанию, поэтому обычно опускается).
- **`descendant::`** — все потомки на любом уровне вложенности.
- **`descendant-or-self::`** — потомки плюс сам узел (эквивалент `//`).
- **`following-sibling::`** — все «соседи» после текущего узла на том же уровне.
- **`preceding-sibling::`** — все «соседи» до текущего узла на том же уровне.
- **`following::`** — все узлы документа, идущие после текущего (в любом месте дерева).
- **`preceding::`** — все узлы документа, идущие до текущего.
- **`self::`** — сам текущий узел.
- **`attribute::`** (сокращённо `@`) — атрибуты текущего узла.

Оси особенно полезны, когда у элемента нет собственного уникального атрибута, но рядом (например, в виде подписи-label) есть текст, по которому легко найти нужный узел, а затем от него перейти к реальному целевому элементу через ось `parent::`, `following-sibling::` и т.д.

## Пример / когда применяется

Классическая задача: найти поле ввода по тексту его label, когда сам input не имеет id или name, совпадающего с текстом:

```html
<div class="form-group">
  <label>Email</label>
  <input type="text" class="dynamic-input-x92j" />
</div>
```

```text
//label[text()='Email']/following-sibling::input
```

Ещё пример — найти родительский `<div>` для кнопки с определённым текстом (чтобы затем проверить состояние всего блока):

```text
//button[text()='Удалить']/parent::div
```

Найти все ячейки таблицы, стоящие перед определённой ячейкой на той же строке:

```text
//td[text()='Итого']/preceding-sibling::td
```

## На что смотрит интервьюер

- Знание конкретных названий осей и их семантики (особенно разница между `following-sibling` и `following`, между `ancestor` и `parent`).
- Понимание того, что оси решают задачу, недоступную в чистом CSS — переход вверх по дереву.
- Частая ошибка — забыть двойное двоеточие `::` после названия оси или перепутать порядок: `following-sibling::input`, а не `input::following-sibling`.
- Follow-up: «Как найти третий по счёту `following-sibling`?» (ответ: `following-sibling::td[3]`), «В чём разница между `descendant::` и `//`?» (в контексте относительного пути они близки по смыслу).
