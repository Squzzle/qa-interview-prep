# Как написать XPath для поиска элемента по тексту, содержащему "Hello"?

**Коротко:** Для поиска любого элемента, текст которого содержит подстроку "Hello", используется функция `contains()` в связке с `text()`: `//*[contains(text(), 'Hello')]`.

## Развёрнутый ответ

XPath умеет находить элементы по их видимому текстовому содержимому — эта возможность отсутствует в стандартном CSS и является одним из главных аргументов в пользу использования XPath. Есть два основных подхода:

1. **Точное совпадение через `text()`**: находит элемент, чей непосредственный текстовый узел равен строке ровно, без лишних символов, пробелов и без учёта вложенных тегов:
```text
//*[text()='Hello']
```
Такой вариант сработает только если текст элемента строго равен "Hello" (без пробелов, без "Hello World" и т.д.).

2. **Частичное совпадение через `contains()`**: находит элемент, чей текст содержит подстроку "Hello" где угодно внутри строки:
```text
//*[contains(text(), 'Hello')]
```
Этот вариант сработает и для "Hello", и для "Hello World!", и для "Say Hello to Bob".

Важный нюанс: и `text()`, и `contains(text(), ...)` работают с непосредственным текстовым узлом элемента, а не с текстом вложенных дочерних элементов. Если структура вложенная, например `<div><span>Hello</span></div>`, то `//div[contains(text(), 'Hello')]` не найдёт `div`, потому что у него самого нет прямого текстового узла "Hello" — текст находится внутри `span`. В этом случае нужно использовать `.` вместо `text()`, который берёт весь текст, включая потомков:
```text
//div[contains(., 'Hello')]
```

Также рекомендуется сужать поиск конкретным тегом вместо универсального `*`, чтобы уменьшить количество совпадений и повысить производительность и предсказуемость локатора: например, `//button[contains(text(), 'Hello')]` вместо `//*[contains(text(), 'Hello')]`.

## Пример / когда применяется

```text
<!-- HTML -->
<button>Hello, World!</button>
```

```text
XPath: //button[contains(text(), 'Hello')]
```

Полный пример на Java:

```java
WebElement element = driver.findElement(
    By.xpath("//button[contains(text(), 'Hello')]")
);
```

Если элементы содержат вложенную разметку:

```html
<div class="message"><span>Hello</span> from the team</div>
```

```text
XPath: //div[contains(@class, 'message') and contains(., 'Hello')]
```

## На что смотрит интервьюер

- Знание точного синтаксиса `contains(text(), '...')` наизусть — это одна из самых базовых, "обязательных" конструкций XPath.
- Понимание разницы между `text()` и `.` в контексте вложенных элементов.
- Частая ошибка — писать `contains(@text, 'Hello')` (текст — не атрибут, синтаксис неверный) или забывать про регистрозависимость сравнения строк.
- Follow-up: «А если текст с большой буквы иногда, а иногда с маленькой?» (ответ: понадобится `translate()` для приведения регистра, так как в XPath 1.0 нет функции `lower-case()`).
