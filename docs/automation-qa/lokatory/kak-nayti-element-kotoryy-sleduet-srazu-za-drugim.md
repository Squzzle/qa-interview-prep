# Как найти элемент, который следует сразу за другим элементом, используя XPath?

**Коротко:** Для поиска элемента, идущего сразу за другим на том же уровне вложенности, используется ось `following-sibling::`, применённая к уже найденному опорному элементу, например: `//label[text()='Email']/following-sibling::input[1]`.

## Развёрнутый ответ

Ось `following-sibling::` возвращает все узлы, которые являются «соседями» текущего (контекстного) элемента и располагаются после него на том же уровне вложенности (то есть имеют того же родителя). Это отличается от оси `following::`, которая возвращает вообще все узлы документа, находящиеся после текущего, независимо от уровня вложенности и родителя.

Типичный сценарий использования — когда нужный элемент (например, `input`) не имеет уникального атрибута, но рядом с ним есть текстовая подпись (`label`, `span`, `td`), которая легко находится по тексту. Тогда стратегия такая:

1. Найти опорный элемент по стабильному признаку (обычно по тексту).
2. От него перейти к следующему элементу через `following-sibling::`.
3. При необходимости сузить результат по тегу и индексу, так как `following-sibling::` может вернуть сразу несколько узлов (все последующие "соседи", а не только ближайший).

Если нужен именно первый (ближайший) следующий элемент, используют индекс `[1]`:

```text
//label[text()='Email']/following-sibling::input[1]
```

Если нужен непосредственно следующий элемент независимо от тега, используют звёздочку `*`:

```text
//label[text()='Email']/following-sibling::*[1]
```

Важно не путать `following-sibling::input[1]` с `(following-sibling::input)[1]`: без скобок индекс `[1]` применяется к каждому найденному input относительно его собственной позиции среди соседей (что обычно и совпадает с нужным результатом, если input единственный после label), но при неоднозначности лучше явно оборачивать выражение в скобки для получения именно первого элемента из всего набора.

## Пример / когда применяется

```html
<div class="form-row">
  <label>Телефон</label>
  <input type="tel" class="dyn-8f21a" />
</div>
```

```java
WebElement phoneInput = driver.findElement(
    By.xpath("//label[text()='Телефон']/following-sibling::input[1]")
);
```

Найти ячейку таблицы, следующую сразу за ячейкой с определённым текстом (в той же строке):

```text
//td[text()='Итого']/following-sibling::td[1]
```

Найти сообщение об ошибке, идущее сразу после конкретного поля ввода:

```text
//input[@name='email']/following-sibling::span[@class='error-message']
```

## На что смотрит интервьюер

- Знание точного названия оси `following-sibling::` и понимание, что она ограничена уровнем родственных узлов (в отличие от `following::`).
- Понимание разницы между `following-sibling::input[1]` и `(following-sibling::input)[1]`.
- Частая ошибка — путать `following-sibling` (только «соседи» после, тот же родитель) с `following` (вообще все узлы документа после текущего).
- Follow-up: «Как найти элемент, идущий непосредственно ПЕРЕД данным?» (`preceding-sibling::input[1]`), «Чем `following-sibling` отличается от `following`?».
