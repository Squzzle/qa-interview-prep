# Какие основные типы локаторов существуют в Selenium WebDriver?

**Коротко:** В Selenium WebDriver класс `By` предоставляет 8 базовых стратегий поиска: id, name, className, tagName, linkText, partialLinkText, cssSelector и xpath.

## Развёрнутый ответ

Selenium WebDriver предоставляет статический класс `By`, который инкапсулирует стратегии поиска элементов. Основные типы локаторов:

- **id** — поиск по атрибуту `id` элемента. Самый быстрый и предпочтительный способ, если id уникален и не генерируется динамически.
- **name** — поиск по атрибуту `name`, часто используется для полей форм.
- **className** — поиск по значению атрибута `class`. Нужно учитывать, что у элемента может быть несколько классов, и className ищет по одному классу из списка.
- **tagName** — поиск по имени HTML-тега (например, `input`, `button`, `a`). Обычно возвращает много элементов, поэтому используется в сочетании с `findElements` и дальнейшей фильтрацией.
- **linkText** — поиск ссылки (`<a>`) по её полному видимому тексту.
- **partialLinkText** — поиск ссылки по части видимого текста.
- **cssSelector** — поиск с помощью CSS-селекторов, поддерживает сложные комбинации по атрибутам, иерархии, псевдоклассам.
- **xpath** — поиск с помощью языка запросов XPath, самый гибкий способ, позволяет искать по тексту, по родственным связям (родитель, потомок, сосед), по осям.

Выбор конкретного локатора зависит от структуры DOM и требований к стабильности теста. На практике чаще всего используются id, cssSelector и xpath, поскольку они наиболее гибкие и предсказуемые.

## Пример / когда применяется

```java
driver.findElement(By.id("username"));
driver.findElement(By.name("password"));
driver.findElement(By.className("btn-primary"));
driver.findElement(By.tagName("input"));
driver.findElement(By.linkText("Forgot password?"));
driver.findElement(By.partialLinkText("Forgot"));
driver.findElement(By.cssSelector("input[type='email']"));
driver.findElement(By.xpath("//button[text()='Submit']"));
```

Пример выбора: если элемент имеет стабильный уникальный `id`, используем `By.id`. Если id отсутствует, но есть удобные CSS-атрибуты — `By.cssSelector`. Если нужно найти элемент по тексту или сложной иерархии — `By.xpath`.

## На что смотрит интервьюер

- Знает ли кандидат все 8 стратегий и их особенности (не путает name и id, понимает ограничения className при множественных классах).
- Понимает ли разницу между `findElement` и `findElements` применительно к локаторам, возвращающим несколько узлов (tagName, className).
- Частая ошибка — считать linkText/partialLinkText универсальными, забывая, что они применимы только к тегу `<a>`.
- Follow-up: «Какой локатор предпочтительнее по производительности?», «Как найти элемент по нескольким классам одновременно через className?» (ответ: className не поддерживает несколько классов через пробел, нужно использовать CSS-селектор или XPath).
