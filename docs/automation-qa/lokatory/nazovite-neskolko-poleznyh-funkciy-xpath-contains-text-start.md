# Назовите несколько полезных функций XPath (`contains()`, `text()`, `starts-with()`, `last()`, `position()`).

**Коротко:** XPath предоставляет встроенные функции для поиска по частичному тексту (`contains`, `starts-with`), по точному текстовому содержимому (`text()`), а также функции для работы с позицией узла в наборе (`position()`, `last()`), что делает XPath значительно более гибким инструментом, чем CSS-селекторы.

## Развёрнутый ответ

Функции XPath используются внутри предикатов (выражений в квадратных скобках `[...]`) для фильтрации найденных узлов по дополнительным условиям:

- **`text()`** — возвращает текстовый узел (видимое текстовое содержимое) элемента. Используется для точного совпадения текста: `//button[text()='Войти']`. Важный нюанс: `text()` учитывает только непосредственный текст элемента, не включая текст вложенных дочерних тегов, и требует точного совпадения (включая пробелы).
- **`contains(строка, подстрока)`** — проверяет, что первая строка содержит вторую как подстроку. Часто применяется как `contains(@class, 'active')` или `contains(text(), 'Привет')` — удобно, когда текст или атрибут динамический и содержит лишние части (например, id с суффиксом или класс с несколькими значениями).
- **`starts-with(строка, подстрока)`** — проверяет, что строка начинается с указанной подстроки. Полезно для динамических id/классов вида `id="item-123"`, `id="item-456"`: `//div[starts-with(@id, 'item-')]`.
- **`position()`** — возвращает порядковый номер узла в текущем наборе (начиная с 1). Используется как `//li[position()=2]` (эквивалент `//li[2]`).
- **`last()`** — возвращает индекс последнего узла в наборе. Используется для получения последнего элемента: `//tr[last()]` или предпоследнего `//tr[last()-1]`.
- **`normalize-space()`** — убирает лишние пробелы (в начале, конце и множественные внутри) перед сравнением текста; полезна, когда в разметке есть непредсказуемые отступы/переносы строк вокруг текста: `//span[normalize-space(text())='Итого']`.
- **`count()`** — возвращает количество узлов в наборе, полезна для проверок (например, что таблица содержит ровно N строк), хотя чаще используется вне локатора, в самой проверке.

Эти функции комбинируются друг с другом и с логическими операторами `and`/`or`/`not()` для построения точных и устойчивых локаторов.

## Пример / когда применяется

```text
// Точное совпадение текста
//button[text()='Отправить']

// Частичное совпадение текста (например, есть скрытый span внутри)
//button[contains(text(), 'Отправ')]

// Совпадение начала динамического id
//div[starts-with(@id, 'product-card-')]

// Второй элемент списка
//ul[@class='menu']/li[position()=2]

// Последний элемент таблицы
//table//tr[last()]

// Текст с пробелами по краям — очищаем перед сравнением
//p[normalize-space(text())='Условия использования']

// Комбинация условий
//input[@type='checkbox' and not(@disabled)]
```

## На что смотрит интервьюер

- Знание разницы между `text()` (точное совпадение) и `contains(text(), ...)` (частичное совпадение).
- Понимание, что `[2]` и `[position()=2]` — эквивалентны, но кандидат должен уметь объяснить, что стоит за сокращённой записью.
- Частая ошибка — использовать `text()` там, где в элементе есть вложенные теги (в этом случае `text()` вернёт пустой узел или не то, что ожидается, нужно использовать `.` вместо `text()` в некоторых случаях, либо `contains(., 'текст')`).
