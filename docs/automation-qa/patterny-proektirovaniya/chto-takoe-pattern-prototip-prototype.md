# Что такое паттерн Прототип (Prototype)?

**Коротко:** Prototype — порождающий паттерн, который позволяет создавать новые объекты путём копирования (клонирования) существующего объекта-образца, а не через создание с нуля, что полезно, когда создание объекта дорогое или сложное.

## Развёрнутый ответ

Идея паттерна: у объекта есть метод `clone()`, который возвращает копию самого себя, при этом клиентский код не обязан знать конкретный класс копируемого объекта — работа идёт через общий интерфейс `Prototype` (или `Cloneable`) с методом `clone()`. Это удобно, когда: создание объекта "с нуля" ресурсоёмко (например, требует обращения к базе данных, сети, сложных вычислений); нужно создать объект, идентичный существующему, но с несколькими изменёнными полями; или система должна оставаться независимой от конкретных классов создаваемых продуктов (аналогично Factory Method, но через копирование, а не инстанцирование).

Важный технический нюанс — разница между поверхностным (shallow) и глубоким (deep) клонированием:
- **Shallow copy** копирует только верхний уровень объекта; вложенные объекты/массивы остаются общими ссылками между оригиналом и копией — изменение вложенного объекта в копии повлияет и на оригинал.
- **Deep copy** рекурсивно копирует все вложенные структуры, делая копию полностью независимой от оригинала.

В JavaScript/TypeScript Prototype часто реализуется через spread-оператор (`{...obj}` — shallow), `structuredClone()` или `JSON.parse(JSON.stringify(obj))` (deep, но с ограничениями — теряются функции, `Date` превращается в строку, не поддерживаются циклические ссылки), либо через явный метод `clone()` с ручной логикой копирования вложенных полей.

В автоматизации тестирования Prototype полезен для генерации тестовых данных на основе "базового" объекта-образца: берём готовый валидный объект (например, "стандартный пользователь") и клонируем его, изменяя 1-2 поля под конкретный тест-кейс, вместо повторного описания всех полей заново.

## Пример / когда применяется

```ts
interface Prototype<T> {
  clone(): T;
}

class TestUser implements Prototype<TestUser> {
  constructor(
    public name: string,
    public email: string,
    public roles: string[],
    public address: { city: string; zip: string }
  ) {}

  // Глубокое клонирование — вложенные структуры копируются, а не разделяются
  clone(): TestUser {
    return new TestUser(this.name, this.email, [...this.roles], {
      ...this.address,
    });
  }
}

const baseUser = new TestUser('John Doe', 'john@example.com', ['user'], {
  city: 'Moscow',
  zip: '123456',
});

// Клонируем базового пользователя и меняем только нужные поля под тест-кейс
const adminUser = baseUser.clone();
adminUser.email = 'admin@example.com';
adminUser.roles.push('admin');

// baseUser.roles остаётся ['user'] — независимая копия благодаря deep clone
console.log(baseUser.roles); // ['user']
console.log(adminUser.roles); // ['user', 'admin']
```

## На что смотрит интервьюер

Ключевой вопрос — знает ли кандидат разницу между shallow и deep copy и умеет ли объяснить, к каким багам приводит поверхностное копирование (мутация вложенного объекта "протекает" в оригинал). Могут спросить про способы клонирования в JS/TS (`structuredClone`, spread, `JSON.parse/stringify` и их ограничения — потеря функций, `undefined`, `Date`, циклических ссылок). Также полезно уметь отличить Prototype от Builder: Prototype копирует существующий объект, Builder собирает новый пошагово с нуля.
