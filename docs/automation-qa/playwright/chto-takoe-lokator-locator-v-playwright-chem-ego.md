# Что такое локатор (Locator) в Playwright? Чем его поведение отличается от `WebElement` в Selenium (ленивость, auto-waiting)?

**Коротко:** `Locator` в Playwright — это ленивое, "отложенное" описание способа найти элемент, которое каждый раз заново ищет элемент в DOM непосредственно перед действием, тогда как `WebElement` в Selenium — это "живая" ссылка на конкретный найденный DOM-узел в момент вызова `findElement`, которая может стать невалидной (`StaleElementReferenceException`) при изменении DOM.

## Развёрнутый ответ

**Ленивость (lazy evaluation).** Когда в Playwright вызывается `page.locator('button.submit')`, поиск элемента в DOM НЕ происходит немедленно — создаётся объект-описание "как найти элемент", хранящий селектор. Реальный поиск в DOM выполняется только в момент вызова действия (`.click()`, `.fill()` и т.д.) или проверки (`expect(locator).toBeVisible()`). Это фундаментально отличается от Selenium, где `driver.findElement(By.cssSelector(...))` немедленно ищет элемент и возвращает конкретную ссылку (`WebElement`) на найденный узел DOM в текущем его состоянии.

**Последствия ленивости для устойчивости к перерисовкам DOM.** Поскольку `Locator` каждый раз заново ищет элемент непосредственно перед действием, он автоматически "переживает" перерисовку DOM (например, React/Vue пересоздал узел после ре-рендера). В Selenium же сохранённая ссылка `WebElement` привязана к конкретному узлу DOM в момент поиска; если этот узел был удалён и заменён новым (даже визуально идентичным), обращение к старой ссылке выбрасывает `StaleElementReferenceException`, и приходится заново вызывать `findElement`.

**Auto-waiting.** Locator в Playwright перед каждым действием выполняет серию "actionability checks": элемент присоединён к DOM, видим, имеет ненулевой размер, стабилен (не в процессе анимации), не перекрыт другими элементами, включён (`enabled`) и, для полей ввода, доступен для редактирования. Playwright автоматически повторяет проверки с поллингом до истечения таймаута (по умолчанию 30 секунд), прежде чем выбросить ошибку. В Selenium аналогичные проверки (`ExpectedConditions.elementToBeClickable` и т.п.) нужно явно описывать через `WebDriverWait`, а базовые методы (`.click()`) не содержат встроенного ожидания готовности — если элемент не готов, тест либо упадёт сразу, либо (при `implicitlyWait`) будет ждать вслепую без проверки конкретных условий готовности.

**Композиция и переиспользование.** Locator можно комбинировать методами `.filter()`, `.first()`, `.last()`, `.nth()`, `.and()`, `.or()`, создавая цепочки без немедленного обращения к DOM — что удобно для построения сложных, но читаемых селекторов (например, "кнопка внутри карточки товара с определённым текстом"). `WebElement` такой композиции не поддерживает — приходится комбинировать сами CSS/XPath строки заранее.

## Пример / когда применяется

Сравнение поведения при перерисовке DOM:

```javascript
// Playwright — Locator переживает перерисовку DOM
const button = page.locator('button.submit'); // не ищет элемент сейчас
await someActionThatRerendersDOM();
await button.click(); // ищет элемент заново прямо сейчас и кликает

// Selenium — WebElement может стать "stale"
WebElement button = driver.findElement(By.cssSelector("button.submit")); // ищет сейчас
someActionThatRerendersDOM();
button.click(); // может выбросить StaleElementReferenceException
```

Композиция локаторов в Playwright:

```javascript
const productCard = page.locator('.product-card').filter({ hasText: 'iPhone' });
await productCard.getByRole('button', { name: 'В корзину' }).click();
```

## На что смотрит интервьюер

- Чёткое понимание разницы "ленивое описание" (Locator) vs "немедленная живая ссылка" (WebElement).
- Знание термина `StaleElementReferenceException` и почему Locator от него не страдает.
- Способность перечислить конкретные actionability checks, которые проверяет Playwright перед действием.
- Follow-up: "Что произойдёт, если элемент перекрыт другим элементом при вызове `.click()`?" (Playwright будет ждать и ретраить проверку до таймаута, затем упадёт с понятной ошибкой).
- Красный флаг — если кандидат говорит, что Locator — это "просто как WebElement, только по-другому называется", не видя разницы в момент выполнения поиска.
