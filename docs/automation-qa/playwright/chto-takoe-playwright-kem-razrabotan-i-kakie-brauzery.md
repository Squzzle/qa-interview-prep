# Что такое Playwright? Кем разработан и какие браузеры поддерживает?

**Коротко:** Playwright — это open-source фреймворк для end-to-end автоматизации браузеров, разработанный Microsoft (командой, ранее создавшей Puppeteer). Он поддерживает Chromium, Firefox и WebKit через единый API.

## Развёрнутый ответ

Playwright — это библиотека для автоматизации тестирования веб-приложений, разработанная компанией Microsoft и впервые выпущенная в 2020 году. Ключевую роль в её создании сыграли инженеры, которые ранее работали над Puppeteer в Google, поэтому архитектурные идеи (управление браузером через протокол, а не через промежуточный драйвер) во многом схожи, но Playwright пошёл дальше и добавил кроссбраузерность.

Playwright поддерживает три основных браузерных движка:

- **Chromium** — движок, лежащий в основе Google Chrome и Microsoft Edge;
- **Firefox** — движок Gecko от Mozilla;
- **WebKit** — движок, используемый в Safari (важно для тестирования поведения на macOS/iOS-подобных браузерах).

Для каждого движка Playwright использует собственные пропатченные билды браузеров, которые скачиваются и устанавливаются локально (`playwright install`). Это позволяет получать предсказуемое поведение независимо от версии браузера, установленного в операционной системе, и гарантирует совместимость новых версий Playwright с новыми возможностями протокола.

Playwright предоставляет официальные биндинги (API) для нескольких языков программирования: JavaScript/TypeScript (нативный, "родной" язык проекта), Python, Java и C#/.NET. Все они реализуют единый набор возможностей: работу с несколькими вкладками и контекстами, эмуляцию мобильных устройств, перехват сетевых запросов, работу с iframes, shadow DOM, а также встроенные механизмы ожидания (auto-waiting).

Помимо самой библиотеки, экосистема Playwright включает Playwright Test — встроенный тест-раннер с параллельным выполнением, ретраями, трейсингом и генерацией отчётов, а также Playwright Inspector, Codegen (генератор кода по действиям пользователя) и Trace Viewer для отладки упавших тестов.

## Пример / когда применяется

Типичный сценарий использования — кроссбраузерное E2E-тестирование веб-приложения одним и тем же набором тестов:

```javascript
const { chromium, firefox, webkit } = require('playwright');

for (const browserType of [chromium, firefox, webkit]) {
  const browser = await browserType.launch();
  const page = await browser.newPage();
  await page.goto('https://example.com');
  console.log(await page.title());
  await browser.close();
}
```

Такой подход позволяет прогонять один и тот же сценарий на всех трёх движках без изменения кода теста — достаточно сменить импортируемый браузер.

## На что смотрит интервьюер

- Знает ли кандидат, что Playwright разработан Microsoft, а не Google (частая путаница с Puppeteer).
- Может ли назвать все три поддерживаемых движка (Chromium, Firefox, WebKit), а не просто перечислить браузеры по названиям.
- Понимает ли, что Playwright использует собственные скачиваемые билды браузеров, а не системные версии.
- Знает ли про поддержку нескольких языков (JS/TS, Python, Java, C#).
- Красный флаг — если кандидат путает Playwright с Selenium WebDriver по архитектуре управления браузером на этом же вопросе (это более глубокий вопрос, но базовое понимание отличия должно быть).
