# Как обрабатывать диалоговые окна (alert, confirm) в Playwright?

**Коротко:** Диалоговые окна обрабатываются через событие `page.on('dialog', handler)`, внутри которого вызывается `dialog.accept()` (подтвердить, опционально с текстом для `prompt`) или `dialog.dismiss()` (отклонить); без обработчика Playwright автоматически закрывает диалоги.

## Развёрнутый ответ

Браузерные нативные диалоговые окна (`alert`, `confirm`, `prompt`, а также `beforeunload`) блокируют выполнение JavaScript на странице до реакции пользователя. Так как Playwright работает асинхронно через протокол CDP, он перехватывает событие появления диалога через `page.on('dialog', async dialog => {...})` до того, как оно фактически заблокирует страницу.

Внутри обработчика доступны:

- `dialog.type()` — тип диалога (`alert`, `confirm`, `prompt`, `beforeunload`).
- `dialog.message()` — текст сообщения диалога.
- `dialog.accept(promptText?)` — принять диалог; для `prompt` можно передать текст, который будет "введён" в поле.
- `dialog.dismiss()` — отклонить/закрыть диалог (аналог нажатия "Отмена" или крестика).

Важная особенность: обработчик диалога нужно зарегистрировать **до** действия, которое вызывает диалог (например, до клика по кнопке), так как событие срабатывает синхронно в момент появления диалога. Если обработчик не зарегистрирован явно, по умолчанию Playwright автоматически отклоняет (`dismiss`) диалог, чтобы тест не завис в ожидании.

Для одноразового сценария удобно использовать `page.once('dialog', handler)`, чтобы обработчик сработал только для ближайшего диалога и автоматически снялся после этого, не мешая последующим диалогам в тесте с другой логикой обработки.

## Пример / когда применяется

```javascript
// Подтверждение confirm-диалога
page.on('dialog', async (dialog) => {
  console.log(dialog.message());
  await dialog.accept();
});
await page.click('#delete-button');

// Ввод текста в prompt и подтверждение
page.once('dialog', async (dialog) => {
  await dialog.accept('Тестовое значение');
});
await page.click('#prompt-trigger');

// Отклонение диалога
page.once('dialog', (dialog) => dialog.dismiss());
await page.click('#confirm-trigger');
```

Применяется при тестировании функциональности удаления с подтверждением (`confirm`), форм с валидацией через `alert`, а также сценариев ухода со страницы с несохранёнными данными (`beforeunload`).

## На что смотрит интервьюер

Интервьюер проверяет знание, что обработчик `page.on('dialog', ...)` должен быть зарегистрирован до действия, вызывающего диалог, иначе возможен race condition или зависание теста. Частая ошибка — не учитывать, что без явного обработчика Playwright сам отклоняет диалог по умолчанию, что может маскировать баги (например, тест "проходит", хотя должен был провалиться на неожиданном alert). Также могут спросить про разницу между `once` и `on` для управления количеством обрабатываемых диалогов и про доступ к `dialog.message()` для проверки текста сообщения в ассерциях.
