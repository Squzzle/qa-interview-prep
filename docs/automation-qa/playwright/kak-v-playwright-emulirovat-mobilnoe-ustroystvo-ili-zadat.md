# Как в Playwright эмулировать мобильное устройство или задать геолокацию?

**Коротко:** Playwright предоставляет встроенный каталог `devices` с готовыми пресетами мобильных устройств (viewport, user-agent, touch-поддержка) и опцию `geolocation` при создании контекста для задания координат, при условии выданного разрешения `permissions: ['geolocation']`.

## Развёрнутый ответ

Для эмуляции мобильного устройства Playwright поставляется с объектом `devices`, содержащим сотни готовых профилей (например, `'iPhone 13'`, `'Pixel 5'`, `'iPad Pro'`). Каждый профиль включает набор параметров: размер viewport, `deviceScaleFactor`, строку `userAgent`, флаги `isMobile` и `hasTouch`. Профиль передаётся при создании контекста через оператор расширения (`...devices['iPhone 13']`), что задаёт сразу все нужные параметры эмуляции.

При необходимости эти параметры можно переопределять вручную (например, изменить только `viewport`, оставив остальные свойства устройства), а также комбинировать эмуляцию устройства с другими опциями контекста — локалью (`locale`), часовым поясом (`timezoneId`), цветовой схемой (`colorScheme`).

Для эмуляции геолокации используется опция `geolocation: { latitude, longitude, accuracy }` при создании контекста. Поскольку геолокация в браузерах защищена системой разрешений, необходимо явно выдать разрешение через `permissions: ['geolocation']` в том же вызове `newContext()` — без этого сайт будет получать отказ в доступе к API геолокации. Изменить координаты можно и после создания контекста через `context.setGeolocation({ latitude, longitude })`, что позволяет симулировать перемещение пользователя во время теста.

## Пример / когда применяется

```javascript
const { chromium, devices } = require('@playwright/test');

const browser = await chromium.launch();
const context = await browser.newContext({
  ...devices['iPhone 13'],
  locale: 'ru-RU',
  geolocation: { latitude: 55.7558, longitude: 37.6173, accuracy: 100 },
  permissions: ['geolocation'],
});

const page = await context.newPage();
await page.goto('https://example.com');

// Изменить координаты во время теста (симуляция перемещения)
await context.setGeolocation({ latitude: 59.9343, longitude: 30.3351 });
```

Такой подход применяется при тестировании адаптивной вёрстки на мобильных разрешениях, специфичного мобильного поведения (touch-жесты, мобильное меню), а также функциональности, зависящей от местоположения — определение ближайшего магазина, региональные цены, локализация контента.

## На что смотрит интервьюер

Проверяется знание готового каталога `devices` и понимание, что за ним стоит просто набор параметров контекста, а не отдельный "режим эмуляции". Частая ошибка — забыть выдать `permissions: ['geolocation']`, из-за чего браузер возвращает ошибку доступа даже при заданных координатах. Также могут спросить про `context.setGeolocation()` для динамического изменения координат в рамках одного теста и про комбинирование эмуляции устройства с другими параметрами контекста (locale, timezone, colorScheme).
