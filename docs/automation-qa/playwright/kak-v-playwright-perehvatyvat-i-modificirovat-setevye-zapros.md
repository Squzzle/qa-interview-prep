# Как в Playwright перехватывать и модифицировать сетевые запросы (`page.route()`)?

**Коротко:** Метод `page.route()` позволяет перехватывать сетевые запросы по маске URL и решать, что с ними делать: пропустить дальше, изменить, оборвать или подменить ответ через объект `route`.

## Развёрнутый ответ

`page.route(url, handler)` регистрирует обработчик, который вызывается при каждом запросе, соответствующем шаблону `url` (строка, glob-паттерн или регулярное выражение). Внутри обработчика доступен объект `route`, представляющий перехваченный запрос, и объект `request` с деталями запроса (метод, заголовки, тело, ресурс-тип).

Основные действия с перехваченным запросом:

- `route.continue(overrides)` — пропустить запрос дальше, при необходимости изменив метод, заголовки, `postData` или URL.
- `route.fulfill(response)` — вернуть браузеру собственный ответ вместо реального (статус, заголовки, тело), не обращаясь к серверу.
- `route.abort(errorCode)` — прервать запрос (например, эмулировать отсутствие сети или ошибку CORS).
- `route.request()` — получить объект запроса для анализа (URL, заголовки, метод, тело POST).

Перехват можно настраивать на уровне страницы (`page.route`) или контекста браузера (`context.route`), последнее удобно, когда нужно применить правило ко всем страницам сразу, включая новые вкладки. Приоритет отдаётся последнему зарегистрированному обработчику, но можно вызвать `route.fallback()`, чтобы передать обработку следующему по цепочке.

Модификация запроса типично применяется для: подмены заголовков авторизации, изменения тела запроса, редиректа на другой хост (например, тестовое окружение вместо продакшена), симуляции медленной сети или ошибок сервера, блокировки лишних ресурсов (аналитика, реклама, шрифты) для ускорения тестов.

## Пример / когда применяется

```javascript
// Изменение заголовка запроса перед отправкой
await page.route('**/api/users', async (route) => {
  const headers = {
    ...route.request().headers(),
    'x-custom-header': 'test-value',
  };
  await route.continue({ headers });
});

// Блокировка загрузки изображений для ускорения теста
await page.route('**/*.{png,jpg,jpeg}', (route) => route.abort());

// Модификация тела POST-запроса
await page.route('**/api/order', async (route) => {
  const postData = JSON.parse(route.request().postData());
  postData.quantity = 1;
  await route.continue({ postData: JSON.stringify(postData) });
});
```

Такой подход используется, когда нужно проверить поведение UI при разных ответах сервера без изменения самого бэкенда, а также для стабилизации тестов, независимых от реальных внешних сервисов.

## На что смотрит интервьюер

Интервьюер проверяет понимание разницы между `route.continue()`, `route.fulfill()` и `route.abort()`, а также знание про уровень регистрации перехвата (page vs context). Частая ошибка — забыть вызвать один из этих методов внутри обработчика, из-за чего запрос "зависает". Также спрашивают про порядок обработчиков и `route.fallback()`, про использование wildcard-паттернов и про отличие перехвата от полноценного мокирования ответа.
