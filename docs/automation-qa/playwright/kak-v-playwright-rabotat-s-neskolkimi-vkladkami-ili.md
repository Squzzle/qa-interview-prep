# Как в Playwright работать с несколькими вкладками или страницами?

**Коротко:** Каждая вкладка в Playwright представлена отдельным объектом `Page`, принадлежащим общему `BrowserContext`; новые вкладки отслеживаются через событие `context.on('page', ...)` или через ожидание события при действии, открывающем вкладку.

## Развёрнутый ответ

В модели Playwright браузерный контекст (`BrowserContext`) может содержать несколько страниц (`Page`), которые соответствуют вкладкам. Все страницы в одном контексте разделяют cookies, localStorage (в рамках контекста) и сессию, но представляют собой независимые объекты DOM с собственным жизненным циклом.

Когда новая вкладка открывается в результате действия пользователя (клик по ссылке с `target="_blank"`, `window.open()`), Playwright не переключается на неё автоматически — нужно явно получить новый объект `Page`. Для этого используется один из подходов:

- Подписка на событие контекста: `context.on('page', page => { ... })` — сработает при появлении любой новой страницы.
- Ожидание события через `context.waitForEvent('page')` в паре с действием, которое инициирует открытие вкладки (обычно оборачивается в `Promise.all`, чтобы не пропустить событие).

После получения объекта новой страницы с ней можно работать так же, как и с исходной: вызывать `waitForLoadState()`, чтобы дождаться полной загрузки, выполнять локаторы, клики и проверки. Важно понимать, что все открытые страницы контекста доступны через `context.pages()`, что позволяет перебирать их и выбирать нужную по URL или заголовку.

Если нужно протестировать сценарий с двумя независимыми пользователями одновременно (не разделяющими сессию), стоит создавать отдельные `BrowserContext` в рамках одного `Browser`, а не просто новые вкладки в одном контексте.

## Пример / когда применяется

```javascript
// Ожидание открытия новой вкладки при клике по ссылке
const [newPage] = await Promise.all([
  context.waitForEvent('page'),
  page.click('a[target="_blank"]'),
]);

await newPage.waitForLoadState();
await expect(newPage).toHaveTitle(/Ожидаемый заголовок/);

// Перебор всех открытых страниц контекста
for (const p of context.pages()) {
  console.log(await p.title());
}
```

Такой подход применяется при тестировании сценариев с внешними ссылками, оплатой через сторонний сервис в новом окне, OAuth-авторизацией через popup, а также при сравнении поведения двух пользователей в одном или разных контекстах.

## На что смотрит интервьюер

Интервьюер проверяет знание разницы между `Page`, `BrowserContext` и `Browser`, а также умение корректно перехватывать открытие новой вкладки через `waitForEvent('page')` вместо простого перебора `context.pages()` сразу после клика (частая ошибка — race condition, когда страница ещё не появилась в списке). Также важно понимание, когда использовать несколько вкладок в одном контексте (для связанных окон одного пользователя), а когда — несколько контекстов (для изоляции сессий разных пользователей).
