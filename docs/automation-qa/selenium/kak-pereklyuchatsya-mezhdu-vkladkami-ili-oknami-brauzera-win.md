# Как переключаться между вкладками или окнами браузера (`WindowHandle`)?

**Коротко:** Каждое окно/вкладка имеет уникальный идентификатор `WindowHandle`; переключение выполняется через `driver.switchTo().window(handle)`, а список всех открытых окон получают методом `driver.getWindowHandles()`.

## Развёрнутый ответ

WebDriver управляет только одним "активным" окном/вкладкой в конкретный момент времени, даже если браузер открыл несколько вкладок (например, после клика по ссылке с `target="_blank"`). Для переключения фокуса используется механизм window handles — строковых идентификаторов, уникальных для каждого окна в рамках сессии драйвера.

Основные методы:
- `driver.getWindowHandle()` — возвращает handle текущего активного окна.
- `driver.getWindowHandles()` — возвращает `Set<String>` со всеми handle всех открытых окон/вкладок в сессии.
- `driver.switchTo().window(handle)` — переключает фокус драйвера на окно с указанным handle.
- `driver.close()` — закрывает текущее активное окно (не всю сессию драйвера).
- `driver.quit()` — закрывает все окна и полностью завершает сессию драйвера.

Поскольку `Set<String>` не гарантирует порядок, типичный паттерн — сохранить handle исходного окна до открытия нового, затем итерироваться по всем handles и переключаться на тот, что не совпадает с исходным:

```java
String originalWindow = driver.getWindowHandle();
driver.findElement(By.linkText("Open in new tab")).click();

// ждём появления нового окна
wait.until(d -> d.getWindowHandles().size() > 1);

for (String handle : driver.getWindowHandles()) {
    if (!handle.equals(originalWindow)) {
        driver.switchTo().window(handle);
        break;
    }
}
```

Важные нюансы:
- После работы с новым окном рекомендуется закрыть его (`driver.close()`) и явно переключиться обратно на исходное (`driver.switchTo().window(originalWindow)`), иначе последующие команды будут выполняться в контексте уже закрытого или неактуального окна и вызовут `NoSuchWindowException`.
- Открытие новой вкладки может происходить не мгновенно, поэтому перед перебором handles стоит дождаться увеличения их количества через `WebDriverWait`.
- Начиная с Selenium 4 появился более удобный `NewWindow` API (`driver.switchTo().newWindow(WindowType.TAB)`) для явного открытия новой вкладки/окна средствами самого Selenium.

## Пример / когда применяется

Классический сценарий — тестирование ссылок, открывающихся в новой вкладке (партнёрские ссылки, документы, сторонние сервисы оплаты), где нужно проверить содержимое новой вкладки и вернуться к исходному сценарию оформления заказа.

## На что смотрит интервьюер

Проверяет знание корректного паттерна переключения (сохранение исходного handle, перебор через `getWindowHandles`), а не просто вызов `switchTo().window(...)` наугад. Важно различие между `close()` и `quit()`. Follow-up: «что произойдёт, если забыть переключиться обратно на исходное окно» и «как дождаться открытия нового окна, если оно появляется асинхронно». Красный флаг — использование `Thread.sleep()` вместо ожидания увеличения количества handles или неверное использование `quit()` вместо `close()` для закрытия одной вкладки.
