# Как работать с элементами внутри iframe?

**Коротко:** Нужно явно переключить контекст драйвера на нужный `iframe` через `driver.switchTo().frame(...)`, после чего искать элементы внутри него, а по завершении вернуться в основной документ через `driver.switchTo().defaultContent()`.

## Развёрнутый ответ

`iframe` создаёт вложенный, изолированный документ (browsing context) внутри страницы. Selenium по умолчанию работает в контексте главного документа и не видит элементы внутри `iframe`, даже если они визуально находятся на той же странице — попытка найти такой элемент вернёт `NoSuchElementException`.

Для доступа к содержимому фрейма используется `driver.switchTo().frame(...)`, который принимает один из вариантов:
- индекс фрейма (`int`, порядковый номер на странице, начиная с 0);
- имя или id фрейма (`String`), заданные атрибутами `name`/`id` в HTML;
- сам `WebElement` фрейма, найденный предварительно через обычный локатор (`driver.findElement(By.cssSelector("iframe#payment"))`).

После переключения все последующие `findElement`/`findElements` выполняются в контексте документа внутри фрейма. Чтобы вернуться к родительскому документу — используется `driver.switchTo().defaultContent()` (переход к самому верхнему документу) или `driver.switchTo().parentFrame()` (переход на один уровень вверх, актуально при вложенных фреймах).

Важные нюансы:
- Если на странице несколько вложенных фреймов (frame внутри frame), нужно последовательно переключаться на каждый уровень.
- Использование `WebElement`-варианта переключения предпочтительнее, так как он не ломается при отсутствии `id`/`name` и более устойчив, чем числовой индекс (индекс может измениться при добавлении новых фреймов на страницу).
- Перед переключением полезно дождаться, что фрейм и его содержимое загрузились, через `ExpectedConditions.frameToBeAvailableAndSwitchToIt(...)`, который сам выполняет ожидание и переключение одним вызовом.
- Некоторые современные фреймворки автоматизации (Selenide, Playwright) упрощают работу с фреймами, скрывая ручное переключение за высокоуровневым API, но в чистом Selenium переключение остаётся явной обязательной операцией.

## Пример / когда применяется

```java
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));

// Переключение на фрейм с ожиданием
wait.until(ExpectedConditions.frameToBeAvailableAndSwitchToIt(By.id("payment-frame")));

driver.findElement(By.id("cardNumber")).sendKeys("4111111111111111");

// Возврат в основной документ
driver.switchTo().defaultContent();
driver.findElement(By.id("submitOrder")).click();
```

Типичный сценарий — виджеты оплаты (Stripe, PayPal) или встроенные редакторы, которые изолируются в отдельный iframe по соображениям безопасности.

## На что смотрит интервьюер

Проверяет знание конкретных методов (`frame`, `defaultContent`, `parentFrame`) и понимание, зачем вообще нужно переключение (изоляция browsing context). Хороший ответ упоминает вложенные фреймы и предпочтение `WebElement`-варианта над индексом. Follow-up: «что произойдёт, если попытаться найти элемент внутри фрейма без переключения» и «как найти элемент, который сам находится внутри iframe, при поиске по всей странице» (стандартный `findElement` не проникает внутрь iframe — это ограничение Selenium, связанное с изоляцией DOM). Красный флаг — попытка искать элемент фрейма напрямую без `switchTo().frame()`.
