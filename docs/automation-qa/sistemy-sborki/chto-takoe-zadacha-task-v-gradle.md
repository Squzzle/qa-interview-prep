# Что такое задача (task) в Gradle?

**Коротко:** Task — это базовая единица работы в Gradle, представляющая собой одно конкретное действие сборки (компиляция, запуск тестов, копирование файлов), которое можно определить, настроить и связать зависимостями с другими задачами.

## Развёрнутый ответ

В отличие от Maven, где единицей работы является goal плагина, привязанный к фазе жизненного цикла, в Gradle всё построено вокруг задач (tasks) и их графа зависимостей (Directed Acyclic Graph, DAG). Плагины Gradle (например, плагин `java`) регистрируют стандартный набор задач: `compileJava`, `test`, `jar`, `build`, `clean` и другие.

Каждая задача имеет:

- **имя** — уникальный идентификатор в рамках проекта;
- **действия (actions)** — что конкретно выполняется при запуске задачи;
- **входы и выходы (inputs/outputs)** — Gradle отслеживает их для кеширования: если входные данные задачи не изменились с прошлого запуска, задача помечается как `UP-TO-DATE` и не выполняется повторно, что существенно ускоряет повторные сборки;
- **зависимости от других задач** — через `dependsOn`, что формирует граф выполнения. Например, задача `test` зависит от `compileTestJava`, а та — от `compileJava`.

Gradle позволяет как использовать готовые задачи из плагинов, так и определять собственные — кастомные задачи, написанные прямо в build.gradle, что даёт большую гибкость по сравнению с Maven, где кастомная логика обычно требует написания отдельного плагина.

Ключевое отличие от Maven-фаз: в Gradle нет жёстко фиксированного линейного жизненного цикла — граф задач гибкий и строится динамически на основе зависимостей между конкретными задачами, что и объясняет более высокую скорость сборки (за счёт инкрементальности и параллельного выполнения независимых задач) и гибкость конфигурации.

## Пример / когда применяется

Пример определения кастомной задачи в `build.gradle`:

```groovy
task smokeTest(type: Test) {
    description 'Запускает только smoke-тесты'
    useTestNG() {
        suites 'testng-smoke.xml'
    }
    dependsOn compileTestJava
}
```

Запуск конкретной задачи из командной строки:

```bash
gradle smokeTest
gradle test --tests "*ApiTests"
gradle clean build
```

Команда `gradle clean build` выполнит задачу `clean`, а затем `build`, которая в свою очередь потянет за собой весь граф зависимых задач: компиляцию, тесты, упаковку.

## На что смотрит интервьюер

Интервьюер оценивает понимание разницы между task-ориентированной моделью Gradle и phase-ориентированной моделью Maven. Часто спрашивают про механизм `UP-TO-DATE` и инкрементальные сборки — если кандидат не знает, за счёт чего Gradle быстрее Maven, это заметный пробел. Также хороший ответ включает пример написания кастомной задачи и понимание `dependsOn` для построения графа выполнения.
