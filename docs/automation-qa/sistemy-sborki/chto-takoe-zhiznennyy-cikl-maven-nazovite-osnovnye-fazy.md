# Что такое жизненный цикл Maven? Назовите основные фазы (clean, compile, test, package, install, deploy).

**Коротко:** Жизненный цикл Maven — это упорядоченная последовательность фаз сборки, каждая из которых выполняет определённый шаг; вызов фазы автоматически выполняет все предыдущие фазы по порядку.

## Развёрнутый ответ

В Maven есть три встроенных жизненных цикла: `default` (основной, отвечает за сборку и деплой проекта), `clean` (удаление артефактов предыдущей сборки) и `site` (генерация документации проекта). Чаще всего на собеседовании речь идёт про `default`-цикл.

Основные фазы default-цикла в порядке выполнения:

1. **validate** — проверка корректности структуры проекта и наличия всей необходимой информации.
2. **compile** — компиляция исходного кода проекта (из `src/main/java`).
3. **test** — запуск unit-тестов с помощью Surefire-плагина (тесты из `src/test/java`); при этом код ещё не упаковывается.
4. **package** — упаковка скомпилированного кода в артефакт нужного формата (jar, war).
5. **verify** — запуск дополнительных проверок, в том числе интеграционных тестов (Failsafe-плагин), для подтверждения качества артефакта.
6. **install** — установка артефакта в локальный репозиторий (`~/.m2/repository`), чтобы его могли использовать другие локальные проекты как зависимость.
7. **deploy** — публикация артефакта в удалённый репозиторий (Nexus, Artifactory) для использования другими командами/проектами.

Отдельно стоит фаза `clean` из одноимённого жизненного цикла — она удаляет директорию `target`, где хранятся результаты предыдущей сборки. Именно поэтому часто вызывают команду `mvn clean install` — сначала очищаем старые артефакты, затем последовательно выполняем все фазы default-цикла вплоть до install.

Ключевая особенность: фазы выполняются последовательно и кумулятивно. Если вызвать `mvn package`, Maven сначала выполнит validate, compile, test и только потом package. Это гарантирует, что нельзя случайно упаковать проект без прохождения тестов (если явно не указано пропустить их флагом `-DskipTests`).

## Пример / когда применяется

Типичные команды в CI-пайплайне:

```bash
mvn clean test           # очистка + компиляция + запуск unit-тестов
mvn clean verify         # + интеграционные тесты
mvn clean install        # + установка в локальный репозиторий
mvn clean deploy         # + публикация в удалённый репозиторий
```

Например, на этапе Pull Request в CI обычно достаточно `mvn clean verify`, а на этапе релиза выполняется `mvn clean deploy`, чтобы опубликовать библиотеку с общими степами автотестов для других команд.

## На что смотрит интервьюер

Проверяется понимание того, что вызов поздней фазы автоматически прогоняет все предыдущие. Частая ошибка — путать `install` и `deploy` (install — локально, deploy — в удалённый репозиторий) или не знать, что тесты запускаются на фазе `test`, а не `package`. Хороший кандидат также упомянет `-DskipTests` vs `-Dmaven.test.skip=true` (второй флаг вообще не компилирует тесты) и разницу между Surefire (unit) и Failsafe (integration, фаза verify).
