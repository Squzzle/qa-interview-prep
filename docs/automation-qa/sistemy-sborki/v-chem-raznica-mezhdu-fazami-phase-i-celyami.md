# В чем разница между фазами (phase) и целями (goal) в Maven?

**Коротко:** Фаза (phase) — это этап жизненного цикла сборки (например, compile или test), а цель (goal) — это конкретное действие, выполняемое плагином (например, `compiler:compile`); одна фаза может запускать одну или несколько целей разных плагинов.

## Развёрнутый ответ

Жизненный цикл Maven состоит из упорядоченного набора фаз (validate, compile, test, package, verify, install, deploy и т.д.). Фаза сама по себе не выполняет никакой работы — она является лишь именованным этапом, к которому привязываются (bind) конкретные цели плагинов.

Цель (goal) — это единица работы, реализованная конкретным плагином, записывается в формате `plugin:goal`, например `compiler:compile`, `surefire:test`, `jar:jar`. Каждый плагин может иметь несколько целей, каждая из которых выполняет своё конкретное действие.

Связь между фазами и целями называется binding (привязкой). Например, по умолчанию:

- фаза `compile` привязана к цели `compiler:compile`;
- фаза `test` привязана к цели `surefire:test`;
- фаза `package` привязана к целям `jar:jar` или `war:war` в зависимости от типа packaging.

Когда вызывается `mvn test`, Maven выполняет все фазы жизненного цикла по порядку вплоть до `test` включительно, и на каждой фазе запускает все привязанные к ней цели (как встроенные по умолчанию для данного типа packaging, так и добавленные явно в конфигурации плагинов в pom.xml).

Важное отличие в использовании: можно вызвать конкретную фазу (`mvn test`) — тогда выполнятся все предыдущие фазы и их цели; а можно вызвать конкретную цель напрямую, без прохождения всего жизненного цикла (`mvn dependency:tree` или `mvn surefire:test`) — в этом случае выполнится только эта одна цель, без привязки к фазам жизненного цикла (если явно не указано иное).

## Пример / когда применяется

```bash
mvn compile              # выполняет фазы validate, compile
                          # (внутри compile вызывается цель compiler:compile)

mvn dependency:tree       # выполняет ТОЛЬКО цель dependency:tree,
                          # не проходя по фазам жизненного цикла

mvn org.apache.maven.plugins:maven-surefire-plugin:3.1.2:test
                          # прямой вызов конкретной цели конкретной версии плагина,
                          # полезно для отладки без пересборки всего проекта
```

Практический пример из проекта автотестов: если нужно быстро посмотреть дерево зависимостей без полной пересборки, вызывают `mvn dependency:tree` — это отдельная цель, не связанная с фазами compile/test/package.

## На что смотрит интервьюер

Интервьюер проверяет понимание того, что фаза — это лишь "точка привязки", а реальную работу выполняют цели плагинов. Частая ошибка — считать, что фаза и цель это одно и то же, или не понимать, что вызов голой цели (`plugin:goal`) не проходит через весь жизненный цикл. Хорошим ответом также будет упоминание, что один и тот же плагин может привязывать разные свои цели к разным фазам через конфигурацию `<executions>` в pom.xml.
