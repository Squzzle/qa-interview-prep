# Что такое ООП? Назовите его принципы с примерами?

**Коротко:** ООП (объектно-ориентированное программирование) — парадигма, где программа строится как набор взаимодействующих объектов, каждый из которых имеет состояние (поля) и поведение (методы). Базируется на четырёх принципах: инкапсуляция, наследование, полиморфизм и абстракция.

## Развёрнутый ответ

Объектно-ориентированное программирование — это подход к проектированию и написанию кода, при котором вместо набора функций и глобальных данных используются объекты — экземпляры классов, объединяющие данные и логику работы с ними. Такой подход упрощает повторное использование кода, тестирование и поддержку больших систем.

Четыре базовых принципа ООП:

1. **Инкапсуляция** — сокрытие внутренней реализации объекта и предоставление доступа к ней только через публичный интерфейс (методы). Поля обычно делают `private`, а доступ к ним предоставляют через геттеры и сеттеры. Это защищает объект от некорректного использования и позволяет менять внутреннюю реализацию, не ломая внешний код.

2. **Наследование** — механизм, позволяющий одному классу (наследнику, `subclass`) перенимать поля и методы другого класса (родителя, `superclass`), расширяя или переопределяя его поведение. Используется для повторного использования кода и построения иерархий типов.

3. **Полиморфизм** — способность объектов разных классов обрабатывать одно и то же сообщение (вызов метода) по-разному. Различают:
   - **Полиморфизм времени компиляции (перегрузка, overloading)** — несколько методов с одинаковым именем, но разными сигнатурами параметров.
   - **Полиморфизм времени выполнения (переопределение, overriding)** — метод дочернего класса замещает реализацию метода родителя, и конкретная реализация выбирается в рантайме на основе реального типа объекта.

4. **Абстракция** — выделение существенных характеристик объекта и скрытие несущественных деталей реализации. Реализуется через абстрактные классы и интерфейсы: пользователь работает с абстракцией («что делает объект»), не зная, «как именно» это реализовано внутри.

## Пример / когда применяется

Пример на Java, иллюстрирующий все четыре принципа:

```java
// Абстракция + основа для наследования
abstract class Animal {
    private String name; // инкапсуляция

    public Animal(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public abstract String makeSound(); // абстрактный метод
}

// Наследование
class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    public String makeSound() { // полиморфизм (overriding)
        return "Гав!";
    }
}

class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    @Override
    public String makeSound() {
        return "Мяу!";
    }
}

public class Main {
    public static void main(String[] args) {
        Animal[] animals = { new Dog("Рекс"), new Cat("Мурка") };
        for (Animal a : animals) {
            System.out.println(a.getName() + ": " + a.makeSound());
        }
    }
}
```

Такой подход применяется практически в любом реальном проекте: например, в тестовом фреймворке базовый класс `BasePage` с общими методами (`click`, `waitForElement`) наследуется конкретными Page Object классами (`LoginPage`, `MainPage`), которые переопределяют специфичное поведение.

## На что смотрит интервьюер

- Умение не просто перечислить принципы, а объяснить их своими словами и привести реальный пример из кода.
- Частая ошибка — путать перегрузку и переопределение методов, или не различать наследование и полиморфизм.
- Follow-up вопросы: «Чем абстрактный класс отличается от интерфейса?», «Приведите пример полиморфизма без наследования (через интерфейсы)», «Зачем нужна инкапсуляция, если можно сделать все поля public?».
- Красный флаг — кандидат заучил определения, но не может привести собственный пример или путается в терминах при уточняющих вопросах.
