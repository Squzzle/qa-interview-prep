# Какая иерархия Collections?

**Коротко:** в основе иерархии Java Collections Framework лежит интерфейс `Collection` с тремя основными ветвями — `List`, `Set` и `Queue`; отдельно, не наследуя `Collection`, существует иерархия `Map` для пар ключ-значение.

## Развёрнутый ответ

Java Collections Framework — это унифицированная архитектура для хранения и обработки групп объектов. Корневой интерфейс — `Iterable`, от которого наследуется `Collection`. Из `Collection` вырастают три основные ветви:

**`List`** — упорядоченная коллекция с индексированным доступом, допускающая дубликаты:
- `ArrayList` — реализация на основе динамического массива, быстрый доступ по индексу (`O(1)`), медленная вставка/удаление в середине (`O(n)`).
- `LinkedList` — реализация на основе двусвязного списка, быстрая вставка/удаление (`O(1)` при наличии ссылки на узел), медленный доступ по индексу (`O(n)`). Также реализует `Deque`.
- `Vector` — устаревший синхронизированный аналог `ArrayList` (legacy, обычно не используется в новом коде).

**`Set`** — коллекция без дубликатов (уникальность определяется через `equals()`/`hashCode()`):
- `HashSet` — на основе `HashMap`, без гарантии порядка элементов, самая быстрая операция вставки/поиска (`O(1)` в среднем).
- `LinkedHashSet` — сохраняет порядок вставки, чуть медленнее `HashSet`.
- `TreeSet` — хранит элементы в отсортированном порядке (реализует `SortedSet`/`NavigableSet`), операции `O(log n)`.

**`Queue`** — коллекция для обработки элементов по определённому порядку (FIFO по умолчанию):
- `LinkedList` — как уже упомянуто, реализует и `List`, и `Deque`.
- `PriorityQueue` — элементы извлекаются по приоритету, а не по порядку добавления.
- `Deque` (расширение `Queue`) — двусторонняя очередь, реализуется `ArrayDeque` и `LinkedList`.

**`Map`** — отдельная иерархия (не наследует `Collection`), хранит пары ключ-значение с уникальными ключами:
- `HashMap` — без гарантии порядка, `O(1)` в среднем для операций.
- `LinkedHashMap` — сохраняет порядок вставки.
- `TreeMap` — сортированная по ключам карта (реализует `SortedMap`/`NavigableMap`).
- `Hashtable` — устаревший синхронизированный аналог `HashMap` (legacy).

## Пример / когда применяется

```java
import java.util.*;

public class CollectionsExample {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>();       // допускает дубликаты, порядок вставки
        names.add("Иван");
        names.add("Иван"); // дубликат — ок

        Set<String> uniqueNames = new HashSet<>(names); // уберёт дубликаты

        Map<String, Integer> ages = new HashMap<>();
        ages.put("Иван", 30);
        ages.put("Иван", 31); // перезапишет значение по тому же ключу

        Queue<Integer> tasks = new LinkedList<>();
        tasks.offer(1);
        tasks.offer(2);
        System.out.println(tasks.poll()); // 1 — FIFO
    }
}
```

`ArrayList` чаще всего используется для хранения списка тестовых данных или результатов проверок; `HashMap` — для сопоставления ожидаемых и фактических значений по ключу; `HashSet` — для проверки отсутствия дубликатов в тестовых данных (например, уникальность ID элементов на странице).

## На что смотрит интервьюер

- Знание, что `Map` не наследует `Collection` — частая ошибка новичков включать её в общую иерархию как прямого наследника.
- Понимание сложности операций (`Big O`) для основных реализаций и способность обосновать выбор конкретной структуры данных под задачу.
- Follow-up: «Чем `HashMap` отличается от `TreeMap` и `LinkedHashMap`?», «Как работает `HashSet` изнутри?» (на основе `HashMap`, где элементы — ключи, а значение — фиктивная константа), «Потокобезопасны ли `ArrayList` и `HashMap`?» (нет, для этого есть `Collections.synchronizedList`/`ConcurrentHashMap`).
- Красный флаг — кандидат не может объяснить разницу между `List`, `Set` и `Map` на уровне «зачем нужна каждая структура», а только перечисляет названия классов.
