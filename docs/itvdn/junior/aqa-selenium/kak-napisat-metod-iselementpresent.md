# Как написать метод isElementPresent?

**Коротко:** метод проверяет наличие элемента на странице без выброса `NoSuchElementException`, используя `findElements` (во множественном числе) и проверку размера возвращённого списка.

## Развёрнутый ответ

В стандартном API Selenium WebDriver нет готового метода `isElementPresent` — его нужно писать самостоятельно как обёртку (helper) в базовом классе тестов или в Page Object. Ключевая идея: метод `driver.findElement(By)` бросает исключение `NoSuchElementException`, если элемент не найден, а метод `driver.findElements(By)` — нет, он просто возвращает пустой список. Поэтому надёжная реализация строится именно на `findElements`, а не на try/catch вокруг `findElement`.

Есть два основных подхода:

1. **Через `findElements` и проверку размера списка** — предпочтительный способ, так как он не использует исключения для управления потоком выполнения (антипаттерн) и работает быстрее, потому что не нужно ждать таймаут перед выбросом исключения.
2. **Через try/catch вокруг `findElement`** — рабочий, но менее элегантный вариант; исключения — дорогая операция, и злоупотребление ими ухудшает читаемость и производительность тестов.

Важный нюанс: при использовании `findElements` нужно правильно настроить `implicitlyWait` — если он большой (например, 10 секунд), метод будет ждать полный таймаут перед тем, как вернуть пустой список в случае отсутствия элемента. Поэтому иногда перед проверкой временно уменьшают implicit wait до 0-1 секунды, а после проверки возвращают исходное значение.

## Пример / когда применяется

Java-реализация (JUnit/TestNG стиль, Selenium 4):

```java
public boolean isElementPresent(By locator) {
    return driver.findElements(locator).size() > 0;
}
```

Вариант с временным уменьшением implicit wait для ускорения проверки:

```java
public boolean isElementPresent(By locator) {
    driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(1));
    boolean present = !driver.findElements(locator).isEmpty();
    driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
    return present;
}
```

Использование в тесте:

```java
if (isElementPresent(By.id("error-message"))) {
    Assert.fail("Обнаружено сообщение об ошибке");
}
```

Применяется, когда нужно проверить условное поведение UI: показывается ли баннер, появилось ли всплывающее сообщение об ошибке, есть ли определённый элемент только для одного из состояний приложения (например, для авторизованного/неавторизованного пользователя).

## На что смотрит интервьюер

Интервьюер обычно проверяет, понимает ли кандидат разницу между `findElement` и `findElements` и почему `findElements` предпочтительнее для такой проверки. Частая ошибка — реализация через try/catch с `findElement`, что работает, но считается плохой практикой из-за использования исключений как управляющей конструкции и потери времени на ожидание таймаута перед исключением. Ещё один red flag — незнание, что `implicitlyWait` влияет на время выполнения такой проверки, и непонимание, как это уменьшить для ускорения тестов. Могут спросить про альтернативу — использование `WebDriverWait` с `ExpectedConditions.presenceOfElementLocated`, а также уточнить разницу между "элемент присутствует в DOM" и "элемент виден" (`isDisplayed`).
