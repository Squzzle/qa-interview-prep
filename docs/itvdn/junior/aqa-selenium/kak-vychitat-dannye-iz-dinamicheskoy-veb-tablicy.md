# Как вычитать данные из динамической веб-таблицы?

**Коротко:** нужно локализовать строки и ячейки таблицы через XPath или CSS-локаторы, использующие относительные индексы (`tr`, `td`), пройтись по ним циклом через `findElements` и собрать текст в структуру данных (например, `List<List<String>>` или `Map`).

## Развёрнутый ответ

Динамическая таблица — это таблица, содержимое которой формируется на лету (данные из БД, API, фильтров) и может меняться по количеству строк/столбцов между запусками теста. Поэтому нельзя жёстко хардкодить индексы строк и столбцов — локаторы должны быть построены так, чтобы работать независимо от текущего размера таблицы.

Общий алгоритм:

1. Найти корневой элемент таблицы (`table`), обычно по `id`, `class` или другому стабильному атрибуту.
2. Получить список всех строк через `findElements(By.xpath(".//tr"))` относительно таблицы (или `By.cssSelector("tr")`).
3. Для каждой строки получить список ячеек через `findElements(By.tagName("td"))` (или `th` для заголовков).
4. Извлечь текст каждой ячейки методом `getText()` и сложить в удобную структуру — список списков, список строк-объектов (маппинг по колонкам) или `Map<String, String>`, если нужно искать по заголовку колонки.
5. При необходимости — обработать конкретную ячейку по номеру строки/столбца через XPath вида `//table[@id='data']//tr[3]/td[2]`.

Важные нюансы:
- Индексация в XPath начинается с 1, а не с 0.
- Если в таблице есть `thead`/`tbody`, нужно учитывать это в XPath (`.//tbody/tr`), иначе можно случайно захватить строку заголовков.
- Для таблиц с пагинацией или ленивой подгрузкой (lazy loading, infinite scroll) нужно дополнительно обрабатывать переход по страницам или скроллинг с ожиданием подгрузки новых строк перед сбором данных.
- Стоит использовать явные ожидания (`WebDriverWait`), чтобы не читать таблицу до её полной отрисовки.

## Пример / когда применяется

```java
public List<List<String>> getTableData(WebElement table) {
    List<List<String>> tableData = new ArrayList<>();
    List<WebElement> rows = table.findElements(By.xpath(".//tbody/tr"));

    for (WebElement row : rows) {
        List<WebElement> cells = row.findElements(By.tagName("td"));
        List<String> rowData = new ArrayList<>();
        for (WebElement cell : cells) {
            rowData.add(cell.getText());
        }
        tableData.add(rowData);
    }
    return tableData;
}
```

Поиск значения в конкретной ячейке по строке и столбцу:

```java
WebElement cell = driver.findElement(
    By.xpath("//table[@id='report-table']//tr[3]/td[2]")
);
String value = cell.getText();
```

Применяется при проверке результатов поиска/фильтрации (например, таблица заказов, список пользователей в админке), сверке данных UI с ожидаемыми данными из БД или API, а также при тестировании сортировки и пагинации.

## На что смотрит интервьюер

Интервьюер ожидает, что кандидат понимает разницу между статичной и динамической таблицей и не будет предлагать хардкод индексов строк. Часто спрашивают про обработку случаев, когда таблица подгружается асинхронно (нужны явные ожидания), про работу с пагинацией и про то, как избежать `StaleElementReferenceException` при повторном обращении к элементам после обновления DOM. Плюсом считается упоминание оптимизации — извлечение данных одним XPath-запросом там, где это возможно, вместо множества обращений к DOM в цикле, что снижает количество round-trip'ов к браузеру.
