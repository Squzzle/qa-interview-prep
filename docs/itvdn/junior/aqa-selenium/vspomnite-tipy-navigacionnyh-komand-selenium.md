# Вспомните типы навигационных команд Selenium.

**Коротко:** навигация в Selenium WebDriver реализована через вложенный интерфейс `WebDriver.Navigation`, доступный по вызову `driver.navigate()`, и включает четыре основные команды: `to()`, `back()`, `forward()`, `refresh()`.

## Развёрнутый ответ

Помимо метода `driver.get(url)`, который открывает страницу и блокирует выполнение до полной загрузки, Selenium предоставляет отдельный набор навигационных команд через `driver.navigate()`. Они логически моделируют действия пользователя с кнопками браузера (назад/вперёд/обновить) и переход по адресной строке.

Основные команды:

1. **`navigate().to(String url)`** — открывает указанный URL. По сути функционально эквивалентен `driver.get(url)`, но семантически ближе к "перейти по ссылке в адресной строке", а не "загрузить новую сессию страницы". Есть перегрузка, принимающая `java.net.URL`.
2. **`navigate().back()`** — имитирует нажатие кнопки "Назад" в браузере, возвращает на предыдущую страницу из истории браузера.
3. **`navigate().forward()`** — имитирует нажатие кнопки "Вперёд", переход на следующую страницу в истории (если до этого был `back()`).
4. **`navigate().refresh()`** — обновляет текущую страницу (эквивалент F5).

Важный нюанс: `back()` и `forward()` работают с историей браузера конкретной сессии драйвера, и их поведение зависит от того, использует ли страница `history.pushState`/hash-навигацию (SPA-приложения) — иногда навигация происходит "виртуально" без полной перезагрузки страницы, что нужно учитывать при синхронизации теста (могут понадобиться дополнительные ожидания).

Разница между `driver.get()` и `driver.navigate().to()`: функционально они делают одно и то же (загружают URL и ждут `document.readyState == complete` или аналог), но исторически `get()` считается "чистой" загрузкой, а `navigate().to()` — частью API, симулирующего пользовательскую навигацию, наравне с `back`/`forward`/`refresh`.

## Пример / когда применяется

```java
driver.get("https://example.com/login");

driver.navigate().to("https://example.com/dashboard");
// эквивалентно driver.get("https://example.com/dashboard")

driver.navigate().back();
// вернулись на /login

driver.navigate().forward();
// снова на /dashboard

driver.navigate().refresh();
// перезагрузили /dashboard
```

Практический сценарий: тест проверяет, что после успешной отправки формы и перехода на страницу подтверждения, кнопка "Назад" браузера возвращает пользователя на форму, а данные формы не задваиваются при повторной отправке — для этого используются `navigate().back()` и `navigate().refresh()` в комбинации с проверкой состояния страницы.

## На что смотрит интервьюер

Интервьюер проверяет, знает ли кандидат все четыре команды навигации, а не только `get()`/`navigate().to()`. Частая ошибка — забыть про `refresh()` или перепутать его с повторным вызовом `get()` с тем же URL (что тоже сработает, но не то же самое семантически и может обойти кеш иначе). Могут спросить про edge-cases: что произойдёт при `back()`, если истории браузера ещё нет (ничего не произойдёт/останется на той же странице), и как правильно синхронизировать тест после навигационных команд в SPA-приложениях, где DOM обновляется асинхронно без полной перезагрузки (нужны явные ожидания вместо предположения, что страница уже готова сразу после вызова команды).
