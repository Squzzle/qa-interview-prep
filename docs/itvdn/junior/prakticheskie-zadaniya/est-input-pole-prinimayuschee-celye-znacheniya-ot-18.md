# Есть Input поле, принимающее целые значения от 18 до 99 включительно. Надо протестировать с помощью техники тест-дизайна Boundary Values ​​Analysis и Equivalence Partitioning.

**Коротко:** классы эквивалентности — "меньше 18" (невалидно), "18–99" (валидно), "больше 99" (невалидно); граничные значения — 17, 18, 19, 98, 99, 100.

## Развёрнутый ответ

**Equivalence Partitioning (EP)** делит область входных данных на классы, внутри которых поведение системы предполагается одинаковым, что позволяет не проверять каждое значение, а взять по одному представителю из класса.

Классы эквивалентности для поля 18–99:
- Класс 1 — невалидный: значения меньше 18 (например, ...-1, 0, 17).
- Класс 2 — валидный: значения от 18 до 99 включительно.
- Класс 3 — невалидный: значения больше 99 (100, 101, ...).
- Дополнительные невалидные классы, специфичные для input-поля: нечисловые значения (буквы, спецсимволы), дробные числа, пустое значение, значения с ведущими нулями/пробелами.

**Boundary Value Analysis (BVA)** проверяет значения непосредственно на границах классов и рядом с ними — именно там чаще всего возникают ошибки off-by-one. Для диапазона 18–99 рассматриваются:
- Нижняя граница: 17 (невалидно, n−1), 18 (валидно, граница n), 19 (валидно, n+1).
- Верхняя граница: 98 (валидно, n−1), 99 (валидно, граница n), 100 (невалидно, n+1).

Эти техники дополняют друг друга: EP даёт общее покрытие классов, BVA — точечную проверку самых рискованных точек (границ).

## Пример / когда применяется

Итоговый набор тестовых данных:

| № | Значение | Класс / граница | Ожидаемый результат |
|---|---|---|---|
| 1 | 17 | Нижняя граница − 1 | Невалидно (ошибка/отклонено) |
| 2 | 18 | Нижняя граница | Валидно (принято) |
| 3 | 19 | Нижняя граница + 1 | Валидно (принято) |
| 4 | 50 | Середина валидного класса | Валидно (принято) |
| 5 | 98 | Верхняя граница − 1 | Валидно (принято) |
| 6 | 99 | Верхняя граница | Валидно (принято) |
| 7 | 100 | Верхняя граница + 1 | Невалидно (ошибка/отклонено) |
| 8 | -5 | Невалидный класс (отрицательные) | Невалидно |
| 9 | 0 | Невалидный класс | Невалидно |
| 10 | 18.5 | Невалидный класс (дробное) | Невалидно |
| 11 | "abc" | Невалидный класс (не число) | Невалидно |
| 12 | "" (пусто) | Невалидный класс | Ошибка "поле обязательно" (если применимо) |

Такое сочетание EP + BVA — стандартный подход для тестирования любого числового поля с ограниченным диапазоном (возраст, количество товара, процент скидки и т. д.).

## На что смотрит интервьюер

- Правильное построение граничных троек (n−1, n, n+1) для обеих границ диапазона, а не только "18 и 99".
- Понимание, что EP и BVA — разные, но взаимодополняющие техники, а не одно и то же.
- Учёт невалидных классов помимо чисел вне диапазона: нечисловой ввод, дробные значения, пустое поле — это демонстрирует более зрелое мышление тестировщика.
- Частая ошибка: кандидат путает границы (проверяет 17/18/19 только с одной стороны или забывает про верхнюю границу 98/99/100).
