# Протестировать «предмет» относительно различных видов тестирования. (Предмет - лифт, карандаш, калькулятор и т. д.)

**Коротко:** это задание на демонстрацию понимания видов тестирования (функциональное, нагрузочное, юзабилити, безопасности и т. д.) через применение каждого к простому бытовому объекту, например к лифту.

## Развёрнутый ответ

Задача не в том, чтобы протестировать объект буквально, а в том, чтобы для каждого вида тестирования сформулировать релевантные проверки, показывая понимание сути каждого вида. Возьмём в качестве предмета **лифт**.

- **Функциональное тестирование** — работает ли лифт согласно назначению: нажатие кнопки вызывает приезд лифта, кнопки этажей внутри кабины работают, двери открываются/закрываются, лифт останавливается на нужном этаже.
- **Нагрузочное тестирование (load testing)** — поведение при максимально допустимой загрузке (вес, количество пассажиров), при одновременном вызове с нескольких этажей.
- **Стресс-тестирование** — поведение при превышении номинальной нагрузки (перегруз), при аварийном отключении электричества, при экстремально частых вызовах подряд.
- **Тестирование юзабилити** — насколько понятны кнопки и индикация этажей, читаемость подсветки, удобство для людей с ограниченными возможностями (кнопки на нужной высоте, шрифт Брайля).
- **Тестирование безопасности** — срабатывание датчика веса при перегрузе, аварийная кнопка связи с диспетчером, автоматическая остановка при препятствии в дверях, поведение при обрыве троса (аварийные тормоза).
- **Тестирование надёжности / долговременное (reliability, soak testing)** — работа лифта в течение длительного времени (месяцы непрерывной эксплуатации) без сбоев.
- **Регрессионное тестирование** — после ремонта/обновления ПО контроллера лифта проверяется, что базовая функциональность не сломалась.
- **Тестирование совместимости** — работа лифта с разными типами картоприёмников/пропусков, интеграция с системой контроля доступа здания.
- **Тестирование восстановления (recovery testing)** — поведение после сбоя питания: возвращается ли лифт в рабочее состояние, срабатывает ли резервное питание для аварийного спуска.
- **Тестирование удобства обслуживания (maintainability)** — насколько легко сервисной службе диагностировать и устранить неисправность.
- **A/B или сравнительное тестирование** — сравнение старой и новой версии панели управления по времени реакции пользователей.

## Пример / когда применяется

Для другого предмета — **карандаша** — те же виды тестирования переносятся аналогично:

| Вид тестирования | Проверка для карандаша |
|---|---|
| Функциональное | Пишет ли карандаш, стирает ли ластик на другом конце |
| Нагрузочное/стресс | Не ломается ли грифель при сильном нажатии, при падении |
| Юзабилити | Удобство хвата, вес, диаметр |
| Безопасность | Нетоксичность краски покрытия, отсутствие острых заусенцев |
| Совместимость | Подходит ли для разной бумаги, для точилок разного диаметра |
| Надёжность | Не крошится ли грифель после длительного использования |

Такое задание часто дают на собеседовании вместо классического "протестируйте ручку", чтобы посмотреть, как кандидат раскладывает абстрактный объект по видам тестирования, а не просто перечисляет тест-кейсы.

## На что смотрит интервьюер

- Знает ли кандидат сами виды тестирования (не путает вид тестирования с уровнем — модульное/интеграционное/системное — и с типом — позитивное/негативное).
- Способность подобрать конкретный, а не абстрактный пример для каждого вида ("при нагрузке — что именно нагружаем").
- Широту охвата: часто забывают про security, recovery, maintainability, ограничиваясь только функциональным и юзабилити.
- Умение при необходимости объяснить отличие вида тестирования от техники тест-дизайна и от уровня тестирования — частый уточняющий вопрос интервьюера.
