# Что такое Configuration Testing?

**Коротко:** Configuration Testing — это проверка работоспособности приложения при различных комбинациях аппаратного и программного окружения (ОС, браузеры, версии драйверов, сети, оборудование), чтобы убедиться, что система корректно функционирует независимо от конфигурации, а не только на одном "эталонном" стенде.

## Развёрнутый ответ

Configuration Testing относится к нефункциональным видам тестирования (иногда рассматривается как разновидность Compatibility Testing). Цель — выявить дефекты, которые проявляются только на определённых конфигурациях: конкретной версии ОС, конкретной видеокарте, определённом разрешении экрана, версии драйвера принтера, конкретной модели мобильного устройства и т.д.

Ключевые особенности:

- Тестируется не логика приложения, а его поведение в разных условиях окружения.
- Используется матрица конфигураций (configuration matrix): комбинации ОС × браузер × разрешение экрана × версия приложения и т.п.
- Из-за комбинаторного взрыва (слишком много возможных комбинаций) применяют методы сокращения набора: попарное тестирование (pairwise testing), выделение приоритетных/наиболее используемых конфигураций на основе аналитики (например, статистика браузеров реальных пользователей).
- Часто выполняется с использованием ферм устройств/виртуальных машин (device farms, BrowserStack, Sauce Labs) или облачных сервисов эмуляции ОС.

Отличие от смежных понятий:

- **Compatibility Testing** — более широкое понятие, включает совместимость с другим ПО, версиями API, форматами файлов, а не только "железо/ОС".
- **Portability Testing** — проверяет, насколько легко перенести систему в другое окружение (например, миграция с одной СУБД на другую), а не поведение в фиксированном наборе конфигураций.

## Пример / когда применяется

Пример: веб-приложение необходимо протестировать на комбинациях: Windows 10/11, macOS, Chrome/Firefox/Safari/Edge последних 2 версий, разрешения 1920x1080 и 1366x768, с включённым и отключённым JavaScript. Вместо полного перебора (десятки комбинаций) применяют pairwise-технику, чтобы покрыть все попарные сочетания параметров минимальным числом тест-кейсов.

Другой пример — мобильное приложение тестируется на разных моделях смартфонов с разными версиями Android/iOS и размерами экранов, чтобы выявить проблемы вёрстки или несовместимость с конкретной версией API ОС.

## На что смотрит интервьюер

- Путают ли Configuration Testing с Compatibility Testing (это близкие, но не идентичные понятия — важно уметь разграничить).
- Знание техник сокращения конфигурационной матрицы (pairwise, ортогональные массивы) — признак зрелого кандидата.
- Понимание, что полное покрытие всех конфигураций физически невозможно, и умение расставлять приоритеты по данным аналитики.
- Может спросить про инструменты (BrowserStack, LambdaTest, эмуляторы) для практического выполнения такого тестирования.
