# Какие существуют UI-стандарты?

**Коротко:** UI-стандарты — это набор рекомендаций и правил, обеспечивающих согласованность, доступность и удобство интерфейса: платформенные гайдлайны (Material Design, Human Interface Guidelines, Fluent Design), стандарты доступности (WCAG), а также общие принципы юзабилити (консистентность, обратная связь, предсказуемость).

## Развёрнутый ответ

При тестировании пользовательского интерфейса тестировщик проверяет не только функциональность, но и соответствие интерфейса принятым стандартам и гайдлайнам. Основные группы UI-стандартов:

- **Платформенные design-гайдлайны**:
  - **Material Design** (Google) — стандарт для Android и веб-приложений: отступы, типографика, поведение компонентов, анимации.
  - **Human Interface Guidelines, HIG** (Apple) — стандарт для iOS/macOS: жесты, навигация, внешний вид нативных компонентов.
  - **Fluent Design System** (Microsoft) — стандарт для Windows-приложений.
- **Стандарты доступности (Accessibility)**:
  - **WCAG (Web Content Accessibility Guidelines)** — международный стандарт W3C, определяющий требования к доступности контента для людей с ограниченными возможностями (контрастность, alt-тексты, навигация с клавиатуры, поддержка screen reader'ов). Имеет уровни соответствия A, AA, AAA.
  - **ARIA (Accessible Rich Internet Applications)** — набор атрибутов для улучшения доступности динамического веб-контента.
  - **Section 508** — американский стандарт доступности для государственных сайтов.
- **Общие принципы юзабилити (Nielsen's heuristics)** — 10 эвристик Якоба Нильсена: видимость статуса системы, соответствие системы реальному миру, контроль пользователя, консистентность и стандарты, предотвращение ошибок, узнаваемость вместо запоминания, гибкость и эффективность использования, эстетичный и минималистичный дизайн, помощь в распознавании и исправлении ошибок, документация и справка.
- **Внутренние UI-гайды компании (Design System / Style Guide)** — корпоративный стандарт: палитра цветов, шрифты, компоненты, отступы (например, собственная библиотека компонентов на основе Material или Ant Design).

## Пример / когда применяется

Пример: при тестировании веб-приложения тестировщик проверяет через инструменты вроде axe DevTools или Lighthouse соответствие WCAG AA — достаточен ли контраст текста (минимум 4.5:1 для обычного текста), есть ли alt-атрибуты у изображений, работает ли навигация по Tab и активируются ли элементы по Enter/Space.

Другой пример: при тестировании Android-приложения проверяется соответствие Material Design — правильные отступы между элементами (8dp grid), корректное поведение FAB (floating action button), анимации переходов между экранами.

## На что смотрит интервьюер

- Знание конкретных названий стандартов (WCAG, Material Design, HIG), а не общих слов "интерфейс должен быть удобным".
- Понимание разницы между функциональным UI-тестированием и проверкой на соответствие гайдлайнам/доступности.
- Может спросить про уровни WCAG (A/AA/AAA) и конкретные критерии (контрастность, keyboard navigation).
- Красный флаг — если кандидат не может назвать ни одного конкретного стандарта или путает UI-тестирование с UX-исследованиями.
