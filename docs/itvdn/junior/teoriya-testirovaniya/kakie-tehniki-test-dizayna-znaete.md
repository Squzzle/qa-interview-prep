# Какие техники тест-дизайна знаете?

**Коротко:** Основные техники тест-дизайна делятся на чёрный ящик (эквивалентное разбиение, граничные значения, таблицы решений, попарное тестирование, диаграммы переходов состояний, use case testing), белый ящик (покрытие операторов, ветвей, путей) и экспертные (исследовательское тестирование, чек-листы, основанные на опыте).

## Развёрнутый ответ

Техники тест-дизайна (test design techniques) — это систематические методы построения тестовых сценариев, которые позволяют получить максимальное покрытие при минимальном числе тестов.

**Black-box техники (основаны на требованиях/спецификации, без знания кода):**

- **Equivalence Partitioning (эквивалентное разбиение)** — входные данные делятся на классы эквивалентности, из каждого класса выбирается один представитель, так как предполагается, что все значения класса обрабатываются системой одинаково.
- **Boundary Value Analysis (анализ граничных значений)** — тестируются значения на границах классов эквивалентности и рядом с ними (минимум, минимум-1, минимум+1, максимум и т.д.), так как ошибки чаще всего возникают именно на границах.
- **Decision Table Testing (таблицы решений)** — используется для проверки комбинаций условий и соответствующих им действий/результатов, особенно эффективна при сложной бизнес-логике с множеством условий.
- **State Transition Testing (тестирование переходов состояний)** — применяется для систем с чётко определёнными состояниями и переходами между ними (например, статусы заказа: создан → оплачен → отправлен → доставлен).
- **Pairwise Testing / Combinatorial Testing (попарное тестирование)** — позволяет резко сократить число комбинаций параметров, сохраняя покрытие всех попарных сочетаний значений, что статистически покрывает большинство дефектов, связанных с взаимодействием параметров.
- **Use Case Testing** — тестовые сценарии строятся на основе пользовательских сценариев использования системы (use cases).
- **Error Guessing (предположение об ошибках)** — тестировщик, опираясь на опыт, предполагает, где вероятнее всего возникнут ошибки (пустые поля, спецсимволы, экстремальные значения).

**White-box техники (основаны на знании внутренней структуры кода):**

- **Statement Coverage (покрытие операторов)** — проверка, что каждая строка/оператор кода выполнена хотя бы раз.
- **Branch/Decision Coverage (покрытие ветвей)** — проверка, что каждая ветвь условного оператора (true/false) была выполнена.
- **Condition Coverage, Path Coverage** — более глубокие уровни покрытия, проверяющие отдельные условия внутри сложных выражений и все возможные пути выполнения кода.

**Experience-based техники (основаны на опыте):**

- **Exploratory Testing** — одновременное обучение, проектирование тестов и их выполнение.
- **Checklist-based Testing** — использование заранее подготовленных чек-листов на основе прошлого опыта тестирования похожей функциональности.

## Пример / когда применяется

Пример применения BVA и Equivalence Partitioning: поле "возраст" принимает значения от 18 до 65 лет. Классы эквивалентности: младше 18 (невалидный), 18–65 (валидный), старше 65 (невалидный). Граничные значения для проверки: 17, 18, 19, 64, 65, 66.

Пример Decision Table: расчёт скидки в интернет-магазине зависит от суммы заказа (больше/меньше 1000) и статуса клиента (обычный/VIP) — таблица решений покажет все 4 комбинации и ожидаемый результат для каждой.

Пример State Transition: тестирование жизненного цикла задачи в таск-трекере (To Do → In Progress → Review → Done), включая проверку недопустимых переходов (например, из To Do сразу в Done).

## На что смотрит интервьюер

Интервьюер проверяет, может ли кандидат не только перечислить техники, но и применить их на практическом примере — обычно просят прямо на месте построить классы эквивалентности или граничные значения для конкретного поля.

Частые ошибки: путать эквивалентное разбиение и граничные значения (считать их одной техникой), не знать, что такое pairwise testing, не уметь построить таблицу решений для простого примера. Follow-up вопросы: "Постройте классы эквивалентности для поля пароля от 8 до 20 символов", "Что такое попарное тестирование и когда его использовать?", "В чём разница между техниками черного и белого ящика?".
