# Что такое AJAX?

**Коротко:** AJAX (Asynchronous JavaScript and XML) — это техника взаимодействия с сервером из JavaScript, позволяющая отправлять и получать данные асинхронно, в фоне, без полной перезагрузки страницы, обновляя только нужную часть DOM.

## Развёрнутый ответ

AJAX — не отдельная технология или язык, а подход (техника), объединяющий несколько существующих веб-технологий: JavaScript для управления логикой, объект `XMLHttpRequest` (или современный `fetch` API) для отправки HTTP-запросов в фоне, и DOM API для обновления содержимого страницы полученными данными. Несмотря на "X" в названии (XML), сегодня почти всегда для обмена данными используется JSON, а не XML — название сохранилось исторически.

До появления AJAX любое взаимодействие с сервером (отправка формы, переход по ссылке) требовало полной перезагрузки страницы браузером — сервер генерировал заново весь HTML-документ. AJAX решил эту проблему: JavaScript отправляет HTTP-запрос к серверу в фоновом потоке (не блокируя интерфейс), получает ответ (обычно в формате JSON) и с помощью DOM API обновляет только нужный фрагмент страницы, не перезагружая её целиком.

Ключевые характеристики AJAX-запросов:
- **Асинхронность.** Запрос выполняется в фоне, пользователь может продолжать взаимодействовать со страницей, пока ожидается ответ; выполнение обрабатывается через колбэки, промисы или `async/await`.
- **Частичное обновление страницы.** Обновляется не вся страница, а конкретный DOM-элемент/блок данными из ответа.
- **Использование стандартных HTTP-методов** (`GET`, `POST` и др.) для взаимодействия с сервером, часто с REST API.

Исторически AJAX реализовывался через объект `XMLHttpRequest`, сегодня чаще используется более современный и удобный `fetch` API, а в реальных приложениях — обёртки над ним (axios, jQuery `$.ajax`).

Для тестировщика AJAX-запросы важны при UI-автоматизации: элементы, подгружаемые асинхронно AJAX-запросом, могут не существовать в DOM сразу после загрузки страницы, поэтому в автотестах (Selenium/Playwright) необходимо использовать явные (explicit) или неявные (implicit) ожидания (`WebDriverWait`) вместо жёстких `Thread.sleep()`, чтобы дождаться завершения запроса и появления элемента в DOM. Также AJAX-запросы важны при API-тестировании и при анализе сетевых вызовов через вкладку Network в DevTools.

## Пример / когда применяется

Классический пример AJAX через современный `fetch`, обновляющий часть страницы без перезагрузки:

```javascript
document.getElementById('loadMoreBtn').addEventListener('click', async () => {
  const response = await fetch('/api/comments?page=2');
  const comments = await response.json();

  const container = document.getElementById('commentsList');
  comments.forEach((comment) => {
    const li = document.createElement('li');
    li.textContent = comment.text;
    container.appendChild(li);
  });
});
```

Пример явного ожидания появления AJAX-подгруженного элемента в Selenium (Java):

```java
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.visibilityOfElementLocated(By.className("comment-item")));
```

Типичные сценарии применения AJAX: подгрузка комментариев без перезагрузки, автодополнение (autocomplete) в поисковой строке, "бесконечная прокрутка" ленты, живая валидация формы (проверка занятости логина без отправки всей формы), обновление корзины интернет-магазина.

## На что смотрит интервьюер

Интервьюер проверяет, понимает ли кандидат, что AJAX — это техника, а не язык/протокол, знает ли, что современный аналог `XMLHttpRequest` — это `fetch`, и умеет ли связать AJAX с практическими проблемами автоматизации (динамически подгружаемые элементы, необходимость explicit wait вместо sleep). Плюсом будет упоминание, что "X" в AJAX сегодня фактически означает JSON. Красный флаг — если кандидат считает AJAX отдельным языком программирования или не понимает связь между AJAX-запросами и нестабильностью (flakiness) UI-автотестов.
