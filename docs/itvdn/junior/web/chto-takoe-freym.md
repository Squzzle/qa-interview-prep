# Что такое фрейм?

**Коротко:** фрейм (frame) — это область веб-страницы, в которую загружается отдельный HTML-документ; исторически фреймы объединялись через тег `frameset`, сейчас вместо них используется тег `iframe`, встраиваемый прямо в разметку страницы.

## Развёрнутый ответ

Изначально в HTML существовал механизм `frameset`/`frame`, который позволял разбить окно браузера на несколько независимых областей, каждая из которых загружала свой собственный HTML-документ по своему URL. Страница-контейнер (`frameset`) описывала сетку из фреймов вместо тела `body`, а каждый `frame` указывал на отдельный файл. Это давало возможность, например, держать неизменное меню в одном фрейме и менять контент в другом без перезагрузки всей страницы.

У классических фреймов было много проблем: сложности с навигацией и историей браузера (кнопка "назад" вела себя непредсказуемо), проблемы с индексацией в поисковых системах, невозможность нормально добавить страницу в закладки, сложности с версткой и адаптивностью, а также проблемы доступности (accessibility). Из-за этого `frameset` и `frame` были признаны устаревшими и исключены из спецификации HTML5.

Вместо них современный HTML использует тег `iframe` (inline frame) — встроенный фрейм, который размещается внутри обычного тела документа как обычный элемент разметки, наравне с `div` или `img`. `iframe` загружает независимый документ (свою DOM, свой JavaScript-контекст, свою историю) внутри указанной прямоугольной области родительской страницы. Он широко применяется для встраивания виджетов, видео (например, YouTube), карт, платёжных форм, рекламы, а также для изоляции стороннего кода по соображениям безопасности (например, sandbox-атрибут ограничивает возможности встроенного документа).

С точки зрения тестирования важно понимать, что содержимое `iframe` — это отдельный документ (contentDocument/contentWindow), и при автоматизации (Selenium, Playwright) для взаимодействия с элементами внутри фрейма нужно явно переключить контекст драйвера на этот фрейм, а после работы — переключиться обратно на основной документ.

## Пример / когда применяется

Пример современного использования `iframe`:

```html
<iframe
  src="https://www.youtube.com/embed/dQw4w9WgXcQ"
  width="560"
  height="315"
  title="Встроенное видео"
  sandbox="allow-scripts allow-same-origin"
></iframe>
```

Устаревший пример `frameset` (только для контекста истории, в продакшне не используется):

```html
<frameset cols="25%,75%">
  <frame src="menu.html" />
  <frame src="content.html" />
</frameset>
```

В Selenium переключение на фрейм для взаимодействия с элементами внутри него:

```java
driver.switchTo().frame("iframeName");
driver.findElement(By.id("submit")).click();
driver.switchTo().defaultContent();
```

## На что смотрит интервьюер

Интервьюер проверяет, отличает ли кандидат устаревший `frameset`/`frame` от актуального `iframe`, и понимает ли он практические последствия использования фреймов для тестировщика: что элементы внутри `iframe` не видны локатору без переключения контекста драйвера, что это частая причина ошибок `NoSuchElementException` в автотестах. Красный флаг — если кандидат путает фреймы с обычными вложенными `div` или не знает про необходимость `switchTo().frame()`.
