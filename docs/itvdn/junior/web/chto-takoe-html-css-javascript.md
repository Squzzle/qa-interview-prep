# Что такое HTML/CSS/JavaScript?

**Коротко:** это три базовых слоя веб-разработки — HTML описывает структуру и содержимое страницы, CSS отвечает за внешний вид и оформление, а JavaScript добавляет интерактивность и логику поведения на стороне клиента (и не только).

## Развёрнутый ответ

**HTML (HyperText Markup Language)** — язык гипертекстовой разметки. Он не является языком программирования: с его помощью описывается структура документа при помощи тегов (`div`, `p`, `h1`, `table`, `form` и т.д.), которые браузер интерпретирует и строит из них DOM-дерево (Document Object Model). HTML определяет семантику содержимого: где заголовок, где абзац, где ссылка, где форма ввода.

**CSS (Cascading Style Sheets)** — каскадные таблицы стилей. CSS отвечает за визуальное представление HTML-разметки: цвета, отступы, шрифты, позиционирование, адаптивность под разные экраны (media queries), анимации и переходы. Правила CSS применяются к элементам через селекторы (по тегу, классу, id, атрибуту) и подчиняются каскаду и специфичности — при конфликте правил побеждает более специфичное или объявленное позже (с учётом `!important`).

**JavaScript** — интерпретируемый (JIT-компилируемый) язык программирования, который выполняется в браузере (а также на сервере через Node.js). Он позволяет динамически изменять DOM и стили после загрузки страницы, обрабатывать события пользователя (клики, ввод, скролл), выполнять асинхронные запросы к серверу (fetch/AJAX), валидировать формы, управлять состоянием приложения (особенно в SPA на React/Vue/Angular). JavaScript однопоточный, но использует event loop и очередь колбэков/промисов для неблокирующей асинхронности.

Вместе эти три технологии образуют классическую модель разделения ответственности во фронтенде: структура (HTML) → представление (CSS) → поведение (JavaScript). Браузер парсит HTML, строит DOM, парсит CSS, строит CSSOM, объединяет их в render tree, затем выполняет JavaScript, который может модифицировать и DOM, и CSSOM в реальном времени.

## Пример / когда применяется

```html
<!-- HTML: структура -->
<button id="likeBtn">Нравится</button>
<span id="counter">0</span>
```

```css
/* CSS: оформление */
#likeBtn {
  background-color: #2f80ed;
  color: white;
  border-radius: 6px;
  padding: 8px 16px;
}
```

```javascript
// JavaScript: поведение
let count = 0;
document.getElementById('likeBtn').addEventListener('click', () => {
  count++;
  document.getElementById('counter').textContent = count;
});
```

Для тестировщика это разделение важно: баги в HTML/структуре часто ловятся статическим анализом верстки и HTML-валидаторами, баги CSS — визуальным (визуальным регрессионным) тестированием, а баги JavaScript — функциональным и UI-автоматизированным тестированием (Selenium, Playwright, Cypress).

## На что смотрит интервьюер

Интервьюер ожидает чёткого разграничения ролей трёх технологий (а не общей фразы "это фронтенд"), понимания, что HTML — не язык программирования, а JavaScript — единственный из трёх исполняемый и Тьюринг-полный язык. Также могут спросить про DOM, CSSOM, event loop или про то, как эти технологии влияют на стратегию тестирования (какие баги где искать). Красный флаг — путаница между HTML и XML, или утверждение, что CSS может выполнять логику (хотя современный CSS умеет условные выражения через `@media`/`:has()`, это не полноценная логика уровня JS).
