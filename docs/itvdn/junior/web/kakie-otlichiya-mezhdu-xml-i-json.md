# Какие отличия между XML и JSON?

**Коротко:** JSON — компактный, легко читаемый и быстро парсящийся формат на основе пар ключ-значение, используемый преимущественно в REST API; XML — более многословный и строгий разметочный язык с поддержкой атрибутов, пространств имён и схем валидации (XSD), используемый в SOAP и enterprise-системах.

## Развёрнутый ответ

**JSON (JavaScript Object Notation)** — текстовый формат обмена данными, основанный на синтаксисе объектов JavaScript. Состоит из пар "ключ-значение", массивов, строк, чисел, булевых значений и `null`.

Преимущества JSON:
- Компактнее XML (меньше служебных символов, нет закрывающих тегов).
- Легче парсится и быстрее обрабатывается большинством языков программирования.
- Нативно поддерживается в JavaScript, легко преобразуется в объекты.
- Проще читается человеком для простых структур данных.

**XML (eXtensible Markup Language)** — язык разметки, использующий вложенные теги для описания структуры данных.

Преимущества XML:
- Поддерживает атрибуты у элементов (`<user id="42">`), что даёт дополнительную гибкость описания метаданных.
- Поддерживает пространства имён (namespaces), что важно при интеграции нескольких систем с пересекающимися именами тегов.
- Есть развитые стандарты валидации структуры — XSD (XML Schema Definition) и DTD (Document Type Definition).
- Поддерживает комментарии внутри документа.
- Исторически используется в SOAP, enterprise-интеграциях, документах (Office Open XML, конфигурационные файлы Maven/Spring).

Ключевые отличия:

| Критерий | JSON | XML |
|---|---|---|
| Многословность | компактный | многословный (открывающие/закрывающие теги) |
| Типы данных | есть (строка, число, булево, null, массив, объект) | все данные — текст, типы не встроены |
| Атрибуты | нет (только вложенные объекты) | есть (`<tag attr="value">`) |
| Комментарии | не поддерживаются | поддерживаются (`<!-- comment -->`) |
| Валидация схемы | JSON Schema (опционально) | XSD/DTD (широко распространено) |
| Пространства имён | нет | есть |
| Скорость парсинга | выше | ниже |
| Использование | REST API, конфиги, NoSQL (MongoDB) | SOAP, документы, легаси-системы |

## Пример / когда применяется

JSON:
```json
{
  "user": {
    "id": 42,
    "name": "Ivan",
    "roles": ["admin", "user"]
  }
}
```

XML:
```xml
<user id="42">
  <name>Ivan</name>
  <roles>
    <role>admin</role>
    <role>user</role>
  </roles>
</user>
```

JSON применяется в большинстве современных REST/GraphQL API, конфигурационных файлах (package.json), NoSQL-базах (MongoDB хранит BSON, производный от JSON). XML применяется в SOAP-сервисах, файлах конфигурации Java-проектов (pom.xml), офисных документах, RSS/Atom-лентах, банковских и государственных интеграциях, где важна строгая валидация через XSD.

## На что смотрит интервьюер

- Знание, что JSON компактнее и быстрее парсится, а XML — более выразителен благодаря атрибутам и пространствам имён.
- Понимание области применения каждого формата (JSON — REST, XML — SOAP/легаси/документы).
- Знание инструментов валидации (JSON Schema против XSD).
- Follow-up вопросы: "Как бы вы валидировали JSON-ответ API в автотестах?", "Что такое XSD и зачем она нужна?", "Может ли REST API возвращать XML?" (да, может, формат не обязательно JSON).
- Красный флаг: утверждение, что "REST — это всегда JSON, а SOAP — всегда XML" без уточнения, что это распространённая практика, а не жёсткое правило для REST.
