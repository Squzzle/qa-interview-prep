# Какие протоколы передачи данных знаете?

**Коротко:** основные протоколы передачи данных — HTTP/HTTPS, TCP, UDP, FTP/SFTP, SMTP/POP3/IMAP, WebSocket, DNS; каждый решает свою задачу на своём уровне сетевой модели OSI/TCP-IP.

## Развёрнутый ответ

Протоколы передачи данных можно классифицировать по уровням модели OSI (или упрощённой TCP/IP):

**Транспортный уровень:**
- **TCP (Transmission Control Protocol)** — надёжный, устанавливает соединение (three-way handshake), гарантирует доставку и порядок пакетов, используется там, где важна целостность данных (HTTP, FTP, email).
- **UDP (User Datagram Protocol)** — ненадёжный, без установления соединения, быстрее TCP, используется там, где важна скорость, а не 100% доставка (стриминг видео, онлайн-игры, VoIP, DNS-запросы).

**Прикладной уровень:**
- **HTTP/HTTPS (HyperText Transfer Protocol / Secure)** — основной протокол веба, работает поверх TCP, HTTPS добавляет шифрование через TLS/SSL.
- **FTP/SFTP (File Transfer Protocol / Secure FTP)** — передача файлов; SFTP работает поверх SSH и шифрует данные.
- **SMTP (Simple Mail Transfer Protocol)** — отправка email.
- **POP3/IMAP (Post Office Protocol / Internet Message Access Protocol)** — получение email; IMAP синхронизирует письма на сервере, POP3 скачивает и обычно удаляет.
- **WebSocket** — протокол полнодуплексной связи поверх одного TCP-соединения, позволяет серверу самому инициировать отправку данных клиенту (чаты, живые уведомления, биржевые котировки).
- **DNS (Domain Name System)** — преобразование доменных имён в IP-адреса, обычно работает по UDP (порт 53).
- **DHCP (Dynamic Host Configuration Protocol)** — автоматическая раздача IP-адресов в сети.

**Дополнительно стоит упомянуть:**
- **gRPC** — современный высокопроизводительный RPC-протокол от Google на основе HTTP/2 и Protocol Buffers, используется в микросервисах.
- **MQTT** — лёгкий протокол публикации-подписки для IoT.

## Пример / когда применяется

Пример использования: тестировщик проверяет API через HTTPS, а при нагрузочном тестировании стриминга видео может анализировать трафик по UDP через Wireshark.

```
Уровень приложения:    HTTP, FTP, SMTP, DNS, WebSocket
Уровень транспорта:    TCP, UDP
Уровень сети:          IP
Уровень канала:        Ethernet, Wi-Fi
```

Практический пример проверки протокола через `curl`:

```bash
curl -v https://example.com
```
Флаг `-v` покажет установление TCP-соединения, TLS handshake и обмен HTTP-заголовками.

## На что смотрит интервьюер

- Знание разницы между TCP и UDP и умение объяснить, когда что применяется.
- Понимание места HTTP в стеке протоколов (что это прикладной уровень поверх TCP).
- Знание хотя бы 5-6 протоколов с областями применения, а не просто перечисление названий.
- Follow-up вопросы: "Чем отличается HTTP/1.1 от HTTP/2?", "Что происходит при TLS handshake?", "Почему DNS использует UDP, а не TCP?".
- Красный флаг: путаница между протоколом и портом (например, "HTTP — это порт 80", без понимания сути протокола).
