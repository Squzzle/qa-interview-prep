# Какие знаете Web elements?

**Коротко:** Web elements — это элементы HTML-разметки страницы (кнопки, поля ввода, ссылки, чекбоксы, списки, таблицы и т.д.), с которыми взаимодействует пользователь и которые тестировщик находит и проверяет через локаторы в автотестах или вручную.

## Развёрнутый ответ

Веб-элементы — это HTML-теги (или их DOM-представление), формирующие интерфейс страницы. С точки зрения тестирования их принято классифицировать по типу взаимодействия:

**Элементы ввода и управления:**
- `input` (текстовое поле, `type="text"`, `email`, `password`, `number`, `checkbox`, `radio`, `date` и т.д.);
- `textarea` — многострочное текстовое поле;
- `select` / `option` — выпадающий список;
- `button` — кнопка;
- `a` (ссылка) — переход по URL;
- `form` — контейнер для группы полей ввода, отправляющий данные.

**Элементы отображения информации:**
- заголовки `h1`–`h6`;
- параграфы `p`, span `span`, div `div` — блочные/строчные контейнеры без семантической нагрузки;
- списки `ul`, `ol`, `li`;
- таблицы `table`, `tr`, `td`, `th`;
- изображения `img`, видео `video`, аудио `audio`.

**Семантические элементы (HTML5):**
- `header`, `footer`, `nav`, `main`, `section`, `article`, `aside` — задают смысловую структуру документа, важны для доступности (accessibility) и SEO.

**Интерактивные/составные элементы (часто кастомные, через JS-фреймворки):**
- модальные окна (modal), выпадающие меню (dropdown), табы (tabs), карусели (carousel), тултипы (tooltip) — как правило, собраны из базовых тегов с добавлением CSS/JS-поведения, не являются нативными HTML-тегами.

Для автоматизации тестирования важно уметь находить эти элементы через локаторы: `id`, `name`, `class`, CSS-селекторы, XPath, а также атрибуты `data-testid`, специально добавляемые для тестов.

## Пример / когда применяется

Пример формы регистрации и связанных с ней Web elements:

```html
<form id="registerForm">
  <input type="text" name="username" placeholder="Имя пользователя" />
  <input type="email" name="email" placeholder="Email" />
  <input type="password" name="password" placeholder="Пароль" />
  <input type="checkbox" name="agree" /> Я согласен с условиями
  <select name="country">
    <option value="ru">Россия</option>
    <option value="by">Беларусь</option>
  </select>
  <button type="submit">Зарегистрироваться</button>
</form>
```

Тестировщик, проверяя эту форму, будет: вводить данные в `input`, отмечать `checkbox`, выбирать значение в `select`, кликать `button` и проверять валидацию каждого поля (обязательность, формат email, минимальная длина пароля).

## На что смотрит интервьюер

- Может ли кандидат назвать конкретные HTML-теги, а не только абстрактно "кнопки и поля".
- Знает ли разницу между семантическими и несемантическими тегами (`div`/`span` vs `header`/`nav`/`article`).
- Понимает ли связь между Web elements и локаторами в автоматизации (id, class, xpath, css selector, data-testid).
- Follow-up: "Как бы вы нашли элемент, у которого нет id и class?", "Что такое data-testid и зачем он нужен?", "Чем div отличается от section?".
- Красный флаг — если кандидат путает HTML-элементы с CSS-свойствами или JS-событиями.
