# Какую структуру имеет веб-страница?

**Коротко:** веб-страница — это HTML-документ, состоящий из объявления типа документа (`doctype`), корневого элемента `html`, который включает `head` (метаданные, стили, скрипты) и `body` (видимое содержимое), а браузер на основе этого строит DOM-дерево.

## Развёрнутый ответ

Стандартная структура HTML-документа выглядит иерархически:

1. **`<!DOCTYPE html>`** — объявление типа документа, указывающее браузеру, что нужно использовать режим стандартов (standards mode) HTML5, а не устаревший quirks mode.
2. **`<html>`** — корневой элемент документа, обычно с атрибутом `lang` для указания языка страницы (важно для доступности и SEO).
3. **`<head>`** — раздел метаданных, невидимый пользователю напрямую. Содержит:
   - `<meta charset>` — кодировку документа;
   - `<title>` — заголовок вкладки браузера;
   - `<meta name="viewport">` — настройки адаптивности для мобильных устройств;
   - `<link>` — подключение внешних CSS-файлов, favicon;
   - `<script>` — подключение JavaScript (может быть и в `head`, и в `body`);
   - `<meta name="description">` и другие метатеги для SEO.
4. **`<body>`** — видимое содержимое страницы: заголовки (`h1`-`h6`), параграфы (`p`), ссылки (`a`), изображения (`img`), формы (`form`), таблицы (`table`), контейнеры (`div`, `section`, `article`, `header`, `nav`, `main`, `footer` — семантические теги HTML5).

Браузер разбирает (парсит) HTML последовательно сверху вниз и строит **DOM (Document Object Model)** — древовидную объектную модель документа, где каждый тег становится узлом дерева. Одновременно браузер парсит подключённые CSS-файлы и строит **CSSOM**. DOM и CSSOM объединяются в **render tree**, на основе которого происходит **layout** (расчёт геометрии) и **paint** (отрисовка пикселей).

Важный нюанс для тестировщиков: расположение `<script>` влияет на производительность и порядок выполнения — синхронные скрипты в `head` без атрибутов `async`/`defer` блокируют парсинг HTML до своей загрузки и выполнения, поэтому их часто помещают перед закрывающим `</body>` либо используют атрибут `defer`.

## Пример / когда применяется

Минимальная валидная структура HTML5-страницы:

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Пример страницы</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <header>
      <nav>Навигация</nav>
    </header>
    <main>
      <h1>Заголовок страницы</h1>
      <p>Основной контент.</p>
    </main>
    <footer>Подвал сайта</footer>
    <script src="app.js" defer></script>
  </body>
</html>
```

Понимание этой структуры нужно тестировщику при написании локаторов (XPath/CSS-селекторов), при проверке SEO-тегов, а также при анализе производительности загрузки страницы через DevTools (вкладка Network/Performance).

## На что смотрит интервьюер

Интервьюер смотрит, знает ли кандидат обязательные элементы (`doctype`, `html`, `head`, `body`), понимает ли разницу между `head` и `body`, и может ли объяснить, как из HTML получается DOM. Дополнительно могут спросить про семантическую вёрстку (`header`, `nav`, `main`, `footer` вместо голых `div`) и про влияние расположения скриптов на рендеринг. Красный флаг — если кандидат не может назвать базовые обязательные теги документа или путает DOM с самим HTML-файлом.
