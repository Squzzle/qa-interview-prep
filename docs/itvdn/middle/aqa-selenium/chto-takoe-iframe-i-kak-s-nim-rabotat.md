# Что такое iframe и как с ним работать в Selenium?

**Коротко:** iframe (`<iframe>`) — это встроенный в страницу отдельный HTML-документ со своим собственным DOM; чтобы взаимодействовать с элементами внутри него, Selenium требует явно переключить контекст драйвера на этот фрейм через `driver.switchTo().frame(...)`, а после работы вернуться в основной документ через `driver.switchTo().defaultContent()`.

## Развёрнутый ответ

`iframe` (inline frame) — HTML-элемент, который встраивает в текущую страницу другой, полностью независимый HTML-документ (со своим `<html>`, `<head>`, `<body>`, скриптами, стилями). С точки зрения браузера это отдельное дерево DOM, изолированное от родительского документа (в том числе по соображениям безопасности — same-origin policy).

Из-за этой изоляции методы `findElement`/`findElements` в Selenium по умолчанию «видят» только DOM основной страницы (или того фрейма, в контексте которого сейчас находится драйвер). Если элемент физически находится внутри `iframe`, попытка найти его без переключения контекста приведёт к `NoSuchElementException`, даже если элемент визуально виден на странице.

Для работы с содержимым фрейма нужно:

1. **Переключиться на фрейм** одним из трёх способов:
   - по индексу: `driver.switchTo().frame(0)`;
   - по атрибуту `name` или `id`: `driver.switchTo().frame("frameName")`;
   - по веб-элементу (наиболее надёжный способ): `driver.switchTo().frame(webElement)`.
2. **Выполнить нужные действия** внутри фрейма — теперь `findElement` работает в контексте DOM этого фрейма.
3. **Вернуться обратно**:
   - `driver.switchTo().parentFrame()` — вернуться на один уровень вверх (полезно при вложенных фреймах);
   - `driver.switchTo().defaultContent()` — вернуться в основной документ верхнего уровня.

Важные нюансы:
- Если фреймов несколько (вложенные iframe), нужно последовательно переключаться на каждый уровень — нельзя «перепрыгнуть» сразу к элементу вложенного фрейма.
- Явные и неявные ожидания (`WebDriverWait`) должны применяться уже после переключения в нужный контекст, иначе можно ждать элемент не в том документе.
- После переключения на фрейм команды `driver.getTitle()`, `driver.getCurrentUrl()` по-прежнему относятся к главной странице, а не к содержимому фрейма (это можно использовать как способ отличить, действительно ли произошло переключение).
- Если атрибут `id`/`name` отсутствует или неуникален, лучше использовать `switchTo().frame(By.cssSelector(...))` через предварительный поиск элемента.

## Пример / когда применяется

```java
// Переключение на фрейм по id
driver.switchTo().frame("payment-frame");

WebElement cardInput = driver.findElement(By.id("cardNumber"));
cardInput.sendKeys("4111111111111111");

// Возврат в основной документ
driver.switchTo().defaultContent();

WebElement submitButton = driver.findElement(By.id("submit"));
submitButton.click();
```

Типичный кейс: виджеты оплаты (Stripe, PayPal), встроенные видео (YouTube), рекламные блоки и капчи (`reCAPTCHA`) часто реализованы как отдельные `iframe` в целях безопасности — их автоматизация требует именно такого переключения контекста.

## На что смотрит интервьюер

- Знание, что iframe — изолированный DOM, и почему `findElement` не находит элементы внутри него без переключения.
- Умение перечислить все три способа переключения (индекс, имя/id, WebElement) и объяснить, какой предпочтительнее.
- Знание разницы между `parentFrame()` и `defaultContent()`.
- Follow-up: как работать с несколькими вложенными фреймами, что будет при `StaleElementReferenceException` после переключения контекста, как автоматизировать reCAPTCHA (в общем случае — никак, это специально защищено).
- Красный флаг: попытка искать элемент внутри фрейма без переключения контекста или незнание, как вернуться назад.
