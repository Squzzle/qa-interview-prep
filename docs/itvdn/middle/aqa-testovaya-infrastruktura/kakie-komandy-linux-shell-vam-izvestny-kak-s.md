# Какие команды Linux Shell вам известны? Как с помощью команд Linux Shell найти лог-файл и строчку с ошибкой в файле?

**Коротко:** базовый набор команд Linux Shell включает навигацию (`cd`, `ls`, `pwd`), работу с файлами (`cp`, `mv`, `rm`, `find`), просмотр содержимого (`cat`, `less`, `tail`, `head`) и поиск текста (`grep`); найти лог-файл и строку с ошибкой в нём проще всего комбинацией `find` и `grep`.

## Развёрнутый ответ

Для эффективной работы QA-инженеру в Linux-окружении необходимо знание нескольких групп команд:

**Навигация и информация о системе:**
- `pwd` — вывести текущую директорию.
- `cd` — сменить директорию.
- `ls -la` — вывести список файлов, включая скрытые, с подробностями (права, владелец, размер, дата).
- `ps aux` — список запущенных процессов.
- `top`/`htop` — мониторинг использования CPU/памяти в реальном времени.
- `df -h` — использование дискового пространства.

**Работа с файлами и директориями:**
- `cp`, `mv`, `rm`, `mkdir`, `rmdir` — копирование, перемещение, удаление файлов и папок.
- `chmod`, `chown` — изменение прав доступа и владельца файла.
- `find` — рекурсивный поиск файлов по имени, типу, дате изменения и другим критериям.

**Просмотр и анализ текстового содержимого:**
- `cat` — вывести содержимое файла целиком.
- `less`/`more` — постраничный просмотр больших файлов.
- `head -n` / `tail -n` — вывести первые/последние N строк файла.
- `tail -f` — следить за файлом в реальном времени (незаменимо для логов, которые пишутся во время выполнения теста).
- `grep` — поиск строк, соответствующих шаблону (регулярному выражению), внутри файла или потока.
- `awk`/`sed` — обработка и трансформация текста построчно, вырезание нужных колонок, замена текста.
- `wc -l` — подсчёт количества строк в файле.

**Перенаправление и конвейеры:**
- `|` (pipe) — передаёт вывод одной команды на вход другой, что позволяет комбинировать простые утилиты в мощные цепочки обработки.
- `>` и `>>` — перенаправление вывода в файл (перезапись/дозапись).

Чтобы найти лог-файл и строку с ошибкой в нём, обычно действуют в два шага: сначала находят сам файл лога, если его точное расположение неизвестно, затем ищут в нём нужный текст ошибки.

## Пример / когда применяется

```bash
# Шаг 1: найти лог-файлы, изменённые за последние 24 часа, в директории /var/log
find /var/log -name "*.log" -mtime -1

# Шаг 2: найти строки с ошибками в конкретном файле (регистронезависимо),
# показав 2 строки контекста до и после каждого совпадения
grep -i -B2 -A2 "error" /var/log/app/application.log

# Комбинация в один пайплайн: найти все лог-файлы проекта и поискать в них "Exception"
find /opt/myapp/logs -name "*.log" | xargs grep -l "Exception"

# Следить за логом в реальном времени во время прогона автотеста и сразу фильтровать ошибки
tail -f /var/log/app/test-run.log | grep --line-buffered -i "error\|fail"
```

Такой подход типичен при расследовании падения автотеста на CI-агенте: сначала находят актуальный лог-файл прогона (`find`), затем быстро локализуют причину падения (`grep -i "error"` или `grep -i "exception"`), после чего смотрят более широкий контекст вокруг найденной строки (`grep -B -A` или открытие файла в `less` с переходом к найденной строке через `/`).

## На что смотрит интервьюер

Интервьюер проверяет практическое, а не книжное знание команд — важно, чтобы кандидат уверенно называл флаги (`-i`, `-r`, `-n`, `-B`/`-A` для grep; `-n`, `-mtime` для find) и понимал разницу между похожими командами (`cat` для маленьких файлов vs `less`/`tail -f` для логов, которые постоянно растут). Частая ошибка — пытаться открыть гигабайтный лог через `cat`, что забивает терминал; правильный подход — `tail`, `less` или сразу `grep`. Хорошим сигналом опытного кандидата является упоминание конвейеров (`|`) и комбинирования нескольких утилит, а не использование только одной команды.
