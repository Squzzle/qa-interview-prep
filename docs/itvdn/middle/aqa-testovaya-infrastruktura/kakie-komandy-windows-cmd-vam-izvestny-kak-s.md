# Какие команды Windows CMD вам известны? Как с помощью команд Windows CMD найти IP-адрес машины?

**Коротко:** базовые команды Windows CMD включают навигацию (`cd`, `dir`), работу с файлами (`copy`, `move`, `del`, `mkdir`), диагностику сети (`ipconfig`, `ping`, `tracert`, `netstat`) и процессы (`tasklist`, `taskkill`); IP-адрес машины проще всего узнать командой `ipconfig`.

## Развёрнутый ответ

Windows CMD (Command Prompt) — стандартный интерпретатор командной строки Windows, менее мощный, чем PowerShell, но широко используемый для базовых административных и диагностических задач.

**Навигация и работа с файлами:**
- `dir` — список файлов и папок в текущей директории (аналог `ls` в Linux).
- `cd` — сменить текущую директорию.
- `copy`, `move`, `del`, `mkdir`, `rmdir` — копирование, перемещение, удаление файлов и папок.
- `type` — вывести содержимое текстового файла (аналог `cat`).
- `findstr` — поиск строк по шаблону внутри файла (аналог `grep` в Linux).

**Сетевая диагностика:**
- `ipconfig` — вывод сетевой конфигурации: IP-адрес, маска подсети, шлюз по умолчанию.
- `ipconfig /all` — расширенный вывод, включая MAC-адрес, DNS-серверы, DHCP.
- `ping <host>` — проверка доступности узла и измерение задержки (RTT).
- `tracert <host>` — трассировка маршрута пакетов до узла через промежуточные роутеры.
- `nslookup <host>` — разрешение DNS-имени в IP-адрес и наоборот.
- `netstat -an` — список активных сетевых соединений и открытых портов.

**Процессы и система:**
- `tasklist` — список запущенных процессов (аналог `ps` в Linux).
- `taskkill /PID <pid> /F` — принудительное завершение процесса по его PID.
- `systeminfo` — подробная информация об операционной системе и железе.
- `set` — вывод/установка переменных окружения.

Чтобы найти IP-адрес машины, самая простая и распространённая команда — `ipconfig`. Она выводит IP-адрес по каждому сетевому адаптеру (Ethernet, Wi-Fi, VPN-адаптеры), маску подсети и шлюз по умолчанию.

## Пример / когда применяется

```cmd
:: Базовый вывод: IP-адрес, маска подсети, шлюз для каждого адаптера
ipconfig

:: Расширенный вывод: MAC-адрес, DNS-серверы, статус DHCP
ipconfig /all

:: Отфильтровать только строки с IPv4-адресом
ipconfig | findstr "IPv4"

:: Проверить доступность тестового сервера по имени и узнать его IP
ping test-server.local

:: Посмотреть, какие процессы слушают порт 4444 (например, Selenium Grid hub)
netstat -ano | findstr ":4444"
```

Такие команды часто применяются QA-инженером при настройке локального окружения для тестирования — например, чтобы узнать свой IP для подключения мобильного устройства к локальному Appium-серверу через локальную сеть, или чтобы диагностировать, почему тестовый агент CI не может достучаться до сервиса (`ping`, `tracert`, `netstat`).

## На что смотрит интервьюер

Ожидается, что кандидат уверенно назовёт `ipconfig` как основную команду для получения IP-адреса и сможет объяснить разницу между `ipconfig` и `ipconfig /all`. Частая ошибка — путать команды CMD с PowerShell-командлетами (`Get-NetIPAddress`, `Test-Connection`) или Linux-командами (`ifconfig`, которого нет в Windows). Может прозвучать follow-up про диагностику сетевых проблем целиком: сначала `ping` до целевого хоста, затем `tracert` для локализации проблемного узла в маршруте, `netstat` для проверки открытых портов — кандидат должен уметь выстроить последовательность диагностики, а не называть команды разрозненно.
