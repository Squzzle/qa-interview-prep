# Что такое REgexp?

**Коротко:** RegExp (regular expression, регулярное выражение) — это формальный язык описания шаблонов текста, используемый для поиска, проверки и извлечения подстрок в строках по заданным правилам.

## Развёрнутый ответ

Регулярное выражение — это последовательность символов, задающая шаблон поиска. Оно состоит из литеральных символов (которые совпадают сами с собой) и метасимволов (спецсимволов), задающих правила: количество повторений, альтернативы, границы, классы символов.

Основные элементы regexp:

- **Классы символов**: `.` — любой символ, `\d` — цифра, `\w` — буква/цифра/подчёркивание, `\s` — пробельный символ, `[abc]` — один из символов a, b или c, `[^abc]` — любой символ, кроме указанных.
- **Квантификаторы**: `*` — 0 или более повторений, `+` — 1 или более, `?` — 0 или 1 (опционально), `{n,m}` — от n до m повторений.
- **Якоря**: `^` — начало строки, `$` — конец строки, `\b` — граница слова.
- **Группы и альтернативы**: `(...)` — группа захвата, `(?:...)` — группа без захвата, `|` — логическое ИЛИ.
- **Экранирование**: `\` перед метасимволом делает его литеральным (например, `\.` означает точку как символ, а не "любой символ").

В тестировании regexp применяется в трёх основных сценариях: валидация форматов данных (email, телефон, паспорт), поиск и фильтрация логов/данных, а также в автоматизации (локаторы элементов в Selenium/Playwright, парсинг ответов API, настройка тестовых данных).

Важный нюанс для собеседования — понимание "жадности" (greedy) и "ленивости" (lazy) квантификаторов: `.*` жадно захватывает максимально возможную подстроку, а `.*?` — минимально возможную, что критично при парсинге, например, HTML-тегов.

## Пример / когда применяется

Примеры практического использования:

```text
Проверка email:        ^[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}$
Проверка телефона RU:  ^\+7\d{10}$
Поиск числа с плавающей точкой: -?\d+(\.\d+)?
```

Пример использования в автотесте (Java, поиск текста в логах):

```java
import java.util.regex.Matcher;
import java.util.regex.Pattern;

Pattern pattern = Pattern.compile("ERROR:\\s+(.*)");
Matcher matcher = pattern.matcher(logLine);
if (matcher.find()) {
    String errorMessage = matcher.group(1);
    System.out.println("Найдена ошибка: " + errorMessage);
}
```

В автоматизации UI-тестирования regexp часто используется в XPath/CSS-локаторах для частичного совпадения текста, а также в конфигурациях тест-фреймворков для фильтрации тестов по имени/тегу.

## На что смотрит интервьюер

Смотрят, может ли кандидат не только назвать определение, но и написать простое выражение "на лету" (например, для email или телефона). Частые ошибки: путают `*` и `+`, не знают о жадности/ленивости квантификаторов, не понимают разницу между `find()`/`matches()`/`search()` в разных языках. Хороший ответ также упоминает практическое применение в контексте QA: валидация тестовых данных, парсинг логов, локаторы в автотестах.
