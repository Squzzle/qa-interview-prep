# Что такое case function?

**Коротко:** `CASE` — это условное выражение в SQL, аналог `if/else` или `switch`, которое возвращает разные значения в зависимости от выполнения условий, прямо внутри запроса.

## Развёрнутый ответ

`CASE` — стандартное SQL-выражение условной логики (SQL:1992+), позволяющее вычислять значение "на лету" в зависимости от одного или нескольких условий. Оно может использоваться практически везде, где допустимо скалярное выражение: в списке `SELECT`, в `WHERE`, `ORDER BY`, `GROUP BY`, `HAVING`, а также внутри агрегатных функций.

Существует две формы записи:

1. **Простая форма (simple CASE)** — сравнивает одно выражение со списком значений:

```sql
CASE column_name
    WHEN 'A' THEN 'Active'
    WHEN 'I' THEN 'Inactive'
    ELSE 'Unknown'
END
```

2. **Поисковая форма (searched CASE)** — проверяет произвольные булевы условия, более гибкая:

```sql
CASE
    WHEN age < 18 THEN 'Minor'
    WHEN age BETWEEN 18 AND 65 THEN 'Adult'
    ELSE 'Senior'
END
```

Важные нюансы:

- Ветки проверяются последовательно сверху вниз, выполняется первое подходящее условие, остальные не вычисляются (short-circuit).
- Ветка `ELSE` необязательна; если она отсутствует и ни одно условие не сработало, результатом будет `NULL`.
- Все ветки `THEN` (и `ELSE`) должны возвращать совместимые по типу значения — СУБД приводит их к общему типу.
- `CASE` — это выражение, а не оператор потока управления, поэтому его нельзя использовать для выполнения нескольких операторов (для этого в хранимых процедурах есть `IF...THEN...ELSE`).
- Часто применяется для "условной агрегации" — например, для построения сводных отчётов (pivot) без специального оператора `PIVOT`.

## Пример / когда применяется

Классический пример условной агрегации для построения отчёта по продажам по регионам:

```sql
SELECT
    product_id,
    SUM(CASE WHEN region = 'EU' THEN amount ELSE 0 END) AS eu_sales,
    SUM(CASE WHEN region = 'US' THEN amount ELSE 0 END) AS us_sales
FROM sales
GROUP BY product_id;
```

Пример использования в `ORDER BY` для кастомной сортировки статусов:

```sql
SELECT * FROM tasks
ORDER BY
    CASE status
        WHEN 'urgent' THEN 1
        WHEN 'normal' THEN 2
        ELSE 3
    END;
```

## На что смотрит интервьюер

Интервьюер проверяет, знает ли кандидат обе формы `CASE` (simple и searched) и понимает ли, что это выражение, а не оператор управления потоком. Частая ошибка — забывать про `ELSE` и не понимать, что тогда результатом будет `NULL`. Также ожидается умение применить `CASE` для условной агрегации (pivot-like отчёты) — это частый практический вопрос на собеседовании.
