# Что такое collation?

**Коротко:** Collation — это набор правил, определяющих, как СУБД сравнивает, сортирует и оценивает на равенство строковые данные с учётом кодировки, регистра, ударений и локали.

## Развёрнутый ответ

Collation (правило сортировки/сравнения) задаёт три ключевых аспекта работы со строковыми значениями в базе данных:

- **Порядок сортировки (order)** — например, должна ли буква "ё" идти после "е" или в конце алфавита, как сортируются символы с диакритическими знаками (á, é, ü) относительно базовых латинских букв.
- **Чувствительность к регистру (case sensitivity, CS/CI)** — считается ли `'Apple' = 'apple'` истинным или ложным при сравнении.
- **Чувствительность к акцентам (accent sensitivity, AS/AI)** — считается ли `'café' = 'cafe'` равенством.

Collation обычно указывается в имени вида `<локаль>_<чувствительность>`, например `SQL_Latin1_General_CP1_CI_AS` (CI — case insensitive, AS — accent sensitive) в SQL Server, или `utf8mb4_general_ci` / `utf8mb4_bin` в MySQL, или `en_US.UTF-8` в PostgreSQL (где collation по большей части управляется через локаль ОС/ICU).

Collation можно задавать на нескольких уровнях иерархии, причём более специфичный уровень переопределяет общий:

1. Уровень сервера (инстанса СУБД) — collation по умолчанию для всех новых баз.
2. Уровень базы данных — переопределяет серверный.
3. Уровень столбца таблицы — переопределяет collation базы данных.
4. Уровень конкретного выражения в запросе (`COLLATE` в предложении) — переопределяет всё вышестоящее для этого сравнения.

Практическая значимость collation: неправильный или несовпадающий collation при `JOIN` двух таблиц с разными collation вызовет ошибку "Cannot resolve collation conflict" (в SQL Server) или потребует явного `COLLATE` для приведения к общему виду. Также collation напрямую влияет на поведение `LIKE`, `ORDER BY`, уникальных индексов (регистронезависимый collation сделает `'User'` и `'user'` дубликатом в уникальном индексе) и на производительность (некоторые collation, например бинарные, сравниваются быстрее лексикографических).

## Пример / когда применяется

Пример конфликта collation при join:

```sql
SELECT a.name, b.value
FROM table_a a COLLATE SQL_Latin1_General_CP1_CI_AS a
JOIN table_b b COLLATE French_CI_AS b
    ON a.name = b.name; -- ошибка конфликта collation
```

Решение — явно привести collation в предложении сравнения:

```sql
SELECT a.name, b.value
FROM table_a a
JOIN table_b b
    ON a.name COLLATE SQL_Latin1_General_CP1_CI_AS = b.name COLLATE SQL_Latin1_General_CP1_CI_AS;
```

Пример регистронезависимого поиска в PostgreSQL через явное указание:

```sql
SELECT * FROM users WHERE email = 'Test@Mail.com' COLLATE "und-x-icu";
```

## На что смотрит интервьюер

Интервьюер проверяет понимание того, что collation влияет не только на сортировку, но и на равенство строк, уникальность и индексы. Частая ошибка — путать collation с кодировкой символов (charset/encoding) — это разные, хотя и связанные, понятия: кодировка определяет, как символы хранятся в байтах, а collation — как они сравниваются и сортируются. Красный флаг — незнание, что разный collation у сравниваемых столбцов может привести к ошибке при `JOIN`.
