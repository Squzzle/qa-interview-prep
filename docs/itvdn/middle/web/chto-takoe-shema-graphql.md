# Что такое схема GraphQL?

**Коротко:** Схема GraphQL — это строго типизированное описание всех данных и операций, доступных в API: какие типы существуют, какие поля у них есть и какие запросы, мутации и подписки может выполнить клиент.

## Развёрнутый ответ

Схема (Schema) — центральный контракт между сервером и клиентом в GraphQL. Она описывается на специальном языке определения схемы (SDL — Schema Definition Language) и определяет:

- **Типы объектов (Object types)** — сущности предметной области с набором типизированных полей, например `type User { id: ID!, name: String!, email: String }`.
- **Скалярные типы (Scalars)** — базовые типы данных: `Int`, `Float`, `String`, `Boolean`, `ID`, а также кастомные скаляры (например, `Date`).
- **Корневые типы операций** — `Query` (точки входа для чтения данных), `Mutation` (точки входа для изменения данных) и `Subscription` (точки входа для realtime-подписок через WebSocket).
- **Enum, Interface, Union, Input types** — дополнительные конструкции для моделирования более сложных структур данных и переиспользуемых входных параметров мутаций.
- **Модификаторы обязательности** — восклицательный знак `!` означает non-nullable поле (не может вернуть `null`), квадратные скобки `[Type]` означают список.

Схема служит одновременно: (1) документацией API — через неё генерируется интроспекция (`introspection query`), на основе которой работают инструменты вроде GraphiQL, GraphQL Playground, Apollo Studio; (2) контрактом валидации — сервер отклонит запрос, если он не соответствует схеме (несуществующее поле, неверный тип аргумента); (3) основой для строгой типизации на клиенте — генераторы кода (например, GraphQL Code Generator) создают TypeScript-типы прямо из схемы.

Схема строится по подходу "schema-first" (сначала описывается SDL, затем реализуются резолверы) либо "code-first" (схема генерируется из кода — классов/декораторов, как в NestJS с `@ObjectType()`).

## Пример / когда применяется

Пример простой SDL-схемы интернет-магазина:

```graphql
type Product {
  id: ID!
  name: String!
  price: Float!
  inStock: Boolean!
}

type Query {
  products(category: String): [Product!]!
  product(id: ID!): Product
}

input ProductInput {
  name: String!
  price: Float!
}

type Mutation {
  createProduct(input: ProductInput!): Product!
}
```

Такой контракт позволяет тестировщику заранее понять, какие поля обязательны (`String!`), какие запросы доступны, и написать негативный тест — например, отправить запрос без обязательного поля `name` и проверить, что сервер вернёт ошибку валидации ещё до вызова резолвера.

## На что смотрит интервьюер

Интервьюер оценивает понимание кандидатом того, что схема — это строгий типизированный контракт, обеспечивающий валидацию на уровне сервера ещё до выполнения бизнес-логики. Часто проверяют знание разницы между `Query`, `Mutation` и `Subscription`, а также смысл модификатора `!` (non-null). Красный флаг — путаница между GraphQL-схемой и REST-спецификацией (OpenAPI/Swagger) как эквивалентными по возможностям — GraphQL-схема встроена в протокол и обязательна для работы сервера, тогда как OpenAPI — внешнее описание поверх REST.
