# Что вы понимаете под Self Join? Приведите примеры.

**Коротко:** Self Join — это соединение таблицы саму с собой (через алиасы), которое применяется для сравнения строк внутри одной таблицы, например для построения иерархий или поиска дубликатов.

## Развёрнутый ответ

Self Join (самосоединение) не является отдельным видом синтаксиса `JOIN` — это обычный `INNER JOIN`, `LEFT JOIN` и т.д., в котором таблица соединяется сама с собой. Поскольку в SQL нельзя дважды сослаться на одно и то же имя таблицы в одном запросе без разграничения, таблице обязательно присваиваются два разных алиаса (псевдонима), которые СУБД трактует как две независимые копии для целей соединения.

Типичные сценарии применения Self Join:

- **Иерархические/древовидные данные** — таблица сотрудников, где у каждого сотрудника есть `manager_id`, ссылающийся на `id` того же сотрудника-менеджера в той же таблице.
- **Поиск дубликатов** — сравнение строк друг с другом по совпадающим полям, но с разными первичными ключами.
- **Сравнение пар записей** — например, поиск пар товаров с одинаковой ценой, но разными артикулами, или сравнение показателей "текущий месяц vs предыдущий месяц" в одной таблице временных рядов.
- **Обнаружение пересечений/конфликтов** — например, поиск пересекающихся по времени бронирований одного и того же ресурса.

Важно, что self join логически ничем не отличается от join двух разных таблиц — оптимизатор запросов обрабатывает его так же, опираясь на индексы по ключам соединения. Единственная специфика — обязательное использование алиасов и внимательность к направлению `LEFT/RIGHT JOIN` (например, `LEFT JOIN`, чтобы получить сотрудников без менеджера, если для них `manager_id IS NULL`).

## Пример / когда применяется

Классический пример — иерархия "сотрудник — менеджер":

```sql
SELECT
    e.name AS employee_name,
    m.name AS manager_name
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.id;
```

Здесь `e` — алиас "сотрудник", `m` — алиас "менеджер этого же сотрудника"; `LEFT JOIN` гарантирует, что топ-менеджер (у которого `manager_id IS NULL`) тоже попадёт в результат с `NULL` в колонке `manager_name`.

Пример поиска дубликатов email в таблице пользователей:

```sql
SELECT DISTINCT a.id, a.email
FROM users a
JOIN users b ON a.email = b.email AND a.id <> b.id;
```

Условие `a.id <> b.id` необходимо, чтобы не сравнивать строку саму с собой (иначе каждая запись "совпадёт" сама с собой).

## На что смотрит интервьюер

Интервьюер проверяет, понимает ли кандидат, что self join — это не особый оператор, а обычный join с двумя алиасами одной таблицы, и умеет ли кандидат написать рабочий пример (обычно иерархию сотрудник-менеджер или поиск дубликатов). Частая ошибка — забыть условие `a.id <> b.id` при поиске пар/дубликатов, из-за чего строка соединяется сама с собой. Красный флаг — непонимание, зачем нужны алиасы именно в self join.
