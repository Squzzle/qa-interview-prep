# Что такое NIC?

**Коротко:** NIC (Network Interface Card, сетевой адаптер) — это аппаратный или виртуальный компонент, обеспечивающий физическое или логическое подключение устройства к сети и обмен данными по сетевому протоколу (обычно Ethernet).

## Развёрнутый ответ

NIC — сокращение от Network Interface Controller/Card. Это устройство (плата, встроенный чип на материнской плате, USB-адаптер или виртуальный интерфейс в виртуальной машине/контейнере), которое реализует физический и канальный уровни модели OSI: преобразует данные из компьютера в электрические, оптические или радиосигналы и обратно, формирует и обрабатывает кадры (frames) канального уровня.

Ключевые характеристики и функции NIC:

- Каждая физическая NIC имеет уникальный аппаратный адрес — MAC-адрес (48 бит, назначается производителем), который используется для адресации на канальном уровне (уровень 2 OSI).
- NIC поддерживает определённый тип среды передачи: проводной Ethernet (`RJ-45`), Wi-Fi (беспроводной адаптер), оптоволокно (SFP-модули) и т.д.
- Современные NIC умеют выполнять offloading — часть работы по обработке трафика (checksum offload, TCP segmentation offload, RSS — Receive Side Scaling) берёт на себя сама карта, снижая нагрузку на CPU.
- В серверах и системах виртуализации существуют виртуальные NIC (vNIC) — программные интерфейсы, которые эмулируют физическую карту для виртуальных машин или контейнеров (например, `veth`-пары в Docker/Kubernetes, virtio-net в KVM).
- Драйвер NIC — программное обеспечение, которое связывает операционную систему с аппаратной частью, обрабатывает прерывания и передаёт данные в стек протоколов ОС.

Для тестировщика важно понимать NIC в контексте: диагностики сетевых проблем (недоступность хоста, потеря пакетов), настройки тестовых окружений (multiple NIC для изоляции трафика, VLAN), нагрузочного тестирования (пропускная способность интерфейса как узкое место), а также при тестировании отказоустойчивости (bonding/teaming нескольких NIC).

## Пример / когда применяется

Пример из практики: на тестовом сервере установлено две сетевые карты — одна (`eth0`) подключена к внутренней сети для управления, вторая (`eth1`) — к сети тестового стенда, где генерируется нагрузочный трафик. Это позволяет изолировать служебный трафик от тестового и избежать искажения результатов нагрузочного тестирования.

Просмотр сетевых интерфейсов в Linux:

```bash
ip link show
# или устаревший вариант
ifconfig -a
```

Пример вывода:

```
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc mq state UP
    link/ether 00:1a:2b:3c:4d:5e brd ff:ff:ff:ff:ff:ff
```

## На что смотрит интервьюер

- Понимает ли кандидат разницу между MAC-адресом (уровень 2) и IP-адресом (уровень 3) — частая путаница.
- Знает ли про существование виртуальных NIC и их роль в контейнерах/виртуальных машинах — важно для тестировщиков, работающих с Docker/Kubernetes.
- Может ли объяснить, зачем нужно несколько NIC на одном хосте (изоляция трафика, отказоустойчивость, балансировка нагрузки).
- Красный флаг: если кандидат путает NIC с сетевым протоколом (TCP/IP) или не может назвать хотя бы базовую команду для просмотра интерфейсов.
- Follow-up: "Что произойдёт, если два устройства в одной сети получат одинаковый MAC-адрес?", "Как NIC участвует в ARP-резолюции?".
