# Назовите основные артефакты Scrum.

**Коротко:** В Scrum три основных артефакта: Product Backlog (полный список требований к продукту), Sprint Backlog (задачи, выбранные на текущий спринт) и Increment (готовый прирост продукта по итогам спринта); каждый артефакт содержит "обязательство" (commitment), повышающее прозрачность.

## Развёрнутый ответ

Артефакты Scrum представляют работу или ценность, обеспечивая прозрачность и точки для проверки прогресса:

1. **Product Backlog** — упорядоченный, постоянно уточняемый список всего, что может понадобиться в продукте: фичи, требования, улучшения, исправления. Ведёт его Product Owner. Обязательство (commitment), связанное с этим артефактом — **Product Goal** (цель продукта), которая описывает будущее состояние продукта и служит долгосрочным ориентиром при выборе элементов бэклога.

2. **Sprint Backlog** — набор элементов Product Backlog, выбранных для текущего спринта, вместе с планом по их реализации. Формируется на Sprint Planning силами Development Team. Обязательство здесь — **Sprint Goal** (цель спринта), единая цель, которая объединяет отдельные задачи спринта и даёт команде гибкость в деталях реализации.

3. **Increment (инкремент)** — сумма всех выполненных элементов Product Backlog за текущий спринт вместе со всеми предыдущими инкрементами, соответствующая критериям Definition of Done. Обязательство здесь — **Definition of Done** (определение готовности) — формальное описание того, каким критериям должен соответствовать инкремент, чтобы считаться завершённым (например, код прошёл ревью, тесты написаны и пройдены, документация обновлена).

Каждый из трёх commitment-элементов (Product Goal, Sprint Goal, Definition of Done) был формализован в редакции Scrum Guide 2020 года, чтобы усилить эмпирический контроль процесса — прозрачность, инспекцию и адаптацию.

## Пример / когда применяется

Пример: Product Backlog интернет-магазина содержит 50 user stories с общей Product Goal "Стать удобной платформой для покупок с мобильного устройства". На текущий спринт команда берёт 8 историй в Sprint Backlog с целью (Sprint Goal) "Реализовать оформление заказа в одно касание". По итогам спринта готовые и протестированные фичи, соответствующие Definition of Done (например, "покрыты автотестами, прошли code review, задеплоены на staging"), формируют Increment.

| Артефакт | Что содержит | Кто ведёт | Связанный commitment |
|---|---|---|---|
| Product Backlog | Весь список требований к продукту | Product Owner | Product Goal |
| Sprint Backlog | Задачи текущего спринта + план | Development Team | Sprint Goal |
| Increment | Готовый прирост продукта | Development Team | Definition of Done |

## На что смотрит интервьюер

Частая ошибка — забывать про commitment-элементы (Product Goal, Sprint Goal, Definition of Done) и называть только сами артефакты. Красный флаг — если кандидат путает Increment с релизом (инкремент может не выкатываться в продакшн сразу). Возможные follow-up вопросы: "Что такое Definition of Done и чем оно отличается от Definition of Ready", "Может ли один спринт содержать несколько инкрементов", "Кто решает, входит ли задача в определение готовности".
