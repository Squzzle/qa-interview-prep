# Что такое cURL и для чего он используется?

**Коротко:** cURL — это консольная утилита и библиотека для передачи данных по различным сетевым протоколам (HTTP, HTTPS, FTP и др.), которая используется для отправки запросов к API прямо из командной строки без графического интерфейса.

## Развёрнутый ответ

cURL (Client URL) — кроссплатформенный инструмент командной строки, позволяющий формировать и отправлять сетевые запросы, а также получать и обрабатывать ответы. Поддерживает множество протоколов: HTTP, HTTPS, FTP, SFTP и другие, но в контексте тестирования API чаще всего используется именно для HTTP(S)-запросов.

Для QA и разработчиков cURL полезен по нескольким причинам:

- **Быстрая ручная проверка API** без необходимости открывать Postman или писать код — достаточно одной команды в терминале.
- **Полный контроль над запросом**: можно задать любой HTTP-метод, заголовки, тело запроса, куки, параметры аутентификации, таймауты.
- **Воспроизводимость и автоматизация** — команды cURL легко вставить в bash-скрипты, CI/CD pipeline, документацию (многие API-провайдеры дают примеры именно в виде cURL-команд).
- **Диагностика проблем** — можно посмотреть полный HTTP-ответ, включая заголовки, статус-код, время выполнения запроса (флаг `-v` для verbose-режима, `-i` для вывода заголовков ответа).
- **Отсутствие зависимостей от GUI** — cURL доступен практически на любой Unix-подобной системе "из коробки" и часто предустановлен, что удобно для тестирования на серверах или в контейнерах без графического интерфейса.

Основные полезные флаги cURL: `-X` (HTTP-метод), `-H` (заголовок), `-d`/`--data` (тело запроса), `-i` (показать заголовки ответа), `-v` (подробный вывод, включая процесс установки соединения), `-o` (сохранить ответ в файл), `-u` (базовая аутентификация), `-L` (следовать редиректам), `-k` (игнорировать проверку SSL-сертификата, полезно на тестовых окружениях с самоподписанными сертификатами).

## Пример / когда применяется

Простой GET-запрос:

```bash
curl https://api.example.com/users/42
```

GET-запрос с выводом заголовков ответа и подробным логом:

```bash
curl -i -v https://api.example.com/users/42
```

Запрос с авторизацией через Bearer-токен:

```bash
curl -H "Authorization: Bearer eyJhbGciOi..." https://api.example.com/orders
```

cURL применяется для быстрой проверки эндпоинтов при ручном тестировании, воспроизведения багов из логов (когда нужно повторить конкретный запрос), а также как часть shell-скриптов для smoke-тестов или проверок в CI/CD (health-check эндпоинта после деплоя).

## На что смотрит интервьюер

- Понимание, что cURL — это инструмент для отправки любых сетевых запросов, а не только "аналог Postman".
- Знание базовых флагов (`-X`, `-H`, `-d`, `-i`, `-v`) и умение объяснить их назначение.
- Способность объяснить практическую пользу cURL в тестировании: воспроизводимость, автоматизация, диагностика, отсутствие зависимости от GUI.
- Красный флаг — если кандидат не может назвать ни одного флага или не понимает, чем отличается `-d` от `-H`.
