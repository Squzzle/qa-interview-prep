# Что такое Postman? Для чего он используется в тестировании API?

**Коротко:** Postman — это графический инструмент (GUI-клиент) для разработки, отправки, тестирования и документирования HTTP-запросов к API, широко используемый для ручного и автоматизированного тестирования веб-сервисов.

## Развёрнутый ответ

Postman предоставляет удобный интерфейс для работы с API, снимающий необходимость писать код или запоминать синтаксис командной строки (как в cURL). Основные возможности, важные для тестирования API:

- **Формирование и отправка запросов** любого HTTP-метода (GET, POST, PUT, PATCH, DELETE и др.) с настройкой заголовков, параметров, тела запроса (JSON, form-data, x-www-form-urlencoded, raw, binary), авторизации (Basic Auth, Bearer Token, OAuth 2.0, API Key).
- **Коллекции (Collections)** — группировка связанных запросов в логические наборы, что удобно для организации тестов по фичам/модулям API.
- **Переменные (Variables)** — окружения (Environments), глобальные и локальные переменные, позволяющие параметризовать запросы (например, base URL, токены) и переключаться между окружениями (dev/staging/prod) без правки самих запросов.
- **Тестовые скрипты** — встроенный JavaScript-движок (`pm.test`, библиотека Chai для ассертов) позволяет писать проверки прямо внутри запроса: статус-код, структура ответа, значения полей, время ответа.
- **Pre-request скрипты** — код, который выполняется перед отправкой запроса (например, генерация токена, подготовка данных).
- **Автоматизация и CI/CD** — коллекции можно экспортировать и запускать через Newman (CLI-раннер Postman) в pipeline, что позволяет включать проверки API в автоматическую сборку.
- **Mock-серверы** — Postman умеет поднимать mock-эндпоинты на основе примеров запросов/ответов, что полезно, когда реальный бэкенд ещё не готов.
- **Документация** — автоматическая генерация читаемой документации API на основе коллекций.
- **Monitors** — возможность запускать коллекции по расписанию для мониторинга доступности и работоспособности API.

Postman используется как для ручного разового тестирования отдельных эндпоинтов (быстро проверить, что вернёт API при определённых параметрах), так и для построения полноценных наборов регрессионных тестов API, интегрированных в CI/CD.

## Пример / когда применяется

Пример теста внутри запроса Postman (вкладка Tests):

```javascript
pm.test("Статус ответа 200", function () {
  pm.response.to.have.status(200);
});

pm.test("Тело содержит корректный email", function () {
  const json = pm.response.json();
  pm.expect(json.email).to.eql("ivan@example.com");
});
```

Пример запуска коллекции через Newman в CI:

```bash
newman run collection.json -e staging.postman_environment.json
```

Postman применяется на этапе ручного исследовательского тестирования API, при отладке интеграций, а также как инструмент регрессионного тестирования API в CI/CD через Newman.

## На что смотрит интервьюер

- Понимание, что Postman — не только "отправитель запросов", а полноценная платформа с коллекциями, переменными, тестовыми скриптами и CI-интеграцией.
- Умение объяснить разницу между окружениями (Environments) и глобальными переменными.
- Знание, как автоматизировать запуск коллекций (Newman) вне интерфейса Postman.
- Способность привести пример теста на JavaScript внутри запроса.
- Красный флаг — если кандидат знает Postman только как "кнопку Send" и не упоминает тестовые скрипты, переменные или автоматизацию.
