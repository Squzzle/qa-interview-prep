# Что такое SOAP API? В каком формате передаются данные?

**Коротко:** SOAP (Simple Object Access Protocol) — протокол обмена структурированными сообщениями между приложениями, все данные передаются строго в формате XML по чётко описанной спецификации.

## Развёрнутый ответ

SOAP — это протокол (в отличие от REST, который является архитектурным стилем), поэтому у него есть жёсткая спецификация сообщений. Каждое SOAP-сообщение представляет собой XML-документ (SOAP envelope), состоящий из следующих частей:

- **Envelope** — корневой элемент, обязательная обёртка сообщения.
- **Header** (опционально) — метаданные: аутентификация, транзакции, маршрутизация.
- **Body** — само содержимое сообщения: вызываемый метод и его параметры (в запросе) или результат/ошибка (в ответе).
- **Fault** (внутри Body, опционально) — стандартизированная структура для описания ошибок.

SOAP чаще всего работает поверх HTTP или HTTPS, но протокол не привязан к конкретному транспорту — может использоваться и через SMTP, JMS и другие. Контракт (описание доступных операций, типов параметров, адреса сервиса) описывается в отдельном XML-документе WSDL (Web Services Description Language).

Ключевые особенности SOAP:
- Строгая типизация данных через XML Schema (XSD).
- Встроенная поддержка расширений: WS-Security (шифрование, подпись сообщений), WS-ReliableMessaging (гарантированная доставка), WS-AtomicTransaction (распределённые транзакции).
- Протокол-независимость от транспорта.
- Более высокая избыточность (verbose) сообщений по сравнению с JSON в REST — что увеличивает объём трафика и снижает производительность.

SOAP исторически широко применялся в enterprise-системах, банковском секторе, госуслугах, там, где критичны строгая контрактность, безопасность и надёжность доставки сообщений.

## Пример / когда применяется

Пример SOAP-запроса для получения курса валюты:

```xml
<?xml version="1.0"?>
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Header/>
  <soap:Body>
    <GetExchangeRate xmlns="http://example.com/rates">
      <FromCurrency>USD</FromCurrency>
      <ToCurrency>RUB</ToCurrency>
    </GetExchangeRate>
  </soap:Body>
</soap:Envelope>
```

Пример ответа с ошибкой (Fault):

```xml
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Body>
    <soap:Fault>
      <faultcode>Client</faultcode>
      <faultstring>Invalid currency code</faultstring>
    </soap:Fault>
  </soap:Body>
</soap:Envelope>
```

SOAP до сих пор широко используется в банковских интеграциях, платёжных системах, госуслугах (например, СМЭВ в России), где важны безопасность и формальная контрактность.

## На что смотрит интервьюер

- Знает ли кандидат, что SOAP всегда XML, и не путает его с REST+XML.
- Понимает ли структуру SOAP-сообщения (Envelope, Header, Body, Fault).
- Знает ли про WSDL как описание контракта SOAP-сервиса.
- Follow-up: "почему SOAP тяжелее REST", "где до сих пор используется SOAP и почему".
- Красный флаг: утверждение, что SOAP "устарел и нигде не используется" — на практике он активно применяется в банках, страховании, госсекторе.
