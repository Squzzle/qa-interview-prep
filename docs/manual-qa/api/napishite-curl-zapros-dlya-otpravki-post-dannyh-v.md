# Напишите cURL-запрос для отправки POST-данных в формате JSON.

**Коротко:** Нужно указать метод `POST` через `-X`, заголовок `Content-Type: application/json` через `-H` и передать тело запроса в JSON через `-d` (или `--data`).

## Развёрнутый ответ

Чтобы отправить POST-запрос с телом в формате JSON через cURL, необходимо явно указать три вещи:

1. **HTTP-метод** — флаг `-X POST` (при передаче данных через `-d` метод POST подставляется по умолчанию, но явное указание считается хорошим тоном и обязательно, если нужен другой метод, например `PUT` или `PATCH`, при отправке тела).
2. **Заголовок Content-Type** — обязательно `-H "Content-Type: application/json"`, иначе сервер может интерпретировать тело как `application/x-www-form-urlencoded` (значение по умолчанию для `-d`) и не сможет корректно распарсить JSON.
3. **Тело запроса** — флаг `-d` (или `--data`, `--data-raw`) с JSON-строкой.

Дополнительно часто добавляют:
- Заголовок авторизации, если эндпоинт защищён: `-H "Authorization: Bearer <token>"`.
- Флаг `-i`, чтобы увидеть заголовки и статус-код ответа.
- Для сложного/многострочного JSON удобнее вынести тело в отдельный файл и передать его через `-d @file.json`, чтобы не экранировать кавычки вручную.

Важный нюанс: при использовании `--data-raw` cURL не интерпретирует символ `@` в начале строки как указание на файл (в отличие от `--data`/`-d`), что полезно, если само значение в JSON начинается с `@` (например, email-адрес).

## Пример / когда применяется

Базовый пример POST-запроса с JSON-телом:

```bash
curl -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Иван", "email": "ivan@example.com"}'
```

С выводом статус-кода и заголовков ответа, а также авторизацией:

```bash
curl -i -X POST https://api.example.com/orders \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOi..." \
  -d '{"product_id": 101, "quantity": 2}'
```

С телом запроса из отдельного файла (удобно для больших/сложных JSON):

```bash
curl -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -d @payload.json
```

Такой запрос применяется при ручной проверке эндпоинтов создания ресурсов (регистрация пользователя, создание заказа), а также в smoke-тестах и скриптах CI/CD для проверки работоспособности API после деплоя.

## На что смотрит интервьюер

- Знание, что без заголовка `Content-Type: application/json` сервер может неправильно распарсить тело запроса.
- Понимание разницы между `-d`, `--data-raw` и `--data-binary` (последний не удаляет переносы строк и не интерпретирует `@`/`#`).
- Умение объяснить, что метод `POST` подставляется автоматически при использовании `-d`, но явное указание `-X POST` делает команду более читаемой и обязательно при использовании PUT/PATCH.
- Красный флаг — если кандидат забывает про заголовок `Content-Type` или не может написать синтаксически корректный JSON в теле запроса.
