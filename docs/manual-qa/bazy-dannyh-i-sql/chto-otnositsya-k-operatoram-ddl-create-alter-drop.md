# Что относится к операторам DDL (CREATE, ALTER, DROP)?

**Коротко:** DDL (Data Definition Language) — группа операторов SQL, отвечающая за создание и изменение структуры базы данных: таблиц, индексов, представлений, схем. Основные операторы: `CREATE`, `ALTER`, `DROP`, а также `TRUNCATE` и `RENAME`.

## Развёрнутый ответ

Операторы DDL работают не с содержимым данных, а со структурой (схемой) базы данных.

- **`CREATE`** — создаёт новый объект базы данных: таблицу, индекс, представление, схему, хранимую процедуру, триггер и т.д. Например, `CREATE TABLE`, `CREATE INDEX`, `CREATE VIEW`, `CREATE DATABASE`.
- **`ALTER`** — изменяет структуру уже существующего объекта: добавляет или удаляет столбцы, изменяет типы данных, добавляет ограничения (constraints), переименовывает объекты. Например, `ALTER TABLE ... ADD COLUMN`, `ALTER TABLE ... DROP COLUMN`, `ALTER TABLE ... MODIFY COLUMN`.
- **`DROP`** — полностью удаляет объект вместе со всеми его данными и структурой. Например, `DROP TABLE`, `DROP INDEX`, `DROP DATABASE`. Это необратимая операция (если не восстанавливать из резервной копии).
- **`TRUNCATE`** — быстро удаляет все строки из таблицы, сохраняя саму структуру таблицы. В отличие от `DELETE`, `TRUNCATE` не логирует удаление построчно, обычно сбрасывает счётчики автоинкремента и работает значительно быстрее на больших таблицах, но не позволяет использовать условие `WHERE`.
- **`RENAME`** — переименовывает объект (таблицу, столбец), поддерживается не во всех СУБД как отдельный оператор (иногда реализуется через `ALTER TABLE ... RENAME TO`).

Важная особенность DDL: в большинстве СУБД (например, MySQL) операторы DDL выполняют неявный commit — то есть они автоматически фиксируются и не могут быть отменены командой `ROLLBACK`, даже если выполняются внутри блока транзакции. В некоторых СУБД (например, PostgreSQL) DDL можно выполнять транзакционно и откатить, что является отличительной особенностью этой СУБД.

## Пример / когда применяется

```sql
-- Создание таблицы
CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    salary DECIMAL(10,2)
);

-- Изменение структуры: добавление столбца
ALTER TABLE employees ADD COLUMN hired_at DATE;

-- Изменение типа столбца
ALTER TABLE employees ALTER COLUMN salary TYPE NUMERIC(12,2);

-- Быстрая очистка таблицы без удаления структуры
TRUNCATE TABLE employees;

-- Полное удаление таблицы
DROP TABLE employees;
```

## На что смотрит интервьюер

- Понимание, что DDL работает со структурой, а не с содержимым данных (в отличие от DML).
- Знание разницы между `DROP`, `TRUNCATE` и `DELETE` — это очень частый follow-up вопрос: `DROP` удаляет объект целиком, `TRUNCATE` очищает данные без построчного лога и без `WHERE`, `DELETE` — DML-оператор, удаляющий строки построчно и поддерживающий `WHERE` и откат в транзакции.
- Понимание, что DDL-операции обычно необратимы или не откатываются транзакцией (зависит от СУБД).
- Частая ошибка — считать, что `ROLLBACK` всегда отменит `DROP TABLE` или `ALTER TABLE`; в большинстве СУБД это не так.
- Хороший ответ приводит конкретные примеры операторов и объясняет практические последствия (потеря данных при `DROP`, блокировки таблицы при `ALTER` на больших объёмах данных).
