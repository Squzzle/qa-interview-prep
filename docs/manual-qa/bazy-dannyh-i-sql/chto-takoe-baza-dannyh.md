# Что такое база данных?

**Коротко:** база данных — это организованная, структурированная совокупность данных, хранимая и управляемая таким образом, чтобы её можно было эффективно создавать, читать, изменять, удалять и защищать от повреждений.

## Развёрнутый ответ

База данных (БД) — это структурированный набор данных, организованный по определённым правилам и предназначенный для длительного хранения во внешней памяти компьютера. Данные в БД организуются так, чтобы обеспечить удобный доступ к ним, целостность, непротиворечивость и возможность совместного использования несколькими пользователями и приложениями одновременно.

Ключевые характеристики базы данных:

- Данные структурированы по определённой модели (реляционной, документной, графовой и т.д.).
- Обеспечивается независимость данных от конкретных приложений — одни и те же данные могут использоваться разными программами.
- Поддерживается целостность данных (integrity) — данные соответствуют бизнес-правилам и ограничениям.
- Реализуется контроль доступа и безопасность — разграничение прав пользователей.
- Обеспечивается устойчивость к сбоям и возможность восстановления (резервное копирование, журналирование).

Важно отличать базу данных от системы управления базами данных (СУБД). База данных — это сами данные и их структура, а СУБД — программное обеспечение, которое позволяет создавать, изменять и администрировать эти данные (например, MySQL, PostgreSQL, MongoDB).

## Пример / когда применяется

Например, интернет-магазин хранит в базе данных информацию о товарах, пользователях, заказах и платежах. Структура может выглядеть так:

```sql
CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL
);

CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    customer_id INT REFERENCES customers(id),
    total NUMERIC(10, 2),
    created_at TIMESTAMP DEFAULT now()
);
```

Такая база данных позволяет быстро находить заказы конкретного клиента, считать выручку, строить отчёты и гарантировать, что заказ не будет создан без существующего клиента.

## На что смотрит интервьюер

Интервьюер проверяет, понимает ли кандидат разницу между "база данных" и "СУБД" — частая ошибка новичков в тестировании путать эти понятия. Также важно, чтобы кандидат упомянул структурированность и организованность данных, а не просто сказал "это место, где хранятся данные".
