# Что такое NoSQL базы данных? Какие типы NoSQL БД бывают?

**Коротко:** NoSQL (Not Only SQL) — это класс баз данных, отличных от реляционных, которые не требуют жёсткой схемы данных, легче масштабируются горизонтально и оптимизированы под конкретные сценарии доступа к данным.

## Развёрнутый ответ

NoSQL-базы данных появились как ответ на потребности высоконагруженных веб-приложений, которым требовалась горизонтальная масштабируемость, гибкая схема данных и высокая производительность на больших объёмах данных. В отличие от реляционных БД, NoSQL обычно жертвует частью строгих гарантий согласованности (следуя модели **BASE** — Basically Available, Soft state, Eventual consistency — вместо ACID), но при этом обеспечивает лучшую масштабируемость и гибкость.

Основные типы NoSQL баз данных:

- **Документо-ориентированные (Document Store)** — данные хранятся как документы (обычно JSON/BSON) с произвольной вложенной структурой. Примеры: MongoDB, CouchDB. Подходят для каталогов товаров, профилей пользователей, CMS.
- **Ключ-значение (Key-Value Store)** — простейшая модель, где данные хранятся как пары "ключ - значение". Примеры: Redis, DynamoDB, Memcached. Подходят для кэширования, сессий, счётчиков.
- **Колоночные (Column-family Store)** — данные хранятся по столбцам, а не по строкам, что ускоряет аналитические выборки по отдельным колонкам. Примеры: Cassandra, HBase, ClickHouse. Подходят для больших аналитических хранилищ и временных рядов.
- **Графовые (Graph Database)** — данные представлены узлами и рёбрами с атрибутами, оптимизированы для запросов по связям. Примеры: Neo4j, ArangoDB. Подходят для социальных сетей, рекомендательных систем, анализа мошенничества.

Основные отличия NoSQL от SQL: отсутствие жёсткой схемы (schema-less или schema-flexible), горизонтальное масштабирование (шардирование по кластеру), часто отказ от полноценных JOIN-операций и транзакций между несколькими документами (хотя современные версии, например MongoDB, поддерживают транзакции).

## Пример / когда применяется

Документ пользователя в MongoDB:

```json
{
  "_id": "u123",
  "name": "Иван Иванов",
  "orders": [
    { "product": "Ноутбук", "price": 55000 },
    { "product": "Мышь", "price": 800 }
  ]
}
```

Такая структура удобна, когда у разных пользователей могут быть разные наборы полей и не требуется строгая нормализация, как в реляционной БД. NoSQL часто применяется в кэшировании сессий (Redis), логировании и метриках (Cassandra), поиске (Elasticsearch, который тоже относится к NoSQL-семейству).

## На что смотрит интервьюер

Интервьюер оценивает, понимает ли кандидат, что NoSQL — это не единая технология, а семейство разных подходов, и умеет ли он назвать хотя бы 3-4 типа с примерами СУБД. Красный флаг — утверждение, что NoSQL "быстрее SQL всегда" без понимания компромиссов (CAP-теорема, eventual consistency).
