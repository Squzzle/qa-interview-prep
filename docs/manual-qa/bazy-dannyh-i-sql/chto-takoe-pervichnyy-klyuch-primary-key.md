# Что такое первичный ключ (Primary Key)?

**Коротко:** первичный ключ — это столбец или набор столбцов таблицы, значение которого уникально идентифицирует каждую строку и не может быть пустым (NULL).

## Развёрнутый ответ

Первичный ключ (Primary Key, PK) обеспечивает уникальную идентификацию каждой записи в таблице реляционной БД. Он является одним из фундаментальных механизмов обеспечения целостности данных (entity integrity).

Свойства первичного ключа:

- **Уникальность** — значение первичного ключа не может повторяться среди строк таблицы.
- **Обязательность (NOT NULL)** — первичный ключ не может содержать значение NULL, так как NULL означает "неизвестно" и не может однозначно идентифицировать строку.
- **Неизменяемость (желательно)** — значение первичного ключа по возможности не должно изменяться со временем, чтобы не нарушать ссылочную целостность (внешние ключи, ссылающиеся на него).
- **Минимальность** — если ключ составной (из нескольких столбцов), ни один из столбцов нельзя исключить без потери уникальности.

Первичный ключ может быть:

- **Естественным (natural key)** — использует реально существующий уникальный атрибут (например, номер паспорта, ИНН, email).
- **Суррогатным (surrogate key)** — искусственно созданное значение, не имеющее смысловой нагрузки (автоинкрементное целое число, UUID).
- **Составным (composite key)** — состоит из нескольких столбцов, комбинация которых уникальна (часто используется в связующих таблицах для связи "многие ко многим").

СУБД автоматически создаёт уникальный индекс на столбце(ах) первичного ключа, что также ускоряет поиск по этому полю.

## Пример / когда применяется

```sql
CREATE TABLE employees (
    id SERIAL PRIMARY KEY,       -- суррогатный первичный ключ
    email VARCHAR(100) UNIQUE,   -- естественный уникальный атрибут, но не PK
    full_name VARCHAR(150) NOT NULL
);

-- составной первичный ключ в связующей таблице
CREATE TABLE order_items (
    order_id INT REFERENCES orders(id),
    product_id INT REFERENCES products(id),
    quantity INT NOT NULL,
    PRIMARY KEY (order_id, product_id)
);
```

Первичный ключ используется как "точка входа" для внешних ключей других таблиц, а также как основной способ однозначно найти, обновить или удалить конкретную запись.

## На что смотрит интервьюер

Интервьюер проверяет знание базовых свойств PK (уникальность, NOT NULL) и понимание разницы между первичным ключом и просто уникальным столбцом (UNIQUE constraint может допускать NULL, а PK — нет; в таблице может быть несколько UNIQUE-столбцов, но только один PK). Также важно понимание разницы естественного и суррогатного ключа и когда что лучше использовать. Красный флаг — если кандидат путает PK с обычным индексом или не знает, что PK не может быть NULL.
