# Что такое SQL (Structured Query Language)?

**Коротко:** SQL — декларативный язык структурированных запросов для работы с реляционными базами данных: он позволяет описывать, что нужно получить или изменить, а не как именно это выполнить.

## Развёрнутый ответ

SQL (Structured Query Language) — стандартизированный язык, предназначенный для взаимодействия с реляционными системами управления базами данных (РСУБД). С его помощью можно определять структуру данных (таблицы, схемы, ограничения), манипулировать данными (добавлять, изменять, удалять, выбирать), управлять доступом и транзакциями.

Ключевая особенность SQL — декларативность: разработчик описывает, какой результат нужен (например, "выбрать всех пользователей старше 18 лет, отсортированных по имени"), а СУБД сама решает, как оптимально выполнить этот запрос (какой индекс использовать, в каком порядке соединять таблицы). Это отличает SQL от императивных языков программирования, где нужно пошагово описывать алгоритм получения результата.

SQL стандартизирован ANSI/ISO (стандарт SQL-92, SQL:1999, SQL:2003 и последующие ревизии), но каждая СУБД (PostgreSQL, MySQL, Oracle, SQL Server, SQLite) реализует стандарт с собственными расширениями и особенностями (диалектами).

Операторы SQL традиционно делятся на группы:

- **DDL (Data Definition Language)** — определение структуры: `CREATE`, `ALTER`, `DROP`.
- **DML (Data Manipulation Language)** — работа с данными: `SELECT`, `INSERT`, `UPDATE`, `DELETE`.
- **DCL (Data Control Language)** — управление правами доступа: `GRANT`, `REVOKE`.
- **TCL (Transaction Control Language)** — управление транзакциями: `COMMIT`, `ROLLBACK`, `SAVEPOINT`.

SQL используется практически во всех промышленных реляционных базах данных и остаётся одним из самых распространённых языков для работы с данными.

## Пример / когда применяется

Простой пример типового SQL-запроса, объединяющего фильтрацию, сортировку и объединение таблиц:

```sql
SELECT u.name, o.total
FROM users u
JOIN orders o ON o.user_id = u.id
WHERE u.age > 18
ORDER BY u.name;
```

Кандидат должен понимать, что этот запрос декларативно описывает результат, а конкретный план выполнения (какой индекс применить, в каком порядке join) выбирает оптимизатор запросов СУБД.

## На что смотрит интервьюер

- Понимание декларативной природы SQL в противовес императивным языкам.
- Знание базовой классификации операторов (DDL/DML/DCL/TCL) — часто это отдельный follow-up вопрос.
- Понимание, что SQL — стандарт, а конкретные СУБД реализуют его с расширениями (диалектами), из-за чего запросы не всегда переносимы между СУБД без изменений.
- Частая ошибка — путать SQL с самой СУБД (например, говорить "SQL" вместо "MySQL"/"PostgreSQL").
- Хороший ответ показывает понимание, что за декларативным SQL-запросом стоит оптимизатор запросов, строящий план выполнения.
