# Что такое СУБД? Приведите примеры.

**Коротко:** СУБД (система управления базами данных) — это программное обеспечение, которое обеспечивает создание, хранение, изменение, поиск и защиту данных в базе данных, а также управление доступом к ним.

## Развёрнутый ответ

СУБД (DBMS, Database Management System) — это комплекс программ, предоставляющий пользователям и приложениям интерфейс для работы с базой данных. СУБД скрывает от пользователя детали физического хранения данных и предоставляет логический уровень доступа через язык запросов (чаще всего SQL).

Основные функции СУБД:

- Определение структуры данных (создание таблиц, схем, ограничений).
- Манипулирование данными (вставка, обновление, удаление, выборка).
- Обеспечение целостности данных (ограничения, триггеры, транзакции).
- Управление параллельным доступом (блокировки, изоляция транзакций).
- Обеспечение безопасности (аутентификация, авторизация, роли).
- Резервное копирование и восстановление после сбоев.
- Оптимизация выполнения запросов (планировщик запросов, индексы).

СУБД делятся на несколько типов:

- **Реляционные (RDBMS)** — данные хранятся в таблицах, связанных через ключи; используют SQL. Примеры: MySQL, PostgreSQL, Oracle, Microsoft SQL Server, SQLite.
- **NoSQL** — документные (MongoDB), ключ-значение (Redis), колоночные (Cassandra), графовые (Neo4j).
- **NewSQL** — сочетают масштабируемость NoSQL с гарантиями ACID реляционных СУБД (например, CockroachDB, Google Spanner).

## Пример / когда применяется

Тестировщик взаимодействует с СУБД, когда проверяет корректность данных после действий в приложении. Например, после оформления заказа в интернет-магазине можно напрямую проверить БД:

```sql
SELECT id, status, total
FROM orders
WHERE customer_id = 42
ORDER BY created_at DESC
LIMIT 1;
```

Это позволяет убедиться, что заказ действительно создан и его данные соответствуют ожидаемым (сумма, статус, количество позиций), минуя UI.

## На что смотрит интервьюер

Интервьюер хочет услышать чёткое определение СУБД (а не путаницу с "базой данных") и хотя бы 3-4 конкретных примера с указанием типа (реляционная/NoSQL). Красный флаг — если кандидат называет только одну СУБД или не может объяснить, зачем она нужна. Возможные follow-up: "чем реляционная СУБД отличается от NoSQL?", "что такое ACID?", "какие СУБД вы использовали в работе тестировщика?".
