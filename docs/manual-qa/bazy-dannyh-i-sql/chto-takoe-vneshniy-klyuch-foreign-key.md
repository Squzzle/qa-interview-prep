# Что такое внешний ключ (Foreign Key)?

**Коротко:** внешний ключ — это столбец (или набор столбцов) в одной таблице, значение которого ссылается на первичный (или уникальный) ключ другой таблицы, обеспечивая ссылочную целостность между связанными таблицами.

## Развёрнутый ответ

Внешний ключ (Foreign Key, FK) — это механизм, устанавливающий и контролирующий связь между двумя таблицами. Он гарантирует, что значение в дочерней таблице соответствует существующему значению первичного ключа в родительской таблице — это называется **ссылочной целостностью (referential integrity)**.

Основные свойства и правила внешнего ключа:

- Значение внешнего ключа должно либо совпадать с существующим значением первичного/уникального ключа в родительской таблице, либо быть NULL (если столбец допускает NULL).
- СУБД не позволит вставить в дочернюю таблицу строку со значением FK, которого нет в родительской таблице (ошибка нарушения ограничения).
- При попытке удалить или изменить строку в родительской таблице, на которую есть ссылки, СУБД применяет одну из стратегий, заданных при создании ограничения:
  - `ON DELETE/UPDATE CASCADE` — автоматически удалить/обновить связанные строки в дочерней таблице.
  - `ON DELETE/UPDATE RESTRICT` (или `NO ACTION`) — запретить операцию, если есть связанные строки.
  - `ON DELETE SET NULL` — установить NULL во внешнем ключе связанных строк.
  - `ON DELETE SET DEFAULT` — установить значение по умолчанию.
- Внешний ключ не обязан быть уникальным в дочерней таблице (в отличие от первичного ключа) — именно это и позволяет реализовать связь "один ко многим".

Внешние ключи — основной механизм реализации связей между таблицами (1:N, N:M через связующие таблицы) и защиты от "осиротевших" записей (когда дочерняя запись ссылается на несуществующего родителя).

## Пример / когда применяется

```sql
CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100)
);

CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL,
    total NUMERIC(10,2),
    CONSTRAINT fk_customer
        FOREIGN KEY (customer_id)
        REFERENCES customers(id)
        ON DELETE CASCADE
);

-- Эта вставка вызовет ошибку, если customer_id = 999 не существует
INSERT INTO orders (customer_id, total) VALUES (999, 500.00);
```

При тестировании внешних ключей важно проверять, что БД действительно отклоняет вставку с несуществующим FK и корректно обрабатывает удаление родительской записи согласно заданной стратегии (CASCADE/RESTRICT/SET NULL).

## На что смотрит интервьюер

Интервьюер оценивает понимание ссылочной целостности и знание стратегий каскадного поведения (CASCADE, RESTRICT, SET NULL). Часто спрашивают, что произойдёт при попытке вставить некорректный внешний ключ или удалить родительскую запись — кандидат должен уверенно объяснить поведение СУБД. Красный флаг — если кандидат не знает про каскадное удаление/обновление или путает FK с PK. Follow-up: "что будет, если удалить родительскую запись без указания ON DELETE?", "может ли внешний ключ содержать NULL?".
