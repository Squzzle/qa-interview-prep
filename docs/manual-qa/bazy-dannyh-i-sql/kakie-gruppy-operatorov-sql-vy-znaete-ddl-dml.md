# Какие группы операторов SQL вы знаете (DDL, DML, DCL, TCL)?

**Коротко:** Операторы SQL делятся на четыре основные группы: DDL — определение структуры данных, DML — манипуляция данными, DCL — управление правами доступа, TCL — управление транзакциями.

## Развёрнутый ответ

**DDL (Data Definition Language) — язык определения данных.** Отвечает за создание и изменение структуры базы данных: таблиц, индексов, схем, представлений. Основные операторы: `CREATE` (создать объект), `ALTER` (изменить структуру существующего объекта), `DROP` (удалить объект), `TRUNCATE` (быстро очистить таблицу, удалив все строки без построчного логирования). Операторы DDL обычно автоматически фиксируются (auto-commit) и не могут быть отменены через `ROLLBACK` в большинстве СУБД.

**DML (Data Manipulation Language) — язык манипуляции данными.** Отвечает за добавление, изменение, удаление и выборку данных внутри существующих таблиц. Основные операторы: `SELECT` (выбрать данные), `INSERT` (добавить строки), `UPDATE` (изменить существующие строки), `DELETE` (удалить строки). Некоторые классификации выделяют `SELECT` в отдельную группу DQL (Data Query Language), так как он только читает данные и не изменяет их.

**DCL (Data Control Language) — язык управления доступом.** Отвечает за назначение и отзыв прав пользователей на объекты базы данных. Основные операторы: `GRANT` (выдать права, например на чтение/запись конкретной таблицы), `REVOKE` (отозвать ранее выданные права).

**TCL (Transaction Control Language) — язык управления транзакциями.** Отвечает за управление границами и результатом транзакций. Основные операторы: `BEGIN`/`START TRANSACTION` (начать транзакцию), `COMMIT` (зафиксировать изменения), `ROLLBACK` (откатить изменения), `SAVEPOINT` (создать точку сохранения внутри транзакции для частичного отката).

Эта классификация помогает структурировать понимание того, какие операции обратимы (можно откатить внутри транзакции — обычно DML), а какие фиксируются немедленно (обычно DDL).

## Пример / когда применяется

```sql
-- DDL
CREATE TABLE products (id INT PRIMARY KEY, name VARCHAR(100));
ALTER TABLE products ADD COLUMN price DECIMAL(10,2);
DROP TABLE products;

-- DML
INSERT INTO products (id, name, price) VALUES (1, 'Book', 9.99);
UPDATE products SET price = 12.99 WHERE id = 1;
DELETE FROM products WHERE id = 1;
SELECT * FROM products;

-- DCL
GRANT SELECT, INSERT ON products TO analyst_role;
REVOKE INSERT ON products FROM analyst_role;

-- TCL
BEGIN;
UPDATE products SET price = price * 1.1;
COMMIT;
```

## На что смотрит интервьюер

- Знание всех четырёх групп (многие кандидаты называют только DDL и DML, забывая про DCL и TCL).
- Понимание, что конкретно относится к каждой группе, и умение привести операторы без подсказки.
- Понимание нюанса, что DDL-операторы в большинстве СУБД не откатываются транзакцией (автокоммит), в отличие от DML.
- Знание про отдельную иногда выделяемую группу DQL (Data Query Language) для `SELECT`.
- Частая ошибка — отнести `TRUNCATE` к DML (на самом деле это DDL-подобная операция в большинстве СУБД, так как она не логируется построчно и не всегда откатывается).
