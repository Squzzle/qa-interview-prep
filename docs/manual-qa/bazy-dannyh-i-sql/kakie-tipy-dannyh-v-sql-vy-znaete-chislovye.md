# Какие типы данных в SQL вы знаете (числовые, строковые, дата/время)?

**Коротко:** В SQL типы данных делятся на несколько основных групп: числовые (целые, дробные, точные), строковые (фиксированной и переменной длины, текстовые), дата/время, логические (boolean), а также бинарные и специальные (JSON, UUID, массивы).

## Развёрнутый ответ

**Числовые типы:**
- Целочисленные: `INT` (`INTEGER`), `SMALLINT`, `BIGINT`, `TINYINT` — различаются диапазоном допустимых значений и объёмом занимаемой памяти.
- Дробные с плавающей точкой: `FLOAT`, `REAL`, `DOUBLE PRECISION` — приближённые значения, могут давать погрешности округления.
- Точные дробные: `DECIMAL(p, s)` / `NUMERIC(p, s)` — хранят точное число с заданной точностью (p — общее количество значащих цифр, s — количество цифр после запятой). Используются там, где важна точность (деньги, финансовые расчёты), в отличие от `FLOAT`.

**Строковые типы:**
- `CHAR(n)` — строка фиксированной длины, дополняется пробелами до указанной длины.
- `VARCHAR(n)` — строка переменной длины с ограничением максимума.
- `TEXT` — строка неограниченной (или очень большой) длины, без строгого лимита в определении столбца.

**Дата и время:**
- `DATE` — только дата (год, месяц, день).
- `TIME` — только время (часы, минуты, секунды).
- `DATETIME` / `TIMESTAMP` — дата и время вместе. `TIMESTAMP` часто хранится с учётом часового пояса или автоматически конвертируется в UTC (например, `TIMESTAMPTZ` в PostgreSQL), в то время как `DATETIME` — "наивное" значение без часового пояса.
- `INTERVAL` — промежуток времени (поддерживается, например, в PostgreSQL).

**Логический тип:**
- `BOOLEAN` — значения `TRUE`/`FALSE` (в некоторых СУБД, например MySQL до определённых версий, эмулируется через `TINYINT(1)`).

**Прочие/специальные типы:**
- `BLOB` / `BYTEA` — бинарные данные (файлы, изображения).
- `JSON` / `JSONB` — структурированные данные в формате JSON (JSONB в PostgreSQL хранится в бинарном виде и поддерживает индексацию).
- `UUID` — уникальный идентификатор фиксированного формата.
- `ARRAY` — массив значений (поддерживается, например, в PostgreSQL).
- `ENUM` — перечисляемый тип с фиксированным набором допустимых значений.

Выбор типа данных влияет на объём хранимых данных, скорость выполнения запросов, корректность сравнений и сортировки, а также на возможность применения определённых индексов и ограничений.

## Пример / когда применяется

Пример таблицы с использованием разных групп типов данных:

```sql
CREATE TABLE orders (
    id BIGINT PRIMARY KEY,
    customer_code CHAR(6),
    description VARCHAR(255),
    notes TEXT,
    total_amount DECIMAL(10, 2) NOT NULL,
    is_paid BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    delivery_date DATE,
    metadata JSONB
);
```

Здесь `DECIMAL(10, 2)` используется для денежной суммы (точность важнее производительности), `CHAR(6)` — для кода фиксированной длины, `TIMESTAMP` — для отметки времени создания записи.

## На что смотрит интервьюер

- Знание разницы между `FLOAT`/`DOUBLE` и `DECIMAL`/`NUMERIC` и понимание, что для денег нужно использовать точный тип, а не тип с плавающей точкой.
- Понимание разницы между `CHAR` и `VARCHAR` (часто это отдельный follow-up вопрос).
- Знание разницы между `TIMESTAMP` и `DATETIME`, а также про часовые пояса.
- Умение назвать типы, специфичные для конкретной СУБД (JSONB, ARRAY, ENUM в PostgreSQL) — показывает более глубокое знание, чем просто "стандартные" типы.
- Частая ошибка — не упомянуть точные числовые типы (DECIMAL/NUMERIC) вовсе, забыть про булев тип или про бинарные данные.
