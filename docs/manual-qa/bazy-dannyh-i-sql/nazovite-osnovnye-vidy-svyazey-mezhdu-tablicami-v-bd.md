# Назовите основные виды связей между таблицами в БД (1:1, 1:N, N:M).

**Коротко:** в реляционных БД существует три основных типа связей между таблицами: один к одному (1:1), один ко многим (1:N) и многие ко многим (N:M).

## Развёрнутый ответ

Связь между таблицами определяет, как строки одной таблицы соотносятся со строками другой. Реализуется через первичные и внешние ключи.

- **Один к одному (1:1)** — одной строке первой таблицы соответствует ровно одна строка второй таблицы, и наоборот. Используется редко, обычно для разделения таблицы на "основную" и "дополнительную" (например, по соображениям безопасности или производительности), либо когда часть атрибутов используется редко. Реализуется через внешний ключ с ограничением UNIQUE в дочерней таблице.

- **Один ко многим (1:N)** — одной строке первой таблицы (родительской) может соответствовать несколько строк второй таблицы (дочерней), но каждая строка дочерней таблицы связана только с одной строкой родительской. Это самый распространённый тип связи. Реализуется добавлением внешнего ключа в дочернюю таблицу, ссылающегося на первичный ключ родительской.

- **Многие ко многим (N:M)** — множеству строк первой таблицы может соответствовать множество строк второй таблицы, и наоборот. Напрямую в реляционной модели такая связь не реализуется — для неё создаётся промежуточная (связующая, junction/associative) таблица, содержащая внешние ключи на обе исходные таблицы. Часто такая таблица имеет составной первичный ключ из пары внешних ключей.

Правильный выбор типа связи и его корректная реализация через ограничения (FOREIGN KEY, UNIQUE, составные ключи) — основа целостности данных в реляционной БД.

## Пример / когда применяется

- 1:1 — таблица `users` и `user_profiles` (расширенные, редко используемые данные о пользователе).
- 1:N — таблица `customers` и `orders`: один клиент может сделать много заказов, но заказ принадлежит одному клиенту.
- N:M — таблицы `students` и `courses`: студент может посещать несколько курсов, а курс может посещать несколько студентов.

```sql
CREATE TABLE students (id SERIAL PRIMARY KEY, name VARCHAR(100));
CREATE TABLE courses (id SERIAL PRIMARY KEY, title VARCHAR(100));

CREATE TABLE student_courses (
    student_id INT REFERENCES students(id),
    course_id INT REFERENCES courses(id),
    PRIMARY KEY (student_id, course_id)
);
```

## На что смотрит интервьюер

Интервьюер проверяет, что кандидат может не только назвать три типа связей, но и объяснить, как каждая из них реализуется физически (особенно N:M через связующую таблицу — частая точка непонимания у junior-специалистов). Красный флаг — если кандидат считает, что N:M можно реализовать просто через два внешних ключа без промежуточной таблицы.
