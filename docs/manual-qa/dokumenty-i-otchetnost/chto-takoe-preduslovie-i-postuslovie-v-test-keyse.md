# Что такое **предусловие** и **постусловие** в тест-кейсе?

**Коротко:** предусловие (precondition) — это состояние системы или данных, которое должно быть достигнуто до начала выполнения теста, а постусловие (postcondition) — состояние, которое должно быть верным после его завершения, включая необходимые действия по возврату системы в исходное состояние.

## Развёрнутый ответ

Предусловие описывает контекст, необходимый для корректного выполнения тест-кейса: что должно быть настроено, какие данные должны существовать, в каком состоянии должна находиться система, прежде чем тестировщик начнёт выполнять шаги теста. Без выполненного предусловия тест либо невозможно выполнить, либо результат будет некорректным. Примеры предусловий: пользователь зарегистрирован и авторизован, в базе данных есть тестовый товар с определённым артикулом, у пользователя оформлена подписка определённого уровня, приложение установлено и запущено, тестовое окружение доступно.

Постусловие описывает ожидаемое состояние системы после выполнения теста и, что важно, действия по очистке/восстановлению состояния (teardown), чтобы последующие тесты не были затронуты побочными эффектами текущего теста. Примеры постусловий: заказ создан и виден в истории заказов пользователя, тестовые данные удалены из базы, пользователь разлогинен, файл, загруженный в ходе теста, удалён с сервера, состояние корзины сброшено.

Значение этих элементов:

- **Воспроизводимость** — точно описанное предусловие гарантирует, что тест-кейс можно выполнить в любой момент с одинаковым стартовым состоянием.
- **Изоляция тестов** — постусловие предотвращает "загрязнение" окружения, которое могло бы повлиять на следующие тесты (особенно критично в автоматизации, где тесты часто выполняются в произвольном порядке).
- **Экономия времени** — если предусловие уже выполнено предыдущим тестом или фикстурой, не нужно повторять подготовительные шаги внутри самого теста.

В автоматизированном тестировании предусловия и постусловия часто реализуются через фикстуры (setup/teardown, beforeEach/afterEach), что позволяет переиспользовать подготовительный код между тестами.

## Пример / когда применяется

Пример тест-кейса "Отмена заказа пользователем":

| Элемент | Содержание |
|---|---|
| Предусловие | Пользователь авторизован; у пользователя есть оформленный заказ в статусе "В обработке" |
| Шаги | 1. Открыть раздел «Мои заказы» 2. Выбрать заказ 3. Нажать «Отменить заказ» 4. Подтвердить отмену |
| Ожидаемый результат | Статус заказа меняется на «Отменён», приходит уведомление |
| Постусловие | Заказ остаётся в БД в статусе «Отменён» для проверки истории; тестовые данные (заказ) удаляются после прогона автотестов |

## На что смотрит интервьюер

Интервьюер проверяет понимание того, зачем нужна явная фиксация предусловий и постусловий, особенно в контексте автоматизации и изоляции тестов. Красный флаг — путаница между предусловием и первым шагом теста, или полное игнорирование постусловия/teardown. Возможные follow-up вопросы: "Что будет, если не выполнить teardown после автотеста?", "Как вы организуете общие предусловия для группы тестов?", "Приведите пример, когда отсутствие постусловия привело к падению других тестов (flaky tests)".
