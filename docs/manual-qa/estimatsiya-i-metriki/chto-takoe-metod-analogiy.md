# Что такое метод аналогий?

**Коротко:** метод аналогий — это оценка трудозатрат на новую задачу путём сравнения с уже выполненной похожей задачей, фактические трудозатраты которой известны.

## Развёрнутый ответ

Метод аналогий (Analogy-based estimation, аналоговая оценка) основан на предположении, что похожие по характеру и сложности задачи требуют примерно одинаковых трудозатрат. Оценщик находит в истории проекта (или личном опыте) задачу, максимально похожую на текущую, и берёт её фактические трудозатраты за основу, корректируя их с учётом различий.

Алгоритм применения метода:
1. Найти одну или несколько похожих задач, выполненных ранее, по которым известны реальные (фактические) трудозатраты.
2. Определить ключевые параметры сходства и различия: объём функциональности, сложность интеграций, количество тест-кейсов, качество требований, состав команды.
3. Скорректировать оценку с учётом отличий — например, если новая задача сложнее на 20% (больше интеграций), увеличить базовую оценку соответствующим образом.
4. Зафиксировать итоговую оценку и допущения, на основе которых она сделана.

Метод хорошо работает, когда в компании накоплена статистика по прошлым проектам/задачам (метрики трудозатрат, длительность тестирования аналогичных модулей). Чем больше релевантных исторических данных, тем точнее оценка.

Ограничения метода: он неточен для принципиально новых, уникальных задач, для которых нет аналогов; также он подвержен ошибке, если "похожая" задача выбрана некорректно или если условия (команда, инструменты, требования) существенно изменились с момента выполнения аналога.

Метод аналогий часто используется как быстрый и дешёвый способ первичной, грубой оценки на ранних этапах, когда детальная декомпозиция ещё невозможна или нецелесообразна.

## Пример / когда применяется

Пример: команда тестирования оценивает регрессионное тестирование нового модуля "Личный кабинет пользователя v2". В прошлом релизе тестирование аналогичного модуля "Личный кабинет v1" заняло 5 дней при 40 тест-кейсах. Новый модуль содержит на 20% больше функциональности (48 тест-кейсов), поэтому оценка корректируется пропорционально: 5 дней × 1.2 ≈ 6 дней.

Метод аналогий применяется на этапе предварительной оценки проекта (до детального анализа требований), при оценке типовых, повторяющихся задач (например, тестирование очередного релиза мобильного приложения) и при составлении бюджета проекта на старте.

## На что смотрит интервьюер

Интервьюер оценивает, понимает ли кандидат, что для метода аналогий нужны исторические данные и корректный выбор похожей задачи. Важно, чтобы кандидат мог назвать ограничения метода (неприменим для уникальных задач). Красный флаг — если кандидат путает метод аналогий с экспертной оценкой "на глаз" без опоры на реальные исторические данные. Частый follow-up: "А что если аналогичной задачи не было раньше?", "Как учитывать различия между задачами при корректировке оценки?".
