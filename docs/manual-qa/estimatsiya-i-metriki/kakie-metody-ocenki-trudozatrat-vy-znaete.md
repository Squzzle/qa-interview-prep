# Какие методы оценки трудозатрат вы знаете?

**Коротко:** основные методы — экспертная оценка, метод аналогий, декомпозиция задач (WBS), Planning Poker, оценка по трём точкам (PERT), а также относительная оценка в story points и группировка по T-shirt sizes.

## Развёрнутый ответ

Методы оценки трудозатрат делятся на две большие группы: экспертные (качественные) и математические/статистические (количественные).

Экспертные методы основаны на опыте и знаниях специалистов:
- **Экспертная оценка (Expert Judgement)** — оценку даёт один или несколько экспертов на основе личного опыта работы с похожими задачами.
- **Метод аналогий (Analogy-based estimation)** — оценка новой задачи производится путём сравнения с уже выполненными похожими задачами, по которым известны фактические трудозатраты.
- **Planning Poker** — командная техника, при которой каждый участник независимо выбирает карту с оценкой (обычно числа Фибоначчи), после чего расхождения обсуждаются до достижения консенсуса.
- **T-shirt sizing** — грубая относительная оценка задач по размерам (XS, S, M, L, XL), удобна на ранних этапах, когда деталей мало.
- **Story Points** — относительная оценка сложности/объёма задачи в условных единицах (часто числа Фибоначчи), используется в Scrum вместе со скоростью команды (velocity) для перевода в реальные сроки.

Математические/статистические методы:
- **Декомпозиция (WBS — Work Breakdown Structure)** — задача разбивается на более мелкие подзадачи, каждая оценивается отдельно, затем оценки суммируются. Снижает погрешность за счёт уменьшения размера единицы оценки.
- **PERT (Program Evaluation and Review Technique)** — оценка по трём точкам: оптимистичная, пессимистичная и наиболее вероятная, из которых вычисляется средневзвешенная оценка.
- **Метод дельфи (Delphi/Wideband Delphi)** — несколько экспертов оценивают задачу анонимно в несколько раундов, что снижает влияние группового давления и авторитета.
- **Метод контрольных точек/бенчмаркинга** — использование исторических метрик (например, среднее время на тест-кейс) для расчёта оценки по количеству аналогичных единиц работы.

На практике команды часто комбинируют методы: сначала декомпозируют задачу (WBS), затем оценивают каждую часть через Planning Poker или экспертную оценку, а для критичных задач дополнительно считают PERT.

## Пример / когда применяется

| Метод | Когда применяется |
|---|---|
| Экспертная оценка | Быстрая прикидка, мало данных |
| Метод аналогий | Есть похожая завершённая задача |
| Planning Poker | Командная оценка user story в Scrum |
| PERT | Проекты с высокой неопределённостью, нужен диапазон |
| WBS | Крупные задачи/эпики, требующие декомпозиции |
| Story Points | Долгосрочное планирование в Agile через velocity |

Пример: перед спринтом команда декомпозирует эпик на 5 историй (WBS), затем оценивает каждую в Planning Poker, а для истории с высокой неопределённостью (интеграция с новым внешним API) дополнительно считает PERT, чтобы дать диапазон вместо одной цифры.

## На что смотрит интервьюер

Интервьюер ожидает, что кандидат назовёт минимум 3-4 метода и объяснит разницу между относительной (story points) и абсолютной (часы/дни) оценкой. Плюс — если кандидат приводит примеры из своей практики. Красный флаг — если кандидат знает только один метод или путает эстимацию со сроками релиза. Частые follow-up вопросы: "Чем story points отличаются от часов?", "Как считается velocity команды?", "Что делать, если участники Planning Poker сильно расходятся в оценках?".
