# Объясните метод оценки по трем точкам (PERT). Приведите формулу.

**Коротко:** PERT — метод оценки, при котором для задачи даются три оценки (оптимистичная, пессимистичная, наиболее вероятная), а итоговая оценка вычисляется как их средневзвешенное значение по формуле PERT.

## Развёрнутый ответ

PERT (Program Evaluation and Review Technique) — статистический метод оценки трудозатрат, разработанный ещё в 1950-х годах для управления проектами с высокой степенью неопределённости. Вместо одной точечной оценки эксперт даёт три значения:

- **O (Optimistic, оптимистичная оценка)** — минимально возможное время выполнения задачи при самом благоприятном стечении обстоятельств, без задержек и проблем;
- **P (Pessimistic, пессимистичная оценка)** — максимальное время при самом неблагоприятном сценарии (проблемы, блокеры, недоступность окружения и т.п.);
- **M (Most Likely, наиболее вероятная оценка)** — реалистичная оценка при обычном ходе работы без экстремальных отклонений.

Итоговая (ожидаемая) оценка вычисляется по формуле:

```text
E = (O + 4M + P) / 6
```

Формула придаёт наиболее вероятной оценке (M) вес в 4 раза больше, чем крайним значениям, что соответствует бета-распределению, которое используется как модель для описания подобных оценок.

Дополнительно вычисляется стандартное отклонение, отражающее степень неопределённости оценки:

```text
SD = (P - O) / 6
```

Стандартное отклонение позволяет построить доверительный интервал вокруг итоговой оценки E, например E ± SD (примерно 68% доверия) или E ± 2*SD (примерно 95% доверия).

Преимущества PERT:
- учитывает неопределённость и риски явно, а не игнорирует их;
- даёт диапазон, а не одну точку, что честнее по отношению к заказчику;
- подходит для задач с высокой степенью неизвестности (новые технологии, интеграции с внешними системами).

Ограничения: метод требует, чтобы эксперт корректно оценил все три точки — если пессимистичная или оптимистичная оценка выбраны небрежно, итоговый результат будет искажён. Также PERT не учитывает системные риски (например, одновременную занятость исполнителя на нескольких проектах).

## Пример / когда применяется

Пример расчёта: тестировщик оценивает тестирование нового модуля интеграции с платёжным шлюзом:
- O = 3 дня (всё идёт гладко, документация полная);
- M = 5 дней (обычный сценарий с небольшими доработками);
- P = 10 дней (проблемы со стороны внешнего провайдера, недоступность тестового стенда).

Расчёт:

```text
E = (3 + 4*5 + 10) / 6 = 33 / 6 = 5.5 дней
SD = (10 - 3) / 6 = 1.17 дня
```

Итоговая оценка — 5.5 ± 1.17 дня, то есть с вероятностью ~68% задача займёт от 4.3 до 6.7 дней.

PERT применяется в проектах с высокой неопределённостью: новые интеграции, работа с незнакомыми технологиями, тестирование в условиях нестабильных требований, а также при составлении Test Plan для крупных релизов, когда важно показать заказчику диапазон сроков, а не единственную цифру.

## На что смотрит интервьюер

Интервьюер проверяет, помнит ли кандидат формулу наизусть и понимает ли смысл весового коэффициента 4 у наиболее вероятной оценки. Важно, чтобы кандидат мог объяснить, зачем нужны три точки, а не одна. Красный флаг — если кандидат не может воспроизвести формулу или путает PERT с методом критического пути (CPM). Частые follow-up вопросы: "Как рассчитывается стандартное отклонение и зачем оно нужно?", "В каких ситуациях PERT предпочтительнее экспертной оценки?".
