# Чем **HTTPS** отличается от HTTP? Что такое SSL/TLS?

**Коротко:** HTTPS — это HTTP, работающий поверх шифрующего слоя SSL/TLS, который обеспечивает конфиденциальность, целостность и аутентичность передаваемых данных.

## Развёрнутый ответ

HTTP передаёт данные в открытом (незашифрованном) виде, поэтому любой посредник в сети — провайдер, атакующий в общедоступном Wi-Fi, прокси — может прочитать или подменить содержимое запросов и ответов. HTTPS (HTTP Secure) решает эту проблему, оборачивая обычный HTTP-трафик в криптографический слой SSL (Secure Sockets Layer) или его современного преемника TLS (Transport Layer Security).

SSL/TLS обеспечивает три ключевых свойства:

- **Конфиденциальность (encryption)** — данные шифруются симметричным алгоритмом с использованием сессионного ключа, поэтому перехваченный трафик нельзя прочитать без ключа.
- **Целостность (integrity)** — с помощью MAC (message authentication code) или AEAD-шифров гарантируется, что данные не были изменены при передаче.
- **Аутентификация (authentication)** — сервер (а иногда и клиент) подтверждает свою подлинность с помощью цифрового сертификата, выданного доверенным центром сертификации (Certificate Authority, CA).

SSL — устаревший протокол (SSL 2.0, SSL 3.0), считается небезопасным и не должен использоваться. TLS (1.0, 1.1, 1.2, 1.3) — его развитие; на практике термин «SSL» часто используется по инерции, хотя фактически применяется TLS, обычно версии 1.2 или 1.3.

Отличия HTTPS от HTTP:

| Критерий | HTTP | HTTPS |
|---|---|---|
| Порт по умолчанию | 80 | 443 |
| Шифрование | Нет | Есть (TLS/SSL) |
| Сертификат | Не требуется | Требуется (выдаётся CA) |
| Защита от MITM | Отсутствует | Обеспечивается |
| Влияние на SEO | Нейтрально/хуже | Google отдаёт приоритет HTTPS |

## Пример / когда применяется

Интернет-магазин, принимающий данные банковских карт, обязан использовать HTTPS: без шифрования номер карты и CVV передавались бы открытым текстом и могли быть перехвачены атакующим в незащищённой сети (например, публичный Wi-Fi). Браузеры также помечают HTTP-сайты как «Небезопасно» в адресной строке.

При тестировании QA-инженер проверяет: наличие валидного сертификата, отсутствие смешанного контента (mixed content — когда HTTPS-страница подгружает ресурсы по HTTP), корректные редиректы с HTTP на HTTPS (301), а также срок действия сертификата.

## На что смотрит интервьюер

- Понимание, что HTTPS — это не отдельный протокол, а HTTP + TLS/SSL.
- Знание разницы между шифрованием, целостностью и аутентификацией — кандидаты часто путают эти понятия.
- Осведомлённость, что SSL считается устаревшим и небезопасным, актуален TLS.
- Follow-up: «Что произойдёт, если сертификат просрочен?», «Что такое mixed content?», «Как проверить сертификат сайта в браузере?».
