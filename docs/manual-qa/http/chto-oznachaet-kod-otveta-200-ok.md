# Что означает код ответа **200 OK**?

**Коротко:** `200 OK` означает, что запрос клиента успешно обработан сервером, и в теле ответа обычно возвращается запрошенный результат.

## Развёрнутый ответ

`200 OK` — базовый и самый частый успешный код ответа HTTP, относится к группе 2xx (Success). Он говорит о том, что сервер получил запрос, понял его и обработал так, как и ожидалось, без ошибок.

Смысл кода зависит от метода запроса:

- Для `GET` — ресурс найден, и его представление возвращено в теле ответа.
- Для `PUT` или `PATCH` — ресурс успешно обновлён, тело ответа может содержать обновлённое представление ресурса.
- Для `POST` — действие выполнено успешно, и результат (не обязательно созданный ресурс) возвращён в теле; если запрос создал новый ресурс, правильнее вернуть `201 Created`, а не `200`.
- Для `DELETE` — ресурс удалён, и в теле есть какое-то описание статуса (если тело пустое, корректнее использовать `204 No Content`).

Важный нюанс: `200 OK` — не универсальный "всё хорошо", это конкретно означает, что операция выполнена и в ответе есть тело с результатом. Если тела нет, семантически правильнее `204`. Если запрос создал сущность — правильнее `201`.

## Пример / когда применяется

Классический пример: запрос `GET /users/123` к REST API, пользователь с id=123 существует — сервер возвращает `200 OK` с JSON-телом, содержащим данные пользователя.

```text
GET /users/123 HTTP/1.1
Host: api.example.com

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 123,
  "name": "Иван Иванов"
}
```

## На что смотрит интервьюер

Частая ошибка — путать `200` с `201` при создании ресурсов через `POST`, а также использовать `200` с пустым телом там, где нужен `204`. Интервьюер может спросить: "Чем `200` отличается от `201` и `204`?" — важно показать понимание семантики, а не просто "200 значит успех".
