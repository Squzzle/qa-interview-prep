# Что означает код ответа **201 Created**?

**Коротко:** `201 Created` сообщает, что запрос успешно выполнен и в результате был создан новый ресурс на сервере.

## Развёрнутый ответ

`201 Created` относится к группе успешных кодов 2xx и используется, когда обработка запроса привела к созданию одного или нескольких новых ресурсов. Чаще всего этот код возвращают в ответ на `POST` (реже — на `PUT`, если он создаёт ресурс по заданному URI).

Ключевые технические детали:

- Согласно спецификации, ответ должен содержать заголовок `Location` с URI-адресом только что созданного ресурса, чтобы клиент мог сразу к нему обратиться.
- Тело ответа обычно содержит представление созданного ресурса (например, JSON с его данными и присвоенным идентификатором), хотя это не строго обязательно.
- Важно отличать `201` от `200`: `200` уместен, когда запрос обработан, но не создал новый ресурс (например, обновление), а `201` — именно про факт создания.

## Пример / когда применяется

Пример: клиент отправляет `POST /users` с данными нового пользователя, сервер создаёт запись и возвращает `201` с адресом созданного ресурса.

```text
POST /users HTTP/1.1
Host: api.example.com
Content-Type: application/json

{
  "name": "Иван Иванов"
}

HTTP/1.1 201 Created
Location: /users/124
Content-Type: application/json

{
  "id": 124,
  "name": "Иван Иванов"
}
```

## На что смотрит интервьюер

Проверяют, знает ли кандидат про обязательный (по стандарту) заголовок `Location`, и понимает ли разницу между `200` и `201` при тестировании API создания ресурсов. Частая ошибка тестировщиков — не проверять код `201` отдельно, довольствуясь просто "запрос прошёл успешно" (код 200), что маскирует баг на бэкенде.
