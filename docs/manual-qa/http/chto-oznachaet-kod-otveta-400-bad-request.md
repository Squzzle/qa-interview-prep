# Что означает код ответа **400 Bad Request**?

**Коротко:** `400 Bad Request` означает, что сервер не смог обработать запрос из-за ошибки на стороне клиента: некорректный синтаксис, невалидные данные или неверный формат запроса.

## Развёрнутый ответ

`400` относится к группе клиентских ошибок (4xx) и является общим кодом, сигнализирующим, что запрос сформирован неправильно и сервер не может (или не будет) его обрабатывать в текущем виде. Причины могут быть разными:

- Некорректный синтаксис JSON или XML в теле запроса.
- Отсутствие обязательных параметров или полей.
- Невалидные значения полей (например, буквы в поле, где ожидается число, или дата в неверном формате).
- Нарушение бизнес-правил валидации на уровне API (хотя для таких случаев некоторые API используют `422 Unprocessable Entity`, если сервер их различает).

Важно понимать отличие `400` от других кодов 4xx: `400` — про сам запрос как таковой (его нельзя даже корректно распарсить или он логически некорректен), в отличие от `401` (нет аутентификации), `403` (нет прав) или `404` (ресурс не найден при в целом валидном запросе).

Хороший API возвращает вместе с `400` тело с описанием, что именно не так — например, список невалидных полей, чтобы клиент мог их исправить.

## Пример / когда применяется

Пример: клиент отправляет запрос на регистрацию без обязательного поля email.

```text
POST /users HTTP/1.1
Host: api.example.com
Content-Type: application/json

{
  "name": "Иван"
}

HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "Field 'email' is required"
}
```

При тестировании API это один из ключевых негативных кейсов: проверка граничных значений, отсутствующих обязательных полей, некорректных типов данных, невалидного JSON.

## На что смотрит интервьюер

Смотрят, умеет ли кандидат отличать `400` от `401`, `403`, `404`, `422` и `500`, а также приводит ли примеры конкретных негативных тест-кейсов. Частая ошибка — путать клиентскую ошибку `400` с серверной `500`, либо не проверять содержимое тела ответа с описанием ошибки, довольствуясь только кодом.
