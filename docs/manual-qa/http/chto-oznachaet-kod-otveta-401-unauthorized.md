# Что означает код ответа **401 Unauthorized**?

**Коротко:** `401 Unauthorized` означает, что запрос требует аутентификации, а клиент её не предоставил или предоставил неверные учётные данные.

## Развёрнутый ответ

Несмотря на название, `401` фактически относится к отсутствию или некорректности аутентификации (проверки "кто вы"), а не авторизации (проверки "что вам разрешено"). Это распространённая терминологическая путаница: правильнее было бы назвать код "Unauthenticated".

Сервер возвращает `401`, если:

- запрос вообще не содержит данных для аутентификации (например, отсутствует заголовок `Authorization`);
- предоставленные учётные данные (токен, логин/пароль, API-ключ) неверны или просрочены.

По стандарту сервер, возвращающий `401`, должен также передать заголовок `WWW-Authenticate`, описывающий, какую схему аутентификации использовать (например, `Basic`, `Bearer`).

Отличие от `403 Forbidden`: `401` говорит "я не знаю, кто вы, аутентифицируйтесь", а `403` — "я знаю, кто вы, но у вас недостаточно прав для этого действия". Если после успешного логина пользователь пытается получить доступ к чужим данным или административному разделу — это `403`, а не `401`.

## Пример / когда применяется

Пример: запрос к защищённому эндпоинту без токена или с истёкшим токеном.

```text
GET /api/profile HTTP/1.1
Host: api.example.com

HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm="api"
Content-Type: application/json

{
  "error": "Access token is missing or expired"
}
```

При тестировании важно проверять: доступ без токена, доступ с истёкшим/невалидным токеном, поведение при повторной аутентификации.

## На что смотрит интервьюер

Ключевой вопрос — умение чётко разграничить `401` и `403` (аутентификация vs авторизация). Частая ошибка кандидатов — путать эти два кода местами или объяснять их одинаково. Также проверяют знание заголовка `WWW-Authenticate` и понимание того, что "Unauthorized" по факту означает "не аутентифицирован".
