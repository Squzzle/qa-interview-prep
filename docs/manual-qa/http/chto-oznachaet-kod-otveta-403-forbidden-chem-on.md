# Что означает код ответа **403 Forbidden**? Чем он отличается от 401?

**Коротко:** `403 Forbidden` означает, что сервер понял, кто отправил запрос, но отказывается его выполнять из-за недостатка прав доступа, в отличие от `401`, который означает отсутствие или неверность аутентификации.

## Развёрнутый ответ

`403` относится к группе клиентских ошибок 4xx и сигнализирует об отказе в доступе на уровне авторизации: сервер идентифицировал клиента (или клиент вовсе не обязан быть аутентифицирован для этого запроса), но у него нет прав на выполнение конкретного действия или доступ к конкретному ресурсу. В отличие от `401`, повторная отправка учётных данных ничего не изменит — проблема не в том, "кто вы", а в том, "что вам разрешено".

Основное отличие `403` от `401`:

| Код | Что означает | Что решит проблему |
|---|---|---|
| 401 Unauthorized | Клиент не аутентифицирован или предоставил неверные данные | Пройти аутентификацию / обновить токен |
| 403 Forbidden | Клиент аутентифицирован, но у него нет прав на действие | Изменить права доступа (аутентификация не поможет) |

Дополнительный нюанс: некоторые API из соображений безопасности намеренно возвращают `404 Not Found` вместо `403` для приватных ресурсов, чтобы не раскрывать сам факт их существования неавторизованному пользователю. Это важно учитывать при анализе поведения конкретного сервиса.

## Пример / когда применяется

Пример: обычный пользователь, успешно залогиненный, пытается зайти в административную панель.

```text
GET /admin/users HTTP/1.1
Host: api.example.com
Authorization: Bearer <valid_user_token>

HTTP/1.1 403 Forbidden
Content-Type: application/json

{
  "error": "You do not have permission to access this resource"
}
```

Тест-кейсы: доступ пользователя с недостаточными правами к защищённым ресурсам/действиям других пользователей (IDOR-проверки), доступ к административным разделам без соответствующей роли.

## На что смотрит интервьюер

Ключевая проверка — чёткое разграничение аутентификации и авторизации на примере `401` vs `403`. Интервьюер также может спросить про сценарий, когда API отдаёт `404` вместо `403` для скрытия факта существования ресурса. Красный флаг — кандидат называет оба кода "ошибками доступа" без разъяснения разницы.
