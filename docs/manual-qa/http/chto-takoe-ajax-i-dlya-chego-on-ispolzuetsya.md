# Что такое **AJAX** и для чего он используется?

**Коротко:** AJAX (Asynchronous JavaScript and XML) — это подход к обмену данными с сервером из JavaScript без перезагрузки страницы, что позволяет обновлять часть контента асинхронно.

## Развёрнутый ответ

AJAX — не отдельная технология, а название техники, объединяющей несколько существующих веб-технологий: JavaScript для логики, `XMLHttpRequest` (или современный `fetch`) для отправки асинхронных HTTP-запросов, и обычно JSON (исторически XML, отсюда и название) как формат обмена данными, а также DOM для обновления части страницы без полной перезагрузки.

До появления AJAX любое взаимодействие с сервером требовало полной перезагрузки страницы: пользователь кликал на ссылку или отправлял форму, браузер делал полный переход, весь HTML пересобирался и заново отрисовывался. AJAX решил эту проблему — теперь можно отправить запрос в фоне и обновить только нужную часть DOM, не прерывая работу пользователя со страницей (не сбрасывая скролл, состояние других элементов и т.д.).

Ключевые характеристики AJAX-запроса:
- **Асинхронность** — запрос выполняется в фоне, не блокируя выполнение остального JS-кода и взаимодействие пользователя со страницей.
- **Частичное обновление** — обновляется не вся страница, а конкретный элемент DOM.
- **Формат данных** — чаще всего JSON, реже XML, HTML-фрагменты или обычный текст.

Технически AJAX реализуется через `XMLHttpRequest` (классический, более многословный API) или через современный `fetch API` (более лаконичный, основан на промисах). С точки зрения тестирования оба механизма видны на вкладке Network в DevTools как XHR/fetch-запросы.

## Пример / когда применяется

Классический пример — автодополнение поиска: пока пользователь печатает в поле поиска, JS отправляет AJAX-запросы к серверу и обновляет список подсказок без перезагрузки страницы.

```text
fetch('/api/search?q=telefon')
  .then(response => response.json())
  .then(data => renderSuggestions(data));
```

Другие типичные случаи использования: бесконечная прокрутка (infinite scroll), лайки/подписки без перезагрузки, валидация формы в реальном времени (проверка занятости логина), обновление корзины покупок, чаты и уведомления в реальном времени (хотя для настоящего реального времени чаще используют WebSocket).

## На что смотрит интервьюер

- Понимание, что AJAX — это техника/подход, а не конкретная технология или библиотека.
- Знание, что современный аналог `XMLHttpRequest` — это `fetch`, и оба относятся к "AJAX" в широком смысле.
- Умение объяснить, как QA обнаруживает AJAX-запросы при тестировании (вкладка Network в DevTools, фильтр по XHR/Fetch).
- Follow-up: "В чём разница между `XMLHttpRequest` и `fetch`?", "Как тестировать асинхронные AJAX-запросы, если UI не показывает явной загрузки (loader)?".
- Красный флаг — если кандидат путает AJAX с самим форматом JSON/XML или не может объяснить, зачем он вообще нужен.
