# Что такое **HTTP**? Какой порт используется по умолчанию?

**Коротко:** HTTP (HyperText Transfer Protocol) — прикладной протокол передачи данных между клиентом и сервером по модели «запрос-ответ», работающий поверх TCP; порт по умолчанию — 80 (для HTTPS — 443).

## Развёрнутый ответ

HTTP — это протокол прикладного уровня модели OSI (7-й уровень), который используется для обмена гипертекстовыми документами, а на практике — для передачи почти любых данных: HTML-страниц, JSON, файлов, изображений и т.д. Протокол является текстовым, основан на модели «клиент-сервер»: клиент (например, браузер) отправляет запрос, сервер обрабатывает его и возвращает ответ.

Ключевые характеристики HTTP:

- **Без сохранения состояния (stateless)** — каждый запрос обрабатывается независимо от предыдущих, сервер не хранит информацию о сессии между запросами (состояние эмулируется через cookies, токены, сессии).
- **Транспорт** — работает поверх TCP (в HTTP/3 — поверх QUIC, который использует UDP), что гарантирует надёжную доставку данных.
- **Версии** — HTTP/1.0, HTTP/1.1 (наиболее распространён долгое время, поддерживает keep-alive), HTTP/2 (мультиплексирование, бинарный формат, сжатие заголовков) и HTTP/3 (на базе QUIC).

Порты по умолчанию:

- **80** — стандартный порт для нешифрованного HTTP.
- **443** — стандартный порт для HTTPS (HTTP поверх TLS/SSL).

Порт можно переопределить явно в URL, например `http://example.com:8080`, что часто встречается в тестовых и dev-окружениях.

## Пример / когда применяется

Классический пример — открытие сайта в браузере: браузер выполняет DNS-резолвинг домена, устанавливает TCP-соединение с сервером на порт 80 (или 443 для HTTPS) и отправляет HTTP-запрос вида:

```text
GET /index.html HTTP/1.1
Host: example.com
```

В контексте тестирования QA-инженер должен уметь объяснить, почему при работе с API через Postman или curl по умолчанию подставляется порт 80/443, и как изменить порт при тестировании локального стенда, например `http://localhost:3000`.

## На что смотрит интервьюер

- Знает ли кандидат разницу между протоколом и портом (частая путаница: «HTTP — это порт 80»).
- Понимание, что HTTP — прикладной протокол, работающий поверх TCP, а не транспортный протокол сам по себе.
- Знание, что 443 — это порт HTTPS, а не отдельного протокола шифрования.
- Красный флаг — если кандидат не может назвать номера портов или путает их местами.
