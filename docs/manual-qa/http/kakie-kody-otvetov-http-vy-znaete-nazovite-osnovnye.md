# Какие **коды ответов HTTP** вы знаете? Назовите основные группы (1xx, 2xx, 3xx, 4xx, 5xx).

**Коротко:** коды ответов HTTP делятся на пять групп по первой цифре: 1xx — информационные, 2xx — успех, 3xx — редиректы, 4xx — ошибки клиента, 5xx — ошибки сервера.

## Развёрнутый ответ

Код состояния HTTP (HTTP status code) — это трёхзначное число в первой строке ответа сервера, которое сообщает клиенту результат обработки запроса. Первая цифра кода определяет класс ответа:

- **1xx (Informational)** — информационные, промежуточные ответы. Означают, что запрос принят и обрабатывается, клиент должен ждать продолжения. Пример: `100 Continue`, `101 Switching Protocols`.
- **2xx (Success)** — запрос успешно получен, понят и обработан. Пример: `200 OK`, `201 Created`, `204 No Content`.
- **3xx (Redirection)** — для завершения запроса клиенту нужно выполнить дополнительные действия, обычно перейти по другому адресу. Пример: `301 Moved Permanently`, `302 Found`, `304 Not Modified`.
- **4xx (Client Error)** — ошибка на стороне клиента: неверный запрос, отсутствие авторизации, запрашиваемый ресурс не найден. Пример: `400 Bad Request`, `401 Unauthorized`, `403 Forbidden`, `404 Not Found`.
- **5xx (Server Error)** — сервер не смог выполнить корректный запрос из-за собственной ошибки. Пример: `500 Internal Server Error`, `502 Bad Gateway`, `503 Service Unavailable`.

Это деление задано стандартом RFC 7231 (ранее RFC 2616). Знание группы кода важно даже без знания конкретного значения: например, если тестировщик видит незнакомый код `418`, по первой цифре он поймёт, что это ошибка клиента, и будет анализировать запрос, а не сервер.

## Пример / когда применяется

При тестировании API это база для проверки контрактов: например, POST на создание ресурса должен возвращать `201`, а не `200`; невалидные данные — `400`, а не `500`. При анализе логов сервера резкий рост доли 5xx сигнализирует о проблемах на бэкенде, а рост 4xx — о некорректных запросах клиентов или проблемах в UI.

## На что смотрит интервьюер

Часто путают 401 и 403, не знают разницу между 301 и 302, забывают про 1xx и 5xx группы целиком, называют только 200 и 404. Хороший ответ — не просто перечисление кодов, а понимание смысла каждой группы и умение привести примеры конкретных кодов. Красный флаг — кандидат не может объяснить, зачем вообще нужны коды ответов, кроме "сервер так отвечает".
