# Какие виды куки вы знаете?

**Коротко:** Куки классифицируют по сроку жизни (session / persistent), по источнику (first-party / third-party) и по назначению (аутентификационные, трекинговые, функциональные, рекламные).

## Развёрнутый ответ

Куки можно классифицировать по нескольким независимым признакам.

**По сроку жизни:**
- **Session cookies** — не имеют атрибута `Expires`/`Max-Age`, удаляются автоматически при закрытии браузера. Используются, например, для хранения ID сессии авторизации.
- **Persistent (постоянные) cookies** — имеют заданный срок жизни через `Expires` или `Max-Age`, сохраняются между запусками браузера. Используются для "запомнить меня", настроек, долгосрочной аналитики.

**По источнику (домену):**
- **First-party cookies** — устанавливаются доменом, который пользователь непосредственно посещает (например, `example.com` устанавливает куку для `example.com`).
- **Third-party cookies** — устанавливаются доменом, отличным от того, что в адресной строке (например, рекламный или аналитический скрипт, встроенный на странице, ставит куку от своего домена). Активно блокируются современными браузерами (Safari ITP, Chrome постепенно отказывается от них) из-за проблем с приватностью.

**По назначению:**
- **Строго необходимые (essential/strictly necessary)** — обеспечивают базовую работу сайта (аутентификация, корзина покупок), не требуют согласия пользователя по GDPR.
- **Функциональные (functional)** — хранят пользовательские предпочтения (язык, регион, тема оформления).
- **Аналитические/статистические (analytics)** — используются для сбора статистики посещений (Google Analytics и т.п.).
- **Рекламные/таргетинговые (advertising/targeting)** — используются для показа персонализированной рекламы и ретаргетинга, часто это third-party куки.

**По безопасности (флаги):**
- **Secure cookies** — передаются только по HTTPS.
- **HttpOnly cookies** — недоступны из JavaScript, защита от XSS.
- **SameSite cookies** (Strict / Lax / None) — регулируют отправку в кросс-доменных запросах, защита от CSRF.

## Пример / когда применяется

Пример баннера согласия на куки (cookie consent), который QA должен протестировать: сайт при первом визите должен ставить только строго необходимые куки, а аналитические и рекламные — только после явного согласия пользователя (актуально для проверки соответствия GDPR/152-ФЗ).

```text
Set-Cookie: consent=necessary; Max-Age=31536000; SameSite=Lax
Set-Cookie: analytics_id=xyz789; Max-Age=2592000; SameSite=Lax   ; ставится только после клика "Принять всё"
```

## На что смотрит интервьюер

- Умение назвать классификацию хотя бы по двум признакам (срок жизни и first/third-party), а не просто перечислить одно деление.
- Понимание, зачем браузеры блокируют third-party cookies и как это влияет на тестирование (например, кросс-доменная авторизация может сломаться в Safari).
- Follow-up: "Как проверить, что сайт не ставит лишние куки до согласия пользователя?" (DevTools → Application → Cookies, до и после клика на баннере).
- Красный флаг — путаница между видами куки и куки по атрибутам безопасности (это разные оси классификации).
