# Какие заголовки ответа вы знаете (`Content-Type`, `Set-Cookie`, `Cache-Control`)?

**Коротко:** заголовки ответа передают клиенту метаданные о теле ответа и указания по его обработке — например, формат данных (`Content-Type`), установку cookie (`Set-Cookie`) и правила кеширования (`Cache-Control`).

## Развёрнутый ответ

Заголовки ответа (response headers) отправляются сервером клиенту и описывают сам ответ либо дают инструкции по его дальнейшей обработке. Разберём основные из вопроса:

- **`Content-Type`** — указывает MIME-тип и (опционально) кодировку тела ответа, например `application/json; charset=utf-8` или `text/html`. Клиент использует это значение, чтобы правильно распарсить тело.
- **`Set-Cookie`** — устанавливает cookie на стороне клиента. Важные атрибуты: `HttpOnly` (запрещает доступ к cookie из JavaScript, защита от XSS), `Secure` (передача только по HTTPS), `SameSite` (защита от CSRF, значения `Strict`/`Lax`/`None`), `Max-Age`/`Expires` (время жизни).
- **`Cache-Control`** — управляет кешированием ответа: `no-store` (не кешировать вовсе), `no-cache` (кешировать, но перед использованием проверять актуальность у сервера), `max-age=N` (время жизни кеша в секундах), `private`/`public` (кешировать только у клиента или также у прокси/CDN).

Другие важные заголовки ответа:

- **`Location`** — URI для редиректа (3xx) или адрес созданного ресурса (`201 Created`).
- **`ETag`** — идентификатор версии ресурса, используется вместе с `If-None-Match` для условных запросов и кеширования.
- **`Content-Length`** — размер тела ответа в байтах.
- **`Server`** — информация о серверном ПО (иногда скрывается из соображений безопасности).
- **`Access-Control-Allow-Origin`** — заголовок CORS, разрешающий кросс-доменные запросы с указанных источников.
- **`Strict-Transport-Security`** — предписывает браузеру использовать только HTTPS для домена (HSTS).

## Пример / когда применяется

```text
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Cache-Control: private, max-age=60
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Strict
ETag: "5d8c72a5"
```

При тестировании важно проверять: соответствие `Content-Type` фактическому телу ответа, наличие атрибутов безопасности у `Set-Cookie` (`HttpOnly`, `Secure`, `SameSite`) на продакшене, корректность директив `Cache-Control` для чувствительных данных (например, банковские данные не должны кешироваться публично).

## На что смотрит интервьюер

Проверяют знание практических нюансов, например атрибутов cookie для безопасности, а не только факта их существования. Частая ошибка — не знать разницы между `no-cache` и `no-store`, а также не упоминать атрибуты `Set-Cookie`, отвечающие за безопасность (`HttpOnly`, `Secure`, `SameSite`). Более сильные кандидаты дополнительно вспоминают `ETag` и условное кеширование, а также `CORS`-заголовки.
