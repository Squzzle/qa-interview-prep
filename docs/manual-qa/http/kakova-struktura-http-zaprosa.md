# Какова **структура HTTP-запроса**?

**Коротко:** HTTP-запрос состоит из стартовой строки (метод, путь, версия протокола), набора заголовков и опционального тела запроса.

## Развёрнутый ответ

Структура HTTP-запроса включает три основные части:

1. **Стартовая строка (request line)** — содержит:
   - HTTP-метод (GET, POST, PUT, DELETE и т.д.);
   - путь запроса (URI/path), например `/api/users/1`, может включать query-параметры (`?page=2&limit=10`);
   - версию протокола, например `HTTP/1.1`.

2. **Заголовки (headers)** — набор пар «ключ: значение», передающих метаданные о запросе. Наиболее часто встречающиеся:
   - `Host` — обязательный в HTTP/1.1, указывает домен назначения (важен при виртуальном хостинге);
   - `Content-Type` — MIME-тип тела запроса, например `application/json`;
   - `Content-Length` — размер тела в байтах;
   - `Authorization` — данные аутентификации (Bearer-токен, Basic auth);
   - `Cookie` — передаваемые сессионные данные;
   - `Accept`, `Accept-Language`, `Accept-Encoding` — предпочтения клиента по формату ответа;
   - `User-Agent` — идентификация клиентского ПО.

3. **Пустая строка** — разделяет заголовки и тело запроса (согласно спецификации, обязательна даже при отсутствии тела).

4. **Тело запроса (body)** — опциональная часть, содержащая данные, которые передаются серверу (JSON, form-data, XML и т.д.). Тело обычно присутствует в POST, PUT, PATCH запросах и отсутствует в GET, HEAD, DELETE (хотя технически тело в GET не запрещено спецификацией, на практике оно игнорируется большинством серверов).

## Пример / когда применяется

Пример HTTP-запроса на создание пользователя:

```text
POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
Content-Length: 47

{"name": "Ivan", "email": "ivan@example.com"}
```

При тестировании API через Postman или curl QA-инженер должен уметь собрать такой запрос вручную, указать корректные заголовки (особенно `Content-Type`) и проверить, что тело сериализовано правильно:

```bash
curl -X POST https://example.com/api/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -d '{"name": "Ivan", "email": "ivan@example.com"}'
```

## На что смотрит интервьюер

- Знание обязательности заголовка `Host` в HTTP/1.1.
- Понимание, что query-параметры — часть пути, а не отдельный компонент.
- Умение назвать типичные заголовки и их назначение (особенно `Content-Type`, `Authorization`).
- Follow-up: «Может ли GET-запрос иметь тело?», «Чем отличаются query-параметры от path-параметров?», «Что произойдёт, если `Content-Length` не совпадает с фактическим размером тела?».
