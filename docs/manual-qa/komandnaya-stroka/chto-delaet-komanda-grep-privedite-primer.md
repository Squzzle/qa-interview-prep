# Что делает команда `grep`? Приведите пример.

**Коротко:** `grep` (Global Regular Expression Print) ищет строки, соответствующие заданному шаблону (в том числе регулярному выражению) в тексте или файлах, и выводит совпавшие строки — это одна из самых часто используемых команд для анализа логов.

## Развёрнутый ответ

`grep` — утилита Linux/Unix (доступна и в Windows через WSL, Git Bash или PowerShell как алиас), которая построчно сканирует входной текст (файл или поток) и выводит строки, соответствующие заданному шаблону поиска. Шаблон может быть простой подстрокой или полноценным регулярным выражением.

Часто используемые ключи `grep`:

- `-i` — игнорировать регистр (case-insensitive).
- `-r` / `-R` — рекурсивный поиск по всем файлам в директории и её поддиректориях.
- `-n` — выводить номер строки, где найдено совпадение.
- `-v` — инвертировать поиск: показать строки, которые НЕ соответствуют шаблону.
- `-c` — вывести только количество совпадающих строк.
- `-E` — использовать расширенные регулярные выражения (ERE), эквивалент `egrep`.
- `-l` — вывести только имена файлов, где есть совпадение (без самих строк).
- `-A <N>` / `-B <N>` / `-C <N>` — вывести N строк после/до/вокруг найденной строки (контекст, полезно для анализа стектрейсов).
- `--color` — подсветка совпадений в выводе.

`grep` часто используется в связке с конвейером (`|`), принимая на вход вывод другой команды. Например, `ps aux | grep java` найдёт процессы, содержащие "java" в описании.

Для тестировщика `grep` — незаменимый инструмент при анализе логов: вместо того чтобы вручную листать многотысячные логи, можно быстро отфильтровать только строки с ошибками, конкретным ID запроса, временным диапазоном или кодом ответа.

## Пример / когда применяется

Найти все строки с ошибками в логе, с номерами строк и без учёта регистра:

```bash
grep -in "error" app.log
```

Найти в логе конкретный код ответа 500 вместе с 3 строками контекста после найденной строки (полезно, чтобы увидеть стектрейс):

```bash
grep -A 3 "HTTP 500" app.log
```

Найти во всех файлах проекта упоминания устаревшего API-эндпоинта:

```bash
grep -rn "api/v1/legacy" ./src
```

Показать строки, которые НЕ содержат "INFO" (то есть все не-информационные записи):

```bash
grep -v "INFO" app.log
```

## На что смотрит интервьюер

- Знание ключевых флагов (`-i`, `-r`, `-v`, `-n`, `-A/-B/-C`) и умение объяснить, когда какой использовать.
- Понимание, что `grep` работает построчно и поддерживает регулярные выражения.
- Умение комбинировать `grep` с другими командами через конвейер (`|`), например с `ps`, `cat`, `tail -f`.
- Follow-up: "Как найти строки, которые содержат и 'ERROR', и 'timeout' одновременно?" (ответ: `grep "ERROR" app.log | grep "timeout"` или через регулярное выражение с `grep -E "ERROR.*timeout"`).
- Красный флаг: кандидат не может объяснить разницу между простым текстовым поиском и поиском по регулярному выражению.
