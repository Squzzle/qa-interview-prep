# Что делают операторы `>`, `>>` и `|` в командной строке?

**Коротко:** `>` перенаправляет вывод команды в файл, перезаписывая его; `>>` дописывает вывод в конец файла, не удаляя старое содержимое; `|` (пайп) передаёт стандартный вывод одной команды на стандартный вход другой.

## Развёрнутый ответ

В Unix-подобных системах (Linux, macOS) каждая команда работает с тремя стандартными потоками: `stdin` (ввод), `stdout` (вывод) и `stderr` (ошибки). Операторы `>`, `>>` и `|` управляют тем, куда направляются эти потоки.

- `>` — оператор перенаправления вывода в файл. Если файл существует, его содержимое будет полностью стёрто и заменено новым выводом команды. Если файла нет — он будет создан.
- `>>` — оператор добавления (append). Работает так же, как `>`, но не перезаписывает файл, а дописывает вывод в его конец, сохраняя существующее содержимое. Часто используется для логирования.
- `|` (pipe, конвейер) — соединяет стандартный вывод левой команды со стандартным вводом правой команды. Это позволяет строить цепочки обработки данных из нескольких простых утилит без создания промежуточных файлов.

Важные нюансы:

- По умолчанию `>` перенаправляет только `stdout`, но не `stderr`. Чтобы перенаправить ошибки, используют `2>`, а чтобы перенаправить оба потока — `> file 2>&1` (порядок важен: сначала stdout уходит в файл, затем stderr перенаправляется туда же, куда сейчас указывает stdout).
- Перед выполнением команды shell открывает файл для `>` в режиме перезаписи, поэтому если исходный и целевой файл совпадают (например, `cat file > file`), содержимое файла может быть потеряно.
- В конвейере `|` каждая команда запускается как отдельный процесс, и все они выполняются параллельно, обмениваясь данными через буфер (канал, pipe) в памяти ядра.
- Код возврата всего конвейера по умолчанию — это код возврата последней команды в цепочке (это можно изменить опцией `set -o pipefail` в bash).

## Пример / когда применяется

```bash
# Перезаписать файл результатом команды
ls -la > files.txt

# Дописать новую строку в лог, не стирая старые записи
echo "run finished" >> app.log

# Передать вывод одной команды на вход другой
ps aux | grep node

# Комбинация: найти процессы node и сохранить в файл, добавляя к предыдущим запускам
ps aux | grep node >> node_processes.log

# Перенаправить и stdout, и stderr в один файл
./build.sh > build.log 2>&1
```

## На что смотрит интервьюер

- Понимает ли кандидат разницу между перезаписью (`>`) и дозаписью (`>>`) — частая ошибка новичков потерять данные из-за `>`.
- Знает ли, что `>` не перенаправляет `stderr` по умолчанию и как это исправить (`2>&1`).
- Понимает ли механизм конвейера как соединение процессов через `stdin`/`stdout`, а не просто "магическую передачу текста".
- Может ли привести практический пример использования (фильтрация логов, поиск процессов, обработка вывода команд).
- Красный флаг: если кандидат путает `|` с `&&` или `;` (это операторы управления последовательностью выполнения команд, а не потоками данных).
