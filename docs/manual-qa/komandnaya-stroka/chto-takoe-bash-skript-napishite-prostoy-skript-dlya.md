# Что такое **bash-скрипт**? Напишите простой скрипт для запуска тестов.

**Коротко:** bash-скрипт — это текстовый файл с последовательностью команд для интерпретатора bash, который позволяет автоматизировать повторяющиеся действия, например запуск набора тестов, сборку проекта или деплой.

## Развёрнутый ответ

Bash-скрипт — это обычный текстовый файл, первая строка которого обычно содержит шебанг (`#!/bin/bash` или `#!/usr/bin/env bash`), указывающий системе, каким интерпретатором выполнять файл. Внутри скрипта можно использовать переменные, условия (`if`), циклы (`for`, `while`), функции, а также вызывать любые системные утилиты и другие программы.

Основные элементы, которые обычно встречаются в тестовых скриптах:

- Переменные окружения и параметры (`$1`, `$2`, `$ENV_VAR`) для настройки окружения запуска.
- Проверка кода возврата (`$?`) после выполнения команды, чтобы понять, успешно ли прошли тесты.
- Условные конструкции `if [ условие ]; then ... fi` для ветвления логики (например, разное поведение для CI и локального запуска).
- `set -e` — заставляет скрипт немедленно завершаться при первой ошибке (ненулевой код возврата любой команды), что важно для CI-пайплайнов, чтобы не пропустить упавший шаг.
- `set -u` — завершает выполнение при обращении к неинициализированной переменной, что помогает ловить опечатки.
- Права на выполнение: файл должен иметь флаг `x` (`chmod +x script.sh`), либо запускаться явно через интерпретатор (`bash script.sh`).

## Пример / когда применяется

Простой скрипт для запуска тестов (например, набора автотестов на pytest или npm test), с обработкой кода возврата и логированием:

```bash
#!/bin/bash
set -euo pipefail

echo "Запуск тестов..."

# Можно передать окружение первым параметром: ./run_tests.sh staging
ENVIRONMENT=${1:-local}
echo "Окружение: $ENVIRONMENT"

# Запуск тестов, например через pytest
pytest tests/ --junitxml=report.xml

# Проверка кода возврата предыдущей команды
if [ $? -eq 0 ]; then
  echo "Все тесты прошли успешно"
else
  echo "Тесты завершились с ошибкой"
  exit 1
fi
```

Такой скрипт применяется в CI/CD (Jenkins, GitLab CI, GitHub Actions) для автоматического запуска регрессионных или smoke-тестов при каждом коммите или деплое.

## На что смотрит интервьюер

- Знает ли кандидат про шебанг и права на исполнение файла.
- Использует ли `set -e`/`set -euo pipefail` для надёжной остановки при ошибках — это показатель практического опыта написания production-скриптов, а не только "hello world".
- Проверяет ли код возврата команд (`$?` или напрямую в `if`), а не полагается на визуальный вывод.
- Может ли объяснить, зачем нужны параметры (`$1`) для гибкости скрипта (запуск в разных окружениях).
- Красный флаг: скрипт без обработки ошибок, который "тихо" продолжит выполнение после упавшего теста и может дать ложноположительный результат в CI.
