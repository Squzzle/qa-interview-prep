# Что такое **cmdlet** в PowerShell?

**Коротко:** cmdlet (командлет) — это встроенная в PowerShell лёгковесная команда-функция .NET, реализующая одну конкретную операцию, названная по схеме "Глагол-Существительное" (например, `Get-Process`), которая принимает и возвращает объекты, а не текст.

## Развёрнутый ответ

Cmdlet (произносится "командлет") — базовая единица функциональности в PowerShell. В отличие от обычных исполняемых файлов (.exe) или batch-скриптов CMD, cmdlet — это скомпилированный .NET-класс, встроенный в PowerShell или подключаемый через модуль.

Ключевые характеристики cmdlet:

- **Соглашение об именовании "Глагол-Существительное".** Например: `Get-Process` (получить процессы), `Stop-Process` (остановить процесс), `New-Item` (создать элемент), `Remove-Item` (удалить элемент), `Set-Location` (сменить директорию). Такая единообразная схема делает команды предсказуемыми и легко угадываемыми.
- **Работа с объектами, а не с текстом.** Cmdlet принимает на вход и возвращает объекты .NET со свойствами и методами, что позволяет передавать результат одного cmdlet в другой по конвейеру (`|`) без парсинга строк.
- **Стандартизированные параметры.** Многие cmdlet поддерживают общие параметры вроде `-Verbose`, `-WhatIf`, `-Confirm`, `-ErrorAction`, что даёт единообразное поведение по всей системе.
- **Расширяемость через модули.** Разработчики и Microsoft поставляют дополнительные cmdlet в виде модулей (например, модуль `Az` для управления Azure, `ActiveDirectory` для работы с AD).

Отличие cmdlet от функции и от обычной программы:
- Функция в PowerShell — это пользовательский скриптовый блок кода, написанный на языке PowerShell.
- Cmdlet — как правило, скомпилированный .NET-класс (хотя есть и "advanced functions", которые ведут себя как cmdlet, но написаны на самом PowerShell).
- Обычная программа (.exe) — внешний исполняемый файл, не интегрированный в объектную модель PowerShell; её вывод — обычный текст.

Базовые cmdlet, которые полезно знать тестировщику:

- `Get-Process` / `Stop-Process` — просмотр и завершение процессов.
- `Get-Content` / `Set-Content` — чтение/запись содержимого файла.
- `Get-ChildItem` — список файлов и папок (алиас `ls`, `dir`).
- `Get-Service` — статус служб Windows.
- `Test-Connection` — аналог `ping` с объектным результатом.
- `Invoke-WebRequest` / `Invoke-RestMethod` — HTTP-запросы, полезны для API-тестирования.

## Пример / когда применяется

Тестировщик хочет проверить доступность API-эндпоинта и получить статус-код ответа:

```powershell
$response = Invoke-WebRequest -Uri "https://api.example.com/health" -Method GET
$response.StatusCode
```

Или найти и завершить зависший процесс тестового приложения:

```powershell
Get-Process -Name "myapp" | Stop-Process -Force
```

## На что смотрит интервьюер

- Понимает ли кандидат соглашение "Глагол-Существительное" и может привести несколько примеров навскидку.
- Понимание разницы между cmdlet, функцией и обычным исполняемым файлом.
- Знание, что cmdlet работают с объектами, а не строками — это ключевая идея, отличающая PowerShell.
- Follow-up: "Как узнать список параметров конкретного cmdlet?" (ответ: `Get-Help <cmdlet> -Full` или `-Examples`).
