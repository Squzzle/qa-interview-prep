# Что такое `curl` и как с его помощью отправить GET-запрос?

**Коротко:** `curl` — консольная утилита для отправки HTTP(S)- и других сетевых запросов (FTP, и т.д.) и получения ответа сервера; GET-запрос по умолчанию отправляется простым указанием URL: `curl https://example.com/api/users`.

## Развёрнутый ответ

`curl` (Client URL) — универсальный инструмент командной строки для взаимодействия с удалёнными серверами по различным протоколам, чаще всего HTTP и HTTPS. В тестировании API он широко используется для ручной проверки эндпоинтов, отладки запросов, воспроизведения багов и написания smoke-тестов в CI без установки специализированных фреймворков.

По умолчанию `curl` отправляет GET-запрос, если не указано иное. Полезные флаги для работы с GET-запросами:

- `-X GET` — явно указать метод (необязательно, так как GET используется по умолчанию, но полезно для читаемости).
- `-i` — вывести заголовки ответа вместе с телом.
- `-I` — отправить HEAD-запрос и показать только заголовки, не загружая тело.
- `-v` (verbose) — подробный вывод, включая заголовки запроса, ответ сервера, этапы TLS-хендшейка — полезно для отладки проблем с сетью или сертификатами.
- `-s` (silent) — подавить индикатор прогресса, оставить только сам ответ (удобно в скриптах).
- `-o file.json` — сохранить тело ответа в файл вместо вывода в терминал.
- `-w "%{http_code}"` — вывести дополнительную информацию, например HTTP-статус код ответа, что полезно для проверок в скриптах.
- `-G` вместе с `-d` — передать параметры запроса в query string (например, `curl -G -d "id=5" https://example.com/api/users`).

## Пример / когда применяется

```bash
# Простой GET-запрос, вывод тела ответа в терминал
curl https://api.example.com/users

# GET-запрос с выводом заголовков ответа и статус-кода
curl -i https://api.example.com/users/42

# GET-запрос с параметрами через query string
curl -G -d "page=2" -d "limit=10" https://api.example.com/users

# Проверка HTTP-статуса в скрипте (полезно для smoke-теста в CI)
STATUS=$(curl -s -o /dev/null -w "%{http_code}" https://api.example.com/health)
if [ "$STATUS" -eq 200 ]; then
  echo "Сервис доступен"
else
  echo "Сервис недоступен, статус: $STATUS"
  exit 1
fi
```

## На что смотрит интервьюер

- Знает ли кандидат, что GET — метод по умолчанию, и когда стоит явно указывать `-X GET`.
- Умеет ли получить не только тело, но и заголовки/статус-код ответа (`-i`, `-I`, `-w`) — это ключевой навык для QA при отладке API.
- Может ли объяснить разницу между `-s` (тихий режим) и `-v` (подробный вывод) и когда какой уместен.
- Умеет ли использовать `curl` в скриптах для автоматизированных проверок (smoke-тесты, health-check), а не только вручную в терминале.
- Красный флаг: если кандидат не знает базового синтаксиса `curl` и не может привести ни одного примера использования флагов сверх голого URL.
