# Что такое **PowerShell**? Чем он отличается от CMD?

**Коротко:** PowerShell — современная командная оболочка и язык сценариев от Microsoft, построенная на .NET, которая в отличие от CMD оперирует не текстом, а объектами, поддерживает полноценные скрипты, работу с командлетами (cmdlet) и глубокую интеграцию с системой и облачными сервисами.

## Развёрнутый ответ

CMD (Command Prompt) — это простая текстовая оболочка, унаследованная от MS-DOS, с ограниченным набором встроенных команд и слабыми возможностями скриптинга. PowerShell был создан Microsoft как замена CMD для более сложных задач автоматизации, администрирования систем и работы с .NET-объектами.

Ключевые отличия PowerShell от CMD:

| Характеристика | CMD | PowerShell |
|---|---|---|
| Что передаётся по конвейеру | Обычный текст | Объекты .NET |
| Команды | Встроенные команды и внешние утилиты | Cmdlet'ы (глагол-существительное, например `Get-Process`) |
| Язык сценариев | Ограниченный batch-синтаксис (.bat) | Полноценный скриптовый язык (.ps1) с переменными, циклами, функциями, обработкой ошибок |
| Работа с объектами | Нет — только строки | Да — можно обращаться к свойствам и методам объектов |
| Расширяемость | Слабая | Модули, доступ к .NET и COM, интеграция с Azure/Windows API |
| Кроссплатформенность | Только Windows | PowerShell Core (с версии 6) работает на Windows, Linux, macOS |
| Управление правами | Ограниченное | Встроенная система политик выполнения скриптов (Execution Policy), поддержка удалённого управления (PowerShell Remoting) |

Главное концептуальное отличие: в CMD результат команды — это просто текстовый вывод, который приходится парсить (например, через `find` или `findstr`), чтобы извлечь нужные данные. В PowerShell результат команды — это объект с полноценными свойствами и методами, поэтому можно сразу фильтровать, сортировать и обрабатывать данные без парсинга строк:

```powershell
Get-Process | Where-Object {$_.CPU -gt 100} | Sort-Object CPU -Descending
```

PowerShell также поддерживает алиасы для привычных команд, поэтому многие команды CMD и Bash работают в нём "по инерции" (`dir`, `ls`, `cd`, `cls` — всё это алиасы к соответствующим cmdlet'ам).

## Пример / когда применяется

Тестировщику нужно найти все процессы, потребляющие больше 500 МБ памяти, и завершить их. В CMD пришлось бы парсить текстовый вывод `tasklist` вручную, а в PowerShell это делается объектно и лаконично:

```powershell
Get-Process | Where-Object {$_.WorkingSet -gt 500MB} | Stop-Process -Force
```

Другой пример — автоматизация проверки логов в рамках регресс-тестирования:

```powershell
Get-Content app.log -Wait | Select-String "ERROR"
```

## На что смотрит интервьюер

- Понимание ключевого отличия: объекты против текста в конвейере — это основной концептуальный вопрос, который отличает поверхностное знание от глубокого.
- Знает ли кандидат синтаксис cmdlet'ов (глагол-Существительное: `Get-Process`, `Stop-Process`, `Get-Content`).
- В курсе ли кандидат, что PowerShell кроссплатформенный начиная с PowerShell Core / PowerShell 7.
- Красный флаг: кандидат считает PowerShell просто "более новой версией CMD с другими названиями команд", не понимая объектной модели.
- Follow-up: "Как в PowerShell отфильтровать список файлов больше 10 МБ?" — ожидается использование `Get-ChildItem` и `Where-Object`.
