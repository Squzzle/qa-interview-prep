# Что значит `$?` в bash?

**Коротко:** `$?` — специальная переменная bash, которая хранит код завершения (exit code) последней выполненной команды: `0` означает успех, любое ненулевое значение — ошибку.

## Развёрнутый ответ

Каждая команда в Unix-подобной системе по завершении возвращает числовой код завершения (exit status) операционной системе. По соглашению код `0` означает, что команда выполнилась успешно, а любое значение от `1` до `255` сигнализирует об ошибке (конкретное значение может означать разные типы ошибок в зависимости от программы).

`$?` — это переменная оболочки, которая всегда содержит код возврата последней выполненной команды. Она обновляется автоматически после каждой команды, поэтому проверять её нужно сразу после нужной команды, не выполняя между ними ничего лишнего — иначе значение перезапишется.

Практическое применение:

- Проверка успешности выполнения команды в скриптах: `if [ $? -eq 0 ]; then ... fi`.
- Использование в тестовых и деплой-скриптах для принятия решения — продолжать выполнение дальше или прервать процесс (`exit 1`).
- В конвейерах (`|`) `$?` по умолчанию отражает код возврата только последней команды в цепочке; чтобы учитывать ошибки любой команды в конвейере, используется `set -o pipefail`.
- Более идиоматичный способ проверки — не сохранять `$?` в переменную и потом сравнивать, а сразу использовать команду в условии: `if command; then ... fi`, что эквивалентно, но короче и надёжнее (нет риска случайно "затереть" `$?` промежуточной командой).

## Пример / когда применяется

```bash
#!/bin/bash

grep "ERROR" app.log
if [ $? -eq 0 ]; then
  echo "В логе найдены ошибки"
else
  echo "Ошибок не найдено"
fi

# Более идиоматичный вариант без явного использования $?
if grep -q "ERROR" app.log; then
  echo "В логе найдены ошибки"
fi

# Использование $? для завершения скрипта с тем же кодом ошибки
cp source.txt /backup/
exit $?
```

## На что смотрит интервьюер

- Знает ли кандидат, что `0` — это успех, а не «ошибка», как иногда интуитивно думают новички из других языков.
- Понимает ли, что `$?` нужно проверять сразу после команды, иначе значение будет перезаписано следующей командой (даже `echo` или `[`).
- Знает ли про `set -o pipefail` для корректной проверки ошибок внутри конвейеров.
- Умеет ли предложить более идиоматичный подход — `if command; then` вместо явного сравнения `$?` — что говорит об опыте написания качественных shell-скриптов.
- Красный флаг: если кандидат не может объяснить, откуда берётся это значение (что это код возврата процесса на уровне ОС).
