# Для чего нужна командная строка тестировщику?

**Коротко:** командная строка нужна тестировщику для быстрой диагностики окружения, работы с файлами и логами, запуска скриптов и утилит, а также для автоматизации рутинных проверок без графического интерфейса.

## Развёрнутый ответ

Командная строка (CLI, command-line interface) — это текстовый интерфейс для взаимодействия с операционной системой и приложениями через ввод команд. Для тестировщика это один из базовых инструментов повседневной работы, потому что многие задачи либо невозможно, либо неудобно выполнять через графический интерфейс.

Основные причины, зачем тестировщику нужна командная строка:

- **Диагностика сети и окружения.** Команды вроде `ping`, `ipconfig`/`ifconfig`, `traceroute`, `netstat` позволяют быстро проверить доступность сервера, состояние сети, занятые порты — это критично при локализации багов, связанных с окружением.
- **Работа с логами.** Логи серверов и приложений часто лежат на удалённых машинах без GUI. Команды `cat`, `grep`, `tail -f`, `less` позволяют быстро найти нужную ошибку в многогигабайтном логе.
- **Управление файлами и процессами.** Создание/удаление тестовых файлов, распаковка архивов, проверка запущенных процессов (`ps aux`), завершение зависших процессов (`kill`) — всё это часто нужно для подготовки тестового окружения или для расследования падения приложения.
- **Запуск тестов и скриптов.** Автотесты, сборка проекта, запуск утилит командной строки (curl, Postman CLI, Selenium в headless-режиме, Docker) — всё это делается через терминал.
- **Работа с серверами по SSH.** Тестировщику часто нужно подключиться к тестовому или staging-серверу, где графического интерфейса нет вовсе.
- **Ускорение рутинных операций.** Массовое переименование файлов, поиск по содержимому директорий, фильтрация данных — через CLI это делается быстрее и воспроизводимее, чем руками через проводник.

Кроме практической пользы, знание CLI показывает уровень технической зрелости кандидата: способность работать без GUI считается признаком более глубокого понимания того, как устроена система под капотом.

## Пример / когда применяется

Типичная ситуация: тестировщик обнаружил, что после деплоя приложение отвечает с ошибкой 500. Вместо того чтобы просить разработчика прислать логи, тестировщик подключается по SSH к серверу и сам находит причину:

```bash
ssh user@test-server
cd /var/log/app
tail -f app.log | grep ERROR
```

Или для диагностики сетевой проблемы:

```bash
ping api.example.com
ipconfig /all      # Windows
ifconfig            # Linux/Mac
```

## На что смотрит интервьюер

- Может ли кандидат назвать конкретные команды и объяснить, зачем каждая нужна, а не только сказать общую фразу "это удобно".
- Понимание разницы между CMD, PowerShell и Bash — знает ли кандидат, что в разных ОС используются разные оболочки.
- Есть ли у кандидата реальный опыт использования CLI для диагностики багов, а не только теоретическое знание.
- Красный флаг: кандидат утверждает, что "командная строка не нужна, всё есть в GUI" — это говорит о недостатке практического опыта в реальных проектах, особенно с бэкендом или DevOps-окружением.
- Follow-up вопросы: "Как найти строку с ошибкой в логе за вчерашний день?", "Как проверить, слушает ли сервис нужный порт?".
