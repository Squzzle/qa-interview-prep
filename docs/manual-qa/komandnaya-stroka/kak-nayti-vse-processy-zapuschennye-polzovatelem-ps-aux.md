# Как найти все процессы, запущенные пользователем (`ps aux | grep user`)?

**Коротко:** список всех процессов пользователя получают командой `ps aux`, а конкретного пользователя или процесса фильтруют через конвейер с `grep`: `ps aux | grep <имя>`, либо более точным способом — флагом `-u` (`ps -u <имя_пользователя>`).

## Развёрнутый ответ

`ps` (process status) — стандартная утилита Linux/Unix для получения снимка (snapshot) списка процессов, запущенных в системе на момент выполнения команды.

Разбор ключей `ps aux`:

- `a` — показать процессы всех пользователей (не только текущего).
- `u` — вывести подробный формат с указанием владельца процесса (столбец USER), использования CPU и памяти.
- `x` — включить процессы, не привязанные к терминалу (фоновые демоны, сервисы).

Столбцы типичного вывода `ps aux`:

- `USER` — владелец процесса.
- `PID` — идентификатор процесса.
- `%CPU`, `%MEM` — использование процессора и памяти.
- `VSZ`, `RSS` — виртуальный и реальный размер памяти.
- `STAT` — статус процесса (R — running, S — sleeping, Z — zombie и т.д.).
- `START` — время запуска.
- `COMMAND` — команда, которой был запущен процесс.

Для фильтрации по пользователю или по имени процесса чаще всего комбинируют `ps aux` с `grep`:

```bash
ps aux | grep john
```

Однако у этого подхода есть известный нюанс: в результате поиска часто попадает и сам процесс `grep john`, ищущий строку "john" — это раздражающий побочный эффект. Чтобы его избежать, используют:

```bash
ps aux | grep '[j]ohn'
```

(регулярное выражение с символьным классом `[j]` не совпадает с самим текстом команды grep, так как в аргументах grep строка "[j]ohn" не равна "john").

Более чистый и правильный способ — использовать встроенную фильтрацию `ps` без grep:

```bash
ps -u john
```

или найти процессы по имени команды:

```bash
pgrep -u john
pgrep -l nginx
```

## Пример / когда применяется

Тестировщик хочет найти все процессы, запущенные под пользователем "appuser", чтобы проверить, не осталось ли зависших процессов после падения теста:

```bash
ps -u appuser
```

Найти процесс конкретного Java-приложения с исключением самого grep из результата:

```bash
ps aux | grep '[j]ava'
```

Комбинация с подсчётом количества запущенных экземпляров процесса:

```bash
ps aux | grep -c '[n]ginx'
```

## На что смотрит интервьюер

- Знание значения ключей `a`, `u`, `x` в `ps aux`.
- Понимание "проблемы grep grep" — что результат `ps aux | grep name` часто включает сам процесс grep, и как этого избежать (`grep '[n]ame'` или `pgrep`).
- Знание альтернатив: `ps -u <user>`, `pgrep`, `top`/`htop` для интерактивного просмотра.
- Follow-up: "Как узнать, сколько памяти потребляет конкретный процесс?" (ответ: столбец `%MEM` или `RSS` в выводе `ps aux`, либо `ps -o rss= -p <PID>`).
