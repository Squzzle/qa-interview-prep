# Как посмотреть содержимое файла в Linux?

**Коротко:** содержимое файла в Linux можно посмотреть командами `cat` (вывести целиком), `less`/`more` (постраничный просмотр), `head`/`tail` (начало/конец файла) или `tail -f` (слежение в реальном времени) — выбор команды зависит от размера файла и задачи.

## Развёрнутый ответ

В Linux нет единственного способа "открыть файл" как в GUI — есть набор специализированных команд, каждая из которых подходит под конкретный сценарий:

- **`cat <файл>`** — выводит содержимое файла целиком в терминал. Подходит для небольших файлов. Для больших файлов неудобен, так как весь текст сразу "пролетает" по экрану.
- **`less <файл>`** — открывает файл в постраничном интерактивном просмотрщике. Позволяет прокручивать вверх/вниз, искать текст (`/шаблон`), не загружая файл в память целиком — оптимален для больших файлов. Выход — клавиша `q`.
- **`more <файл>`** — более старый и ограниченный аналог `less` (например, не поддерживает прокрутку назад в некоторых системах).
- **`head <файл>`** — выводит первые 10 строк файла по умолчанию (`head -n 50 файл` — первые 50 строк).
- **`tail <файл>`** — выводит последние 10 строк файла по умолчанию (`tail -n 100 файл` — последние 100 строк). Особенно полезен для логов, так как новые записи обычно добавляются в конец файла.
- **`tail -f <файл>`** — "следящий" режим: выводит новые строки по мере их появления в файле в реальном времени, не завершая работу (используется для мониторинга логов "вживую").
- **`nano <файл>` / `vim <файл>`** — полноценные текстовые редакторы в терминале, позволяют не только смотреть, но и редактировать содержимое.
- **`strings <файл>`** — для бинарных файлов, выводит только читаемые текстовые фрагменты.

Выбор команды зависит от контекста: для быстрого взгляда на маленький конфиг подойдёт `cat`, для многогигабайтного лога — `less` или `tail`, а для мониторинга в реальном времени — `tail -f`.

## Пример / когда применяется

Тестировщик хочет посмотреть последние 50 строк лога после воспроизведения бага:

```bash
tail -n 50 /var/log/app/app.log
```

Если файл большой и нужно искать конкретное слово с возможностью прокрутки:

```bash
less /var/log/app/app.log
```

(внутри `less` можно нажать `/ERROR` и Enter для поиска слова "ERROR")

Быстро вывести небольшой конфигурационный файл целиком:

```bash
cat config.yaml
```

## На что смотрит интервьюер

- Знание разницы между `cat`, `less`, `head` и `tail` и понимание, когда какую использовать (особенно важно для больших файлов, где `cat` неэффективен).
- Знание ключа `-f` у `tail` для слежения за логами в реальном времени.
- Умение сочетать команды через конвейер: например, `tail -f app.log | grep ERROR`.
- Follow-up: "Файл занимает 10 ГБ, нужно посмотреть последние строки — что будете делать?" (ответ: `tail`, а не `cat`, так как `cat` загрузит весь файл).
