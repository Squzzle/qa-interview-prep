# Как в bash-скрипте передать параметр и обратиться к нему (`$1`, `$2`)?

**Коротко:** параметры передаются скрипту как аргументы командной строки при вызове (`./script.sh arg1 arg2`), а внутри скрипта к ним обращаются через позиционные переменные `$1`, `$2` и так далее, где `$0` — это имя самого скрипта.

## Развёрнутый ответ

Когда bash-скрипт запускается с аргументами, оболочка автоматически помещает их в специальные позиционные переменные:

- `$0` — имя (путь) самого скрипта.
- `$1`, `$2`, `$3`, ... — первый, второй, третий и так далее переданные аргументы.
- `$#` — количество переданных аргументов.
- `$@` — все аргументы как отдельные слова (список).
- `$*` — все аргументы как одна строка.
- `"$@"` (в кавычках) — корректно сохраняет границы аргументов, если они содержат пробелы; это предпочтительный способ передавать аргументы дальше в другие команды.

Если аргументов больше девяти, обращаться к десятому и последующим нужно через фигурные скобки: `${10}`, `${11}` (просто `$10` будет интерпретировано как `$1` с добавленным символом `0`).

Часто используемые приёмы:

- Значения по умолчанию: `NAME=${1:-default_value}` — если `$1` не передан, используется `default_value`.
- Проверка количества аргументов: `if [ $# -lt 2 ]; then echo "Использование: $0 <arg1> <arg2>"; exit 1; fi`.
- Сдвиг аргументов командой `shift`, чтобы обрабатывать их по очереди в цикле (полезно для парсинга флагов).

## Пример / когда применяется

```bash
#!/bin/bash
# Использование: ./deploy.sh production v1.2.3

ENVIRONMENT=$1
VERSION=$2

if [ $# -lt 2 ]; then
  echo "Ошибка: нужно передать окружение и версию"
  echo "Использование: $0 <environment> <version>"
  exit 1
fi

echo "Разворачиваем версию $VERSION в окружении $ENVIRONMENT"
echo "Скрипт называется: $0"
echo "Всего аргументов передано: $#"
```

Вызов: `./deploy.sh production v1.2.3` выведет окружение `production`, версию `v1.2.3`, имя скрипта и количество аргументов (`2`).

## На что смотрит интервьюер

- Знает ли кандидат, что `$0` — это имя скрипта, а не первый аргумент (частая ошибка — путать нумерацию).
- Умеет ли обращаться к параметрам после девятого через `${10}`.
- Понимает ли разницу между `$@` и `$*`, особенно в контексте кавычек (`"$@"` сохраняет отдельные аргументы, `"$*"` склеивает всё в одну строку).
- Проверяет ли количество переданных аргументов (`$#`) перед использованием, чтобы избежать пустых значений и падения скрипта.
- Красный флаг: отсутствие валидации входных параметров в production-скриптах, что может привести к неявным ошибкам при запуске без нужных аргументов.
