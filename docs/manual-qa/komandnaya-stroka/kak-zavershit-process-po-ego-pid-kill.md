# Как завершить процесс по его PID (`kill`)?

**Коротко:** процесс завершают командой `kill <PID>`, которая по умолчанию отправляет сигнал SIGTERM (корректное завершение); если процесс не реагирует, используют `kill -9 <PID>` (сигнал SIGKILL — принудительное немедленное завершение).

## Развёрнутый ответ

`kill` — команда Linux/Unix, которая отправляет процессу сигнал операционной системы. Несмотря на название, `kill` не всегда "убивает" процесс жёстко — по умолчанию она посылает сигнал SIGTERM, который просит процесс корректно завершиться самостоятельно (закрыть файлы, освободить ресурсы, сохранить состояние).

Основные сигналы:

- `kill <PID>` или `kill -15 <PID>` (SIGTERM) — "вежливая" просьба завершиться. Процесс может перехватить этот сигнал и корректно завершить работу (например, закрыть соединения с базой данных).
- `kill -9 <PID>` (SIGKILL) — принудительное немедленное завершение процесса на уровне ядра ОС. Процесс не может перехватить или проигнорировать этот сигнал; используется, когда процесс завис и не реагирует на SIGTERM.
- `kill -1 <PID>` (SIGHUP) — часто используется для того, чтобы заставить процесс перечитать конфигурацию без полной остановки (многие демоны обрабатывают SIGHUP именно так).
- `kill -l` — вывести список всех доступных сигналов с их номерами.

Как найти PID процесса перед тем, как его завершить:

```bash
ps aux | grep myapp
```

или

```bash
pgrep myapp
```

Альтернативные команды:

- `pkill <имя_процесса>` — завершить процесс по имени, без необходимости искать PID вручную.
- `killall <имя_процесса>` — завершить все процессы с указанным именем.
- `xkill` (в GUI-окружении) — интерактивно завершить процесс кликом по окну.

В Windows аналог — `taskkill /PID <номер>` или `taskkill /IM <имя.exe> /F` (флаг `/F` — принудительное завершение, аналог SIGKILL).

Важный практический нюанс: SIGKILL нужно использовать как крайнюю меру, потому что процесс не успевает освободить ресурсы и сохранить данные — это может привести к повреждению файлов, незакрытым соединениям с БД или потере несохранённых данных. Правильный порядок действий: сначала попробовать `kill` (SIGTERM), подождать несколько секунд, и только если процесс не завершился — применить `kill -9`.

## Пример / когда применяется

Тестировщик обнаружил зависший процесс тестового сервера, который не отвечает на запросы:

```bash
ps aux | grep test-server
# нашли PID = 4821
kill 4821
sleep 5
ps aux | grep test-server
# если процесс всё ещё жив:
kill -9 4821
```

Массовое завершение всех процессов Node.js, оставшихся от предыдущего запуска автотестов:

```bash
pkill -f "node.*test-runner"
```

## На что смотрит интервьюер

- Понимание разницы между SIGTERM (по умолчанию, "мягкое" завершение) и SIGKILL (`kill -9`, принудительное).
- Знание, что перед `kill -9` разумно сначала попробовать обычный `kill` и подождать.
- Умение найти PID нужного процесса перед завершением (`ps aux | grep`, `pgrep`).
- Красный флаг: кандидат считает, что `kill` без параметров всегда "жёстко убивает" процесс — это распространённое заблуждение.
- Follow-up: "Почему злоупотребление `kill -9` может быть опасно для базы данных или файловой системы?" (ответ: процесс не успевает корректно закрыть транзакции и файловые дескрипторы, что может привести к повреждению данных).
