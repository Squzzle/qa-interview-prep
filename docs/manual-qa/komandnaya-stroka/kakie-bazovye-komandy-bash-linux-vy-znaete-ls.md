# Какие базовые команды **Bash (Linux)** вы знаете (`ls`, `cd`, `pwd`, `mkdir`)?

**Коротко:** базовый набор команд Bash включает навигацию (`cd`, `pwd`, `ls`), работу с файлами и папками (`mkdir`, `rm`, `cp`, `mv`, `touch`), просмотр содержимого (`cat`, `less`, `tail`), поиск (`grep`, `find`), а также работу с правами и процессами (`chmod`, `ps`, `kill`).

## Развёрнутый ответ

Bash (Bourne Again Shell) — самая распространённая командная оболочка в Linux и macOS. Она сочетает интерактивный режим работы с полноценным скриптовым языком (bash-скрипты, .sh-файлы).

Команды навигации:

- `pwd` (print working directory) — вывести путь текущей директории.
- `cd <путь>` — сменить директорию (`cd ~` — домашняя директория, `cd -` — предыдущая директория, `cd ..` — на уровень выше).
- `ls` — вывести список файлов и папок (`ls -l` — подробный вывод с правами и размером, `ls -a` — включая скрытые файлы, `ls -lh` — размеры в человекочитаемом виде).

Команды работы с файлами и папками:

- `mkdir <имя>` — создать директорию (`mkdir -p a/b/c` — создать вложенные директории сразу).
- `rm <файл>` — удалить файл (`rm -r` — рекурсивно для папок, `rm -rf` — принудительно без подтверждений).
- `rmdir` — удалить пустую директорию.
- `cp <источник> <назначение>` — скопировать файл (`cp -r` — для директорий).
- `mv <источник> <назначение>` — переместить или переименовать файл.
- `touch <файл>` — создать пустой файл или обновить время модификации существующего.

Просмотр содержимого и поиск:

- `cat <файл>` — вывести содержимое файла целиком.
- `less` / `more` — постраничный просмотр файла.
- `head` / `tail` — первые/последние N строк файла (`tail -f` — слежение за файлом в реальном времени).
- `grep <шаблон> <файл>` — поиск текста по шаблону (регулярному выражению).
- `find <путь> -name "<шаблон>"` — поиск файлов по имени, дате, размеру и т.д.

Права доступа и процессы:

- `chmod` — изменить права доступа к файлу (например, `chmod +x script.sh` — сделать исполняемым).
- `chown` — изменить владельца файла.
- `ps aux` — список запущенных процессов.
- `kill <PID>` — завершить процесс по идентификатору.
- `top` / `htop` — интерактивный мониторинг процессов и загрузки системы.

Работа с конвейерами и перенаправлением:

- `|` — конвейер, передаёт вывод одной команды на вход другой.
- `>` — перенаправить вывод в файл (перезаписать).
- `>>` — перенаправить вывод в файл (дописать).

## Пример / когда применяется

Тестировщик заходит на тестовый сервер по SSH, создаёт директорию для тестовых артефактов и ищет в логах ошибки за сегодня:

```bash
pwd
mkdir -p ~/test-artifacts/2026-07-09
cd ~/test-artifacts/2026-07-09
grep -i "error" /var/log/app/app.log | tail -20
```

## На что смотрит интервьюер

- Знает ли кандидат ключевые флаги (`-r`, `-f`, `-l`, `-a`) и разницу между похожими командами (`rm` vs `rmdir`, `mv` vs `cp`).
- Понимание конвейеров и перенаправления вывода (`|`, `>`, `>>`) — базовый навык для комбинирования команд.
- Практический опыт: может ли кандидат объяснить сценарий использования, а не просто перечислить команды.
- Красный флаг: незнание разницы между `rm -rf` и обычным `rm` — потенциально опасное отсутствие понимания последствий команд.
- Follow-up: "Как удалить все файлы с расширением .tmp в директории и её поддиректориях?" (ответ через `find . -name "*.tmp" -delete` или `find . -name "*.tmp" -exec rm {} \;`).
