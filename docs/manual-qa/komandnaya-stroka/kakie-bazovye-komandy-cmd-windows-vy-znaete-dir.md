# Какие базовые команды **CMD (Windows)** вы знаете (`dir`, `cd`, `copy`, `del`)?

**Коротко:** базовый набор команд CMD включает навигацию по каталогам (`cd`, `dir`), работу с файлами (`copy`, `move`, `del`, `ren`, `md`, `rd`), просмотр содержимого (`type`), а также сетевые и системные утилиты (`ipconfig`, `ping`, `tasklist`, `taskkill`).

## Развёрнутый ответ

CMD (Command Prompt) — стандартная командная оболочка Windows, унаследованная от MS-DOS. Она работает с текстовыми командами и поддерживает базовые операции с файловой системой, процессами и сетью.

Основные команды для навигации и просмотра:

- `dir` — вывести список файлов и папок в текущей директории (аналог `ls` в Linux).
- `cd <путь>` — сменить текущую директорию (`cd ..` — подняться на уровень выше, `cd \` — перейти в корень диска).
- `cls` — очистить экран терминала.

Команды для работы с файлами и папками:

- `copy <источник> <назначение>` — скопировать файл.
- `move <источник> <назначение>` — переместить или переименовать файл.
- `del <файл>` (или `erase`) — удалить файл.
- `ren <старое имя> <новое имя>` — переименовать файл.
- `md <папка>` (или `mkdir`) — создать директорию.
- `rd <папка>` (или `rmdir`) — удалить директорию (пустую; с ключом `/s` — рекурсивно).
- `type <файл>` — вывести содержимое текстового файла в консоль.
- `xcopy` / `robocopy` — расширенное копирование файлов и директорий, включая рекурсивное копирование с сохранением атрибутов.

Системные и служебные команды:

- `echo` — вывести текст на экран или в файл.
- `find` / `findstr` — поиск текста внутри файлов (аналог `grep`).
- `attrib` — изменить атрибуты файла (скрытый, только для чтения и т.д.).
- `tasklist` — показать список запущенных процессов.
- `taskkill /PID <номер>` или `taskkill /IM <имя.exe>` — завершить процесс.

Важный нюанс: CMD исторически ограничен по сравнению с Bash — нет полноценного скриптового языка, слабая поддержка конвейеров и объектов (только текстовые потоки), из-за чего Microsoft развивает PowerShell как более мощную замену.

## Пример / когда применяется

Тестировщик готовит тестовое окружение: создаёт папку для логов, копирует туда конфигурационный файл и проверяет его содержимое:

```bat
md C:\test-logs
copy C:\app\config.ini C:\test-logs\config_backup.ini
cd C:\test-logs
dir
type config_backup.ini
```

## На что смотрит интервьюер

- Знает ли кандидат разницу между похожими командами: `del` удаляет файл, а `rd` — директорию; `copy` не удаляет исходный файл, а `move` удаляет.
- Понимание ключей команд (например, `/s` для рекурсии, `/q` для тихого режима) — показывает практический опыт, а не поверхностное знание.
- Может ли кандидат провести параллель с аналогичными командами в Linux (`dir` = `ls`, `del` = `rm`, `type` = `cat`) — это показывает системное понимание, а не зазубривание.
- Красный флаг: путаница между CMD и PowerShell синтаксисом (например, использование `Remove-Item` в контексте CMD).
