# Что такое KQL (Kibana Query Language)? Напишите пример запроса.

**Коротко:** KQL — это упрощённый язык запросов Kibana для поиска и фильтрации документов в Elasticsearch прямо из интерфейса Discover, более простой и читаемый, чем полноценный Query DSL или Lucene syntax.

## Развёрнутый ответ

KQL (Kibana Query Language) — язык запросов, разработанный специально для Kibana как более удобная альтернатива классическому Lucene query syntax. Основные особенности KQL:

- **Простой синтаксис поиска по полю:** `поле: значение` — например, `level: "ERROR"`.
- **Логические операторы** `and`, `or`, `not` для комбинирования условий.
- **Поддержка вложенных полей и группировки** через скобки: `(level: "ERROR" or level: "WARN") and service: "auth"`.
- **Поиск по диапазону** для числовых и датовых полей: `response_time >= 500` или `@timestamp >= "2026-07-09"`.
- **Wildcard-поиск** с использованием `*`: `message: "connection*"` — найдёт "connection refused", "connection timeout" и т.п.
- **Поиск по существованию поля:** `userId: *` — найти документы, где поле `userId` заполнено (не пустое).
- Автодополнение полей и значений прямо в строке поиска Kibana, что упрощает построение запросов без знания точной структуры документов.

KQL по умолчанию используется в строке поиска Discover, в фильтрах визуализаций и в условиях алертов. Он менее мощный, чем полный Elasticsearch Query DSL (не поддерживает сложные агрегации или скрипты), но покрывает подавляющее большинство повседневных задач поиска и фильтрации логов.

## Пример / когда применяется

Примеры запросов KQL:

```text
level: "ERROR" and service: "payment-service"
```
— найти все ошибки в сервисе платежей.

```text
level: "ERROR" and not message: "timeout"
```
— все ошибки, кроме связанных с таймаутами.

```text
response_time >= 1000 and http.status_code: 500
```
— запросы с временем ответа от 1000 мс и статус-кодом 500.

```text
userId: "u12345" and (level: "ERROR" or level: "WARN")
```
— все предупреждения и ошибки для конкретного пользователя.

## На что смотрит интервьюер

Интервьюер проверяет, умеет ли кандидат реально писать поисковые запросы, а не только знает название языка. Важно показать синтаксис `поле: значение`, логические операторы и, желательно, разницу между KQL и Lucene syntax. Красный флаг — если кандидат не может привести ни одного корректного примера запроса.
