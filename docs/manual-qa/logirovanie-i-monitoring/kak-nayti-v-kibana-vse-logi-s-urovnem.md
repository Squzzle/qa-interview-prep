# Как найти в Kibana все логи с уровнем ERROR за последний час?

**Коротко:** нужно открыть раздел Discover, выбрать корректный индекс-паттерн, установить временной диапазон "Last 1 hour" и ввести в строке поиска запрос KQL вида `level: "ERROR"`.

## Развёрнутый ответ

Практический алгоритм действий в Kibana:

1. **Открыть Discover** — основной раздел для интерактивного просмотра "сырых" данных из Elasticsearch.
2. **Выбрать index pattern (или data view)** — набор индексов, по которым будет производиться поиск (например, `logs-*`, объединяющий все ежедневные индексы логов).
3. **Установить временной диапазон** в правом верхнем углу — выбрать пресет "Last 1 hour" (последний час) либо задать произвольный диапазон дат/времени. Kibana фильтрует документы по полю `@timestamp` (или другому полю времени, указанному в index pattern).
4. **Ввести запрос в строку поиска** на языке KQL: `level: "ERROR"`. Если название поля отличается (например, `log.level` или `severity`), нужно использовать точное имя поля из схемы документов — это можно проверить через автодополнение, которое предлагает Kibana при вводе.
5. **Опционально уточнить запрос** дополнительными условиями через `and`, например, отфильтровать по конкретному сервису: `level: "ERROR" and service: "order-service"`.
6. Результат отображается в виде таблицы записей с возможностью развернуть каждую и посмотреть все поля документа (стек трейс, request id, дополнительные метаданные).

Дополнительно можно сохранить такой поиск как "Saved Search" для повторного использования, или добавить визуализацию поверх этого фильтра (например, гистограмму количества ошибок по времени), а также настроить алерт, который будет присылать уведомление при превышении порога количества ERROR-записей за интервал.

## Пример / когда применяется

Итоговый запрос в строке поиска Discover с временным фильтром "Last 1 hour":

```text
level: "ERROR"
```

Более точный вариант, с явным указанием диапазона времени прямо в запросе (если не пользоваться пикером времени):

```text
level: "ERROR" and @timestamp >= "now-1h"
```

Практический сценарий: во время нагрузочного тестирования QA-инженер хочет быстро проверить, не появились ли новые ошибки за последний час — открывает Discover, применяет фильтр `level: "ERROR"` с диапазоном "Last 1 hour" и видит все инциденты в реальном времени.
