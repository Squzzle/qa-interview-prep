# Какую роль в ELK играет Elasticsearch?

**Коротко:** Elasticsearch — это ядро ELK Stack, распределённая база данных и поисковый движок, который хранит логи в виде JSON-документов, индексирует их и обеспечивает быстрый поиск, фильтрацию и агрегацию по огромным объёмам данных.

## Развёрнутый ответ

Elasticsearch построен на движке Apache Lucene и является NoSQL, документо-ориентированной поисковой системой. В контексте ELK его роль — центральное хранилище и "мозг" всего стека, отвечающий за:

- **Хранение данных.** Каждая запись лога сохраняется как документ в формате JSON внутри индекса (аналог таблицы в реляционных БД, хотя структура более гибкая — схема может выводиться автоматически, mapping).
- **Индексирование.** При добавлении документа Elasticsearch строит инвертированный индекс (inverted index) по текстовым полям, что позволяет выполнять полнотекстовый поиск за миллисекунды даже по терабайтам данных.
- **Поиск и фильтрацию.** Поддерживает богатый Query DSL (JSON-запросы), а также более простой язык KQL/Lucene query syntax для интерактивного поиска в Kibana. Можно искать по точному совпадению, диапазону дат, wildcard, regex, полнотекстовому совпадению с релевантностью.
- **Агрегации.** Позволяет строить статистику "на лету": сколько ошибок за час, среднее время ответа, top-10 самых частых сообщений об ошибках, группировка по сервисам и т.д. Именно на этих агрегациях строятся графики в Kibana.
- **Масштабируемость и отказоустойчивость.** Elasticsearch распределяет данные по шардам (shards) на разных узлах кластера и реплицирует их (replicas) для устойчивости к сбоям, что позволяет хранить и обрабатывать очень большие объёмы логов.

Важная особенность — управление жизненным циклом индексов (Index Lifecycle Management, ILM): логи обычно разбиваются на индексы по дням/неделям (например, `logs-2026.07.09`), а старые индексы автоматически архивируются или удаляются, чтобы не переполнять диск.

## Пример / когда применяется

Пример упрощённого поискового запроса к Elasticsearch (Query DSL) для поиска ошибок за последний час:

```json
GET /logs-*/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "level": "ERROR" } },
        { "range": { "@timestamp": { "gte": "now-1h" } } }
      ]
    }
  }
}
```

Kibana под капотом транслирует пользовательский запрос (введённый в поисковой строке) именно в такие запросы к Elasticsearch.

## На что смотрит интервьюер

Интервьюер проверяет понимание, что Elasticsearch — это не просто "часть ELK", а именно хранилище+поисковый движок, и что Kibana сама по себе данные не хранит, а лишь визуализирует то, что лежит в Elasticsearch. Красный флаг — путаница ролей компонентов.
