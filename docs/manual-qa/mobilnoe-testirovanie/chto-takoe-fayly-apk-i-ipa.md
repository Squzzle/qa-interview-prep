# Что такое файлы **.apk** и **.ipa**?

**Коротко:** APK (Android Package) — формат установочного пакета приложений для Android, а IPA (iOS App Store Package) — аналогичный формат для iOS; оба представляют собой архивы, содержащие скомпилированный код, ресурсы, манифест и цифровую подпись, необходимые для установки приложения на устройство.

## Развёрнутый ответ

**APK (Android Package Kit)** — это архив в формате ZIP с расширением `.apk`, содержащий всё необходимое для установки и запуска Android-приложения:
- `AndroidManifest.xml` — манифест с описанием компонентов приложения, разрешений, минимальной и целевой версии SDK, package name.
- `classes.dex` — скомпилированный байт-код приложения (Dalvik/ART).
- Ресурсы (`res/`) — изображения, строки, layout-файлы, стили.
- `lib/` — нативные библиотеки для разных архитектур процессора (arm64-v8a, armeabi-v7a, x86 и т.д.).
- Цифровая подпись (META-INF) — сертификат разработчика, которым подписан пакет; при обновлении приложения подпись новой версии должна совпадать с установленной, иначе система откажет в установке.

Современная альтернатива для публикации в Google Play — формат **AAB (Android App Bundle)**: это не устанавливаемый напрямую файл, а формат публикации, из которого Google Play сам генерирует оптимизированные APK под конкретное устройство (Dynamic Delivery), уменьшая размер скачивания.

**IPA (iOS App Store Package)** — это архив в формате ZIP с расширением `.ipa`, содержащий:
- Скомпилированное приложение (`Payload/AppName.app`) с бинарным файлом и ресурсами (изображения, шрифты, storyboard/xib-файлы).
- `Info.plist` — конфигурационный файл с метаданными приложения: bundle identifier, версия, минимальная поддерживаемая версия iOS, используемые разрешения (usage descriptions).
- Provisioning profile и цифровую подпись сертификатом разработчика/дистрибутива — без корректной подписи и профиля приложение не установится на устройство (кроме случая через App Store, где подпись проверяется и переподписывается Apple).

Основные различия между форматами:

| Параметр | APK | IPA |
|---|---|---|
| Платформа | Android | iOS |
| Формат архива | ZIP | ZIP |
| Установка вне стора | Возможна (side-loading) | Ограничена (Ad Hoc/Enterprise, требует подписи и/или регистрации UDID устройства) |
| Подпись | Подпись разработчика (self-signed допустим для отладки) | Обязательна подпись сертификатом Apple Developer Program |
| Формат публикации в сторе | AAB (Android App Bundle) | IPA |

## Пример / когда применяется

Тестировщику эти файлы нужны на практике, например: для установки тестовой сборки APK на устройство командой `adb install app-release.apk`, для проверки содержимого манифеста и разрешений внутри APK (можно распаковать как ZIP-архив или использовать `aapt dump badging app.apk`), либо для загрузки IPA в TestFlight через Transporter/Xcode при подготовке релиза на iOS.

```bash
# Установка APK на подключённое Android-устройство
adb install -r app-release.apk

# Просмотр информации о пакете (package name, версия, разрешения)
aapt dump badging app-release.apk
```

## На что смотрит интервьюер

Интервьюер проверяет понимание, что это не просто "установочные файлы", а архивы определённой структуры с манифестом/Info.plist и обязательной подписью. Плюсом считается знание разницы между APK и AAB, а также понимание, почему подпись критична (несовпадение подписи при обновлении, необходимость provisioning profile на iOS). Красный флаг — если кандидат не может объяснить назначение подписи или путает APK с AAB.
