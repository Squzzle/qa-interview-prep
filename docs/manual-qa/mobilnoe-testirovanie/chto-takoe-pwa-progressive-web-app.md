# Что такое **PWA (Progressive Web App)**?

**Коротко:** PWA — это веб-приложение, которое благодаря Service Worker, манифесту и современным браузерным API ведёт себя как нативное: работает офлайн, устанавливается на домашний экран и умеет отправлять push-уведомления.

## Развёрнутый ответ

Progressive Web App — это концепция и набор технологий, позволяющих обычному сайту приобрести свойства нативного приложения, оставаясь при этом веб-приложением, доступным по URL. Ключевые технологии PWA:

- **Service Worker** — фоновый скрипт, работающий отдельно от основного потока страницы, который перехватывает сетевые запросы, кэширует ресурсы и обеспечивает работу приложения в офлайн-режиме или при нестабильной сети.
- **Web App Manifest** — JSON-файл с метаданными приложения (имя, иконки, цвет темы, стартовый URL), который позволяет "установить" PWA на домашний экран без магазина приложений.
- **HTTPS** — обязательное условие: PWA работает только по защищённому протоколу.
- **Push API / Notifications API** — позволяют отправлять push-уведомления даже когда вкладка браузера закрыта.

Критерии "качественной" PWA обычно проверяются через Lighthouse: быстрая загрузка, адаптивность под любые экраны, работоспособность офлайн, безопасность (HTTPS), возможность установки.

Для тестировщика PWA — гибрид между обычным веб-тестированием и мобильным: нужно проверять как классические веб-аспекты (кроссбраузерность, адаптивность), так и специфичные для PWA сценарии — работу Service Worker (кэширование, инвалидация кэша при обновлении версии), поведение при полном отсутствии сети, корректность установки на разные ОС (Android поддерживает полноценно, iOS — с ограничениями), доставку push-уведомлений и работу манифеста (иконка, splash screen).

## Пример / когда применяется

Примеры известных PWA: Twitter Lite, Starbucks, Pinterest, AliExpress. Такой подход применяется, когда компании нужно охватить пользователей с медленным интернетом или слабыми устройствами, снизить порог входа (не нужно скачивать из стора) и одновременно дать опыт, близкий к нативному приложению.

Чек-лист тестирования PWA:

```text
1. Установка на домашний экран (Android/iOS) — корректность иконки и имени
2. Работа приложения в режиме "нет сети" (Airplane mode)
3. Инвалидация кэша при выходе новой версии Service Worker
4. Получение push-уведомлений при закрытом приложении
5. Проверка через Lighthouse на соответствие критериям PWA
6. Поведение при переключении Wi-Fi <-> мобильная сеть
```

## На что смотрит интервьюер

- Знание, что PWA — это не отдельная платформа, а расширение возможностей обычного сайта.
- Понимание ограничений PWA на iOS (нет полноценных push до недавнего времени, ограниченный доступ к некоторым API).
- Follow-up: "Чем PWA отличается от гибридного приложения?" (PWA не требует установки из стора и не оборачивается в нативный контейнер).
- Красный флаг — если кандидат путает PWA с обычным адаптивным сайтом и не может назвать роль Service Worker.
