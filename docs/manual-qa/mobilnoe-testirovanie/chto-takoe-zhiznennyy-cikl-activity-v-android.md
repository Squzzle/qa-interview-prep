# Что такое **жизненный цикл Activity** в Android?

**Коротко:** это последовательность состояний, через которые проходит экран приложения (Activity) от создания до уничтожения, управляемая системой через методы-колбэки: `onCreate`, `onStart`, `onResume`, `onPause`, `onStop`, `onDestroy` и `onRestart`.

## Развёрнутый ответ

Activity — это компонент Android, представляющий отдельный экран с пользовательским интерфейсом. Система вызывает предопределённые методы жизненного цикла при переходах между состояниями (создание, переход на передний план, сворачивание, восстановление, уничтожение), и разработчик переопределяет их для инициализации ресурсов, сохранения состояния и освобождения памяти.

Основные методы жизненного цикла:

- **onCreate()** — вызывается при первом создании Activity: инициализация UI (`setContentView`), привязка данных, восстановление сохранённого состояния из `Bundle`.
- **onStart()** — Activity становится видимой пользователю, но ещё не находится в фокусе взаимодействия.
- **onResume()** — Activity выходит на передний план и получает фокус, пользователь может с ней взаимодействовать; это "активное" состояние.
- **onPause()** — Activity частично теряет фокус (например, поверх появился диалог или всплывающее уведомление), но остаётся частично видимой; здесь принято сохранять несохранённые данные и останавливать ресурсоёмкие операции (анимации, камеру).
- **onStop()** — Activity полностью скрыта от пользователя (например, открыта другая Activity или пользователь свернул приложение); здесь освобождают тяжёлые ресурсы.
- **onRestart()** — вызывается перед повторным onStart(), если Activity возвращается из состояния Stopped.
- **onDestroy()** — финальный вызов перед уничтожением Activity — либо из-за явного завершения (`finish()`), либо из-за уничтожения системой (например, при повороте экрана или нехватке памяти); здесь освобождают все оставшиеся ресурсы.

Важный нюанс для тестировщика: при повороте экрана, смене языка или других configuration changes система по умолчанию уничтожает и заново создаёт Activity (`onDestroy` → `onCreate`), поэтому именно в таких сценариях чаще всего проявляются баги с потерей введённых пользователем данных, если состояние не сохраняется в `onSaveInstanceState()`. Также при уходе приложения в фон система может уничтожить процесс целиком для освобождения памяти, и при возврате пользователя произойдёт пересоздание Activity с попыткой восстановления состояния.

## Пример / когда применяется

Классический тестовый сценарий — проверка сохранения состояния экрана при повороте устройства: пользователь вводит текст в форму, поворачивает экран, и приложение не должно "терять" введённые данные. Это напрямую завязано на корректную обработку `onSaveInstanceState()` / `onRestoreInstanceState()` в паре с жизненным циклом.

Также жизненный цикл важен при тестировании сценариев прерывания: входящий звонок или пуш-уведомление во время работы с приложением переводит Activity в `onPause`/`onStop`, и после возврата приложение должно корректно восстановить своё состояние (`onResume`), не теряя данные и не показывая пустой экран.

## На что смотрит интервьюер

Интервьюер проверяет знание порядка вызова методов и понимание, в каком состоянии Activity видна/невидима/в фокусе. Частая ошибка — путать `onPause` и `onStop` (частичная видимость против полного скрытия) или не знать, что поворот экрана приводит к пересозданию Activity. Хороший ответ связывает жизненный цикл с реальными тестовыми сценариями: потеря данных при повороте, поведение при сворачивании/разворачивании, при low memory kill.
