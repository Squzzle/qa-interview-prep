# Как с помощью ADB эмулировать нажатие на экран?

**Коротко:** нажатие эмулируется командой `adb shell input tap x y`, где x и y — координаты точки на экране в пикселях.

## Развёрнутый ответ

Команда `input` в ADB shell позволяет программно генерировать события ввода — тапы, свайпы, ввод текста, нажатия аппаратных клавиш — без реального взаимодействия с экраном устройства. Это полезно для автоматизации простых сценариев, воспроизведения багов по шагам или удалённого тестирования устройств без физического доступа.

Основные команды семейства `input`:

- `adb shell input tap x y` — одиночное нажатие (тап) по координатам x, y.
- `adb shell input swipe x1 y1 x2 y2 [duration_ms]` — свайп от одной точки к другой за указанное время.
- `adb shell input text "строка"` — ввод текста в активное текстовое поле (без пробелов, либо экранированных).
- `adb shell input keyevent KEYCODE` — эмуляция нажатия аппаратной или системной клавиши (например, `KEYCODE_BACK`, `KEYCODE_HOME`, `KEYCODE_ENTER`).
- `adb shell input press` (реже используется) — аналог tap для некоторых версий.

Координаты x, y отсчитываются от левого верхнего угла экрана в пикселях физического разрешения устройства, поэтому перед использованием полезно узнать разрешение через `adb shell wm size`.

## Пример / когда применяется

Тестировщик хочет быстро воспроизвести баг, который проявляется после серии тапов по определённым координатам, без написания полноценного автотеста:

```bash
adb shell wm size
adb shell input tap 540 1200
adb shell input swipe 500 1500 500 300 300
adb shell input keyevent KEYCODE_BACK
```

Здесь сначала узнаётся разрешение экрана, затем выполняется тап по кнопке, свайп вверх (например, для прокрутки ленты) и нажатие системной кнопки "Назад".

## На что смотрит интервьюер

Интервьюер оценивает, знает ли кандидат, что координаты зависят от разрешения экрана конкретного устройства и не переносятся между устройствами напрямую, а также понимает разницу между `tap`, `swipe` и `keyevent`. Частая ошибка — путать `input tap` с `input keyevent` или не знать, откуда брать координаты. Возможные follow-up: "как узнать разрешение экрана устройства?", "как эмулировать долгое нажатие (long press)?", "чем этот подход хуже полноценных фреймворков автоматизации (Appium, Espresso)?".
