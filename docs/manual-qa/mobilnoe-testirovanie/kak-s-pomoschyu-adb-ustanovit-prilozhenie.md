# Как с помощью **ADB** установить приложение?

**Коротко:** приложение устанавливается командой `adb install путь-к-файлу.apk`, при повторной установке с сохранением данных используется флаг `-r`.

## Развёрнутый ответ

ADB (Android Debug Bridge) — консольный инструмент из Android SDK, который позволяет взаимодействовать с эмулятором или физическим устройством: устанавливать приложения, снимать логи, управлять файловой системой, эмулировать ввод и многое другое. Перед использованием команд устройство должно быть подключено по USB (с включённой отладкой по USB) или через Wi-Fi, а также должно определяться командой `adb devices`.

Базовая команда установки APK-файла:

```bash
adb install path/to/app.apk
```

Полезные флаги:

- `-r` — переустановить приложение, сохранив пользовательские данные (replace).
- `-t` — разрешить установку тестовых APK (test-only).
- `-d` — разрешить установку с понижением версии (downgrade), обычно требует debug-сборку.
- `-g` — сразу выдать все runtime-разрешения (grant permissions) при установке.

Если подключено несколько устройств одновременно, нужно указать конкретное через `-s <serial>`, серийный номер которого можно узнать из вывода `adb devices`.

Для установки на конкретное устройство при нескольких подключённых:

```bash
adb -s emulator-5554 install app.apk
```

Если приложение состоит из нескольких APK (App Bundle / split APK), используется `adb install-multiple` со списком файлов.

## Пример / когда применяется

Тестировщик получает от разработчика новую сборку в виде APK и должен установить её на тестовое устройство, не удаляя ранее сохранённые тестовые данные (логин, локальную БД):

```bash
adb install -r ./builds/release-1.2.3.apk
```

Если нужно установить приложение поверх уже стоящей версии с другой подписью, потребуется сначала удалить старую версию (`adb uninstall <package>`), так как Android не позволяет заменить приложение с несовпадающей подписью.

## На что смотрит интервьюер

Интервьюер проверяет базовое владение ADB, знание флага `-r` и понимание, что установка может завершиться ошибкой `INSTALL_FAILED_UPDATE_INCOMPATIBLE` при несовпадении подписи сборки. Плюсом будет упоминание работы с несколькими устройствами и split APK / App Bundle. Частый follow-up: "что делать, если установка падает с ошибкой конфликта подписи?", "как установить приложение без предварительного удаления данных?", "как понять имя пакета для последующих команд?".
