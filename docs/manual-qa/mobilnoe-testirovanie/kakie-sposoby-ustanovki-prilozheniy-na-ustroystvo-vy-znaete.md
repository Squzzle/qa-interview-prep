# Какие **способы установки приложений** на устройство вы знаете?

**Коротко:** Приложение можно установить через официальный магазин (Google Play, App Store), напрямую с помощью файла APK/IPA (side-loading, TestFlight), через инструменты разработки (Android Studio, Xcode) или из командной строки (adb install, xcrun/altool), а также через внутренние корпоративные и тестовые каналы распространения.

## Развёрнутый ответ

Способы установки приложений различаются в зависимости от платформы и цели (продакшн, тестирование, разработка):

**Android:**
- **Google Play Store** — основной канал для конечных пользователей; поддерживает внутреннее (internal), закрытое (closed) и открытое (open) тестирование, а также поэтапный раскат (staged rollout).
- **Прямая установка APK/AAB-файла (side-loading)** — установка через `adb install app.apk` из командной строки, либо копированием файла на устройство и запуском через файловый менеджер (требует разрешения "установка из неизвестных источников").
- **Android Studio** — запуск приложения на подключённом устройстве/эмуляторе прямо из IDE в режиме debug.
- **Firebase App Distribution** — сервис для распространения тестовых сборок ограниченному кругу тестировщиков без публикации в сторе.
- **Корпоративные MDM-решения** — установка приложений на управляемые корпоративные устройства через системы Mobile Device Management.
- **Сторонние магазины/маркеты** (для отдельных регионов, например, устройства без сервисов Google) — альтернативные магазины приложений.

**iOS:**
- **App Store** — основной канал для конечных пользователей.
- **TestFlight** — официальный инструмент Apple для бета-тестирования, позволяет распространять сборки внутренним и внешним тестировщикам (до определённого лимита) без публикации в сторе.
- **Xcode** — прямая установка на подключённое устройство или симулятор через IDE в режиме разработки (требует подписи сертификатом разработчика).
- **Ad Hoc / Enterprise-распространение** — установка IPA-файла на список зарегистрированных устройств (Ad Hoc) или в рамках корпоративной программы (Apple Developer Enterprise Program) без App Store.
- **xcrun / altool / Transporter** — инструменты командной строки/приложения для загрузки сборок в App Store Connect.

Дополнительно, для тестирования на множестве устройств без физического доступа используются **облачные фермы устройств** (Firebase Test Lab, BrowserStack App Live, Sauce Labs), куда сборка загружается и устанавливается на удалённые реальные или виртуальные устройства.

## Пример / когда применяется

Пример выбора способа установки в зависимости от задачи:

| Задача | Способ установки |
|---|---|
| Разработчик отлаживает новую фичу | Запуск через Android Studio / Xcode на подключённом устройстве |
| QA получает сборку для регрессионного тестирования | Установка через `adb install` (Android) или TestFlight (iOS) |
| Бета-тестирование с внешними пользователями | Google Play Closed/Open testing, TestFlight |
| Проверка на 20+ моделях устройств без их физического наличия | Загрузка сборки в облачную ферму устройств (Firebase Test Lab, BrowserStack) |
| Финальный релиз | Публикация в Google Play / App Store с поэтапным раскатом |

## На что смотрит интервьюер

Интервьюер проверяет, знает ли кандидат разницу между тестовыми каналами (TestFlight, internal/closed testing) и side-loading, а также базовые команды (`adb install`). Плюсом будет упоминание облачных ферм устройств и понимание, что side-loading на iOS требует подписи сертификатом и включённого доверия разработчику в настройках устройства. Красный флаг — если кандидат знает только про установку "через магазин" и не может назвать альтернативные способы для тестирования.
