# Напишите команду ADB для очистки данных приложения.

**Коротко:** данные приложения очищаются командой `adb shell pm clear <package_name>`, что аналогично нажатию "Очистить данные" в системных настройках Android.

## Развёрнутый ответ

Команда `pm clear` (Package Manager) удаляет все данные приложения: локальную базу данных, SharedPreferences, кэш, файлы во внутреннем хранилище приложения, а также сбрасывает выданные разрешения (permissions) до состояния "не запрошено". Само приложение при этом не удаляется — деинсталляции не происходит, в отличие от `adb uninstall`.

Синтаксис команды:

```bash
adb shell pm clear com.example.myapp
```

Где `com.example.myapp` — точное имя пакета (package name), которое можно узнать несколькими способами:

- из файла `AndroidManifest.xml` сборки;
- командой `adb shell pm list packages | grep myapp`;
- через `aapt dump badging app.apk | grep package`.

Важные нюансы для тестировщика:

- После `pm clear` приложение переходит в состояние "первого запуска" — это удобно для тестирования онбординга, экранов приветствия, первичной настройки разрешений.
- Команда не трогает данные, сохранённые вне песочницы приложения (например, файлы на внешнем хранилище, если приложение писало туда напрямую, а не через scoped storage).
- Есть отдельная команда для очистки только кэша: `adb shell pm clear --cache-only <package_name>` (доступна не на всех версиях API).

## Пример / когда применяется

Тестировщик проверяет сценарий "первый запуск после установки" (флоу онбординга, запрос разрешений, отображение туториала) без переустановки приложения:

```bash
adb shell pm clear com.example.myapp
adb shell am start -n com.example.myapp/.MainActivity
```

Первая команда очищает данные, вторая — сразу запускает приложение с "чистого листа", что ускоряет цикл регрессионного тестирования онбординга.

## На что смотрит интервьюер

Интервьюер проверяет, отличает ли кандидат `pm clear` от `uninstall` (данные vs всё приложение целиком) и знает ли, как узнать имя пакета. Частая ошибка — путать очистку данных с очисткой кэша или считать, что после `pm clear` приложение полностью удаляется. Возможные follow-up вопросы: "чем отличается очистка данных от очистки кэша?", "что произойдёт с выданными permissions после clear?", "как найти package name без исходников?".
