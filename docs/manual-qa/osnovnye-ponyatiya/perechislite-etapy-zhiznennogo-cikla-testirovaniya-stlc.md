# Перечислите этапы **жизненного цикла тестирования (STLC)**.

**Коротко:** STLC (Software Testing Life Cycle) состоит из шести последовательных этапов: анализ требований, планирование тестирования, разработка тест-кейсов, настройка тестового окружения, выполнение тестирования и завершение тестирования (Test Closure).

## Развёрнутый ответ

STLC — это структурированная последовательность фаз, которую проходит процесс тестирования в рамках разработки ПО, обычно параллельно и в тесной связи с жизненным циклом разработки (SDLC).

1. **Анализ требований (Requirement Analysis).** QA-команда изучает требования (функциональные и нефункциональные), выявляет тестируемые сценарии, задаёт уточняющие вопросы аналитикам и заказчику, определяет критерии приёмки. На этом этапе также решают, что можно автоматизировать, а что тестировать вручную.
2. **Планирование тестирования (Test Planning).** Формируется тест-план: определяются объём работ (scope), стратегия тестирования, распределение ресурсов, сроки, риски, критерии входа/выхода (entry/exit criteria), необходимые инструменты и окружения. Оценивается трудозатраты (estimation).
3. **Разработка тест-кейсов (Test Case Development / Design).** Пишутся тест-кейсы, чек-листы, тестовые сценарии на основе требований; готовятся тестовые данные. Тест-кейсы проходят ревью (peer review, review с аналитиком).
4. **Настройка тестового окружения (Test Environment Setup).** Подготавливается инфраструктура для тестирования: разворачивается стенд, настраиваются конфигурации, тестовые данные, доступы. Часто выполняется параллельно с разработкой тест-кейсов, чтобы не тратить время впустую. Проводится smoke-тестирование окружения (проверка, что среда готова к тестированию).
5. **Выполнение тестирования (Test Execution).** Тестировщики выполняют подготовленные тест-кейсы, фиксируют результаты, заводят баг-репорты на найденные дефекты, проводят повторное тестирование (retest) после исправлений и регрессионное тестирование, чтобы убедиться, что исправления не сломали другую функциональность.
6. **Завершение тестирования (Test Closure).** Подводятся итоги цикла тестирования: анализируется покрытие, статистика по дефектам, готовится итоговый отчёт (Test Summary Report), фиксируются извлечённые уроки (lessons learned), архивируются тестовые артефакты для будущих проектов.

Важно понимать, что STLC — итеративный процесс: фазы могут пересекаться, а внутри одного релиза может быть несколько циклов выполнения тестирования (например, при работе по Agile-спринтам STLC "сжимается" и повторяется каждый спринт).

## Пример / когда применяется

Пример применения STLC в спринте Agile-команды: в начале спринта тестировщик анализирует пользовательские истории (requirement analysis), оценивает трудозатраты на тестирование в рамках planning poker (test planning), пишет тест-кейсы по acceptance criteria (test case development), поднимает тестовое окружение с последним билдом (environment setup), выполняет функциональное и регрессионное тестирование в течение спринта (test execution), а в конце спринта на ретроспективе и в отчёте фиксирует итоги и найденные проблемы процесса (test closure).

Таблица кратко:

| Этап                        | Основной результат                     |
|-----------------------------|------------------------------------------|
| Requirement Analysis        | Список тестируемых требований, вопросы   |
| Test Planning                | Тест-план, оценка, стратегия             |
| Test Case Development       | Тест-кейсы, тестовые данные              |
| Test Environment Setup      | Готовый тестовый стенд                   |
| Test Execution               | Результаты прогонов, баг-репорты         |
| Test Closure                 | Итоговый отчёт, метрики, lessons learned |

## На что смотрит интервьюер

- Знание правильной последовательности этапов и умение кратко описать результат каждого.
- Понимание, что STLC не изолирован, а тесно связан с SDLC (например, V-модель показывает соответствие этапов разработки и тестирования).
- Способность привести пример из своей практики, как применялся STLC в реальном проекте/спринте.
- Частая ошибка — путать STLC с STLC отдельных тест-кейсов или пропускать этап Test Closure, считая, что тестирование заканчивается на выполнении тестов.
