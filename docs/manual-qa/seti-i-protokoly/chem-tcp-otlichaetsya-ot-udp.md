# Чем TCP отличается от UDP?

**Коротко:** TCP — надёжный протокол с установлением соединения, гарантирующий доставку и порядок данных; UDP — быстрый протокол без соединения и без гарантий доставки.

## Развёрнутый ответ

Оба протокола работают на транспортном уровне и используются поверх IP, но решают разные задачи:

TCP (Transmission Control Protocol):
- ориентирован на соединение (нужен handshake перед передачей данных);
- гарантирует доставку — потерянные сегменты повторно передаются;
- гарантирует порядок доставки данных;
- имеет контроль потока и перегрузки;
- больше накладных расходов (заголовок от 20 байт, подтверждения, ретрансмиссии) — как следствие, выше задержки.

UDP (User Datagram Protocol):
- не ориентирован на соединение — отправитель просто шлёт датаграммы без предварительного согласования;
- не гарантирует доставку — потерянный пакет не будет переотправлен на уровне протокола;
- не гарантирует порядок — датаграммы могут прийти в другом порядке;
- нет управления потоком/перегрузкой на уровне протокола;
- минимальный заголовок (8 байт), меньше задержек, выше пропускная способность для небольших сообщений.

Выбор между TCP и UDP определяется требованиями приложения: если критична целостность и полнота данных (файл, веб-страница, транзакция) — используется TCP; если критична скорость и допустима потеря части данных (голос, видео в реальном времени, игровые пакеты положения) — используется UDP.

## Пример / когда применяется

| Критерий           | TCP                          | UDP                        |
|--------------------|-------------------------------|-----------------------------|
| Соединение          | Устанавливается (handshake)  | Отсутствует                |
| Надёжность          | Гарантирована                | Не гарантирована           |
| Порядок доставки    | Гарантирован                 | Не гарантирован            |
| Скорость            | Ниже                          | Выше                       |
| Заголовок           | от 20 байт                   | 8 байт                     |
| Примеры протоколов  | HTTP, HTTPS, FTP, SSH, SMTP   | DNS (запросы), DHCP, VoIP, стриминг, онлайн-игры |

Например, видеозвонок использует UDP: потеря одного кадра видео некритична и не стоит того, чтобы задерживать весь поток ради его повторной передачи. А загрузка файла использует TCP: потеря даже одного байта делает файл повреждённым.

## На что смотрит интервьюер

Ожидается не просто "TCP надёжный, UDP нет", а понимание механизмов (handshake, ACK, ретрансмиссия) и конкретные примеры применения каждого протокола. Частый follow-up — "почему DNS использует UDP, а не TCP", "может ли приложение реализовать надёжность поверх UDP" (да, например QUIC/HTTP3), "что произойдёт с потерянным UDP-пакетом".
