# Что такое CMS (Content Management System)? Приведите примеры.

**Коротко:** CMS (Content Management System, система управления контентом) — это программная платформа, позволяющая создавать, редактировать, публиковать и управлять контентом сайта без глубоких знаний программирования; примеры — WordPress, Joomla, Drupal, Bitrix, Magento.

## Развёрнутый ответ

CMS предоставляет пользователю (часто не программисту) интерфейс для управления содержимым сайта: текстами, изображениями, страницами, товарами, новостями — без необходимости писать код вручную для каждой публикации. CMS обычно состоит из двух частей:

- **Административная панель (back-end)** — интерфейс для редактирования контента, настройки структуры сайта, управления пользователями, плагинами/модулями, темами оформления.
- **Публичная часть сайта (front-end)** — то, что видят посетители сайта, сгенерированное на основе контента, хранящегося в CMS (обычно в базе данных).

CMS решает типовые задачи: хранение контента в структурированном виде (в базе данных), разграничение прав доступа (например, редактор может публиковать статьи, но не может менять настройки сервера), версионирование контента, SEO-настройки, расширяемость через плагины/модули/темы.

Примеры популярных CMS:

- **WordPress** — самая распространённая CMS в мире, изначально ориентирована на блоги, сейчас используется для сайтов любого типа благодаря огромной экосистеме плагинов и тем.
- **Joomla** — универсальная CMS с гибкой структурой, популярна для средних и крупных сайтов.
- **Drupal** — CMS, ориентированная на сложные и высоконагруженные проекты, требует больше технической экспертизы при настройке.
- **1C-Битрикс (Bitrix)** — популярна в России и СНГ, часто используется для интернет-магазинов и корпоративных порталов, включает встроенные бизнес-инструменты (CRM и т.д.).
- **Magento** — специализированная CMS для интернет-магазинов (e-commerce).
- **Tilda, Wix** — CMS с уклоном в конструкторы сайтов (no-code), ориентированы на простоту создания лендингов без технических знаний.

Отдельно стоит выделить **headless CMS** (например, Contentful, Strapi) — систему, которая управляет только контентом и отдаёт его через API (обычно REST или GraphQL), без собственного front-end рендеринга; front-end реализуется отдельно (на React, Vue и т.д.). Это удобно для мультиканальной публикации контента (сайт, мобильное приложение, умные устройства).

## Пример / когда применяется

Пример: компания создаёт корпоративный блог и интернет-магазин. Для блога выбирают WordPress с плагином WooCommerce для базовой торговли, либо более специализированный Magento для крупного e-commerce с сложной логикой (скидки, склад, интеграции с платёжными системами).

В тестировании CMS-based сайтов важно учитывать специфику: контент может динамически формироваться из БД, значит нужно проверять как отображение (front-end), так и корректность работы административной панели (создание/редактирование/удаление контента, права доступа ролей, работу плагинов и их совместимость друг с другом после обновлений).

```text
Типовой стек CMS:
  Админ-панель -> База данных (контент, настройки)
                        |
                        v
            Публичная часть сайта (рендеринг)
```

## На что смотрит интервьюер

Частые ошибки: кандидат не может привести конкретные примеры CMS, путает CMS с фреймворком (например, React) или с CRM-системой; не понимает разницу между headless и классической CMS. Follow-up вопросы: "Чем отличается CMS от CRM?", "Что такое headless CMS и зачем она нужна?", "На что стоит обращать внимание при тестировании сайта на CMS (например, плагины, обновления, права доступа)?". Красный флаг — путаница базовых терминов (CMS/CRM/CDN) или отсутствие понимания практической роли CMS в веб-разработке.
