# Что такое модель OSI? Перечислите ее 7 уровней.

**Коротко:** OSI (Open Systems Interconnection) — эталонная концептуальная модель, описывающая сетевое взаимодействие через 7 уровней абстракции, от физической передачи сигналов до прикладных программ.

## Развёрнутый ответ

Модель OSI была разработана ISO для стандартизации сетевого взаимодействия и облегчения понимания того, какие функции выполняются на каждом этапе передачи данных. Это справочная модель — не все реальные протоколы строго следуют её границам, но она удобна для анализа и обучения.

Семь уровней (снизу вверх):
1. Физический (Physical) — передача битов по физической среде: кабели, разъёмы, радиосигналы, напряжение, частоты. Пример: Ethernet-кабель, Wi-Fi-сигнал.
2. Канальный (Data Link) — передача кадров между узлами в одном сегменте сети, обнаружение ошибок передачи, MAC-адресация. Пример: Ethernet, коммутаторы (switches).
3. Сетевой (Network) — маршрутизация пакетов между разными сетями, логическая адресация. Пример: IP, роутеры.
4. Транспортный (Transport) — доставка данных между процессами на конечных узлах, сегментация, надёжность. Пример: TCP, UDP.
5. Сеансовый (Session) — установление, поддержание и завершение сеансов связи между приложениями. Пример: управление сессиями в RPC, NetBIOS.
6. Представления (Presentation) — преобразование форматов данных, шифрование, сжатие. Пример: кодировки, SSL/TLS (частично), форматы JPEG.
7. Прикладной (Application) — взаимодействие с конечными приложениями, протоколы, которые видит пользователь. Пример: HTTP, FTP, DNS, SMTP.

Для запоминания порядка часто используют мнемонику: "Пожалуйста, Не Стоит Трогать Провода Дорогих Приятелей" или английскую "Please Do Not Throw Sausage Pizza Away" (Physical, Data Link, Network, Transport, Session, Presentation, Application).

## Пример / когда применяется

Модель OSI используется как инструмент для диагностики сетевых проблем: специалисты последовательно проверяют уровни снизу вверх — "есть ли физическое подключение (L1)", "видит ли коммутатор MAC-адрес (L2)", "проходит ли пинг по IP (L3)", "открыт ли TCP-порт (L4)", "отвечает ли приложение (L7)". Это помогает локализовать, на каком уровне возникла неисправность.

## На что смотрит интервьюер

Нужно перечислить все 7 уровней в правильном порядке с примерами протоколов/устройств для каждого. Частая ошибка — перепутать порядок сеансового и представления уровней или не привести примеры. Follow-up вопросы: "на каком уровне работает свитч/роутер/файрвол", "чем отличается модель OSI от TCP/IP", "приведите протокол для каждого уровня".
