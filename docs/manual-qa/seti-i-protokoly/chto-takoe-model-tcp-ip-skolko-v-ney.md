# Что такое модель TCP/IP? Сколько в ней уровней?

**Коротко:** модель TCP/IP — практическая (в отличие от эталонной OSI) модель сетевого взаимодействия, лежащая в основе интернета, обычно состоит из 4 уровней (иногда её представляют как 5).

## Развёрнутый ответ

Модель TCP/IP (также называемая моделью DoD или Internet Protocol Suite) сложилась исторически вместе с развитием протоколов сети ARPANET/Интернет и является более практичной и упрощённой по сравнению с теоретической моделью OSI. В классическом виде она включает 4 уровня:

1. Уровень сетевого доступа (Network Access / Link) — объединяет функции физического и канального уровней OSI: передача данных в рамках локальной сети, MAC-адресация, кадры. Протоколы/технологии: Ethernet, Wi-Fi, ARP.
2. Межсетевой уровень (Internet) — соответствует сетевому уровню OSI: маршрутизация пакетов между сетями, логическая адресация. Протокол: IP (IPv4/IPv6), ICMP.
3. Транспортный уровень (Transport) — аналогичен транспортному уровню OSI: доставка данных между процессами конечных узлов. Протоколы: TCP, UDP.
4. Прикладной уровень (Application) — объединяет сеансовый, представления и прикладной уровни OSI: конкретные протоколы взаимодействия приложений. Протоколы: HTTP, HTTPS, FTP, SSH, DNS, SMTP.

Иногда модель TCP/IP представляют из 5 уровней, разделяя уровень сетевого доступа на физический и канальный отдельно (по аналогии с OSI) — это упрощает сопоставление моделей, но не является строгим стандартом.

Ключевое отличие от OSI: TCP/IP разрабатывалась не как теоретическая модель "сверху вниз", а выросла из реально работающих протоколов — она проще, менее детализирована, но именно она описывает реальную архитектуру современного интернета.

## Пример / когда применяется

Когда браузер отправляет HTTP-запрос: данные формируются на прикладном уровне (HTTP), передаются транспортному уровню (TCP оборачивает их в сегмент с портами отправителя/получателя), затем межсетевому уровню (IP добавляет заголовок с адресами источника/назначения), и наконец уровню сетевого доступа (Ethernet формирует кадр с MAC-адресами для передачи по локальной сети).

## На что смотрит интервьюер

Важно чётко назвать количество уровней (4, при необходимости упомянуть вариант с 5) и не путать модель TCP/IP с OSI. Частая ошибка — сказать "7 уровней", перепутав с OSI.
