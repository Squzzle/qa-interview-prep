# Что такое протокол TCP? Какие у него особенности (надежность, соединение)?

**Коротко:** TCP (Transmission Control Protocol) — это транспортный протокол с установлением соединения, который гарантирует надёжную, упорядоченную доставку данных между приложениями.

## Развёрнутый ответ

TCP работает на транспортном уровне модели OSI (4-й уровень) и модели TCP/IP. Его ключевая особенность — ориентация на соединение (connection-oriented): перед передачей данных стороны выполняют "тройное рукопожатие" (three-way handshake): SYN → SYN-ACK → ACK, устанавливая логическое соединение с согласованными параметрами (начальные номера последовательности, размер окна).

Основные особенности TCP:
- Надёжность доставки: каждый переданный сегмент подтверждается получателем (ACK); если подтверждение не пришло за таймаут, сегмент передаётся повторно.
- Гарантия порядка: сегменты нумеруются (sequence number), и получатель восстанавливает исходный порядок данных, даже если пакеты пришли не по порядку.
- Контроль целостности: контрольная сумма (checksum) в заголовке обнаруживает повреждённые данные.
- Управление потоком (flow control): механизм скользящего окна (sliding window) не позволяет отправителю "заваливать" получателя данными быстрее, чем тот способен обработать.
- Управление перегрузкой (congestion control): алгоритмы (slow start, congestion avoidance) регулируют скорость отправки в зависимости от состояния сети.
- Корректное закрытие соединения: происходит через обмен FIN/ACK с обеих сторон (four-way termination).

Из-за всех этих механизмов TCP медленнее UDP и создаёт дополнительные накладные расходы (заголовок TCP — 20 байт минимум против 8 у UDP), но обеспечивает гарантии, критичные для большинства прикладных протоколов: HTTP/HTTPS, FTP, SSH, SMTP работают поверх TCP.

## Пример / когда применяется

TCP применяется там, где важна целостность и полнота данных больше, чем скорость: загрузка веб-страниц (HTTP/HTTPS), передача файлов (FTP), электронная почта (SMTP), удалённый доступ (SSH). Если при передаче файла один пакет потеряется, TCP автоматически его перепошлёт — пользователь получит файл целиком, пусть и с небольшой задержкой.

Пример handshake при открытии TCP-соединения:
```text
Клиент → Сервер: SYN (seq=100)
Сервер → Клиент: SYN-ACK (seq=300, ack=101)
Клиент → Сервер: ACK (ack=301)
```

## На что смотрит интервьюер

Ожидается чёткое перечисление механизмов надёжности (ACK, повторная передача, нумерация, контроль потока/перегрузки) и понимание, зачем нужно установление соединения. Частый follow-up — сравнение с UDP, вопрос про three-way handshake по шагам, про то, что произойдёт при потере пакета, про разницу flow control и congestion control.
