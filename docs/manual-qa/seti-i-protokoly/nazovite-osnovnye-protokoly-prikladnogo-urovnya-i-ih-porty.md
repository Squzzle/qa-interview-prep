# Назовите основные протоколы прикладного уровня и их порты (HTTP, HTTPS, FTP, SSH, DNS).

**Коротко:** это протоколы, обеспечивающие конкретные виды взаимодействия приложений (веб, файлы, удалённый доступ, разрешение имён), каждый из которых по умолчанию использует свой стандартный порт.

## Развёрнутый ответ

Протоколы прикладного уровня определяют формат сообщений, которыми обмениваются приложения поверх транспортного уровня (обычно TCP или UDP). Каждому протоколу назначен well-known порт (0–1023), закреплённый IANA, что позволяет ОС и сетевому оборудованию понимать, к какому сервису адресован трафик.

Основные протоколы и их порты:
- HTTP (HyperText Transfer Protocol) — порт 80, TCP. Передача веб-страниц и данных в незашифрованном виде.
- HTTPS (HTTP Secure) — порт 443, TCP. Тот же HTTP, но поверх TLS/SSL — трафик шифруется, обеспечивается аутентичность сервера.
- FTP (File Transfer Protocol) — порты 21 (управляющее соединение) и 20 (передача данных в активном режиме), TCP. Передача файлов между клиентом и сервером.
- SSH (Secure Shell) — порт 22, TCP. Защищённый удалённый доступ к командной строке сервера, туннелирование, безопасное копирование файлов (SCP/SFTP работают поверх SSH).
- DNS (Domain Name System) — порт 53, обычно UDP (для коротких запросов), но может использовать TCP (для больших ответов и зонных передач). Преобразование доменных имён в IP-адреса.

Дополнительно интервьюер может ожидать знание и других портов: SMTP — 25 (587 для отправки с аутентификацией), POP3 — 110, IMAP — 143, Telnet — 23, RDP — 3389.

## Пример / когда применяется

| Протокол | Порт   | Транспорт | Назначение              |
|----------|--------|-----------|--------------------------|
| HTTP     | 80     | TCP       | Веб-страницы (без шифрования) |
| HTTPS    | 443    | TCP       | Веб-страницы (с TLS)     |
| FTP      | 21/20  | TCP       | Передача файлов          |
| SSH      | 22     | TCP       | Удалённый доступ         |
| DNS      | 53     | UDP/TCP   | Разрешение имён          |

Например, при тестировании API нужно знать, что запрос к `https://example.com` по умолчанию идёт на порт 443, а обращение к внутреннему тестовому серверу по HTTP без указания порта — на 80.

## На что смотрит интервьюер

Проверяется точное знание портов (частая ошибка — перепутать 80/443, забыть про 20/21 у FTP, назвать DNS только TCP). Могут спросить про разницу активного/пассивного FTP, про то, почему DNS в основном использует UDP, а не TCP, и про то, как определить порт сервиса командой (например, `netstat` или `ss`).
