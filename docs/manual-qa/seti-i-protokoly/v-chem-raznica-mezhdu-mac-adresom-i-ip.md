# В чем разница между MAC-адресом и IP-адресом?

**Коротко:** MAC-адрес — это уникальный физический (аппаратный) адрес сетевого интерфейса, присваиваемый производителем и работающий на канальном уровне; IP-адрес — это логический адрес, который назначается устройству в сети (вручную или через DHCP) и работает на сетевом уровне для маршрутизации трафика между сетями.

## Развёрнутый ответ

MAC-адрес (Media Access Control) — это 48-битный идентификатор, зашитый в сетевую карту (или другой сетевой интерфейс) производителем. Он записывается в виде шести пар шестнадцатеричных цифр, например `00:1A:2B:3C:4D:5E`. Первые три пары (OUI) идентифицируют производителя оборудования. MAC-адрес считается уникальным для каждого устройства и в норме не меняется, хотя технически его можно подменить программно (MAC spoofing). Он используется для идентификации устройства в пределах одной локальной сети (сегмента, коллизионного домена) и работает на канальном уровне (Data Link Layer, L2).

IP-адрес (Internet Protocol) — это логический адрес, который назначается сетевому интерфейсу для работы в сети на основе протокола IP (IPv4 или IPv6). В отличие от MAC-адреса, IP-адрес не привязан жёстко к оборудованию: он может меняться в зависимости от сети, к которой подключено устройство, назначаться статически администратором или динамически через DHCP. IP-адрес работает на сетевом уровне (Network Layer, L3) и используется для маршрутизации пакетов между разными сетями (в том числе через интернет).

Ключевые отличия:

- **Уровень модели OSI**: MAC — канальный уровень (L2), IP — сетевой уровень (L3).
- **Природа адреса**: MAC — физический, аппаратный, "прошит" в устройство; IP — логический, программно назначаемый.
- **Постоянство**: MAC обычно неизменен (кроме подмены); IP может меняться при смене сети или переподключении.
- **Область действия**: MAC используется для передачи данных внутри локальной сети; IP используется как для локальной сети, так и для маршрутизации между сетями через интернет.
- **Формат**: MAC — 48 бит, шестнадцатеричная запись; IPv4 — 32 бита, десятичная запись через точки; IPv6 — 128 бит.

Связь между ними обеспечивает протокол ARP (Address Resolution Protocol) — он сопоставляет IP-адрес устройства с его MAC-адресом в пределах локальной сети, чтобы кадры канального уровня могли быть доставлены нужному узлу.

## Пример / когда применяется

Пример: компьютер с MAC-адресом `00:1A:2B:3C:4D:5E` подключается к домашней сети Wi-Fi и получает по DHCP IP-адрес `192.168.1.15`. При переподключении к другой сети (например, в офисе) IP-адрес изменится на `10.0.0.42`, но MAC-адрес сетевой карты останется прежним.

В тестировании это важно, например, при работе с DHCP-серверами, при настройке whitelisting по MAC-адресу на роутерах, при анализе сетевого трафика в Wireshark, где видны оба типа адресов на разных уровнях пакета (Ethernet frame содержит MAC, IP-пакет внутри содержит IP-адреса).

```text
Ethernet Frame:
  Source MAC:      00:1A:2B:3C:4D:5E
  Destination MAC:  AA:BB:CC:DD:EE:FF
  Payload -> IP Packet:
    Source IP:      192.168.1.15
    Destination IP: 192.168.1.1
```

## На что смотрит интервьюер

Частые ошибки кандидатов: путают уровни OSI (говорят, что MAC работает на сетевом уровне); утверждают, что MAC-адрес нельзя изменить в принципе (на самом деле возможен MAC spoofing); не могут объяснить связь через ARP. Возможные follow-up вопросы: "Что делает протокол ARP?", "Может ли IP-адрес быть постоянным?" (да, статический IP), "Что такое NAT и как он влияет на видимый внешний IP?". Красный флаг — если кандидат не понимает разницу между логическим и физическим адресом или путает MAC-адрес с IP-адресом в контексте маршрутизации.
