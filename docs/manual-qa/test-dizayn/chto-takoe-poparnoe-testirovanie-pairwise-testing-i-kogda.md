# Что такое **попарное тестирование (Pairwise Testing)** и когда оно эффективно?

**Коротко:** Pairwise Testing — комбинаторная техника, которая формирует минимальный набор тестов, покрывающий все возможные пары значений входных параметров, вместо полного перебора всех комбинаций.

## Развёрнутый ответ

Pairwise Testing (попарное тестирование, всеобщие сокращения — All-Pairs) основано на эмпирическом наблюдении, что подавляющее большинство дефектов вызывается взаимодействием не более чем двух параметров одновременно, а тройные и более сложные взаимодействия дают дефекты значительно реже.

Проблема, которую решает техника: если система имеет несколько независимых параметров с несколькими значениями каждый, полный перебор всех комбинаций (full combinatorial) растет экспоненциально. Например, 5 параметров по 4 значения каждый дают 4^5 = 1024 комбинации — тестировать все нереально.

Pairwise-алгоритм строит существенно меньший набор тестов, в котором каждая пара значений из разных параметров встречается хотя бы один раз. Для построения таких наборов используют:
- ортогональные массивы (orthogonal arrays);
- специализированные инструменты и генераторы (PICT, Hexawise, AllPairs, ACTS).

Технику относят к black-box / комбинаторным техникам тест-дизайна.

Когда pairwise эффективен:
- много независимых конфигурационных параметров (браузер, ОС, версия, локаль, разрешение экрана и т.п.);
- параметры относительно независимы друг от друга (без сложных бизнес-зависимостей между значениями);
- полный перебор комбинаций невозможен по времени/бюджету;
- допустимо (и обычно оправдано) не покрывать редкие тройные взаимодействия параметров.

Когда pairwise малоэффективен или требует осторожности:
- если между параметрами есть жесткие бизнес-зависимости (некоторые комбинации вообще недопустимы) — нужно константное/конструированное покрытие (constraints);
- для критичных систем, где важны редкие многофакторные взаимодействия (например, авиация, медицина) — иногда нужен более полный перебор.

## Пример / когда применяется

Тестирование формы оформления заказа с параметрами:
- Браузер: Chrome, Firefox, Safari;
- ОС: Windows, macOS, Linux;
- Способ оплаты: карта, PayPal, наличные.

Полный перебор: 3 × 3 × 3 = 27 комбинаций.

Pairwise-набор (пример, сгенерированный инструментом типа PICT) сократит число тестов примерно до 9, при этом каждая пара значений (например, "Chrome + PayPal", "macOS + наличные") встретится хотя бы раз.

```text
Test 1: Chrome,  Windows, карта
Test 2: Chrome,  macOS,   PayPal
Test 3: Chrome,  Linux,   наличные
Test 4: Firefox, Windows, PayPal
Test 5: Firefox, macOS,   наличные
Test 6: Firefox, Linux,   карта
Test 7: Safari,  Windows, наличные
Test 8: Safari,  macOS,   карта
Test 9: Safari,  Linux,   PayPal
```

## На что смотрит интервьюер

- Понимание, что pairwise — это компромисс между полнотой покрытия и стоимостью тестирования, а не гарантия отсутствия багов.
- Знание, что технику применяют именно при большом количестве независимых параметров конфигурации.
- Follow-up: "А что если два параметра не могут сочетаться друг с другом?" — правильный ответ: современные генераторы позволяют задавать constraints (ограничения/недопустимые комбинации).
- Красный флаг — если кандидат не может объяснить, почему сокращается количество тестов (то есть не понимает суть комбинаторики "все пары, а не все комбинации").
