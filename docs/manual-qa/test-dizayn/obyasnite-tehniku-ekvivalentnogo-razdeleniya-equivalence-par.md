# Объясните технику **эквивалентного разделения (Equivalence Partitioning)**. Приведите пример.

**Коротко:** входные (или выходные) данные разбиваются на классы эквивалентности — группы значений, которые система должна обрабатывать одинаково; достаточно протестировать по одному представителю из каждого класса.

## Развёрнутый ответ

Equivalence Partitioning (эквивалентное разделение, разбиение на классы эквивалентности) — черноящичная техника тест-дизайна, основанная на предположении, что если система корректно обрабатывает одно значение из класса, то она корректно обработает и все остальные значения этого класса, так как логика их обработки одинакова.

Классы эквивалентности делятся на:
- **валидные (positive)** — значения, которые система должна принять и обработать штатно;
- **невалидные (negative)** — значения, которые система должна отклонить с корректной обработкой ошибки.

Для каждого класса выбирается один representative (репрезентативное значение) — тестировать все значения класса избыточно, так как ожидается одинаковое поведение.

Разбиение может строиться по разным критериям: диапазон чисел, длина строки, тип данных, формат, множество допустимых значений (enum), бизнес-правило.

Главная польза техники — резкое сокращение количества тестов при сохранении приемлемого покрытия, особенно на входных данных с большим диапазоном значений.

## Пример / когда применяется

Поле "Возраст" должно принимать целые числа от 18 до 65.

Классы эквивалентности:
1. Валидный класс: 18–65 (например, представитель — 30).
2. Невалидный класс: меньше 18 (например, 10).
3. Невалидный класс: больше 65 (например, 80).
4. Невалидный класс: нечисловое значение (например, "abc").
5. Невалидный класс: пустое значение.

Вместо перебора всех чисел от 0 до 200 достаточно 5 тестов — по одному на класс.

Применяется на этапе тест-дизайна для любых полей ввода, параметров API, комбобоксов и т.д., особенно в связке с анализом граничных значений.

## На что смотрит интервьюер

- Кандидат должен четко разделить валидные и невалидные классы, а не просто "разбить диапазон пополам".
- Частая ошибка — выбирать несколько представителей из одного класса вместо одного (это избыточно и противоречит сути техники).
- Follow-up: "А что если система обрабатывает пустую строку иначе, чем null?" — правильный ответ: это разные классы эквивалентности, их нужно выделять отдельно.
- Хороший кандидат свяжет технику с последующим анализом граничных значений — обычно они применяются вместе.
