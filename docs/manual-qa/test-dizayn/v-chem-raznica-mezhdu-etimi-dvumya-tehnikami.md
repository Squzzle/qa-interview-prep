# В чем разница между этими двумя техниками?

**Коротко:** эквивалентное разделение проверяет по одному представителю "внутри" каждого класса значений, а анализ граничных значений целенаправленно проверяет значения на границах и рядом с ними, где вероятность ошибки максимальна.

## Развёрнутый ответ

Речь о разнице между Equivalence Partitioning (эквивалентное разделение) и Boundary Value Analysis (анализ граничных значений) — двумя тесно связанными, но разными техниками тест-дизайна.

**Equivalence Partitioning:**
- цель — сократить количество тестов, разбив все возможные входные данные на классы с одинаковым ожидаемым поведением;
- тестовое значение выбирается из "середины" класса, любое значение, репрезентативное для всего класса;
- отвечает на вопрос "какие группы данных вообще существуют?".

**Boundary Value Analysis:**
- цель — найти дефекты, характерные именно для границ диапазонов (off-by-one, неверные операторы сравнения);
- тестовые значения выбираются целенаправленно на границе класса и рядом с ней;
- отвечает на вопрос "где именно внутри/между классами наиболее вероятны ошибки?".

Ключевое отличие: EP — это техника разбиения пространства входных данных на группы для сокращения числа тестов в целом; BVA — техника точечного усиления покрытия именно на границах уже выделенных классов. На практике они почти всегда применяются вместе: сначала строят классы эквивалентности (EP), а затем для каждой границы между классами добавляют граничные тесты (BVA). BVA можно рассматривать как уточнение и продолжение EP, а не как полностью самостоятельную независимую технику.

## Пример / когда применяется

Диапазон "Возраст": 18–65.

- Через EP получаем три класса: `<18` (невалидный), `18–65` (валидный), `>65` (невалидный) — и берем по одному представителю из каждого, например 10, 30, 80.
- Через BVA для тех же границ добавляем точечные проверки: 17, 18, 19 (нижняя граница) и 64, 65, 66 (верхняя граница).

Итоговый набор тестов объединяет оба подхода: представители классов (для общего покрытия функциональности) + граничные значения (для покрытия типичных ошибок реализации).

## На что смотрит интервьюер

- Способность четко сформулировать различие в цели и в способе выбора тестовых значений, а не просто повторить определения обеих техник по отдельности.
- Понимание, что BVA логически "надстраивается" над EP, а не существует изолированно.
- Частая ошибка — путать эти техники или утверждать, что они взаимоисключающие.
- Хороший кандидат отметит, что на практике обе техники комбинируют в одном наборе тестов для оптимального покрытия при минимальном числе тест-кейсов.
