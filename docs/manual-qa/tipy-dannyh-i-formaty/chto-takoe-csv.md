# Что такое **CSV**?

**Коротко:** CSV (Comma-Separated Values) — простой текстовый формат для табличных данных, где значения строки разделены запятой (или другим разделителем), а строки — переносом строки.

## Развёрнутый ответ

CSV — один из старейших и простейших форматов обмена табличными данными. Каждая строка файла представляет одну запись (например, строку таблицы), а поля внутри строки разделены символом-разделителем (обычно запятой, но встречаются точка с запятой, табуляция — TSV, или другие символы).

Особенности формата:

- **Первая строка** часто содержит заголовки колонок (не обязательно по стандарту, но по соглашению).
- **Разделитель** — запятая по умолчанию, но локализованные версии (например, в русской версии Excel) часто используют точку с запятой, так как запятая является десятичным разделителем.
- **Экранирование** — если значение само содержит разделитель, перенос строки или кавычки, оно заключается в двойные кавычки; кавычка внутри значения экранируется удвоением (`""`).
- **Отсутствие типов данных** — все значения по сути строки, типизация (число, дата, boolean) выполняется на стороне парсера/потребителя.
- **Нет вложенности** — CSV подходит только для плоских, табличных структур, в отличие от JSON/XML/YAML, которые поддерживают иерархию.
- **Нет официальной строгой спецификации** — существует RFC 4180, описывающий рекомендации, но на практике много вариаций (кодировка, разделитель, наличие BOM).

CSV широко используется для экспорта/импорта данных между системами, выгрузки отчётов из БД или Excel, миграции данных, тестовых данных для дата-провайдеров в автотестах.

## Пример / когда применяется

Пример CSV-файла:

```text
id,name,email,is_active
1,Иван Иванов,ivan@example.com,true
2,"Петров, Пётр",petrov@example.com,false
```

Обратите внимание: значение `"Петров, Пётр"` заключено в кавычки, так как содержит разделитель-запятую внутри самого значения.

В автотестах CSV часто используют как источник тестовых данных для параметризованных тестов (data-driven testing), например через TestNG DataProvider или pytest fixtures, читая файл построчно.

## На что смотрит интервьюер

Интервьюер проверяет знание нюансов экранирования (кавычки, запятые внутри значений) и понимание, что CSV — плоский формат без вложенности и типов. Частые follow-up: чем CSV отличается от JSON/XML (нет иерархии, нет типов), какие проблемы возникают при разных кодировках и разделителях в разных локалях. Красный флаг — считать CSV "просто таблицей без нюансов" и не знать про экранирование кавычками.
