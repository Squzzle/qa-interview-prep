# Что такое **JSON**? Опишите его структуру и типы данных.

**Коротко:** JSON (JavaScript Object Notation) — текстовый формат обмена данными на основе пар "ключ-значение", легко читаемый человеком и парсимый машинами.

## Развёрнутый ответ

JSON — это независимый от языка программирования текстовый формат представления структурированных данных, изначально основанный на синтаксисе объектов JavaScript, но сейчас используемый повсеместно (REST API, конфиги, файлы обмена данными).

Структура JSON строится из двух базовых конструкций:

- **Объект** — неупорядоченный набор пар `ключ: значение`, заключённый в фигурные скобки `{}`. Ключи всегда строки в двойных кавычках.
- **Массив** — упорядоченный список значений, заключённый в квадратные скобки `[]`.

Типы данных в JSON:

- **Строка (string)** — последовательность символов в двойных кавычках, поддерживает экранирование (`\n`, `\"`, `\\` и т.д.).
- **Число (number)** — целое или дробное, может содержать экспоненциальную запись, без разделения на int/float на уровне синтаксиса.
- **Логическое значение (boolean)** — `true` или `false`.
- **Null** — отсутствие значения, обозначается `null`.
- **Объект (object)** — вложенная структура ключ-значение.
- **Массив (array)** — вложенный список значений.

Важные нюансы: JSON не поддерживает комментарии, даты (передаются как строки в определённом формате, например ISO 8601), а также не различает целые и дробные числа на уровне спецификации — это делает язык-парсер. Ключи должны быть уникальными по соглашению, хотя формально спецификация это не запрещает.

## Пример / когда применяется

Пример JSON-документа:

```json
{
  "id": 42,
  "name": "Иван Иванов",
  "isActive": true,
  "balance": 1234.56,
  "roles": ["admin", "user"],
  "address": {
    "city": "Москва",
    "zip": "101000"
  },
  "middleName": null
}
```

JSON применяется в REST API как формат тела запроса/ответа, в конфигурационных файлах (`package.json`, `tsconfig.json`), для хранения данных в NoSQL БД (MongoDB), при логировании структурированных событий.

## На что смотрит интервьюер

Интервьюер проверяет, знает ли кандидат точные типы данных JSON и отличие от JS-объекта (например, что ключи должны быть в двойных кавычках, а не в одинарных). Частая ошибка — путать JSON с JavaScript-объектом или утверждать, что JSON поддерживает функции, даты как тип или комментарии. Могут спросить про MIME-тип (`application/json`), про валидацию (JSON Schema), про сериализацию/десериализацию и про обработку дублирующихся ключей.
