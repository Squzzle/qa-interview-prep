# Что такое **JSON Schema** и для чего она нужна?

**Коротко:** JSON Schema — это спецификация (сама в формате JSON), описывающая структуру, типы и ограничения полей JSON-документа, используемая для валидации данных.

## Развёрнутый ответ

JSON Schema позволяет декларативно описать, каким должен быть валидный JSON-документ: какие поля обязательны, какого они типа, какие значения допустимы, каковы ограничения по длине, диапазону, формату (например, email, дата) и т.д. Это своего рода "контракт" данных.

Основные ключевые слова JSON Schema:

- `type` — тип данных поля (`string`, `number`, `object`, `array`, `boolean`, `null`).
- `required` — список обязательных полей.
- `properties` — описание полей объекта.
- `items` — описание элементов массива.
- `enum` — ограниченный список допустимых значений.
- `minimum`/`maximum`, `minLength`/`maxLength` — числовые и строковые ограничения.
- `pattern` — регулярное выражение для строки.
- `format` — семантический формат (`date-time`, `email`, `uri` и т.д.).
- `additionalProperties` — разрешены ли поля, не описанные в схеме.

JSON Schema применяется для автоматизации проверки контракта API (contract testing), генерации документации (Swagger/OpenAPI использует похожий подход), генерации тестовых данных и валидации входящих запросов на бэкенде.

Для тестировщика это ключевой инструмент автоматизации проверки ответов API: вместо ручной проверки каждого поля можно один раз описать схему и валидировать любой ответ по ней.

## Пример / когда применяется

Пример JSON Schema для объекта пользователя:

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "id": { "type": "integer" },
    "name": { "type": "string", "minLength": 1 },
    "email": { "type": "string", "format": "email" },
    "isActive": { "type": "boolean" }
  },
  "required": ["id", "name", "email"],
  "additionalProperties": false
}
```

В автотестах на JavaScript такую схему можно проверить, например, библиотекой `ajv`:

```javascript
const Ajv = require("ajv");
const ajv = new Ajv();
const validate = ajv.compile(schema);
const valid = validate(responseBody);
```

## На что смотрит интервьюер

Проверяется понимание того, что JSON Schema — это не сам JSON, а метаописание структуры. Частая ошибка — путать JSON Schema с XSD (это аналог для XML) или не знать, зачем она нужна в автоматизации тестирования API. Хороший ответ включает пример использования в контрактных тестах и упоминание инструментов валидации (ajv, jsonschema для Python).
