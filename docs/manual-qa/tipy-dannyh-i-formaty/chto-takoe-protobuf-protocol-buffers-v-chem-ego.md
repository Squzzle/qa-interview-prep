# Что такое **Protobuf (Protocol Buffers)**? В чем его преимущество перед JSON?

**Коротко:** Protobuf — бинарный формат сериализации данных от Google со строгой схемой (`.proto`), значительно компактнее и быстрее JSON, применяется в основном в связке с gRPC.

## Развёрнутый ответ

Protocol Buffers — механизм сериализации структурированных данных, разработанный Google. В отличие от JSON и XML, Protobuf не является текстовым форматом — данные сериализуются в компактный бинарный вид, непонятный человеку без десериализации.

Ключевые особенности:

- **Схема обязательна** — структура данных описывается в `.proto`-файле с указанием полей, их типов и номеров (тегов). Из схемы кодогенератор создаёт классы для разных языков (Java, Go, Python, C++ и др.).
- **Числовые теги полей** — каждому полю присваивается уникальный номер, который используется в бинарном представлении вместо имени поля, что уменьшает размер сообщения.
- **Строгая типизация** — типы (int32, string, bool, message, repeated и т.д.) фиксированы в схеме.
- **Обратная совместимость** — можно добавлять новые поля, не ломая старых клиентов (если следовать правилам эволюции схемы).
- **Компактность** — бинарный формат значительно меньше по размеру, чем эквивалентный JSON.
- **Скорость** — сериализация/десериализация значительно быстрее, чем парсинг текстового JSON.

**Преимущества Protobuf перед JSON:**

1. Меньший размер передаваемых данных за счёт бинарного формата и числовых тегов вместо имён полей.
2. Более высокая производительность сериализации/десериализации.
3. Строгая типизация и явный контракт данных (схема обязательна, а не опциональна как JSON Schema).
4. Встроенная поддержка эволюции схемы с обратной совместимостью.
5. Кодогенерация для множества языков программирования из единого источника правды.

**Недостатки:** данные не читаются человеком напрямую (нужна десериализация или знание схемы), требует дополнительной инфраструктуры (компиляция `.proto`), менее удобен для отладки через простой просмотр в браузере/curl.

## Пример / когда применяется

Пример `.proto`-файла:

```protobuf
syntax = "proto3";

message User {
  int32 id = 1;
  string name = 2;
  bool is_active = 3;
}
```

Protobuf чаще всего применяется вместе с **gRPC** для межсервисного взаимодействия внутри микросервисной архитектуры, где важны низкая задержка и высокая пропускная способность, а также в мобильных приложениях для экономии трафика.

## На что смотрит интервьюер

Интервьюер ожидает понимания, что Protobuf — бинарный, а не текстовый формат, и что для его использования обязательна схема. Часто спрашивают про связь с gRPC, про то, как тестировать gRPC/Protobuf сервисы (например, инструментами вроде grpcurl, BloomRPC, Postman с поддержкой gRPC). Красный флаг — путать Protobuf с JSON по читаемости или не знать, что для десериализации нужна схема/сгенерированный код.
