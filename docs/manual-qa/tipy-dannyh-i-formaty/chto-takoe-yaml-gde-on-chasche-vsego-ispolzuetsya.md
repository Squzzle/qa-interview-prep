# Что такое **YAML**? Где он чаще всего используется (конфиги, CI/CD)?

**Коротко:** YAML (YAML Ain't Markup Language) — человекочитаемый формат сериализации данных, основанный на отступах, широко применяемый для конфигурационных файлов и описания CI/CD пайплайнов.

## Развёрнутый ответ

YAML ориентирован на максимальную читаемость для человека. В отличие от JSON и XML, структура задаётся через отступы (обычно два пробела, табы запрещены), а не через скобки или теги. Это делает файлы компактными и приятными для написания вручную, но чувствительными к форматированию.

Основные особенности синтаксиса YAML:

- **Отступы** определяют вложенность (аналог блоков в Python).
- **Пары ключ-значение**: `key: value`.
- **Списки**: элементы через дефис `- item` или инлайн `[a, b, c]`.
- **Вложенные объекты**: инлайн `{key: value}` или через отступы.
- **Комментарии**: строка, начинающаяся с `#`.
- **Многострочные строки**: символы `|` (сохраняет переносы строк) и `>` (складывает в одну строку).
- **Якоря и ссылки** (`&anchor`, `*anchor`) — механизм повторного использования блоков данных.
- **Типы данных**: строки, числа, boolean, null, даты — определяются автоматически по содержимому, при необходимости можно явно указать тип через теги (`!!str`).

YAML — надмножество JSON: валидный JSON-документ, как правило, является валидным YAML.

Где применяется:

- **CI/CD пайплайны** — GitHub Actions, GitLab CI, CircleCI, Jenkins (декларативный Pipeline через `Jenkinsfile.yaml`).
- **Оркестрация контейнеров** — манифесты Kubernetes (Deployment, Service, ConfigMap).
- **Docker Compose** — описание сервисов, сетей, томов.
- **Конфигурационные файлы приложений** — Spring Boot (`application.yml`), Ansible playbooks.
- **Тестовая документация** — OpenAPI/Swagger спецификации часто пишутся в YAML.

## Пример / когда применяется

Пример YAML-конфигурации для GitHub Actions:

```yaml
name: CI Pipeline
on:
  push:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install dependencies
        run: npm install
      - name: Run tests
        run: npm test
```

Типичная ошибка новичков — смешать табы и пробелы для отступов, что приводит к ошибке парсинга.

## На что смотрит интервьюер

Интервьюер проверяет практический опыт: работал ли кандидат с YAML в CI/CD или Kubernetes-манифестах, знает ли он про чувствительность к отступам и запрет табуляции. Могут спросить про разницу между `|` и `>` в многострочных строках, про якоря/ссылки, про то, что YAML — надмножество JSON. Красный флаг — незнание, где на практике встречается YAML, кроме "где-то в конфигах".
