# Что такое **функциональное тестирование**? Приведите примеры.

**Коротко:** функциональное тестирование проверяет, что система выполняет свои функции в соответствии с заявленными требованиями и бизнес-логикой — то есть проверяет ЧТО делает продукт.

## Развёрнутый ответ

Функциональное тестирование (Functional testing) — тип тестирования, основанный на анализе спецификации функциональности системы (требований, пользовательских историй, бизнес-правил). Оно проверяет корректность работы конкретных функций: правильность вычислений, обработки данных, навигации, работы бизнес-логики, интеграции с внешними системами на уровне функций.

Функциональное тестирование обычно строится по принципу "чёрного ящика" (black-box): проверяется вход и ожидаемый выход, без анализа внутреннего устройства кода. Оно может проводиться на разных уровнях (unit, integration, system) и включает такие техники, как:
- тестирование по эквивалентным классам (equivalence partitioning);
- анализ граничных значений (boundary value analysis);
- тестирование на основе таблиц решений (decision table testing);
- тестирование переходов состояний (state transition testing).

Основные виды функционального тестирования:
- **Smoke testing** — проверка базовой работоспособности после сборки/деплоя.
- **Sanity testing** — узконаправленная проверка конкретной функциональности после небольших изменений.
- **Тестирование бизнес-логики** — проверка правильности расчётов, обработки правил.
- **Тестирование пользовательского интерфейса на уровне функций** — корректность работы форм, кнопок, переходов.
- **Тестирование API** — корректность обработки запросов и формирования ответов.
- **Тестирование безопасности на уровне функций** — например, проверка прав доступа (может пересекаться с нефункциональным тестированием).

## Пример / когда применяется

Примеры функциональных тестов для формы регистрации:

```text
1. Регистрация с корректными данными -> аккаунт создан, письмо подтверждения отправлено
2. Регистрация с уже существующим email -> система показывает ошибку "email уже используется"
3. Регистрация с паролем короче 8 символов -> система показывает ошибку валидации
4. Восстановление пароля по email -> пользователь получает письмо со ссылкой сброса пароля
```

Другой пример — калькулятор скидок в интернет-магазине: проверка, что при вводе промокода на 20% итоговая цена товара за 1000 рублей корректно пересчитывается на 800 рублей.

## На что смотрит интервьюер

- Умение чётко отделить функциональное тестирование от нефункционального (функциональное — "что делает", нефункциональное — "как делает").
- Знание конкретных техник тест-дизайна (классы эквивалентности, граничные значения) как инструмента для построения функциональных тестов.
- Способность привести реальный, а не абстрактный пример.
- Частый follow-up: "А как вы будете тестировать конкретную форму/фичу функционально?" — ожидается применение техник тест-дизайна.
