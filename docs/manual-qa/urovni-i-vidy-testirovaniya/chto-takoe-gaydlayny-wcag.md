# Что такое гайдлайны **WCAG**?

**Коротко:** WCAG (Web Content Accessibility Guidelines) — это международный набор рекомендаций по обеспечению доступности веб-контента для людей с ограниченными возможностями, разработанный консорциумом W3C.

## Развёрнутый ответ

WCAG — это документ, определяющий, каким требованиям должен соответствовать веб-контент, чтобы быть доступным для максимально широкого круга пользователей, включая людей с нарушениями зрения, слуха, моторики и когнитивными особенностями. Разрабатывается и поддерживается World Wide Web Consortium (W3C) в рамках инициативы WAI (Web Accessibility Initiative).

WCAG построен вокруг четырёх фундаментальных принципов, часто обозначаемых аббревиатурой **POUR**:

- **Perceivable (Воспринимаемость)** — контент должен быть представлен так, чтобы пользователь мог его воспринять (текстовые альтернативы для изображений, субтитры для видео, достаточный контраст).
- **Operable (Управляемость)** — интерфейс должен быть управляем (доступность с клавиатуры, достаточно времени на выполнение действий, отсутствие контента, вызывающего судороги — например, резко мигающих элементов).
- **Understandable (Понятность)** — информация и управление интерфейсом должны быть понятны (предсказуемая навигация, понятный язык, помощь при вводе данных и исправлении ошибок).
- **Robust (Надёжность/совместимость)** — контент должен корректно интерпретироваться широким спектром пользовательских агентов, включая вспомогательные технологии (валидная семантическая разметка).

WCAG определяет три уровня соответствия:

- **Уровень A** — минимальный, базовые требования доступности.
- **Уровень AA** — стандартный уровень, обычно требуемый законодательством и большинством организаций (например, для государственных сайтов).
- **Уровень AAA** — максимальный уровень, наиболее строгий, не всегда достижим для всего контента.

Каждый принцип раскрывается через набор конкретных критериев успеха (success criteria), например: "1.4.3 Контрастность (минимум)" требует соотношения контраста текста к фону не менее 4.5:1 для обычного текста на уровне AA.

## Пример / когда применяется

Пример критерия WCAG уровня AA — "Контрастность текста должна быть не менее 4.5:1 для обычного текста и 3:1 для крупного текста". Тестировщик проверяет это с помощью инструмента проверки контраста (например, встроенного в DevTools или отдельного color contrast checker), сравнивая цвет текста и фона.

Другой пример — критерий "2.1.1 Клавиатура", требующий, чтобы вся функциональность сайта была доступна с клавиатуры без использования мыши. QA-инженер проверяет это, проходя ключевые сценарии только с помощью Tab, Enter и стрелок.

WCAG применяется как база для юридических требований во многих странах (например, ADA в США, EN 301 549 в ЕС) и как ориентир при проведении accessibility testing.

## На что смотрит интервьюер

- Знает ли кандидат расшифровку аббревиатуры WCAG и кто её разрабатывает (W3C).
- Может ли назвать принципы POUR и объяснить их своими словами.
- Понимает ли разницу между уровнями A, AA, AAA.
- Частая ошибка — путать WCAG с ARIA (ARIA — это техническая спецификация HTML-атрибутов, а WCAG — набор требований к доступности в целом, ARIA лишь один из способов их выполнения).
- Follow-up вопросы: какой уровень соответствия WCAG обычно требуется, приведите пример конкретного критерия успеха, как проверить контраст текста на практике.
