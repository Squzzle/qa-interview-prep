# Что такое **нефункциональное тестирование**? Приведите примеры.

**Коротко:** нефункциональное тестирование проверяет качественные характеристики системы — КАК она работает (производительность, надёжность, безопасность, удобство использования), а не что именно она делает.

## Развёрнутый ответ

Нефункциональное тестирование (Non-functional testing) оценивает атрибуты качества системы, которые не связаны напрямую с конкретной бизнес-функцией, но критично влияют на пользовательский опыт и эксплуатационную пригодность продукта. Часто опирается на модель качества ПО (например, ISO/IEC 25010), включающую такие характеристики как производительность, надёжность, удобство использования, безопасность, совместимость, сопровождаемость, портируемость.

Основные виды нефункционального тестирования:

- **Тестирование производительности (Performance testing)** — включает нагрузочное тестирование (load testing, проверка поведения под ожидаемой нагрузкой), стресс-тестирование (stress testing, поведение на пределе и за пределом нагрузки), тестирование стабильности (soak/endurance testing, длительная работа под нагрузкой), тестирование пиковых нагрузок (spike testing).
- **Тестирование удобства использования (Usability testing)** — насколько система интуитивно понятна и удобна для целевой аудитории.
- **Тестирование надёжности (Reliability testing)** — стабильность работы системы в течение долгого времени, устойчивость к сбоям, восстановление после отказов (recovery testing).
- **Тестирование безопасности (Security testing)** — защищённость от несанкционированного доступа, утечек данных, атак.
- **Тестирование совместимости (Compatibility testing)** — работа системы на разных браузерах, ОС, устройствах, с разными версиями зависимостей.
- **Тестирование масштабируемости (Scalability testing)** — способность системы наращивать производительность при увеличении ресурсов.
- **Тестирование установки/удаления (Installability testing)**, **тестирование локализации (Localization testing)** и другие.

## Пример / когда применяется

Примеры нефункциональных тестов для интернет-магазина:

```text
1. Load testing: 5000 пользователей одновременно оформляют заказ,
   время отклика страницы оформления заказа не превышает 2 секунд
2. Usability testing: новый пользователь может оформить заказ
   без подсказок за менее чем 3 минуты
3. Security testing: попытка SQL-инъекции в поле поиска
   не приводит к утечке данных из базы
4. Compatibility testing: сайт корректно отображается
   в Chrome, Firefox, Safari на десктопе и мобильных устройствах
5. Recovery testing: при внезапном отключении сервера оплаты
   система корректно сохраняет состояние заказа и не теряет данные
```

## На что смотрит интервьюер

- Умение назвать конкретные виды нефункционального тестирования (не ограничиваться только "производительность").
- Понимание разницы между load, stress, soak и spike тестированием как частный случай знания темы.
- Способность привести измеримые, конкретные примеры (с цифрами, метриками), а не общие фразы.
- Follow-up: "Как вы будете проводить нагрузочное тестирование?" или "Какие инструменты для этого используются (JMeter, Gatling, k6)?"
