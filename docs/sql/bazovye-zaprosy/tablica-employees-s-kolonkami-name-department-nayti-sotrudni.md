# Таблица Employees с колонками name, department. Найти сотрудников из отделов 'HR' или 'Sales'.

**Коротко:** используется оператор `IN` (или эквивалентная цепочка `OR`) для проверки принадлежности значения набору из нескольких вариантов.

## Развёрнутый ответ

Задача требует объединить два условия равенства через логическое "или". Есть два эквивалентных по результату способа:

1. `WHERE department = 'HR' OR department = 'Sales'` — явная запись через `OR`.
2. `WHERE department IN ('HR', 'Sales')` — компактная и более читаемая форма, предпочтительная при трёх и более значениях.

С точки зрения производительности современные оптимизаторы запросов (PostgreSQL, MySQL, SQL Server) обычно превращают `IN` с константным списком в эквивалентный план выполнения, что и `OR`, поэтому разницы в скорости на практике почти нет — выбор `IN` оправдан читаемостью и удобством сопровождения кода.

Нюансы:

- `IN` работает корректно и с `NULL` в списке значений, но сама проверяемая колонка со значением `NULL` никогда не попадёт в результат (аналогично поведению `=`).
- Если список значений формируется динамически из подзапроса, `IN` также поддерживает синтаксис `department IN (SELECT ... )`.
- Регистрозависимость сравнения зависит от collation колонки/СУБД, как и в случае с одиночным равенством.

## Пример / когда применяется

Типичный сценарий — формирование выборки сотрудников для объединённого отчёта по нескольким отделам, например для совместного тимбилдинга HR и Sales.

```sql
SELECT name, department
FROM Employees
WHERE department IN ('HR', 'Sales');
```

Эквивалентный вариант через OR:

```sql
SELECT name, department
FROM Employees
WHERE department = 'HR' OR department = 'Sales';
```

## На что смотрит интервьюер

- Знает ли кандидат оператор `IN` и когда он предпочтительнее длинной цепочки `OR`.
- Понимает ли, что `IN` и `OR` в данном случае логически эквивалентны.
- Может уточнить: "а если нужно исключить эти отделы?" — ожидается `NOT IN` или `<>` с `AND`, с оговоркой про осторожность при `NULL` в списке для `NOT IN`.
- Проверяет общую грамотность формулировки условий фильтрации.
