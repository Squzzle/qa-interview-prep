# Таблица Employees с колонками name, department, salary. Вывести имена и зарплаты всех сотрудников.

**Коротко:** используется базовый оператор `SELECT` с явным перечислением нужных колонок (`name`, `salary`) без фильтрации строк.

## Развёрнутый ответ

Задача сводится к проекции — выбору подмножества столбцов из таблицы без изменения набора строк. В SQL для этого применяется список колонок после `SELECT`, а не `SELECT *`, потому что явное перечисление колонок:

- делает запрос устойчивым к изменению структуры таблицы (добавление новых колонок не повлияет на результат);
- уменьшает объём передаваемых данных, особенно важно при большом количестве столбцов или наличии тяжёлых типов (BLOB, TEXT);
- улучшает читаемость запроса и облегчает работу с индексами (index-only scan может сработать, если все нужные колонки покрыты индексом).

Так как условие фильтрации отсутствует, предложение `WHERE` не нужно — выводятся все строки таблицы `Employees`. Порядок колонок в результирующем наборе соответствует порядку в списке `SELECT`, а не порядку в определении таблицы.

Важный нюанс: если требуется получить именно "имена и зарплаты", а не всю строку, нельзя использовать `SELECT *`, так как это вернёт лишние колонки (`department`) и может нарушить контракт с потребителем данных (например, приложением, ожидающим ровно два столбца).

## Пример / когда применяется

Такой запрос типичен для отчётов, выгрузок в BI-системы или подготовки данных для расчёта фонда оплаты труда, где не нужна информация об отделе.

```sql
SELECT name, salary
FROM Employees;
```

Пример результата:

| name    | salary |
|---------|--------|
| Иванов  | 5000   |
| Петров  | 6200   |
| Сидорова| 4800   |

## На что смотрит интервьюер

- Использует ли кандидат `SELECT *` вместо явного перечисления колонок — частая ошибка новичков.
- Понимает ли кандидат разницу между проекцией (выбор колонок) и селекцией (фильтрация строк).
- Может спросить про производительность: почему явные колонки быстрее и безопаснее при изменениях схемы.
- Follow-up: "а как отсортировать результат по зарплате?" — ожидается добавление `ORDER BY salary`.
