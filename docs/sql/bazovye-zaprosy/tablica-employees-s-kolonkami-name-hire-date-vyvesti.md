# Таблица Employees с колонками name, hire_date. Вывести сотрудников, нанятых после 1 января 2020 года.

**Коротко:** фильтрация по дате через `WHERE hire_date > DATE '2020-01-01'`, с обязательным вниманием к формату даты и границе интервала.

## Развёрнутый ответ

Работа с датами в SQL требует аккуратности: дата должна передаваться в формате, который однозначно распознаётся СУБД, обычно ISO-8601 (`YYYY-MM-DD`). Использование строкового литерала `'2020-01-01'` в сравнении с колонкой типа `DATE` или `TIMESTAMP` в большинстве СУБД (PostgreSQL, MySQL, SQL Server) работает корректно благодаря неявному приведению типов, но явное указание типа (`DATE '2020-01-01'` в PostgreSQL или `CAST('2020-01-01' AS DATE)`) делает запрос более переносимым и предсказуемым.

Важные нюансы:

- Формулировка "после 1 января 2020 года" означает строгое неравенство `>`, то есть сам день 1 января 2020 года в результат не входит. Если нужно включить эту дату, используется `>=`.
- Если `hire_date` имеет тип `TIMESTAMP` (содержит время), сравнение с `'2020-01-01'` неявно трактуется как полночь этого дня (`2020-01-01 00:00:00`), что обычно совпадает с ожидаемой семантикой "после этой даты".
- Различия форматов дат между локалями (например, американский `MM/DD/YYYY` против европейского `DD/MM/YYYY`) — частый источник ошибок; поэтому предпочтителен ISO-формат.
- Для запросов, часто фильтрующих по диапазону дат, полезен индекс на `hire_date`.

## Пример / когда применяется

Такой запрос используется для отчётов по новым сотрудникам, расчёта стажа или анализа найма за период.

```sql
SELECT name, hire_date
FROM Employees
WHERE hire_date > DATE '2020-01-01';
```

Пример результата:

| name     | hire_date  |
|----------|------------|
| Смирнова | 2020-03-15 |
| Волков   | 2021-07-01 |

## На что смотрит интервьюер

- Правильно ли кандидат трактует "после" как строгое неравенство, а не включительно.
- Знает ли про типы данных для дат и почему не стоит хранить даты как строки.
- Упоминает ли важность единого формата дат и потенциальные проблемы локализации.
- Follow-up: "а как найти нанятых в конкретном диапазоне, например за 2020 год?" — ожидается `BETWEEN` или пара условий `>= и <`.
