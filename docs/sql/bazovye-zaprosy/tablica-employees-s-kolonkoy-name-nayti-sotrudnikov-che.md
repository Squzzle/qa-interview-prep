# Таблица Employees с колонкой name. Найти сотрудников, чьё имя начинается на букву 'А'.

**Коротко:** используется оператор `LIKE` с шаблоном `'А%'`, где символ `%` означает любое количество любых символов после первой буквы.

## Развёрнутый ответ

Для поиска по началу строки в SQL применяется оператор `LIKE` (или `ILIKE` для регистронезависимого поиска в PostgreSQL) с шаблоном, где `%` — джокер, соответствующий нулю или более любых символов, а `_` — ровно одному символу. Условие "начинается на букву 'А'" записывается как `name LIKE 'А%'`.

Важные нюансы, которые стоит проговорить:

- Регистр буквы 'А' имеет значение в большинстве СУБД по умолчанию (например, PostgreSQL с collation `C` или `en_US` регистрозависим для `LIKE`), поэтому для надёжности часто используют `ILIKE` (PostgreSQL) или `UPPER(name) LIKE 'А%'` как переносимый вариант.
- Кириллическая буква 'А' визуально совпадает с латинской 'A', но это разные символы в кодировке Unicode — важно убедиться, что в шаблоне используется именно кириллица, если имена хранятся на русском языке.
- `LIKE 'А%'` (шаблон, не начинающийся с `%`) может использовать обычный B-tree индекс на колонке `name` (index range scan), в отличие от `LIKE '%А%'`, которое требует полного сканирования таблицы или специального индекса (например, `pg_trgm` в PostgreSQL или полнотекстового индекса).
- Альтернатива без `LIKE` — использование `LEFT(name, 1) = 'А'` или `SUBSTRING(name FROM 1 FOR 1) = 'А'`, но такой подход обычно не позволяет использовать индекс без функционального индекса и менее идиоматичен.

## Пример / когда применяется

Такой запрос часто используется в UI с алфавитной навигацией по списку сотрудников или для группировки контактов по первой букве.

```sql
SELECT name
FROM Employees
WHERE name LIKE 'А%';
```

Регистронезависимый вариант (PostgreSQL):

```sql
SELECT name
FROM Employees
WHERE name ILIKE 'А%';
```

## На что смотрит интервьюер

- Знает ли кандидат синтаксис `LIKE`, разницу между `%` и `_`.
- Замечает ли проблему регистрозависимости и путаницы кириллицы/латиницы.
- Понимает ли, какие шаблоны `LIKE` могут использовать индекс, а какие приводят к полному сканированию таблицы.
- Follow-up: "а как найти имена, содержащие 'ан' в любом месте?" — ожидается `LIKE '%ан%'` с пояснением о потере эффективности индекса.
