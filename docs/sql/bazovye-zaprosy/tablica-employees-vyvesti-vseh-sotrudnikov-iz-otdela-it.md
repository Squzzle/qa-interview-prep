# Таблица Employees. Вывести всех сотрудников из отдела 'IT'.

**Коротко:** нужна фильтрация строк по условию равенства через `WHERE department = 'IT'`, при этом `SELECT *` уместен, так как требуется вся информация о сотрудниках.

## Развёрнутый ответ

Задача требует не проекции, а селекции — отбора строк, удовлетворяющих условию. Условие фильтрации задаётся в предложении `WHERE`, которое выполняется до `SELECT` в логическом порядке обработки запроса (`FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY`), поэтому в `WHERE` нельзя использовать алиасы, объявленные в `SELECT`.

Ключевые нюансы:

- Строковое сравнение `department = 'IT'` в большинстве СУБД (PostgreSQL по умолчанию) регистрозависимо. В MySQL с типичной collation (`utf8mb4_general_ci`) сравнение регистронезависимо. Это важно уточнить у интервьюера или указать как допущение.
- Если в колонке `department` могут быть завершающие пробелы или значения `NULL`, простое сравнение `= 'IT'` не найдёт `NULL` (по правилам трёхзначной логики SQL — `NULL = 'IT'` даёт `UNKNOWN`, а не `TRUE`).
- Для ускорения фильтрации по часто используемому предикату стоит создать индекс на колонке `department`, особенно если таблица большая и запрос выполняется регулярно.
- Если возможны разные варианты написания (`'it'`, `'IT '`), можно использовать `UPPER(TRIM(department)) = 'IT'`, но это лишает возможности использовать обычный B-tree индекс без функционального индекса.

## Пример / когда применяется

Типичная задача — выгрузка списка сотрудников конкретного подразделения для HR-отчёта или рассылки.

```sql
SELECT *
FROM Employees
WHERE department = 'IT';
```

Более безопасный вариант с учётом регистра и пробелов:

```sql
SELECT *
FROM Employees
WHERE UPPER(TRIM(department)) = 'IT';
```

## На что смотрит интервьюер

- Знает ли кандидат про трёхзначную логику SQL и поведение `NULL` в сравнениях.
- Поднимает ли вопрос регистрозависимости сравнения строк в разных СУБД.
- Предлагает ли индекс для оптимизации фильтрации по колонке `department`.
- Follow-up: "а если нужно несколько отделов?" — ожидается `IN ('IT', 'HR')` вместо цепочки `OR`.
