# Добавить колонку phone_number типа VARCHAR(20) в таблицу Employees.

**Коротко:** используется команда `ALTER TABLE ... ADD COLUMN`, которая добавляет новую колонку в существующую таблицу без потери данных в уже имеющихся строках.

## Развёрнутый ответ

`ALTER TABLE` — команда DDL для изменения структуры уже созданной таблицы: добавления, изменения или удаления колонок и ограничений. Для добавления новой колонки используется конструкция `ADD COLUMN` (в некоторых СУБД, например MySQL и Oracle, ключевое слово `COLUMN` можно опустить).

Важные нюансы:
- Если новая колонка не имеет ограничения `NOT NULL`, то во всех существующих строках она автоматически заполняется значением `NULL`.
- Если требуется `NOT NULL`, необходимо либо задать `DEFAULT` значение, либо СУБД выдаст ошибку при наличии уже существующих строк (кроме случаев, когда таблица пуста или указано значение по умолчанию).
- В больших таблицах (миллионы строк) операция `ADD COLUMN` может быть дорогой по времени и блокировать таблицу — это зависит от СУБД:
  - PostgreSQL (начиная с 11 версии) — добавление колонки со статическим `DEFAULT` выполняется мгновенно (без перезаписи всей таблицы), так как значение хранится в метаданных;
  - MySQL (InnoDB, начиная с 8.0 с `ALGORITHM=INSTANT`) — тоже может быть почти мгновенным при определённых условиях;
  - в более старых версиях СУБД операция может требовать полной перезаписи таблицы и блокировки.
- VARCHAR(20) ограничивает длину строки максимум 20 символами — при попытке вставить более длинное значение СУБД либо обрежет его (не рекомендуется), либо выдаст ошибку в зависимости от режима строгости (например, `sql_mode` в MySQL).

## Пример / когда применяется

Типичный сценарий — расширение схемы существующей таблицы сотрудников новым полем для контактных данных, без необходимости пересоздавать таблицу.

```sql
ALTER TABLE Employees
ADD COLUMN phone_number VARCHAR(20);
```

Если нужно сразу задать ограничение и значение по умолчанию:

```sql
ALTER TABLE Employees
ADD COLUMN phone_number VARCHAR(20) NOT NULL DEFAULT '';
```

## На что смотрит интервьюер

- Знает ли кандидат разницу между `ADD COLUMN` в разных СУБД (в некоторых слово `COLUMN` не требуется, а в некоторых обязательно).
- Понимает ли, что произойдёт со старыми строками (заполнение `NULL` или `DEFAULT`).
- Осведомлён ли о влиянии операции на блокировки и производительность в больших таблицах (частый вопрос на senior-уровне).
- Красный флаг — предложение "пересоздать таблицу" вместо использования `ALTER TABLE`, что означает потерю данных без дополнительной миграции.
