# Изменить тип колонки salary в таблице Employees с INT на DECIMAL(10,2).

**Коротко:** тип колонки меняется командой `ALTER TABLE ... ALTER COLUMN ... TYPE` (PostgreSQL) или `ALTER TABLE ... MODIFY COLUMN` (MySQL) — синтаксис зависит от СУБД.

## Развёрнутый ответ

Изменение типа данных существующей колонки — операция DDL, синтаксис которой существенно отличается между СУБД:

- **PostgreSQL**: `ALTER TABLE ... ALTER COLUMN ... TYPE новый_тип [USING выражение]`.
- **MySQL**: `ALTER TABLE ... MODIFY COLUMN имя новый_тип [ограничения]`.
- **SQL Server**: `ALTER TABLE ... ALTER COLUMN имя новый_тип`.
- **Oracle**: `ALTER TABLE ... MODIFY (имя новый_тип)`.

При переходе с `INT` на `DECIMAL(10,2)` данные технически совместимы: целые числа корректно преобразуются в десятичные с двумя знаками после запятой (например, 50000 станет 50000.00). Это безопасное расширяющее преобразование, не приводящее к потере данных.

Важные нюансы:
- В PostgreSQL при сложных преобразованиях типов (например, из текста в число) может потребоваться явное указание `USING имя_колонки::новый_тип`, но при INT → DECIMAL это преобразование выполняется автоматически (неявное приведение).
- Операция может требовать блокировки таблицы и полной её перезаписи (ACCESS EXCLUSIVE lock в PostgreSQL), что критично для больших таблиц в продакшене — стоит выполнять в окно обслуживания или через онлайн-инструменты миграции (например, pt-online-schema-change для MySQL, gh-ost).
- DECIMAL(10,2) означает: всего 10 значащих цифр, из которых 2 — после запятой (то есть максимум 99999999.99).
- Если в колонке уже были значения, превышающие точность нового типа, СУБД выдаст ошибку переполнения — это стоит учитывать перед миграцией.

## Пример / когда применяется

Типичный сценарий — уточнение денежного поля, когда изначально зарплата хранилась как целое число (без копеек), а бизнес-требования изменились и потребовалась точность до копеек.

```sql
-- PostgreSQL
ALTER TABLE Employees
ALTER COLUMN salary TYPE DECIMAL(10,2);
```

```sql
-- MySQL
ALTER TABLE Employees
MODIFY COLUMN salary DECIMAL(10,2);
```

## На что смотрит интервьюер

- Знает ли кандидат, что синтаксис отличается: `ALTER COLUMN ... TYPE` в PostgreSQL против `MODIFY COLUMN` в MySQL — частая путаница у джуниоров.
- Понимает ли смысл `DECIMAL(10,2)` (точность и масштаб), а не путает с `FLOAT`/`DOUBLE` (которые имеют погрешность и не подходят для денег).
- Упоминает ли риски блокировки таблицы и деградации производительности на больших объёмах данных.
- Красный флаг — предложение использовать `FLOAT` для денежных сумм (потенциальная потеря точности при вычислениях).
- Follow-up: почему для денег используют DECIMAL, а не FLOAT; как выполнить такую миграцию без даунтайма на очень большой таблице.
