# Удалить таблицу OldLogs.

**Коротко:** для полного удаления таблицы вместе со всеми её данными, индексами и ограничениями используется команда DDL `DROP TABLE OldLogs;`.

## Развёрнутый ответ

`DROP TABLE` — команда языка определения данных (DDL), которая безвозвратно удаляет таблицу целиком: саму структуру, все хранящиеся в ней данные, связанные индексы, триггеры и ограничения (constraints), определённые непосредственно на этой таблице. Это принципиально отличается от `DELETE FROM OldLogs`, который удаляет только строки, сохраняя структуру таблицы, или от `TRUNCATE TABLE`, который быстро очищает данные, но также сохраняет структуру.

Ключевые нюансы:
- `DROP TABLE` — операция DDL, которая в большинстве СУБД автоматически коммитится (implicit commit) и не может быть отменена через `ROLLBACK` в MySQL (в PostgreSQL DDL транзакционен и может быть отменён до коммита).
- Если на таблицу OldLogs ссылаются внешние ключи из других таблиц, попытка удаления завершится ошибкой (в большинстве СУБД по умолчанию), пока не будут удалены зависимые ограничения или сами зависимые таблицы/строки.
- Конструкция `IF EXISTS` позволяет избежать ошибки, если таблицы уже не существует: `DROP TABLE IF EXISTS OldLogs;` — полезно в идемпотентных миграционных скриптах.
- В некоторых СУБД (например, PostgreSQL) можно использовать `DROP TABLE OldLogs CASCADE;`, чтобы автоматически удалить все зависимые объекты (внешние ключи, представления), ссылающиеся на эту таблицу — но это потенциально опасная операция, требующая осторожности.
- Перед удалением таблицы в проде рекомендуется убедиться в наличии актуального бэкапа, так как операция необратима.

## Пример / когда применяется

Типичный сценарий — очистка схемы базы данных от устаревшей таблицы логов, которая больше не используется приложением (например, заменена новой архивной таблицей или вынесена во внешнее хранилище).

```sql
DROP TABLE IF EXISTS OldLogs;
```

Если есть зависимые объекты и нужно принудительное удаление (PostgreSQL):

```sql
DROP TABLE IF EXISTS OldLogs CASCADE;
```

## На что смотрит интервьюер

- Понимает ли кандидат разницу между `DROP TABLE`, `TRUNCATE TABLE` и `DELETE FROM` (это классический вопрос на собеседовании).
- Знает ли, что `DROP TABLE` необратим и удаляет структуру, а не только данные.
- Упоминает ли `IF EXISTS` для безопасных идемпотентных скриптов миграции.
- Осведомлён ли о поведении при наличии внешних ключей, ссылающихся на удаляемую таблицу, и о ключевом слове `CASCADE`.
- Красный флаг — если кандидат не упоминает необходимость бэкапа или не видит разницы между DROP и DELETE/TRUNCATE.
