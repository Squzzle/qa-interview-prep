# Таблица Employees (id, name, manager_id). Вывести список сотрудников с именами начальников (SELF JOIN).

**Коротко:** таблица соединяется сама с собой по условию `e.manager_id = m.id`, при этом обеим "копиям" таблицы даются разные алиасы, чтобы SQL мог отличить строку сотрудника от строки его начальника.

## Развёрнутый ответ

`SELF JOIN` — это не отдельный вид соединения, а обычный `JOIN` (чаще `LEFT JOIN`), в котором таблица соединяется сама с собой. Технически это выглядит как обращение к одной и той же таблице дважды с разными алиасами, что позволяет трактовать одну "копию" как сотрудника, а другую — как его руководителя.

В таблице `Employees(id, name, manager_id)` поле `manager_id` — это внешний ключ, ссылающийся на `id` этой же таблицы (иерархическая связь "сотрудник — его руководитель"). Чтобы вывести имя начальника рядом с именем сотрудника, нужно присоединить таблицу к самой себе по условию `employee.manager_id = manager.id`.

Ключевые нюансы:
- обязательно использовать разные алиасы (например, `e` для сотрудника и `m` для менеджера), иначе СУБД не поймёт, к какой "копии" относится каждый столбец;
- топ-менеджер (например, CEO) обычно имеет `manager_id = NULL`, поэтому для него нужен `LEFT JOIN`, а не `INNER JOIN` — иначе такой сотрудник вообще выпадет из результата, так как для него нет совпадения по `id`;
- при `LEFT JOIN` для сотрудников без руководителя имя начальника в результате будет `NULL`, что и требуется показать как "нет начальника" (или можно обернуть в `COALESCE(m.name, 'Нет начальника')` для более наглядного вывода);
- `SELF JOIN` часто используется для иерархических структур: оргструктура компании, категории/подкатегории товаров, дерево комментариев (хотя для произвольной глубины иерархии одного `SELF JOIN` недостаточно — тогда нужны рекурсивные CTE).

## Пример / когда применяется

```sql
SELECT
    e.name AS employee_name,
    m.name AS manager_name
FROM Employees e
LEFT JOIN Employees m
    ON e.manager_id = m.id
ORDER BY e.name;
```

Для сотрудника без руководителя (`manager_id IS NULL`) колонка `manager_name` будет `NULL`. Если нужно вывести название вместо `NULL`, можно применить `COALESCE`:

```sql
SELECT
    e.name AS employee_name,
    COALESCE(m.name, 'Нет начальника') AS manager_name
FROM Employees e
LEFT JOIN Employees m
    ON e.manager_id = m.id;
```

## На что смотрит интервьюер

- Использование разных алиасов для одной и той же таблицы — без этого запрос вообще не скомпилируется или будет неоднозначным.
- Выбор `LEFT JOIN`, а не `INNER JOIN`, чтобы не терять сотрудников без руководителя (топ-менеджеров).
- Понимание, что `SELF JOIN` решает задачу только для одного уровня иерархии "сотрудник — непосредственный начальник"; для произвольной глубины (например, "вывести всю цепочку начальников до CEO") нужен рекурсивный `WITH RECURSIVE` (recursive CTE), и кандидат должен уметь это озвучить как следующий шаг.
- Follow-up вопрос: "как посчитать, сколько подчинённых у каждого менеджера?" — ожидается `GROUP BY manager_id` с `COUNT`.
