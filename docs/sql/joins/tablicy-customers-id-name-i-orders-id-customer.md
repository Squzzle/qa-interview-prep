# Таблицы Customers (id, name) и Orders (id, customer_id, amount). Вывести всех клиентов и их заказы (клиенты без заказов с NULL).

**Коротко:** нужен `LEFT JOIN` от `Customers` к `Orders` — он сохраняет все строки левой таблицы (клиентов), а для отсутствующих заказов подставляет `NULL` в колонки правой таблицы.

## Развёрнутый ответ

`LEFT JOIN` (он же `LEFT OUTER JOIN`) возвращает все строки из левой таблицы независимо от того, найдено ли совпадение в правой таблице. Если совпадения нет, все столбцы правой таблицы в этой строке результата будут `NULL`. Это прямо противоположно `INNER JOIN`, который отбрасывает несовпадающие строки.

Важные нюансы:
- порядок таблиц имеет значение: `Customers LEFT JOIN Orders` — это не то же самое, что `Orders LEFT JOIN Customers`;
- условие соединения (`ON customers.id = orders.customer_id`) определяет, как строки сопоставляются; если у клиента несколько заказов, он появится в результате несколько раз — по одной строке на каждый заказ (это нормальное поведение JOIN, не дублирование данных);
- если нужно посчитать что-то по клиенту (например, сумму заказов, включая клиентов без заказов), обычно используют `LEFT JOIN` в сочетании с `GROUP BY` и агрегатной функцией, которая корректно обрабатывает `NULL` (например, `SUM` игнорирует `NULL`, `COUNT(orders.id)` даст 0 для клиента без заказов, а `COUNT(*)` даст 1, что часто является частой ошибкой);
- альтернатива — `RIGHT JOIN` с переставленными таблицами даст тот же результат, но `LEFT JOIN` считается более читаемым стилем и используется чаще на практике.

## Пример / когда применяется

Схема: `Customers(id, name)`, `Orders(id, customer_id, amount)`.

```sql
SELECT
    c.name,
    o.id AS order_id,
    o.amount
FROM Customers c
LEFT JOIN Orders o
    ON c.id = o.customer_id
ORDER BY c.id;
```

Для клиента без заказов колонки `order_id` и `amount` будут содержать `NULL`. Такой запрос типичен для отчётов по клиентской базе, где важно видеть "неактивных" клиентов (без единого заказа), а не только тех, у кого есть история покупок.

Если нужно посчитать количество заказов на каждого клиента, включая нулевые:

```sql
SELECT
    c.name,
    COUNT(o.id) AS orders_count
FROM Customers c
LEFT JOIN Orders o
    ON c.id = o.customer_id
GROUP BY c.id, c.name;
```

## На что смотрит интервьюер

- Понимание, что `LEFT JOIN` сохраняет все строки левой таблицы, а не только "первую попавшуюся".
- Осознание разницы между `COUNT(o.id)` и `COUNT(*)` при агрегации после `LEFT JOIN` — частая ошибка кандидатов, которые получают 1 вместо 0 для клиентов без заказов.
- Понимание, что фильтр `WHERE orders.amount > 100` после `LEFT JOIN` фактически превращает его в `INNER JOIN` (отбрасывает строки с `NULL`), а правильный способ фильтрации правой таблицы без потери левых строк — перенести условие в `ON`.
- Follow-up вопрос: "как найти клиентов, у которых вообще нет заказов?" — ожидается `WHERE o.id IS NULL` после `LEFT JOIN`.
