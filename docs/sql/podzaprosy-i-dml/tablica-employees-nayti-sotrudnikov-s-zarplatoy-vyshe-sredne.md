# Таблица Employees. Найти сотрудников с зарплатой выше средней (подзапрос).

**Коротко:** используется коррелированный или простой скалярный подзапрос: сначала вычисляется средняя зарплата по всей таблице через `AVG(salary)`, затем внешний запрос сравнивает зарплату каждого сотрудника с этим значением через `WHERE salary > (SELECT AVG(salary) FROM Employees)`.

## Развёрнутый ответ

Задача решается с помощью скалярного подзапроса — подзапроса, возвращающего одно значение. Скалярный подзапрос `SELECT AVG(salary) FROM Employees` вычисляется один раз (в большинстве СУБД оптимизатор материализует это значение), а затем внешний запрос фильтрует строки, сравнивая зарплату каждого сотрудника с этим единственным числом.

Важные нюансы:

- Подзапрос в данном случае **некоррелированный** (не зависит от строк внешнего запроса), поэтому он выполняется один раз, а не для каждой строки — это эффективно.
- Если требуется сравнение не с общей средней, а со средней зарплатой по отделу сотрудника, подзапрос становится **коррелированным**: `WHERE salary > (SELECT AVG(e2.salary) FROM Employees e2 WHERE e2.department_id = e1.department_id)`. В этом случае подзапрос выполняется для каждой строки внешнего запроса, что может быть медленнее на больших таблицах.
- Нужно учитывать `NULL`: функция `AVG()` игнорирует `NULL`-значения зарплаты, что обычно является желаемым поведением, но об этом стоит упомянуть на собеседовании.
- Альтернатива подзапросу — оконная функция `AVG(salary) OVER ()`, которая вычисляет среднее без группировки и позволяет получить дополнительные колонки (например, отклонение от среднего) в одном проходе, без вложенного запроса.

## Пример / когда применяется

Простой вариант (сравнение со средней по всей компании):

```sql
SELECT id, name, department_id, salary
FROM Employees
WHERE salary > (SELECT AVG(salary) FROM Employees);
```

Вариант со средней по отделу (коррелированный подзапрос):

```sql
SELECT e1.id, e1.name, e1.department_id, e1.salary
FROM Employees e1
WHERE e1.salary > (
    SELECT AVG(e2.salary)
    FROM Employees e2
    WHERE e2.department_id = e1.department_id
);
```

Эквивалент через оконную функцию:

```sql
SELECT id, name, salary
FROM (
    SELECT id, name, salary, AVG(salary) OVER () AS avg_salary
    FROM Employees
) t
WHERE salary > avg_salary;
```

## На что смотрит интервьюер

- Понимает ли кандидат разницу между коррелированным и некоррелированным подзапросом и их влияние на производительность.
- Не забыл ли кандидат, что подзапрос со средней должен вычисляться отдельно, а не внутри `GROUP BY` (частая ошибка — попытка написать `WHERE salary > AVG(salary)` без подзапроса, что синтаксически недопустимо, так как агрегатные функции нельзя использовать в `WHERE` напрямую).
- Знает ли альтернативу через оконные функции.
- Уточняет ли кандидат требование: "средняя по компании" или "средняя по отделу" — хороший кандидат задаёт уточняющий вопрос.
- Обрабатывает ли `NULL` в зарплате корректно.
