# Таблица Employees. Увеличить зарплату на 10% всем в отделе 'IT' (UPDATE).

**Коротко:** используется `UPDATE Employees SET salary = salary * 1.1 WHERE department = 'IT'` — обновление применяется только к строкам, удовлетворяющим условию `WHERE`, при этом расчёт `salary * 1.1` выполняется на основе текущего значения каждой строки.

## Развёрнутый ответ

Оператор `UPDATE` изменяет значения столбцов для всех строк, попавших под условие `WHERE`. Увеличение на 10% означает умножение текущей зарплаты на `1.1` (то есть 100% + 10%). Важно не забыть `WHERE department = 'IT'` — без него зарплата обновится у всех сотрудников во всей таблице, что является одной из самых частых и опасных ошибок при работе с `UPDATE`.

Если структура данных нормализована и отдел хранится не строкой, а через `department_id`, ссылающийся на таблицу `Departments`, то условие превращается в подзапрос: `WHERE department_id = (SELECT id FROM Departments WHERE name = 'IT')`, либо используется `JOIN`-синтаксис обновления, если СУБД его поддерживает (например, `UPDATE ... FROM` в PostgreSQL/SQL Server).

Важные нюансы и best practices:

- Перед выполнением реального `UPDATE` на проде рекомендуется сначала выполнить эквивалентный `SELECT` с тем же `WHERE`, чтобы убедиться, что затрагиваются именно нужные строки.
- Операцию стоит оборачивать в транзакцию (`BEGIN ... COMMIT`), чтобы иметь возможность сделать `ROLLBACK`, если что-то пошло не так.
- Нужно учитывать регистрозависимость сравнения строки `'IT'` — в зависимости от collation базы данных сравнение может быть чувствительным или нечувствительным к регистру.
- Если в столбце `salary` может быть `NULL`, то `NULL * 1.1` даст `NULL` — это ожидаемое поведение, но стоит проговорить на собеседовании.
- Округление: в зависимости от типа столбца (`DECIMAL`, `FLOAT`) результат умножения может потребовать округления, например через `ROUND(salary * 1.1, 2)`.

## Пример / когда применяется

Простой вариант, если отдел хранится строкой:

```sql
UPDATE Employees
SET salary = ROUND(salary * 1.1, 2)
WHERE department = 'IT';
```

Вариант с нормализованной схемой (department хранится через department_id):

```sql
UPDATE Employees
SET salary = ROUND(salary * 1.1, 2)
WHERE department_id = (
    SELECT id FROM Departments WHERE name = 'IT'
);
```

Безопасный сценарий с проверкой перед выполнением:

```sql
-- 1. Сначала проверить, кого затронет обновление
SELECT id, name, salary
FROM Employees
WHERE department = 'IT';

-- 2. Затем выполнить в транзакции
BEGIN;
UPDATE Employees
SET salary = ROUND(salary * 1.1, 2)
WHERE department = 'IT';
COMMIT;
```

## На что смотрит интервьюер

- Не забыл ли кандидат условие `WHERE` (обновление без `WHERE` — классическая "страшилка" собеседований).
- Понимание, что `salary * 1.1`, а не `salary + 10` или `salary * 10 / 100 + salary` — то есть корректная математика повышения на процент.
- Знание практики "сначала SELECT, потом UPDATE" и оборачивания в транзакцию для безопасности.
- Готовность обсудить нормализованную схему (department_id вместо строки) как более правильный дизайн.
- Внимание к округлению и типу данных денежного столбца.
