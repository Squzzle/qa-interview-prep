# Таблица Employees. Присвоить ранг каждому сотруднику внутри отдела по зарплате (оконная функция).

**Коротко:** используется оконная функция `RANK()` (или `DENSE_RANK()` / `ROW_NUMBER()`) с конструкцией `OVER (PARTITION BY department_id ORDER BY salary DESC)`, которая делит строки на партиции по отделу и внутри каждой партиции присваивает ранг по убыванию зарплаты.

## Развёрнутый ответ

Оконные функции (window functions) выполняют вычисление над набором строк, связанных с текущей строкой, но не схлопывают результат в одну строку, как это делает `GROUP BY`. Синтаксис общий: `функция() OVER (PARTITION BY <столбцы> ORDER BY <столбцы>)`.

`PARTITION BY department_id` делит все строки таблицы на группы — по одной группе на каждый отдел. Внутри каждой партиции `ORDER BY salary DESC` задаёт порядок, по которому и присваивается ранг.

Разница между тремя похожими функциями ранжирования важна для интервью:

- `ROW_NUMBER()` — присваивает уникальный порядковый номер каждой строке, даже если зарплаты равны (нет "дырок" и нет одинаковых номеров).
- `RANK()` — строки с одинаковым значением получают одинаковый ранг, но следующий ранг пропускается (например, 1, 2, 2, 4).
- `DENSE_RANK()` — строки с одинаковым значением получают одинаковый ранг, но следующий ранг идёт подряд без пропусков (1, 2, 2, 3).

Выбор конкретной функции зависит от требований: если нужно строго уникальное место в рейтинге — `ROW_NUMBER()`, если важно честно отразить "дубликаты" зарплат — `RANK()` или `DENSE_RANK()`.

Важный нюанс: оконная функция вычисляется после `WHERE`, `JOIN` и `GROUP BY`, но до `ORDER BY` и `LIMIT` всего запроса, поэтому её нельзя напрямую использовать в `WHERE` того же уровня запроса — для фильтрации по рангу нужен подзапрос или CTE.

## Пример / когда применяется

Допустим, таблица `employees(employee_id, name, department_id, salary)`.

```sql
SELECT
    employee_id,
    name,
    department_id,
    salary,
    RANK() OVER (PARTITION BY department_id ORDER BY salary DESC) AS salary_rank
FROM employees
ORDER BY department_id, salary_rank;
```

Если нужно получить только топ-1 сотрудника по зарплате в каждом отделе, оборачиваем в CTE, так как использовать оконную функцию в `WHERE` напрямую нельзя:

```sql
WITH ranked AS (
    SELECT
        employee_id,
        name,
        department_id,
        salary,
        RANK() OVER (PARTITION BY department_id ORDER BY salary DESC) AS salary_rank
    FROM employees
)
SELECT * FROM ranked WHERE salary_rank = 1;
```

## На что смотрит интервьюер

Интервьюер проверяет, понимает ли кандидат разницу между `RANK()`, `DENSE_RANK()` и `ROW_NUMBER()`, и умеет ли объяснить назначение `PARTITION BY` (аналог группировки, но без схлопывания строк). Частая ошибка — попытка сразу отфильтровать по псевдониму оконной функции в `WHERE` того же запроса (это невозможно, так как `WHERE` выполняется раньше вычисления оконных функций). Также могут спросить про производительность — оконные функции требуют сортировки данных внутри партиции, что может быть затратно на больших таблицах без подходящего индекса.
