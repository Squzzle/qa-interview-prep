# Таблица Products. Вывести название, цену и разницу с предыдущей ценой (функция LAG()).

**Коротко:** функция `LAG(price) OVER (ORDER BY ...)` возвращает значение цены из предыдущей строки относительно текущей в заданном порядке сортировки, что позволяет вычислить разницу `price - LAG(price) OVER (...)` без самосоединения таблицы.

## Развёрнутый ответ

`LAG()` и `LEAD()` — это оконные функции доступа к смещённым строкам (offset functions). `LAG(column, offset, default)` возвращает значение `column` из строки, отстоящей на `offset` строк назад от текущей (по умолчанию `offset = 1`); `LEAD()` — аналогично, но вперёд. Третий параметр — значение по умолчанию, если предыдущей строки не существует (например, для самой первой строки в партиции), обычно `NULL` или `0`.

До появления оконных функций такую задачу решали через самосоединение таблицы (`self join`) по номеру строки или по дате, что было громоздко и менее эффективно. `LAG()`/`LEAD()` заменяют это одной конструкцией.

Важные нюансы:

- Обязательно указывать `ORDER BY` внутри `OVER`, иначе понятие "предыдущая строка" не определено — без сортировки СУБД не гарантирует детерминированный порядок.
- Если нужно сравнивать цены в рамках группы (например, по каждому товару отдельно, если строк по товару несколько — история цен), добавляется `PARTITION BY product_id`.
- Разница вычисляется как обычное арифметическое выражение над результатом оконной функции: `price - LAG(price) OVER (...)`. Обернуть в `COALESCE`, если нужно избежать `NULL` для первой строки.

## Пример / когда применяется

Таблица `products(product_id, name, price, updated_at)`, где может храниться история изменения цены.

```sql
SELECT
    name,
    price,
    LAG(price) OVER (PARTITION BY product_id ORDER BY updated_at) AS previous_price,
    price - LAG(price) OVER (PARTITION BY product_id ORDER BY updated_at) AS price_diff
FROM products
ORDER BY product_id, updated_at;
```

Если таблица содержит по одной строке на товар (без истории), а требуется сравнение цены с предыдущим по алфавиту или по id товаром — партиционирование не нужно, сортировка идёт по всему набору:

```sql
SELECT
    name,
    price,
    LAG(price) OVER (ORDER BY product_id) AS previous_price,
    price - LAG(price) OVER (ORDER BY product_id) AS price_diff
FROM products;
```

## На что смотрит интервьюер

Интервьюер проверяет знание того, что `LAG`/`LEAD` требуют `ORDER BY`, и понимание разницы между "предыдущей строкой в результирующем наборе" и "предыдущим значением во времени" — это не одно и то же, если сортировка задана некорректно. Частый вопрос на уточнение: что вернётся для первой строки партиции (по умолчанию `NULL`, если не задан третий аргумент). Также могут спросить про альтернативу через `self join` и попросить объяснить, почему оконная функция эффективнее.
