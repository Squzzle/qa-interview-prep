# Таблица Employees. Найти отделы, в которых средняя зарплата превышает 6000.

**Коротко:** нужно сгруппировать сотрудников по отделу (`GROUP BY department`), вычислить `AVG(salary)` для каждой группы и отфильтровать результат условием `HAVING AVG(salary) > 6000`.

## Развёрнутый ответ

Это классическая задача на разграничение `WHERE` и `HAVING`:

- `WHERE` фильтрует отдельные строки ДО группировки и агрегации. В `WHERE` нельзя использовать агрегатные функции, потому что на этом этапе группы ещё не сформированы.
- `HAVING` фильтрует уже сформированные группы ПОСЛЕ применения `GROUP BY` и вычисления агрегатов. Именно в `HAVING` можно писать условия с `AVG`, `SUM`, `COUNT` и другими агрегатными функциями.

Порядок логического выполнения запроса: `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY`. Понимание этого порядка объясняет, почему условие на среднюю зарплату должно стоять именно в `HAVING`, а не в `WHERE`.

Дополнительно: если в `HAVING` нужно сослаться на алиас, заданный в `SELECT` (например, `avg_salary`), то в некоторых СУБД (PostgreSQL, MySQL) это допустимо, но в строгом ANSI SQL и в ряде СУБД (например, в старых версиях Oracle) в `HAVING` нужно повторно писать выражение агрегатной функции, а не алиас, потому что `HAVING` логически выполняется до вычисления списка `SELECT`.

## Пример / когда применяется

Такой запрос применяется при анализе, какие отделы в среднем переплачивают сотрудникам сверх определённого порога (например, для бюджетного контроля).

```sql
SELECT department, AVG(salary) AS avg_salary
FROM Employees
GROUP BY department
HAVING AVG(salary) > 6000;
```

## На что смотрит интервьюер

- Использование `WHERE AVG(salary) > 6000` вместо `HAVING` — грубая ошибка, приведёт к синтаксической ошибке в большинстве СУБД, так как агрегатные функции недопустимы в `WHERE`.
- Понимание порядка выполнения предложений SQL-запроса.
- Follow-up: "Можно ли использовать HAVING без GROUP BY?" — да, тогда вся таблица считается одной группой.
- Follow-up: "Можно ли отфильтровать по нескольким условиям в HAVING, включая неагрегатные?" — да, HAVING может содержать любые условия, но традиционно используется для агрегатов.
- Красный флаг: попытка комбинировать `WHERE` и `HAVING` там, где условие относится только к агрегату — избыточное усложнение запроса.
