# Таблица Employees. Посчитать общее количество сотрудников.

**Коротко:** используется агрегатная функция `COUNT(*)`, которая возвращает количество строк в таблице (или в результирующем наборе после фильтрации).

## Развёрнутый ответ

`COUNT` — одна из базовых агрегатных функций SQL. Есть несколько вариантов её использования, и разница между ними — частый вопрос на собеседовании:

- `COUNT(*)` — считает все строки, включая те, где есть `NULL` в любых колонках. Не зависит от конкретной колонки.
- `COUNT(column)` — считает только строки, где значение указанной колонки НЕ `NULL`. Если, например, у части сотрудников не указан `department`, то `COUNT(department)` даст меньшее число, чем `COUNT(*)`.
- `COUNT(DISTINCT column)` — считает количество уникальных непустых значений в колонке.

Без `GROUP BY` агрегатная функция применяется ко всей таблице целиком и возвращает одну строку с одним числом. Если требуется подсчёт по группам (например, по отделам), необходимо использовать `GROUP BY`.

`COUNT(*)` обычно оптимизируется СУБД (например, через метаданные или индекс) и работает быстрее, чем подсчёт через `COUNT(column)`, если колонка не индексирована.

## Пример / когда применяется

Такой запрос используется для получения общей статистики, например, для дашборда HR-системы.

```sql
SELECT COUNT(*) AS total_employees
FROM Employees;
```

## На что смотрит интервьюер

- Путают `COUNT(*)` и `COUNT(column)` — не понимают, что вторая форма игнорирует `NULL`.
- Не знают про `COUNT(DISTINCT ...)`.
- Follow-up: "А как посчитать количество сотрудников в каждом отделе?" (проверка понимания `GROUP BY`).
- Follow-up: "Что вернёт `COUNT(*)` для пустой таблицы?" (правильный ответ — 0, а не `NULL`).
- Красный флаг: попытка получить количество строк через `SELECT *` и подсчёт на стороне приложения вместо агрегатной функции в БД.
