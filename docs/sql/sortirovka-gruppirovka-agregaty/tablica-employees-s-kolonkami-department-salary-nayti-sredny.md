# Таблица Employees с колонками department, salary. Найти среднюю зарплату по отделу 'Sales'.

**Коротко:** нужно отфильтровать строки по `WHERE department = 'Sales'` и применить агрегатную функцию `AVG(salary)`.

## Развёрнутый ответ

Здесь важно правильно определить порядок операций и выбрать между `WHERE` и `GROUP BY`/`HAVING`.

Поскольку требуется среднее значение только для одного конкретного отдела (а не для всех отделов сразу), группировка не обязательна — достаточно отфильтровать строки условием `WHERE department = 'Sales'` и затем применить `AVG(salary)` ко всему отфильтрованному набору. Это эффективнее, чем группировать по всем отделам и потом фильтровать через `HAVING`.

Нюансы:

- `AVG` игнорирует `NULL` значения в колонке `salary` — они не участвуют ни в сумме, ни в делителе (количестве). Если для части сотрудников зарплата не указана, среднее будет считаться только по заполненным значениям.
- `WHERE` применяется до агрегации и работает быстрее при наличии индекса на колонку `department`.
- Если требуется получить среднюю зарплату по каждому отделу одновременно, используется `GROUP BY department` без условия `WHERE` на конкретный отдел.
- Тип результата `AVG` для целочисленной колонки может быть дробным (зависит от СУБД): в PostgreSQL `AVG` от `integer` вернёт `numeric` с дробной частью, в некоторых системах может потребоваться явное приведение типов.

## Пример / когда применяется

Такой запрос используется для точечного анализа конкретного отдела, например, в HR-дашборде при выборе отдела из фильтра.

```sql
SELECT AVG(salary) AS avg_salary
FROM Employees
WHERE department = 'Sales';
```

Если нужно получить среднюю зарплату по всем отделам сразу (более общий случай):

```sql
SELECT department, AVG(salary) AS avg_salary
FROM Employees
GROUP BY department;
```

## На что смотрит интервьюер

- Использование `GROUP BY` + `HAVING department = 'Sales'` вместо простого `WHERE` — рабочий, но неоптимальный вариант (обсуждение эффективности).
- Забыли учесть, что `AVG` игнорирует `NULL`, что может исказить трактовку результата.
- Follow-up: "Чем отличается WHERE от HAVING?" — базовый, но важный вопрос.
- Follow-up: "Что если в отделе Sales нет сотрудников?" — правильный ответ: запрос вернёт одну строку со значением `NULL`, а не пустой результат.
- Красный флаг: непонимание разницы между фильтрацией строк и фильтрацией групп.
