# Таблица Employees с колонками department, salary. Посчитать максимальную и минимальную зарплату по каждому отделу.

**Коротко:** используется `GROUP BY department` вместе с агрегатными функциями `MAX(salary)` и `MIN(salary)`, вычисляемыми одновременно в одном запросе.

## Развёрнутый ответ

`MAX` и `MIN` — агрегатные функции, возвращающие соответственно наибольшее и наименьшее значение колонки в пределах группы (или всей таблицы, если группировки нет). Их можно вычислять одновременно в одном `SELECT`, так как обе функции применяются к одной и той же группе строк за один проход.

Особенности:

- `MAX`/`MIN` игнорируют `NULL` — если все значения в группе `NULL`, результат будет `NULL`.
- Функции работают не только с числами, но и со строками (лексикографическое сравнение) и датами (хронологическое сравнение).
- Как и в предыдущих примерах, в `SELECT` рядом с агрегатными функциями допустима только колонка группировки (`department`), иначе будет ошибка.
- Для нахождения "лучшего" сотрудника в отделе (то есть строки с максимальной зарплатой, а не только значения) простого `MAX` недостаточно — потребуется оконная функция (`RANK() OVER (PARTITION BY department ORDER BY salary DESC)`) или подзапрос с соединением по `MAX(salary)`.

## Пример / когда применяется

Такой запрос типичен для отчётов о разбросе зарплат внутри отделов (используется при анализе справедливости оплаты труда).

```sql
SELECT
    department,
    MAX(salary) AS max_salary,
    MIN(salary) AS min_salary
FROM Employees
GROUP BY department;
```

Если дополнительно нужно узнать имя сотрудника с максимальной зарплатой в отделе, потребуется оконная функция:

```sql
SELECT name, department, salary
FROM (
    SELECT
        name,
        department,
        salary,
        RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rnk
    FROM Employees
) t
WHERE rnk = 1;
```

## На что смотрит интервьюер

- Понимание, что `MAX`/`MIN` можно вычислять в одном запросе без дублирования группировки.
- Путают агрегатную функцию `MAX(salary)` (значение) с задачей "найти сотрудника с максимальной зарплатой" (нужна строка целиком) — частая логическая ошибка новичков.
- Follow-up: "Как найти имя сотрудника с максимальной зарплатой в каждом отделе?" — проверка знания оконных функций или подзапросов с JOIN.
- Follow-up: "Что вернёт MAX для группы, где все значения NULL?" — правильный ответ: NULL.
- Красный флаг: попытка получить экстремумы через `ORDER BY` и `LIMIT 1` без `GROUP BY`, что не решает задачу "по каждому отделу".
