# Таблица Employees с колонками name, salary. Вывести сотрудников, отсортированных по зарплате от высокой к низкой.

**Коротко:** используется оператор `ORDER BY salary DESC`, который сортирует строки результата по убыванию значения колонки salary.

## Развёрнутый ответ

`ORDER BY` — это предложение SQL, которое задаёт порядок сортировки строк в результирующем наборе. По умолчанию сортировка выполняется по возрастанию (`ASC`), поэтому для сортировки от большего к меньшему нужно явно указать `DESC`.

Важные нюансы:

- `ORDER BY` выполняется в самом конце обработки запроса — после `WHERE`, `GROUP BY`, `HAVING` и `SELECT`, поэтому можно сортировать по алиасам, заданным в `SELECT`.
- Если в колонке `salary` встречаются `NULL`, поведение зависит от СУБД: в PostgreSQL по умолчанию `NULL` считается "больше" любого значения при `DESC` (то есть окажется первым), в MySQL — `NULL` считается наименьшим значением. При необходимости порядок `NULL` можно задать явно через `NULLS FIRST` / `NULLS LAST` (поддерживается не во всех СУБД).
- Без `ORDER BY` порядок строк в результате не гарантирован — СУБД не обязана сохранять физический порядок хранения.
- Можно сортировать по нескольким колонкам, указывая направление для каждой отдельно, например `ORDER BY salary DESC, name ASC` — это применяется, когда нужно упорядочить одинаковые значения зарплаты по имени.

## Пример / когда применяется

Такой запрос типичен для отчётов "топ сотрудников по зарплате" или для построения рейтинга.

```sql
SELECT name, salary
FROM Employees
ORDER BY salary DESC;
```

Если нужно вывести только топ-5 сотрудников, запрос дополняется ограничением:

```sql
SELECT name, salary
FROM Employees
ORDER BY salary DESC
LIMIT 5;
```

## На что смотрит интервьюер

- Забыли указать `DESC` (по умолчанию сортировка по возрастанию) — частая ошибка.
- Путают `ORDER BY` с `GROUP BY` — сортировка не группирует и не агрегирует данные.
- Не знают, как обрабатываются `NULL` при сортировке в конкретной СУБД.
- Follow-up: как получить только N записей (`LIMIT`/`TOP`/`FETCH FIRST`), как сделать сортировку стабильной при равных значениях (добавить вторичный ключ сортировки).
- Красный флаг: попытка сортировать данные на уровне приложения вместо базы, когда это можно эффективно сделать в SQL с использованием индекса.
