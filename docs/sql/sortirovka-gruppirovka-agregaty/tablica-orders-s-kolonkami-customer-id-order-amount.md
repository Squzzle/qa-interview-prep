# Таблица Orders с колонками customer_id, order_amount. Вывести customer_id и общую сумму заказов только для клиентов с суммой больше 1000.

**Коротко:** нужно сгруппировать заказы по `customer_id` (`GROUP BY customer_id`), вычислить `SUM(order_amount)` и отфильтровать группы условием `HAVING SUM(order_amount) > 1000`.

## Развёрнутый ответ

Задача сочетает группировку, агрегацию и фильтрацию по результату агрегата — типичный паттерн "top customers" / "клиенты выше порога".

Ключевые моменты:

- `SUM(order_amount)` вычисляет суммарную сумму заказов для каждого `customer_id` отдельно, благодаря `GROUP BY customer_id`.
- Условие "только для клиентов с суммой больше 1000" относится к результату агрегатной функции, а не к отдельной строке заказа, поэтому фильтрация обязана идти через `HAVING`, а не `WHERE`.
- `SUM` по умолчанию игнорирует `NULL` значения — если у части заказов `order_amount` не заполнен, они просто не учитываются в сумме (это не приводит к ошибке, но может исказить трактовку "общей суммы").
- Если у клиента вообще нет заказов (гипотетически, если бы данные брались с `LEFT JOIN` из таблицы Customers), `SUM` вернёт `NULL`, а условие `HAVING SUM(...) > 1000` отсеет такую группу, так как сравнение с `NULL` даёт `UNKNOWN`, а не `TRUE`.
- Для дополнительной наглядности результат обычно сортируют по убыванию суммы (`ORDER BY total DESC`), хотя это не было явно запрошено — уточнить у интервьюера, нужен ли порядок вывода.

## Пример / когда применяется

Используется, например, для выявления "ценных" клиентов в интернет-магазине для маркетинговых рассылок или VIP-программ.

```sql
SELECT customer_id, SUM(order_amount) AS total_amount
FROM Orders
GROUP BY customer_id
HAVING SUM(order_amount) > 1000
ORDER BY total_amount DESC;
```

## На что смотрит интервьюер

- Правильность выбора `HAVING` вместо `WHERE` для условия на агрегат.
- Не забыли ли добавить алиас для читаемости результата (`total_amount`).
- Follow-up: "Как изменится запрос, если нужно учитывать только заказы за последний месяц?" — проверка на добавление `WHERE order_date >= ...` до группировки, в дополнение к `HAVING`.
- Follow-up: "Что будет, если один и тот же customer_id встречается в разных регистрах или с пробелами?" — обсуждение качества данных и нормализации перед группировкой.
- Красный флаг: попытка написать `WHERE SUM(order_amount) > 1000` — синтаксическая ошибка, показывающая непонимание порядка выполнения SQL.
