# Расскажите о том, когда в TypeScript используют ключевое слово declare.

**Коротко:** `declare` используется, чтобы сообщить компилятору о существовании переменной, функции, класса, модуля или типа, реализация которых находится вне зоны видимости TypeScript (в другом файле, глобальном скрипте или сторонней JS-библиотеке), без генерации какого-либо JS-кода при компиляции.

## Развёрнутый ответ

`declare` — это способ описать **ambient-объявление** (внешнее объявление): TypeScript-у сообщается "такая сущность существует в рантайме, поверь мне на слово и просто проверяй типы", но сам компилятор не генерирует для неё никакого кода — реализация должна быть предоставлена откуда-то ещё (глобальный `<script>`, сторонняя библиотека, environment вроде Node.js/браузера).

Основные сценарии использования `declare`:

1. **Глобальные переменные, добавленные вне TS-кода** — например, если сторонний `<script>` на странице определяет глобальную переменную `window.myWidget`, можно написать `declare const myWidget: MyWidgetAPI;`, чтобы TypeScript знал о её типе и не ругался при обращении к ней.
2. **Файлы деклараций для JS-библиотек (`.d.ts`)** — большинство пакетов из `@types/*` целиком состоят из `declare module 'lib-name' { export function foo(): void; }` — это описывает публичный API библиотеки без её реализации.
3. **Глобальное расширение существующих типов** — например, `declare global { interface Window { myGlobal: string; } }` внутри модуля, чтобы добавить поле в глобальный `Window` из TS-модуля (обязательно нужен блок `declare global` в контексте модуля).
4. **Модули для нестандартных импортов** — например, `declare module "*.svg" { const content: string; export default content; }`, чтобы TypeScript не ругался при импорте файлов, которые не являются `.ts`/`.js` (изображения, стили и т.д.), обрабатываемых бандлером.
5. **`declare namespace`** — для описания структуры legacy-библиотек, которые размещают API в глобальном пространстве имён (например, jQuery `$`).
6. **`declare function`, `declare class`** — для описания сигнатуры функции или класса, чья реализация подключается отдельно (например, через `<script>` или через сборку в другом языке — WASM, C++ addon).

Важно: `declare` не создаёт значение в рантайме — если объявить `declare const foo: string;` без реальной глобальной переменной `foo`, компиляция пройдёт успешно, но при выполнении получится `ReferenceError`, потому что TypeScript просто доверяет разработчику. Также `declare` не может сочетаться с телом функции/значением инициализации (нельзя написать `declare function foo() { return 1; }` — declare-объявления всегда "пустые", это только описание контракта.

## Пример / когда применяется

```ts
// Объявление глобальной переменной, определённой сторонним скриптом
declare const GOOGLE_MAPS_API_KEY: string;

console.log(GOOGLE_MAPS_API_KEY); // компилируется, но в рантайме зависит от того, что переменная реально существует
```

```ts
// Файл global.d.ts — расширение типа Window
export {};

declare global {
  interface Window {
    analytics: {
      track(event: string, payload?: Record<string, unknown>): void;
    };
  }
}

window.analytics.track("page_view");
```

```ts
// Описание модуля без типов (например, импорт CSS-модулей)
declare module "*.css" {
  const classes: { [key: string]: string };
  export default classes;
}
```

## На что смотрит интервьюер

- Понимает ли кандидат, что `declare` не генерирует рантайм-код и служит только компилятору для проверки типов.
- Знает ли типичные сценарии: `.d.ts` файлы для JS-библиотек, глобальные переменные от сторонних скриптов, расширение `Window`, декларации для нестандартных импортов (svg/css).
- Может ли объяснить разницу между `declare module` для стороннего пакета и `declare global` для расширения глобальной области видимости.
- Частая ошибка — путать `declare` с обычным объявлением переменной, или пытаться дать `declare`-сущности реализацию (тело функции, значение) в одном месте.
- Follow-up: "Как правильно расширить интерфейс из node_modules без модификации самой библиотеки?", "В чём разница между `.d.ts` файлом и обычным `.ts` файлом с `declare`?".
